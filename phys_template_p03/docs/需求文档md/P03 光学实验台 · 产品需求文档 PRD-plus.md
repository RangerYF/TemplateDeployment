## 一、文档目的

这份 PRD-plus 不再只回答“要做哪些功能”，而是重点回答以下问题：

1. 这个产品到底是什么，不是什么。
2. 每个模块的教学目标是什么。
3. 哪些自由度必须给老师，哪些自由度不该开放。
4. 每个模块默认应该怎样进入课堂讲解。
5. 后续开发和评审时，什么算达标，什么不算。

本文件将作为后续逐模块评审的基线文档。在本文件未审清楚前，不建议继续扩大开发范围。

## 二、产品定位重定义

### 2.1 我们要做的产品

P03 的目标产品不是“完全自由搭建的光学工作台”，也不是“只有几个滑杆的公式演示器”，而是：

**面向高中物理课堂的标准实验教学演示器，并在每个标准实验内部，给老师足够但有边界的教学自由度。**

换句话说：

- 核心仍然是标准实验和典型课堂场景。
- 自由度服务于讲解，不服务于无限搭建。
- 产品价值在于“帮助老师把现象讲清楚、把变化讲明白、把结论讲准确”。

### 2.2 我们不做的产品

以下方向明确不作为本阶段目标：

- 不做任意元件无限拼装的通用光学沙盒。
- 不做竞赛级或大学级精密光学求解器。
- 不做超出高中教学范围的大量公式推导交互。
- 不以“酷炫拖拽”代替教学价值。
- 不为了自由度牺牲默认上手性和课堂稳定性。

### 2.3 产品关键词

- `教学优先`
- `标准实验优先`
- `现象正确`
- `变化趋势清晰`
- `有边界的自由度`
- `默认即可上课`

## 三、目标用户与使用方式

### 3.1 目标用户

- 高中物理老师
- 初中阶段透镜成像演示的兼容用户

默认假设：

- 用户不具备编程背景。
- 用户上课时操作时间有限。
- 用户更关心“能不能快速讲清楚”，而不是“能不能无限搭建”。

### 3.2 用户核心任务

老师使用本产品时，最常见的任务不是“创造新实验”，而是以下几类：

1. 快速打开一个标准实验，直接进入讲解。
2. 调整关键参数，让学生看到规律如何变化。
3. 切换几种典型变式，用于对比教学。
4. 用数值、法线、角度、强度曲线辅助解释现象。
5. 避免在黑板静态作图中容易画错或讲错的地方。

### 3.3 用户使用原则

每个模块都必须同时满足两层能力：

1. `标准演示层`
   打开即有正确默认场景，老师不配置也能直接开始讲。
2. `教学变式层`
   允许在不破坏实验本质的前提下，对关键参数、局部结构、显示方式做有限调整。

## 四、总体产品原则

### 4.1 统一交互框架

整个平台保持统一骨架：

- 顶部：模块入口
- 左侧：参数与显示控制
- 中央：实验演示区
- 右侧：实时数值、公式验证、教学提示

模块切换时不改变整体使用逻辑，只替换当前实验的内容。

### 4.2 自由度设计原则

自由度必须满足以下标准，才值得开放：

1. 老师课堂上真的会用到。
2. 能明显提升讲解效果。
3. 不会让模块逻辑变得失控。
4. 不会让用户面对空白画布无从下手。

如果某项自由度只是在技术上“可以做”，但对教学没有明确价值，应视为非优先。

### 4.3 默认优先原则

每个模块必须提供：

- 默认实验场景
- 默认参数
- 默认显示选项
- 默认教学重点

默认状态必须满足“教师首次打开即可直接演示”。

### 4.4 教学向显示原则

中央演示区优先展示“课堂上要指给学生看的东西”，例如：

- 光路
- 法线
- 角度
- 像的位置与性质
- 条纹图样
- 强度分布

右侧数值区优先展示“课堂上要读给学生听的东西”，例如：

- 临界角
- 折射角
- 条纹间距
- 焦距、物距、像距
- 第一极小位置
- 牛顿环半径

## 五、整体范围定义

### 5.1 本期必须覆盖的一级模块

1. 折射与全反射
2. 透镜与成像
3. 双缝干涉
4. 单缝 / 圆孔衍射
5. 薄膜干涉

### 5.2 本期必须覆盖的产品能力

- 模块切换
- 参数调节
- 关键显示开关
- 正确的现象演示
- 关键数值读数
- 公式验证显示
- 少量高价值的教学变式

### 5.3 本期不追求的能力

- 任意添加/删除任意光学元件
- 多模块跨场景自由拼装
- 任意旋转所有元件
- 复杂多元件联立求解
- 存档、分享、自定义场景编辑器

## 六、教师自由度边界

### 6.1 必须开放的自由度类型

- 关键物理参数调节
- 典型光源类型切换
- 少量关键元件的相对位置调整
- 标准模型之间的切换
- 显示层开关
- 对比视图或对比参数能力

### 6.2 谨慎开放的自由度类型

- 元件位置拖拽
- 光源位置拖拽
- 屏幕位置拖拽

这类自由度只能在“有明确教学意义”的模块中开放，且必须有限制范围。

### 6.3 不开放的自由度类型

- 空白画布自由摆放全部元件
- 任意旋转全部元件
- 元件无限复制
- 自定义公式求解器
- 开放与教学目标无关的细节参数

## 七、模块评审方法

后续按模块评审时，每个模块必须依次回答以下 8 个问题：

1. 这个模块解决老师哪一个高频教学任务？
2. 默认打开后，老师能否立刻开始讲？
3. 当前开放的自由度是否都服务于教学？
4. 是否缺少关键变式？
5. 是否存在多余但低价值的交互？
6. 数值、公式、现象三者是否一致？
7. 学生最容易误解的点是否被显式支持讲解？
8. 验收标准是否具体到可测试？

## 八、模块 1：折射与全反射

### 8.1 模块目标

帮助老师讲清以下内容：

- 折射角与折射率关系
- 法线与角度定义
- 全反射发生条件
- 临界角意义
- 玻璃砖中的平行出射和侧移
- 半球形玻璃砖中的曲面法线判断
- 光导纤维的导光原理

### 8.2 标准实验范围

本模块包含以下标准场景：

1. 平行界面折射
2. 矩形玻璃砖
3. 半球形玻璃砖
4. 光导纤维模型

### 8.3 默认讲解脚本

老师进入模块后，应能按如下顺序讲解：

1. 先看入射光、法线、折射光、反射光。
2. 调节入射角，观察折射角变化。
3. 调节介质折射率，观察折射程度变化。
4. 在满足条件时，观察全反射出现。
5. 切换到玻璃砖或半球模型，说明形状改变后的光路特点。
6. 切换到光纤模型，说明多次全反射的意义。

### 8.4 必须支持的自由度

- 入射角调节
- 介质折射率调节
- 折射材料预设切换
- 单束光 / 平行光切换
- 入射点有限范围拖动
- 元件整体有限范围移动
- 法线显示开关
- 角度标注开关
- 公式验证开关
- 颜色显示开关

### 8.5 不应扩大的自由度

- 不开放任意新增介质块
- 不开放任意旋转全部界面
- 不开放多块折射介质自由组合
- 不将本模块扩展为通用几何光学沙盒

### 8.6 必须展示的实时信息

- 入射角
- 折射角
- 反射角
- 临界角
- 介质折射率
- 是否发生全反射
- 玻璃砖侧移量
- 光纤导光 / 漏光判定

### 8.7 教学风险点

- 角度相对法线而不是相对界面测量
- 折射率大小与偏折方向关系易讲反
- 全反射只在从光密到光疏且超过临界角时发生
- 半球曲面法线判断容易讲错

### 8.8 本模块验收标准

- 折射方向与斯涅尔定律一致
- 临界角计算与显示一致
- 入射角超过临界角时，折射光停止显示或明确标识不存在
- 玻璃砖出射光与入射光保持平行
- 半球模式下法线判断与图示一致
- 光纤模式能清楚体现连续全反射与弯曲损耗趋势

## 九、模块 2：透镜与成像

### 9.1 模块目标

帮助老师讲清以下内容：

- 凸透镜成像规律
- 凹透镜成像特点
- 物距变化对像距、倒正、大小、虚实的影响
- 主光轴、焦点、特殊光线的作用
- 屏幕与实像关系

### 9.2 标准实验范围

1. 凸透镜成像
2. 凹透镜成像

### 9.3 默认讲解脚本

1. 先展示主光轴、透镜、焦点、物体、像。
2. 调节物距，观察像的位置与性质变化。
3. 依次覆盖 `u > 2f`、`u = 2f`、`f < u < 2f`、`u = f`、`u < f`。
4. 切换凹透镜，说明始终成正立缩小虚像。
5. 通过屏幕位置判断实像是否落在屏上。

### 9.4 必须支持的自由度

- 凸 / 凹透镜切换
- 焦距调节
- 物距调节
- 物高调节
- 光源类型切换
  - 物体光源
  - 点光源
  - 平行光
- 物体有限拖动
- 透镜有限拖动
- 屏幕有限拖动
- 显示屏幕开关
- 特殊光线显示开关
- 公式验证开关

### 9.5 不应扩大的自由度

- 不开放多透镜自由拼装
- 不开放任意角度放置透镜
- 不开放任意二维画布编辑
- 不做复杂成像题求解器

### 9.6 必须展示的实时信息

- 焦距
- 物距
- 像距
- 像高或放大率
- 像的性质
- 屏幕判定
- 光源类型

### 9.7 教学风险点

- 学生容易混淆实像和虚像
- `u = f` 时“不成像”的含义容易模糊
- 凹透镜像距正负与图中位置关系容易讲乱
- 屏幕上只能接到实像这一点必须明确

### 9.8 本模块验收标准

- 凸透镜各典型物距下成像性质正确
- 凹透镜始终成正立缩小虚像
- 特殊光线方向正确
- 屏幕判定与像位置一致
- 平行光入射时与焦点关系正确

## 十、模块 3：双缝干涉

### 10.1 模块目标

帮助老师讲清以下内容：

- 干涉条纹形成
- 条纹间距与波长、缝间距、屏距关系
- 白光干涉的典型特征
- 现象演示与强度曲线之间的对应关系

### 10.2 标准实验范围

1. 杨氏双缝干涉

### 10.3 默认讲解脚本

1. 默认显示双缝、屏幕和中央条纹图样。
2. 调节缝间距，说明条纹为何变密。
3. 调节波长，说明颜色与条纹疏密关系。
4. 调节屏距，说明条纹间距变化。
5. 打开强度曲线，建立“图样”和“数值趋势”的联系。
6. 切换白光，讲中央亮纹与彩色分布。

### 10.4 必须支持的自由度

- 缝间距调节
- 缝宽调节
- 屏距调节
- 波长调节
- 白光开关
- 颜色显示开关
- 强度曲线开关
- 公式验证开关
- 光源、双缝、屏幕的有限位置调整

### 10.5 不应扩大的自由度

- 不开放多组双缝组合
- 不开放任意屏幕形状
- 不开放完整波动光学推导流程

### 10.6 必须展示的实时信息

- 波长
- 缝间距
- 屏距
- 条纹间距
- 当前是否白光模式
- 强度分布趋势

### 10.7 教学风险点

- 仅看图样不看参数，学生容易只记结果不记关系
- 白光条纹表现容易做假，必须至少满足高中课堂认知
- 缝宽不是主公式核心，但会影响包络，不能完全忽略

### 10.8 本模块验收标准

- `d` 增大时条纹变密
- `lambda` 增大时条纹变宽
- `L` 增大时条纹变宽
- 条纹间距读数与公式一致
- 强度曲线与条纹图样对应合理
- 白光模式中央亮纹特征正确

## 十一、模块 4：单缝 / 圆孔衍射

### 11.1 模块目标

帮助老师讲清以下内容：

- 单缝衍射与圆孔衍射的图样差异
- 缝宽 / 孔径对图样宽度的影响
- 波长变化对图样宽度的影响
- 强度曲线与中央主极大的关系

### 11.2 标准实验范围

1. 单缝衍射
2. 圆孔衍射

### 11.3 默认讲解脚本

1. 先展示单缝衍射默认图样。
2. 调节缝宽，观察中央亮纹宽度变化。
3. 调节波长，观察图样宽度变化。
4. 切换到圆孔，观察艾里斑。
5. 打开强度曲线或多波长对比，说明规律。

### 11.4 必须支持的自由度

- 单缝 / 圆孔切换
- 缝宽调节
- 孔径调节
- 波长调节
- 屏距调节
- 颜色显示开关
- 强度曲线开关
- 多波长对比开关
- 光源、孔径、屏幕的有限位置调整

### 11.5 不应扩大的自由度

- 不开放任意孔径形状
- 不开放复杂衍射光阑编辑器
- 不将本模块做成频谱分析工具

### 11.6 必须展示的实时信息

- 缝宽或孔径
- 波长
- 屏距
- 第一极小位置或艾里盘半径
- 图样变化趋势说明

### 11.7 教学风险点

- 学生容易把干涉和衍射混为一谈
- 圆孔衍射若图样不够典型，会削弱教学说服力
- 单缝衍射需明确中央明纹最宽最亮

### 11.8 本模块验收标准

- 单缝下 `a` 减小时中央主极大变宽
- 单缝下 `lambda` 增大时图样变宽
- 圆孔下 `D` 减小时艾里斑变大
- 圆孔下 `lambda` 增大时艾里斑变大
- 图样与强度曲线变化趋势一致

## 十二、模块 5：薄膜干涉

### 12.1 模块目标

帮助老师讲清以下内容：

- 薄膜干涉的三种典型图样
- 膜厚、楔角、曲率半径与图样关系
- 肥皂泡彩色条纹、楔形薄膜平行条纹、牛顿环同心圆三者差异

### 12.2 标准实验范围

1. 肥皂泡干涉
2. 楔形薄膜干涉
3. 牛顿环

### 12.3 默认讲解脚本

1. 先展示三种典型图样的视觉区别。
2. 在楔形薄膜中调节楔角和波长，观察条纹间距变化。
3. 在牛顿环中调节曲率半径或波长，观察环半径变化。
4. 在肥皂泡中强调“白光下不同波长满足条件的位置不同”。

### 12.4 必须支持的自由度

- 三种模型切换
- 膜厚调节
- 薄膜折射率调节
- 楔角调节
- 波长调节
- 曲率半径调节
- 强度曲线开关
- 公式验证开关

### 12.5 不应扩大的自由度

- 不开放任意三维薄膜建模
- 不开放复杂入射角联动求解
- 不追求研究级精细彩色渲染

### 12.6 必须展示的实时信息

- 膜厚
- 楔角
- 折射率
- 波长
- 牛顿环典型环半径
- 楔形条纹间距
- 肥皂泡的厚度变化含义

### 12.7 教学风险点

- 若三种图样外观差异不明显，模块教学价值会显著下降
- 牛顿环中心明暗必须处理正确
- 肥皂泡彩色效果可以近似，但必须有合理物理解释口径

### 12.8 本模块验收标准

- 牛顿环必须呈现同心圆环
- 楔形薄膜必须呈现近似平行条纹
- 肥皂泡必须呈现随厚度变化的彩色条纹带
- 关键参数变化时图样变化方向正确
- 右侧读数与公式验证逻辑一致

## 十三、通用显示与交互要求

### 13.1 左侧参数区

要求：

- 根据当前模块动态切换
- 参数控件数量适中
- 避免一次出现过多低价值参数
- 每个参数最好带教学提示语

### 13.2 中央演示区

要求：

- 清晰、稳定、实时刷新
- 优先保证课堂投影可读性
- 重要元素不能被装饰信息遮挡
- 拖拽只用于有教学意义的关键元素

### 13.3 右侧数值区

要求：

- 读数优先简洁
- 关键值突出显示
- 公式验证默认可开关
- 教学要点应支持简短卡片式展示

### 13.4 移动端与窄屏

本产品主要面向桌面 / 教室大屏，不以移动端完整体验为优先目标。  
窄屏只要求不崩溃，不要求完整课堂效率。

## 十四、验收总标准

### 14.1 物理正确性

- 关键数值正确
- 图样形态正确
- 趋势方向正确
- 公式显示与计算一致

### 14.2 教学可用性

- 默认状态可直接上课
- 参数不需要复杂学习成本
- 关键自由度能服务课堂讲解
- 不出现明显误导性的视觉表达

### 14.3 交互稳定性

- 调参实时刷新
- 不出现明显残影或错位
- 拖拽范围受控
- 模块切换状态合理

### 14.4 产品边界一致性

- 各模块都遵守“标准实验教学演示器”的产品定位
- 不因局部炫技破坏整体一致性
- 自由度始终受教学目标约束

## 十五、开发前冻结项

在进入下一轮开发前，建议先冻结以下内容：

1. 产品定位是否确定为“标准实验教学演示器 + 有边界自由度”。
2. 五个一级模块是否全部保留。
3. 每个模块的标准场景是否确认。
4. 每个模块允许的自由度边界是否确认。
5. 哪些“看起来高级但暂不做”的能力要明确写进非目标。

这些项如果不先冻结，后续开发很容易再次滑向“演示器”和“搭建台”之间的混合状态。

## 十六、建议的评审顺序

建议按以下顺序逐模块评审：

1. 折射与全反射
   原因：最能检验几何光学框架、法线、角度、全反射逻辑是否清楚。
2. 透镜与成像
   原因：最能检验“标准实验 + 教师自由度”的平衡是否合理。
3. 双缝干涉
   原因：最适合校验“现象 + 读数 + 公式”三者协同。
4. 单缝 / 圆孔衍射
   原因：可进一步检验图样渲染与趋势表达能力。
5. 薄膜干涉
   原因：视觉表现更复杂，建议在前四项边界清楚后再审。

## 十七、结论

P03 的正确方向已经明确：

- 不是通用光学沙盒
- 不是纯公式滑杆 demo
- 而是面向课堂的标准实验教学演示器

它的竞争力不来自“无限自由”，而来自：

1. 默认就能上课
2. 现象和数值都讲得准
3. 老师能方便做典型变式
4. 每个模块都知道自己在帮老师解决什么教学问题

后续所有模块评审与开发判断，都应优先服从这四条。
