## 一、模块定位

本模块是 P03 中的成像教学核心模块，目标不是做复杂光学系统设计器，而是帮助老师围绕“焦距、物距、像距、像的性质、屏幕判定、特殊光线”完成课堂讲解。

本模块应满足两层目标：

1. 标准课堂场景打开即可讲。
2. 老师可在标准场景内部做有边界的教学变式。

本模块应成为后续整个产品中“成像规律是否准确、特殊光线是否规范、实像/虚像与屏幕关系是否清楚”的基准模块。

## 二、模块教学目标

老师使用本模块后，应能稳定讲清以下内容：

1. 凸透镜与凹透镜的基本作用。
2. 主光轴、焦点、焦距的意义。
3. 物距变化时像距和像的性质如何变化。
4. 凸透镜在典型物距条件下的成像规律。
5. 凹透镜始终成正立缩小虚像。
6. 实像和虚像与屏幕之间的关系。
7. 三条特殊光线为何能帮助判断像的位置。

## 三、模块范围定义

### 3.1 本模块纳入范围的场景族

本模块按 5 个场景族组织：

1. `凸透镜成像族`
2. `凹透镜成像族`
3. `光源类型族`
4. `屏幕验证族`
5. `对比教学族`

### 3.2 本模块不纳入范围的内容

- 多透镜自由拼装
- 任意三维光学系统设计
- 大学级厚透镜 / 像差推导
- 复杂题库求解器
- 任意数量元件自由添加

## 四、场景族与子场景清单

### 4.1 凸透镜成像族

**教学任务：**

- 讲清凸透镜典型物距条件下像的位置与性质

**包含子场景：**

1. `u > 2f`
2. `u = 2f`
3. `f < u < 2f`
4. `u = f`
5. `u < f`
6. 连续拖动物体观察像变化

**默认主场景：**

- `OPT-011 凸透镜成像`
- 默认参数建议：`f = 10 cm`, `u = 25 cm`, `h = 40 cm`

### 4.2 凹透镜成像族

**教学任务：**

- 讲清凹透镜始终成正立缩小虚像

**包含子场景：**

1. 凹透镜标准成像
2. 改变物距的对比
3. 与凸透镜并列对比

**默认主场景：**

- `OPT-012 凹透镜成像`
- 默认参数建议：`|f| = 10 cm`, `u = 25 cm`, `h = 40 cm`

### 4.3 光源类型族

**教学任务：**

- 不只讲“物体”，还要讲不同入射条件对成像/光路的影响

**包含子场景：**

1. 物体光源
2. 平行光
3. 点光源
4. 平行光经过凸透镜汇聚
5. 平行光经过凹透镜发散

**默认主场景：**

- 在凸透镜成像场景中切换光源类型

### 4.4 屏幕验证族

**教学任务：**

- 讲清实像 / 虚像与屏幕的关系

**包含子场景：**

1. 屏幕接收到实像
2. 屏幕接不到虚像
3. 移动屏幕寻找成像位置

**默认主场景：**

- 屏幕开启，标准凸透镜成像

### 4.5 对比教学族

**教学任务：**

- 支持老师做课堂变式和左右对比讲解

**包含子场景：**

1. 凸透镜 vs 凹透镜
2. 物体前后移动对比
3. 焦距变化对比
4. 是否落屏对比

**默认主场景：**

- 为第二批深化内容，不阻塞第一批开发

### 5.3 第一批统一要求

第一批正式实现子场景必须全部满足：

1. 有明确默认参数。
2. 有明确默认讲解顺序。
3. 有明确实时读数。
4. 有明确像的性质说明。
5. 有明确屏幕判定逻辑。
6. 默认状态下不依赖点光源也能完成整章主线讲解。

### 5.4 第一批开发优先顺序

建议按如下顺序推进：

1. 凸透镜标准成像主链
2. 凹透镜标准成像
3. 屏幕验证
4. 光源类型切换
5. 第二批对比教学

## 六、统一能力边界

### 6.1 全模块统一必须具备的能力

1. 透镜对象切换
2. 光源对象切换
3. 屏幕对象显示 / 隐藏
4. 对象直接拖动
5. 画布平移与缩放
6. 主光轴与焦点显示
7. 三条特殊光线显示
8. 实时读数区
9. 公式验证开关

### 6.2 受限画布模式约束

模块 2 明确继续复用模块 1 已验证过的交互模式：

- `单主对象`
- `单主光源`
- `受限搭建画布`

也就是说：

- 允许老师直接拖动物体 / 透镜 / 屏幕
- 允许画布平移与缩放
- 不允许进入多透镜自由拼装的通用搭建器状态

该约束是模块 2 的开发前冻结项，不建议在第一批开发中突破。

### 6.3 有边界开放的交互能力

以下能力可开放，但必须限制范围：

1. 物体 / 光源拖动
2. 透镜拖动
3. 屏幕拖动
4. 焦距调整
5. 物高调整

限制原则：

- 不让用户进入完全空白的搭建状态
- 不开放教学目标无关的复杂自由度
- 所有拖动都应服务于成像规律讲解

### 6.4 明确不开放的能力

1. 多透镜自由拼装
2. 任意增加多个屏幕
3. 复杂镜片曲率和像差参数
4. 通用成像求解器

## 七、参数体系

### 7.1 通用参数

- 焦距 `f`
- 物距 `u`
- 物高 `h`
- 光源类型
- 透镜类型
- 显示开关

### 7.2 场景专属参数

**凸透镜 / 凹透镜：**

- 焦距 `f`
- 物距 `u`
- 物高 `h`

**屏幕验证：**

- 屏幕位置 `x`
- 是否显示屏幕

### 7.3 对象模型

本模块画布中只允许以下核心对象：

1. 物体 / 主光源
2. 透镜
3. 屏幕
4. 像
5. 主光轴 / 焦点 / 2F 标记

对象原则：

- 每个对象互相解耦
- 拖动透镜不改变光源位置
- 拖动屏幕不改变成像本身，只改变“是否落屏”的判断
- 第一批不允许同时出现多个透镜或多个屏幕

## 九、各场景族必备读数

### 9.1 全模块通用读数

- 当前透镜类型
- 当前光源类型
- 当前对象状态

### 9.2 成像读数

- 焦距 `f`
- 物距 `u`
- 像距 `v`
- 放大率 `m`
- 像高 `h'`
- 像的性质
- 屏幕判定

其中以下两项必须作为右侧核心高亮卡存在：

1. `像的性质卡`
2. `屏幕判定卡`

### 9.3 路径解释读数

- 当前路径模式
- 当前是否实像
- 当前是否虚像
- 当前是否落屏

### 9.4 右侧信息区优先级

右侧信息区的优先级必须明确如下：

1. 像的性质卡
2. 屏幕判定卡
3. 焦距 / 物距 / 像距
4. 放大率 / 像高
5. 当前路径说明
6. 公式验证

不建议把公式区放在比“像的性质”和“屏幕判定”更高的优先级。

## 十、界面与交互风格要求

模块 2 应继续复用模块 1 已形成的苹果风格总体壳层：

- 顶栏玻璃胶囊
- 左右玻璃卡片
- 中央视台优先
- 右侧信息卡高亮重要数值

模块 2 在右侧的产品重点不是“参数面板”，而是“成像判读区”。

但模块 2 在画布内容上要更“成像导向”：

- 主光轴必须清晰
- 物体、透镜、像、屏幕的相对关系必须一眼可辨
- 特殊光线和反向延长线必须层级分明

不建议：

- 把过多说明文字塞进舞台中央
- 用过度动画干扰课堂讲解

## 十一、中央演示区要求

中央演示区必须优先展示：

1. 主光轴
2. 透镜
3. 焦点与 2F
4. 物体 / 光源
5. 像
6. 屏幕
7. 特殊光线 / 反向延长线

以下表现要求必须满足：

- 物体和像的正倒要明显
- 虚像必须通过反向延长线区分
- 实像与屏幕关系必须清楚
- 平行光场景中焦点汇聚或反向延长必须明确

## 十二、默认课堂脚本

建议老师进入本模块后的默认讲解路径如下：

1. 先从凸透镜标准成像开始，讲主光轴、焦点和物体。
2. 连续调整物距，覆盖五种典型物距条件。
3. 打开三条特殊光线，说明如何判断像的位置。
4. 打开屏幕，讲实像能否落在屏上。
5. 切换到凹透镜，讲始终成正立缩小虚像。
6. 切换平行光，讲焦点意义和光束汇聚 / 发散。

## 十三、主要教学风险点

本模块必须重点避免以下风险：

1. 把像距和物距混淆。
2. 把实像和虚像讲反。
3. `u = f` 的“不成像”表达模糊。
4. 屏幕接不到虚像这一点没有明确讲清。
5. 特殊光线画法不规范。

## 十四、验收标准

### 14.1 物理正确性验收

- 凸透镜五种典型物距情形正确
- 凹透镜始终成正立缩小虚像
- 平行光经过凸透镜汇聚于焦点
- 平行光经过凹透镜发散，反向延长交于焦点
- 屏幕判定与实像 / 虚像关系一致

### 14.2 教学可用性验收

- 默认场景打开即可讲
- 三条特殊光线足够清楚
- 右侧读数能支持课堂讲解
- 当前路径说明清楚
- 像的性质卡与屏幕判定卡足够突出

### 14.3 模块边界验收

- 模块呈现为成像教学模块，而非通用光学编辑器
- 自由度始终围绕成像教学服务
- 不因扩展对比场景破坏主链清晰度

## 十五、建议的开发冻结项

在开始实现模块 2 前，建议先冻结以下项：

1. 是否确认按 5 个场景族组织模块。
2. 是否确认第一批正式实现子场景范围。
3. 是否确认模块 2 继续复用模块 1 的单对象 / 单主光源画布模式。
4. 是否确认右侧读数必须包含像的性质与屏幕判定。
5. 是否确认三条特殊光线是第一批正式能力。
6. 是否确认点光源进入第二批而非第一批。
7. 是否确认公式区按状态切换说明，而不是固定模板。

## 十六、结论

模块 2 不应被定义成“透镜画图器”，而应被定义成：

**一个围绕透镜成像规律、特殊光线、屏幕验证和像的性质展开的完整教学专题模块。**

它的价值不在于做复杂搭建，而在于：

1. 把成像规律讲准
2. 把典型物距条件讲清
3. 把实像 / 虚像和屏幕关系讲透
4. 支持老师做有限但高价值的课堂变式

后续如果继续细化本模块，建议下一轮进入“单子场景规格表”，逐个明确：

- 默认参数
- 必备控件
- 必备读数
- 必备公式与状态说明
- 默认讲解顺序
- 明确不做项
