## 一、模块定位

本模块是 P03 中的几何光学基础模块，核心任务不是做通用光学沙盒，而是帮助老师围绕“折射、法线、临界角、全反射、典型介质形状、光纤应用”完成课堂讲解。

本模块应满足两层目标：

1. 标准实验可直接演示。
2. 老师可在标准实验内部进行有边界的教学变式。

本模块应成为后续整个产品中“法线判断是否清楚、角度定义是否统一、物理方向是否正确”的基准模块。

## 二、模块教学目标

老师使用本模块后，应能稳定讲清以下内容：

1. 入射角、折射角、反射角都以法线为参照。
2. 光由光疏介质进入光密介质时靠近法线，反之远离法线。
3. 全反射只发生在从光密到光疏且入射角超过临界角时。
4. 平行界面、矩形玻璃砖、半球、球体等不同几何结构中，法线的判断方式不同。
5. 玻璃砖中的出射光与入射光平行，但会产生侧向位移。
6. 光导纤维导光的本质是纤芯-包层界面持续满足全反射条件。

## 三、模块范围定义

### 3.1 本模块纳入范围的场景族

本模块按 5 个场景族组织，而不是按零散实验平铺。

1. `界面折射族`
2. `平板介质族`
3. `半球介质族`
4. `球形介质族`
5. `光纤应用族`

### 3.2 本模块不纳入范围的内容

- 多块介质自由拼装
- 任意数量元件自由添加
- 任意角度旋转全部介质体
- 通用二维画布编辑器
- 多透镜、多反射镜、多介质联立求解

## 四、场景族与子场景清单

### 4.1 界面折射族

**教学任务：**

- 讲清斯涅尔定律
- 讲清折射方向变化
- 讲清临界角和全反射

**包含子场景：**

1. 空气 -> 玻璃
2. 玻璃 -> 空气
3. 空气 -> 水
4. 水 -> 空气
5. 自定义双介质折射
6. 临界角演示
7. 全反射演示
8. 单束光与平行光对比

**默认主场景：**

- `OPT-001 平行界面折射`
- 默认参数：`theta1 = 30°`, `n1 = 1.000`, `n2 = 1.500`

### 4.2 平板介质族

**教学任务：**

- 讲清双界面折射
- 讲清平行出射
- 讲清侧向位移

**包含子场景：**

1. 矩形玻璃砖标准演示
2. 厚度变化对侧移影响
3. 折射率变化对侧移影响
4. 入射角变化对侧移影响
5. 单束光与平行光在玻璃砖中的对比

**默认主场景：**

- `OPT-002 矩形玻璃砖`
- 默认参数：`theta = 45°`, `n = 1.500`, `d = 5 cm`

### 4.3 半球介质族

**教学任务：**

- 讲清球心入射
- 讲清平面入射到曲面时法线如何判断
- 讲清曲面处全反射

**包含子场景：**

1. 半球球心入射
2. 半球平面入射
3. 平面入射后到曲面折射
4. 平面入射后到曲面全反射
5. 挖空半球示意

**默认主场景：**

- `OPT-003 半球形玻璃砖`
- 默认参数：`theta = 30°`, `n = 1.500`, `R = 5 cm`
- 默认模式：平面入射

### 4.4 球形介质族

**教学任务：**

- 进一步强化“球面法线就是半径方向”
- 帮助老师对比半球与球体在法线判断上的差异

**包含子场景：**

1. 球形玻璃砖基本折射
2. 不同入射点导致的不同路径
3. 球形与半球法线对比

**默认主场景：**

- 本场景族为模块扩展场景，不强制绑定实验编号
- 默认参数建议：`theta = 25°`, `n = 1.500`, `R = 5 cm`

### 4.5 光纤应用族

**教学任务：**

- 把全反射与真实应用联系起来
- 说明纤芯 / 包层折射率关系
- 说明弯曲导致漏光风险

**包含子场景：**

1. 直光纤导光
2. 弯曲光纤导光
3. 弯曲半径变小时的漏光趋势
4. 纤芯 / 包层折射率差对导光的影响

**默认主场景：**

- `OPT-004 光导纤维模型`
- 默认参数：`n1 = 1.500`, `n2 = 1.300`, `R = 10 cm`

## 六、统一能力边界

### 5.1 全模块统一必须具备的能力

1. 场景族切换
2. 子场景切换
3. 标准默认参数一键恢复
4. 关键参数滑杆调节
5. 材料预设切换
6. 光线路径实时刷新
7. 法线显示开关
8. 角度标注开关
9. 颜色显示开关
10. 公式验证开关
11. 实时读数区

### 5.2 有边界开放的交互能力

以下能力可以开放，但必须限制范围：

1. 入射点拖动
2. 元件整体位置拖动
3. 单束光 / 平行光切换
4. 半球入射方式切换
5. 光纤整体位置或弯曲程度调节

限制原则：

- 只能在预设可讲清的区域内变化
- 不能出现无法解释或明显超纲的结果
- 不能因自由拖拽导致用户进入“空白搭建状态”

### 5.3 明确不开放的能力

1. 空白画布任意新增元件
2. 任意复制无限多个介质体
3. 任意旋转全部几何结构
4. 多介质复杂自由组合
5. 通用几何光学编辑器模式

## 十、界面与交互风格要求

本模块前端界面不应继续沿用“功能堆叠但不够精致”的实现方式。

模块 1 重构时，界面与交互必须明确朝以下方向靠拢：

- 苹果风格
- 教学优先
- 信息层级清晰
- 操作成本低
- 默认状态就足够美观且可信

### 10.1 整体视觉方向

建议采用偏苹果风格的教学工作台设计：

- 大圆角卡片
- 半透明浅色面板
- 清晰分层的磨砂质感
- 节制但明确的强调色
- 更强的留白和秩序感
- 避免工程调试面板式堆叠

视觉关键词：

- `clean`
- `glassmorphism-lite`
- `calm`
- `teaching workstation`

### 10.2 布局要求

模块 1 推荐使用固定三栏结构：

1. 左栏：场景族 / 子场景 / 参数 / 显示选项
2. 中栏：演示舞台
3. 右栏：实时读数 / 公式 / 教学提示 / 风险提醒

要求：

- 中栏必须始终是视觉焦点。
- 左栏操作区应层次明确，不能像长表单一样难扫。
- 右栏读数区应卡片化，关键读数优先突出。

### 10.3 苹果风格细化要求

必须体现以下特征：

1. 顶部导航简洁，不做复杂标签堆砌。
2. 控件外观统一，滑杆、分段选择、开关风格一致。
3. 卡片之间通过留白、阴影、透明度区分层级，而不是重色块硬切。
4. 字体层级清楚，标题、说明、弱提示、数值必须一眼区分。
5. 交互反馈应温和，不要出现粗糙、跳变、工程感过强的反馈。

### 10.4 可用性要求

模块 1 的便捷性必须优先于炫技。

要求：

- 老师首次进入后 5 秒内知道从哪里开始。
- 默认主场景能直接开始讲。
- 常用操作不能埋得太深。
- 切换场景族后，参数区要跟着重组，而不是把所有参数永远铺开。
- 重要读数必须在右侧稳定位置，不随内容跳动过大。

### 10.5 舞台区要求

中央舞台区必须做到：

- 视觉上大、干净、可投影
- 光路、法线、角度有主次层级
- 图形不被过多说明文字遮挡
- 关键结论能通过轻量提示强调

不建议：

- 把太多说明文字塞进舞台中央
- 用过度花哨动画干扰课堂讲解
- 让控件侵占舞台可视面积

### 10.6 读数区要求

右侧读数区建议采用苹果风格的信息卡：

- 关键值大号显示
- 单位清楚
- 重要状态可用柔和高亮
- 低优先级说明使用更轻的文字层级

关键读数如：

- 折射角
- 临界角
- 侧移量
- 导光 / 漏光判定

应有明显的视觉突出，不应和普通说明文字混在一起。

### 10.7 本模块界面验收要求

界面设计验收时，除功能外还必须满足：

1. 第一眼看上去像“教学产品”，不是“工程 Demo”。
2. 老师能快速找到场景、参数、读数。
3. 中央视觉焦点明确。
4. 控件风格统一。
5. 页面整体具有苹果风格的精致感与秩序感。

## 十一、中央演示区要求

中央演示区必须优先展示课堂讲解真正需要指出的对象：

1. 光路主路径
2. 法线
3. 角度弧线与角度值
4. 介质边界
5. 全反射状态
6. 曲面几何关系
7. 光纤中的连续传播路径

以下表现要求必须满足：

- 光路方向清楚
- 界面边界清楚
- 法线与光线不混淆
- 全反射状态视觉上足够明显
- 曲面法线不应模糊成“只是一条辅助线”

## 十二、默认课堂脚本

建议老师进入本模块后的默认讲解路径如下：

1. 从界面折射族开始，讲清角度与法线定义。
2. 通过空气 / 玻璃、水 / 空气切换讲折射方向规律。
3. 逐步增加入射角，引出临界角与全反射。
4. 切换到平板介质族，讲双界面和平行出射。
5. 切换到半球介质族，讲球心入射和曲面法线。
6. 切换到球形介质族，加深曲面法线理解。
7. 最后进入光纤应用族，完成“规律 -> 应用”的教学闭环。

## 十三、主要教学风险点

本模块必须重点避免以下教学风险：

1. 把角度误画成相对界面的角度。
2. 让学生误以为任意从光密到光疏都会全反射。
3. 曲面法线不清，导致球心入射概念讲不清。
4. 玻璃砖中的平行出射表达不准确。
5. 光纤中“连续全反射”和“折射漏光”的分界表达不清。

## 十四、验收标准

### 14.1 物理正确性验收

- 折射方向与斯涅尔定律一致
- 反射角等于入射角
- 临界角数值与公式一致
- 超过临界角后折射光停止或明确消失
- 玻璃砖出射光与入射光平行
- 光纤模式中导光与漏光判定方向正确

### 14.2 教学可用性验收

- 默认场景打开即可讲
- 控件数量不失控
- 重点读数能快速定位
- 显示层能辅助而不是干扰讲解

### 14.3 模块边界验收

- 模块呈现为专题教学模块，而非通用编辑器
- 自由度始终受场景约束
- 新增场景不会破坏整体使用逻辑

## 十五、建议的开发冻结项

在开始实现或重构本模块前，建议先冻结以下项：

1. 是否确认按 5 个场景族组织模块。
2. 是否确认球形介质族与挖空半球纳入本模块深入范围。
3. 是否确认本模块坚持“专题教学模块”而非“光学沙盒”。
4. 是否确认第一批正式实现子场景与第二批深化子场景划分。
5. 是否确认全模块统一的显示开关与读数体系。
6. 是否确认材料预设范围。
7. 是否确认苹果风格的界面重构方向。

## 十六、结论

模块 1 不应被定义为“4 个折射实验”，而应被定义为：

**一个围绕折射、临界角、全反射、曲面法线与光纤应用展开的完整教学专题模块。**

它的价值不在于可任意拼装，而在于：

1. 能把基础规律讲准
2. 能把典型几何体讲清
3. 能把全反射讲到应用
4. 能让老师在课堂上做有限但高价值的变式

当前文档已经可以支持模块 1 开始进入开发准备，尤其适合先启动：

1. 模块结构重构
2. 第一批正式实现子场景
3. 苹果风格的界面与交互重构

后续如果继续细化本模块，建议下一轮进入“单子场景规格表”，逐个明确：

- 默认参数
- 必备控件
- 必备读数
- 必备公式
- 特殊交互
- 默认讲解脚本
- 明确不做项
