## M02 — 函数图形实验室

### Must Do（Bug 修复）—— 全部已修复

| # | 问题描述 | 类型 | 状态 | 解决方案 |
|---|---------|------|------|---------|
| M02-1 | `sinx` 等函数图象有时不生成，输入后画布无图象 | Bug | ✅ 已修复 | `expressionEngine.ts` 中的 `preprocessExpression()` 预处理器增加了隐式乘法规则，自动将 `sinx` 转换为 `sin(x)`、`2x` 转换为 `2*x` 等，共 6 条正则规则覆盖常见简写 |
| M02-2 | 录制功能 Bug：同时选择「循环」和「录制」时，会自动不停录制视频，无法暂停 | Bug | ✅ 已修复 | `useCanvasRecorder.ts` 实现了完整的录制生命周期管理：`startRecording()` / `stopRecording()` / `forceCleanup()` 三态控制，录制与动画循环解耦，循环结束时自动停止录制并导出 webm 文件 |
| M02-3 | 动画参数 c、d 无效果；参数 b 未出现在解析式中但图象却在变化（参数映射逻辑有误） | Bug | ✅ 已修复 | `AnimationControlPanel.tsx` 中参数映射已统一：变换参数严格对应 a / b / h / k（即 `y = a·f(b(x−h)) + k`），自定义系数（如二次函数的 a、b、c）通过 `namedParams` 独立管理，不再与变换参数混淆 |
| M02-4 | 函数图象突然消失，操作过程中图象无故从画布上消失 | Bug | ✅ 已修复 | 问题根源为表达式缓存失效与视口越界。已通过 LRU 缓存（50 条）稳定表达式求值、`sampler.ts` 增加不连续检测（断点处 `moveTo` 而非 `lineTo`）、以及视口变更时强制重绘静态层解决 |
| M02-5 | 函数识别有误，输入的函数表达式解析结果与预期不符 | Bug | ✅ 已修复 | `preprocessExpression()` 增加了完整的隐式乘法和括号补全逻辑，支持 `2sinx`、`xlnx`、`e^x` 等自然书写方式；同时 `coefficientDetector.ts` 通过 AST SymbolNode 遍历准确识别用户自定义参数 |

> 本模块老师反馈均为 Bug，无额外需求建议。

## M03 — 解析几何画板

### Must Do（Bug 修复）—— 已修复

| # | 问题描述 | 类型 | 状态 | 解决方案 |
|---|---------|------|------|---------|
| M03-1 | 含参数直线无法识别，输入带参数的直线方程后系统不响应 | Bug | ✅ 已修复 | `LineParamPanel.tsx` + `lineExpressionEngine.ts` 实现了完整的直线方程解析，支持 `x = 3`（竖直线）、`y = kx + b`（斜截式）、`y = 常数`（水平线）等格式，系数通过 `detectLineCoefficients()` 自动识别为可调参数 |

### Should Do（体验优化）—— 全部已实现

| # | 问题描述 | 类型 | 状态 | 实现方案 |
|---|---------|------|------|---------|
| M03-2 | 增加自定义曲线功能，允许老师自行输入曲线方程 | 功能 | ✅ 已实现 | 新增 `ImplicitCurveEntity` 实体类型，教师可输入任意隐式方程 `F(x,y) = 0`；后端使用 Marching Squares 算法实时渲染，支持自定义参数滑块调节；配套 `ImplicitCurveParamPanel` 面板 |
| M03-3 | 增加动点功能，支持点在曲线上运动并展示轨迹 | 功能 | ✅ 已实现 | 新增 `MovablePointEntity` 实体类型，可将点约束在任意曲线上拖动；支持轨迹记录与显示、参数化运动动画（含缓动函数 + 循环模式）；配套 `MovablePointAnimPanel` 面板和 `MovablePointDragTool` 交互工具 |
| M03-4 | 抛物线离心率动图表意不清，老师看不懂想表达什么 | 体验 | ✅ 已优化 | `EccentricityPanel` 重新设计：增加离心率区间指示条（椭圆区 / 抛物线标记线 / 双曲线区），当前值以彩色指示点标注；实时显示曲线类型文字标签（椭圆 / 抛物线 / 双曲线）；Play 按钮支持 e 从 0.1→1.9 的 2 秒连续动画 |
