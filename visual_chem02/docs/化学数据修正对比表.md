## 三、HIGH — 键线式 H-杂原子键悬空

### 问题描述
键线式模式下，H 原子仅在父原子键数 ≥ 4 时才合并（如 CH₄ → "CH₄"）。对于 O-H、N-H、S-H 等杂原子键，父原子键数 < 4，H 不被合并但标签被隐藏，导致**可见键线连到空处**。

### 影响分子 (15+)
| 分子 | 分子式 | 悬空键 |
|------|--------|--------|
| MOL-012 | H₂O | 2 条 O-H |
| MOL-016 | H₂S | 2 条 S-H |
| MOL-019 | NH₃ | 3 条 N-H |
| MOL-042 | OH⁻ | 1 条 O-H |
| MOL-058 | CH₃OH | 1 条 O-H |
| MOL-059 | C₂H₅OH | 1 条 O-H |
| MOL-062 | HCOOH | 1 条 O-H |
| MOL-063 | CH₃COOH | 1 条 O-H |
| MOL-065 | CH₃NH₂ | 2 条 N-H |
| MOL-066 | C₆H₅NH₂ | 2 条 N-H |
| MOL-068 | CO(NH₂)₂ | 4 条 N-H |
| MOL-071 | C₆H₁₂O₆ | 多条 O-H |
| ... | 其他含 O-H/N-H/S-H 分子 | |

### 修正前
```typescript
// projection2d.ts — 仅两种模式
buildProjected2DFromPositions(positions, model, noMerge: boolean)
// noMerge=false 时：仅 bondCounts[parent] >= 4 才合并 H
// 键线式和结构简式共用同一合并逻辑
```
结果：ethanol 键线式显示 `—O—` 后面一条悬空线（H 不可见）

### 修正后
```typescript
// projection2d.ts — 三种合并模式
buildProjected2DFromPositions(positions, model, mergeMode: 'default' | 'all' | 'none')
// 'none'  = 电子式（不合并任何 H）
// 'default' = 结构简式（仅 ≥4 键合并，保留 O-H 可见）
// 'all'   = 键线式（合并所有 H 到父标签）

// Formula2DView.tsx
const mergeMode = displayMode === 'electron-formula' ? 'none'
  : displayMode === 'skeletal' ? 'all'
  : 'default';
```
结果：ethanol 键线式显示 `—OH`（H 合并进 O 标签）

### 修改文件
| 文件 | 变更 |
|------|------|
| `src/engine/projection2d.ts` | `noMerge: boolean` → `mergeMode: 'default'\|'all'\|'none'`; 合并条件按模式分支 |
| `src/components/formula/Formula2DView.tsx` | 按 displayMode 传入对应 mergeMode |

## 四、HIGH — 离子孤电子对缺失

### 问题描述
孤电子对公式 `lp = floor((ve − ΣbondOrders) / 2)` 是逐原子计算，不考虑分子整体电荷。对于阴离子，额外电子未被分配到终端原子，导致孤电子对偏少。

### 影响分子 (8 个离子)

| 离子 | 原子 | 修正前 lp | 修正后 lp | 说明 |
|------|------|----------|----------|------|
| CO₃²⁻ (MOL-036) | O (单键×2) | 各 2 | 各 3 | −2 电荷 → 2 个多余 e⁻ 对 |
| SO₄²⁻ (MOL-037) | O (单键×2) | 各 2 | 各 3 | −2 电荷 → 2 个多余 e⁻ 对 |
| NO₃⁻ (MOL-038) | O (单键×2) | 各 2 | 各 3 | −1 电荷 → 1 个多余 e⁻ 对（分配给 O⁻） |
| ClO₄⁻ (MOL-039) | O (单键×2) | 各 2 | 各 3 | −1 电荷 |
| PO₄³⁻ (MOL-040) | O (单键×3) | 各 2 | 各 3 | −3 电荷 → 3 个多余 e⁻ 对 |
| MnO₄⁻ (MOL-041) | O (单键×2) | 各 2 | 各 3 | −1 电荷 |
| OH⁻ (MOL-042) | O | 2 | 3 | −1 电荷 |
| NH₄⁺ (MOL-030) | (全部) | 正确 | 正确 | +1 电荷不影响（N 的 lp 已为 0） |

### 修正前
```typescript
// moleculeBuilder.ts / legacyBuilder.ts
function computeAllLonePairs(atoms, bonds, positions) {
  // 仅逐原子公式：lp = floor((ve - sumBO) / 2)
  // 不考虑分子电荷 → 阴离子缺少孤电子对
}
```

### 修正后
```typescript
function computeAllLonePairs(atoms, bonds, positions, charge = 0) {
  // 第一步：逐原子初始计算
  perAtomLp[i] = floor((ve - sumBO) / 2)

  // 第二步：分子级修正
  adjustedVE = Σve - charge              // 阴离子加电子
  targetLpCount = floor((adjustedVE - 2×ΣbondOrders) / 2)
  deficit = targetLpCount - Σ(perAtomLp)

  // 第三步：差额分配（键级低的终端原子优先，不超过八隅体限制）
  candidates.sort(by bondOrders ascending)
  for each candidate: give min(deficit, octet_room)
}
```

### 修改文件
| 文件 | 变更 |
|------|------|
| `src/data/moleculeMetadata.ts` | 8 个离子新增 `charge` 字段 |
| `src/data/molecules.ts` | `MoleculeData` 接口新增 `charge?: number` |
| `src/engine/moleculeBuilder.ts` | `computeAllLonePairs` 增加 charge 参数 + 分子级修正算法 |
| `src/engine/legacyBuilder.ts` | 同上 |

## 五、HIGH — CO 碳上孤电子对缺失

### 问题描述
CO (MOL-009) 的 SDF 使用三键(order=3)。逐原子公式:
- C: `floor((4-3)/2) = 0` → 缺失 1 个孤电子对
- O: `floor((6-3)/2) = 1` → 正确

实际 Lewis 结构 :C≡O: 中 C 和 O 各有 1 个孤电子对。

### 修正
分子级修正自动处理：
```
totalVE = 4+6 = 10, charge = 0
bondElectrons = 3×2 = 6
targetLp = (10-6)/2 = 2
currentSum = 0+1 = 1
deficit = 1 → 分配给 C（C 的 bondOrders=3, maxLp=4-3=1, room=1）
```
修正后：C=1 lp, O=1 lp ✓

## 六、MEDIUM — 纯碳氢化合物键线式全标签

### 问题描述
当分子没有杂原子(全是 C/H)时，`hasHeteroatom=false` 导致 `shouldShowLabel` 对所有原子返回 true，键线式退化为结构简式（所有原子标签都显示）。

### 影响分子
烷烃(6)、烯烃(4)、炔烃(2)、芳烃(3)、部分小分子 = 约 20 个纯碳氢化合物

### 修正前
```typescript
const shouldShowLabel = (el) => {
  if (el === 'C') return !hasHeteroatom; // 无杂原子 → C 显示 → 全标签
  return el !== 'H';
};
```

### 修正后
```typescript
const shouldShowLabel = (el) => {
  if (el === 'H') return false;           // H 已合并
  if (el !== 'C') return true;            // 杂原子显示
  if (!hasHeteroatom && heavyAtomCount <= 2) return true;  // CH₄ 等太小的回退
  return false;                            // C 隐藏（折点隐含）
};
```

小分子（≤2 重原子如 CH₄、C₂H₂）仍显示标签（否则视觉信息不足）。较长链（C₃H₈+）正确显示为碳骨架折线。

## 七、未修改的已知限制

| 限制 | 说明 | 原因 |
|------|------|------|
| NO₂ 单电子 | NO₂ 是自由基，有 1 个未配对电子，当前模型无法表示 | 孤电子对模型只支持成对电子 |
| NO 单电子 | NO 有 11 个价电子（奇数），1 个未配对电子 | 同上 |
| 苯环离域键 | 电子式中离域键显示为 1 对共用电子（非 1.5） | 教学简化，离域键电子对数取整 |
