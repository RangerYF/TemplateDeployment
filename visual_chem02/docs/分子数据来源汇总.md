## 2D 数据与 3D 数据的区别

| 比较项 | 3D SDF 数据 | 2D SDF 数据 |
|--------|------------|------------|
| **坐标维度** | x, y, z 三维空间坐标（Å） | x, y 平面坐标，z 恒为 0 |
| **键角** | 反映真实键角（如 H₂O 104.5°） | 仅画图美观角度，无物理意义 |
| **键长** | 可计算真实键长（pm） | 距离为画图距离，无物理意义 |
| **球棍模型** | 空间结构正确（四面体、三角锥等） | 所有原子压在同一平面，空间结构失真 |
| **VSEPR 几何** | 四面体、三角双锥等自然呈现 | 无法体现 |
| **2D 化学式** | 影响不大（有独立的拓扑布局引擎） | 影响不大（有独立的拓扑布局引擎） |

**结论**：2D 数据的分子在 3D 模式（球棍/空间填充/电子云）下会被压扁为平面结构，键长和键角测量不可用。2D 化学式（结构简式/电子式/键线式）不受影响。

---

## 数据来源分类

- **PubChem 3D**：从 PubChem REST API 获取的 3D conformer SDF（`record_type=3d`），坐标来自 MMFF94 力场优化
- **PubChem 2D**：PubChem 仅提供 2D 拓扑坐标（常见于离子、配合物、晶体结构等），3D 模式下标注"无 3D 构型"提示
- **手工构建**：基于 VSEPR 理论手工编写 3D 坐标（用于 PubChem 无 3D 数据的简单分子/离子）
