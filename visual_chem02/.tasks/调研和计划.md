## 1. 需求摘要

### 1.1 产品定位
面向中学化学教师的 **3D 分子结构查看器**（Viewer，非 Editor），核心价值：**3 秒内调出任意中学化学分子的 3D 模型，配合精确键长键角数据和 VSEPR 构型可视化。**

### 1.2 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 分子检索与输入 | 化学式、中文名、英文名三种搜索方式 |
| P0 | 3D 球棍模型渲染 | CPK 配色，原子球体 + 化学键圆柱体 |
| P0 | 3D 交互操作 | 鼠标拖拽旋转、滚轮缩放、右键平移 |
| P0 | 键长键角标注 | 点选显示键长/键角，附带偏移注释 |
| P1 | 显示模式切换 | 球棍模型 / 空间填充模型 |
| P1 | 分子对比 | 左右分栏 3D 展示 + 下方参数对比表格 |
| P1 | VSEPR 构型可视化 | 孤电子对显示、电子域构型与分子构型联动 |
| P1 | 命名互译面板 | IUPAC 名、中文名、化学式同时显示 |
| P2 | 电子云模型 | 分子轨道等值面渲染（本期不实现） |

### 1.3 数据规模
- **71 个分子**：42 无机 + 29 有机
- **19 种元素**：H, C, N, O, F, Cl, Br, S, P, Si, B, Na, K, Fe, Cu, Zn, Al, Mg, Ca
- **13 种 VSEPR 构型**：从直线形到正八面体
- **6 种键类型**：单键、双键、三键、离域键、配位键、氢键

## 2. 技术方案对比

### 2.1 3D 渲染方案

| 方案 | 说明 | 优点 | 缺点 | 结论 |
|------|------|------|------|------|
| **A: React Three Fiber (R3F)** | React 封装的 Three.js | 与 React 生态无缝集成；声明式 API；组件化渲染 | 略有性能开销 | **推荐 ✅** |
| B: 原生 Three.js | 直接使用 Three.js API | 最大性能；最大灵活性 | 命令式 API 与 React 集成繁琐 | 不推荐 |
| C: 3Dmol.js | 专用分子渲染库 | 内置分子渲染功能 | 教学定制困难；与现有技术栈不一致 | 不推荐 |

**决策：采用 R3F**，与 visual_template 技术栈一致，便于跨模块复用。

### 2.2 分子坐标计算方案

| 方案 | 说明 | 适用范围 | 优点 | 缺点 |
|------|------|----------|------|------|
| **A: VSEPR 模板放置** | 预定义 13 种 VSEPR 几何模板，按键长缩放 | 所有无机分子 | 简单可靠；精确匹配数据 | 不适用链状有机分子 |
| **B: 递推链式构建** | 从中心原子出发，按键长+键角+二面角递推 | 有机链状分子 | 可处理任意链长 | 实现复杂 |
| **C: Z-matrix 转换** | 标准计算化学方法 | 通用 | 学术标准 | 需要完整 Z-matrix 数据（当前数据不含） |
| **D: 手动预置坐标** | 每个分子手动定义 3D 坐标 | 所有分子 | 最精确 | 维护成本高 |

**决策：混合方案 A+B**
- **无机分子（MOL-001 ~ MOL-042）**：VSEPR 模板放置。根据 bond_pairs + lone_pairs 确定几何模板，按 bond_length 缩放。
- **有机分子（MOL-043 ~ MOL-071）**：递推链式构建。从第一个碳出发，根据杂化类型确定键角（sp³=109.5°, sp²=120°, sp=180°），递推放置原子。
- 特殊分子（苯环、萘）：使用预定义正六边形/双六边形模板。

### 2.3 原子渲染方案

| 方案 | 说明 | 优点 | 缺点 | 结论 |
|------|------|------|------|------|
| **A: 独立 SphereGeometry** | 每个原子一个 Mesh | 简单直观 | 70+ 原子时 draw call 多 | 适用（分子最大 ~30 原子） |
| B: InstancedMesh | 同元素原子实例化渲染 | 高性能 | 实现复杂 | 过度优化 |

**决策：方案 A**。中学化学分子最大约 30 个原子，无需 InstancedMesh 优化。

### 2.4 状态管理方案

| 方案 | 说明 | 结论 |
|------|------|------|
| **Zustand** | 与 visual_template 一致 | **推荐 ✅** |
| Context API | React 内置 | 不够灵活 |
| Redux | 重量级 | 过度 |

**决策：Zustand**，与 visual_template 保持一致。

### 2.5 与推荐技术路径的对照

chem-summary.md 推荐的架构为 **chem-3d-viewer monorepo**（core-3d + molecule-viewer + crystal-viewer）。本项目 visual_chem02 **独立实现 molecule-viewer 部分**，但架构设计时考虑未来可抽取的共享层：

| 推荐共享层 | 本项目实现方式 | 可复用性 |
|-----------|---------------|----------|
| Three.js 场景初始化 | MoleculeScene 组件 | 可抽取为 core-3d |
| OrbitControls 相机控制 | @react-three/drei OrbitControls | 直接复用 |
| 原子球体渲染 | AtomRenderer 组件 | 可抽取 |
| 化学键圆柱渲染 | BondRenderer 组件 | 可抽取 |
| CPK 配色系统 | elements.ts 数据 | 可抽取 |
| Raycasting 拾取 | R3F 内置 onPointerOver/onClick | 直接复用 |
| 显示模式切换 | uiStore.displayMode | 可抽取 |
| 文本标注 | @react-three/drei Html | 直接复用 |

### 3.4 关键数据结构

```typescript
// 分子原始数据
interface MoleculeData {
  id: string;           // MOL-XXX
  name_cn: string;      // 中文名
  name_en: string;      // 英文名
  formula: string;      // 分子式
  level: string;        // 学段
  category: string;     // 分类
  vsepr?: string;       // VSEPR 构型
  geometry?: string;    // 分子空间构型
  central_atom?: string;
  bond_pairs?: number;
  lone_pairs?: number;
  bond_lengths: Record<string, number>;  // 键长 pm
  bond_angles?: Record<string, number>;  // 键角 °
  polarity?: string;
  hybridization?: string;
  functional_group?: string;
  features?: string;
  atoms: AtomDef[];     // 原子定义
  bonds: BondDef[];     // 键定义
}

// 3D 构建结果
interface MoleculeModel {
  atoms: Atom3D[];
  bonds: Bond3D[];
  lonePairs: LonePair3D[];
  center: [number, number, number];  // 几何中心
  radius: number;                     // 包围球半径
}

interface Atom3D {
  index: number;
  element: string;
  position: [number, number, number];
  label?: string;
  radius: number;       // 渲染半径
  color: string;         // CPK 颜色
}

interface Bond3D {
  from: number;          // 原子索引
  to: number;
  order: number;         // 1=单键, 2=双键, 3=三键
  length: number;        // 键长 pm
  type: 'single'|'double'|'triple'|'delocalized'|'coordinate';
}
```

## 5. 与 C-05 晶体查看器的复用考虑

以下组件在设计时已考虑未来抽取为共享 core-3d 包：
- `AtomRenderer` — 通用原子球体渲染（CPK 配色 + 半径映射）
- `BondRenderer` — 通用化学键渲染（单/双/三键）
- `elements.ts` — 元素属性数据（19+ 种元素）
- `MoleculeScene` — 场景初始化 + OrbitControls + 灯光
- 设计令牌系统 — 完全共享

C-05 独有而 C-02 不涉及：分数坐标转换、晶胞扩展、配位多面体、堆积动画。

## 6. 风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 有机分子 3D 坐标计算复杂 | 高 | 采用递推链式构建 + 特殊分子预定义模板 |
| 双键/三键渲染方向 | 中 | 计算垂直于键轴且在分子平面内的偏移方向 |
| 苯环离域键渲染 | 中 | 使用 TorusGeometry 半透明环面 |
| 大有机分子（正戊烷等）键角累积误差 | 低 | 使用交错构象默认二面角 180° |
