# Checkpoint Fast

用于低成本但不断链的收尾。

## 适用场景
- 子任务刚做完，需要让后续子任务接得上
- 本轮没有阶段切换
- 本轮没有稳定知识要沉淀
- 想显式避免 `update-knowledge`

## 执行流程

### Step 1：提取关键用户输入

仅提取：
- `[决策]`
- `[Bug]`
- `[调整]`
- `[反馈]`
- `[设计]`

忽略：
- 确认型提问
- 操作指令
- 进度询问
- 对 AI 工作方式的纠正

### Step 2：写入 `.knowledge/user-inputs.md`

若 Step 1 非空，追加写入；为空则跳过。

### Step 3：同步任务文档链

按需更新：
- 当前子任务文档
- 父级阶段文档
- 总计划文档
- 本轮直接相关且确有变化的文档

### Step 3.1：补后续承接信息

若本轮对后续子任务有影响，至少记录命中项：
- 新约束
- 依赖变更
- 必看文件
- 复用结论
- 未决风险

优先写在当前子任务文档，再在阶段文档和总计划文档中简短回填。

### Step 4：输出摘要

```
## Checkpoint 完成
- mode: fast
- user-inputs.md: +{N} 条输入归档
- 任务文档：{更新了哪些文档}
- update-knowledge: 跳过 — fast 模式固定跳过，仅保留文档链连续性
```
