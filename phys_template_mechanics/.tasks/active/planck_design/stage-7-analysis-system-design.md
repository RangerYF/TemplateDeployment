# 阶段7：分析系统设计方案

> 状态：设计中 | 创建：2026-03-26

## 一、设计目标

在现有受力分析视角基础上，扩展为完整的物理分析系统，覆盖：
- **框选与多选**：框选多个物体 + Shift+点击追加/移除
- **分析组/子系统**：多选物体后创建分析组，作为图表数据源
- **运动分析**（P-02）：速度/加速度矢量 + v-t/a-t/x-t 图表 + 轨迹
- **能量分析**（P-14）：Ek/Ep/Ep弹/E总 柱状图 + E-t 曲线
- **动量分析**（P-12）：动量矢量 + 碰撞前后对比 + 系统总动量

支持单体分析和系统（多体）分析。

---

## 二、核心交互模型

### 2.1 分析对象：数据源选择器

**核心概念**：单个物体和子系统是**平级的数据源**，用户自由勾选组合。

```
数据源选择器（底部图表面板内）
┌─────────────────────────────────┐
│ 数据源：                         │
│ ☑ 物块A（红）                    │
│ ☐ 球体B（蓝）                    │
│ ☑ [系统1: A+B]（灰）            │
│ ☐ 物块C（绿）                    │
└─────────────────────────────────┘
```

- 勾选"物块A" → 图表显示物块A的曲线（红色）
- 勾选"系统1" → 图表显示系统汇总曲线（灰色）
- 同时勾选多个 → 同一张图上多条曲线，不同颜色区分

**颜色分配**：每个物体/系统自动分配颜色（可能复用物体本身的渲染色或自动分配图表色板）。

### 2.2 子系统的创建与管理

**创建流程**：
1. 多选物体（Shift+点击，或未来框选）
2. 工具栏/右键菜单出现"创建分析组"按钮
3. 点击后创建子系统，自动命名（可修改）
4. 子系统出现在数据源选择器中

**数据模型**：
```typescript
interface AnalysisGroup {
  id: string;
  name: string;           // 用户可修改，默认"系统1"
  bodyIds: string[];       // 包含的物体 ID 列表
  color: string;           // 图表中的显示颜色
}
```

**管理**：
- 在数据源选择器中可以删除/编辑子系统
- 子系统中的物体被删除时，自动从子系统中移除
- 子系统可以包含 2-N 个物体（实际教学中通常 2-3 个）

### 2.3 框选与多选

**前置能力**：框选是创建分析组的基础交互，在阶段7中实现。

#### 框选交互

在 SelectTool 下：
- **左键拖拽空白区域** → 绘制蓝色半透明矩形框
- 松手时，框内物体全部选中（多选状态）
- **选中判定**：物体 AABB 与框选矩形有交集即选中

#### 多选交互

- **Shift+点击物体** → 追加到选中集合（已选中则移除，toggle）
- **点击空白区域**（非拖拽） → 取消全部选中
- **点击单个物体**（不按 Shift） → 仅选中该物体（取消其他）

#### 多选状态下的行为

| 功能 | 多选时行为 |
|------|-----------|
| 属性面板 | 显示共有属性（如质量），值不同时显示"—" |
| 拖拽移动 | 整体移动所有选中物体（保持相对位置） |
| Delete 键 | 删除所有选中物体 |
| 创建分析组 | 工具栏出现"创建分析组"按钮 |

#### 选中状态的视觉反馈

- 单选：物体显示选中高亮边框（已有）
- 多选：所有选中物体显示选中高亮边框
- 框选过程：蓝色半透明矩形 + 边框虚线

#### selectionStore 扩展

```typescript
// 现有：单选
selected: SelectableObject | null;

// 扩展为：多选
selected: SelectableObject[];  // 数组，支持 0~N 个选中对象
```

### 2.4 视角叠加（Checkbox 控制）

多个分析视角可同时开启，不互斥。在工具栏或侧边提供 checkbox：

```
分析视角：
☑ 受力分析（已有）
☑ 运动分析
☐ 能量分析
☐ 动量分析
```

每个视角控制两件事：
1. **画布叠加层**：在物体上渲染对应矢量（力箭头、速度箭头等）
2. **底部图表面板**：显示对应类型的图表标签页

## 三、底部图表面板

### 3.1 整体布局

```
┌──────┬──────────────────────────────────────────┬────────┐
│ 物体 │              Canvas 画布                  │ 属性   │
│ 面板 │                                          │ 面板   │
│      │                                          │        │
│      │                                          │        │
│      ├─────┬──────────────────────────┬─────────┤        │
│      │数据 │  [v-t] [a-t] [x-t]      │  ▼ 折叠 │        │
│      │源   │  ~~~~~~~~~~~~~~~~~~~~~~~~│         │        │
│      │选择 │   图表渲染区域            │         │        │
│      │器   │                          │         │        │
└──────┴─────┴──────────────────────────┴─────────┘
```

- 面板横跨画布底部（左侧物体面板和右侧属性面板之间）
- 可折叠/展开（折叠时只显示一行标签栏）
- 展开高度约 200-250px，可拖拽调整

### 3.2 面板内部结构

**左侧：数据源选择器**（~120px 宽）
- 列出所有物体 + 所有已创建的子系统
- 每项前有 checkbox + 颜色标记
- 滚动列表（物体多时）

**中间：图表区域**
- 标签页切换不同图表类型
- 当前激活的分析视角决定可用的标签页
- 图表内容根据勾选的数据源显示对应曲线

**标签页映射**：

| 分析视角 | 可用标签页 |
|---------|-----------|
| 运动分析 | v-t、a-t、x-t（速度/加速度/位移 vs 时间） |
| 能量分析 | E-t（能量 vs 时间）、E柱状（各能量分量柱状图） |
| 动量分析 | p-t（动量 vs 时间）、p柱状（碰撞前后对比柱状图） |

多个视角同时开启时，标签页合并显示（如同时开运动+能量 → v-t、a-t、x-t、E-t、E柱状）。

### 3.3 图表交互

- **实时更新**：仿真运行时图表实时绘制
- **暂停后回看**：仿真暂停后可拖拽时间轴回看任意时刻
- **时间指示线**：图表上有竖线标记当前仿真时刻，画布上物体状态同步
- **鼠标悬停**：悬停图表某点显示精确数值（tooltip）
- **自动缩放**：Y 轴根据数据范围自动缩放

### 3.4 子系统在图表中的数据含义

| 图表类型 | 单个物体显示 | 子系统显示 |
|---------|-------------|-----------|
| v-t | 该物体速度 | 质心速度 v_cm = Σ(mi·vi) / Σmi |
| a-t | 该物体加速度 | 质心加速度 |
| x-t | 该物体位移 | 质心位移 |
| E-t | 该物体 Ek+Ep | 系统 ΣEk + ΣEp（各分量可展开） |
| E柱状 | 该物体的 Ek、Ep_重、Ep_弹 | 系统各能量分量求和 |
| p-t | 该物体动量 mv | 系统总动量 Σ(mi·vi) |
| p柱状 | 该物体碰前/碰后动量 | 系统碰前/碰后总动量 |

## 四、画布叠加层

### 4.1 设计决策：运动/能量/动量不渲染矢量箭头

经分析，**画布上只保留受力分析的力箭头**（已有），运动/能量/动量视角不在画布上渲染额外矢量。

**理由**：
- **加速度箭头** = 合力方向 / 质量，与力箭头信息重复
- **动量箭头** = 质量 × 速度，方向与速度完全相同，无额外信息
- **速度箭头** — 直线运动中方向固定意义不大；抛体/圆周中有一定价值，但图表更能表达速度随时间的变化规律
- 去掉这些箭头后，多视角叠加不会造成视觉混乱

**结论**：运动/能量/动量的核心教学价值通过**底部图表面板**实现，画布保持简洁。

| 视角 | 画布渲染 | 图表面板 |
|------|---------|---------|
| 受力分析（已有） | 力箭头 + 标签 | —（受力列表在属性面板） |
| 运动分析 | 轨迹线（可选） | v-t、a-t、x-t |
| 能量分析 | 无 | E-t 曲线、E柱状图 |
| 动量分析 | 无 | p-t 曲线、p柱状图 |

### 4.2 轨迹渲染（运动分析视角专属）

- 物体运动轨迹线：等间隔点阵（每 N 帧记一个点）
- 颜色同物体颜色，透明度渐变（越早越淡）
- 可在运动分析视角的设置中开关

## 五、数据记录框架

### 5.1 架构设计

```
PhysicsBridge.step()
    │
    ├── 每帧记录 ──→ AnalysisRecorder
    │                  ├── 位置 (x, y)
    │                  ├── 速度 (vx, vy, |v|)
    │                  ├── 加速度 (ax, ay, |a|) ← 速度差分
    │                  ├── 力（已有 ForceCollector）
    │                  ├── 动量 (px, py, |p|) ← m × v
    │                  ├── 动能 Ek ← ½mv²
    │                  ├── 重力势能 Ep ← mgy（相对地面）
    │                  ├── 弹性势能 Ep_spring ← 从弹簧 Joint 获取
    │                  └── 时间戳 t
    │
    └── 子系统汇总 ──→ AnalysisRecorder（系统级）
                       ├── 质心位置/速度/加速度
                       ├── 系统总动量
                       ├── 系统总 Ek、总 Ep
                       └── 系统总机械能
```

### 5.2 数据存储

```typescript
interface FrameRecord {
  t: number;                    // 仿真时间（秒）
  bodies: Map<string, BodyFrameData>;
  groups: Map<string, GroupFrameData>;
}

interface BodyFrameData {
  x: number; y: number;        // 位置
  vx: number; vy: number;      // 速度
  ax: number; ay: number;      // 加速度（差分计算）
  px: number; py: number;      // 动量 = m * v
  ek: number;                  // 动能 = ½mv²
  epGravity: number;           // 重力势能 = mgy
  epSpring: number;            // 弹性势能（如果连接了弹簧）
}

interface GroupFrameData {
  // 质心
  cmX: number; cmY: number;
  cmVx: number; cmVy: number;
  cmAx: number; cmAy: number;
  // 系统汇总
  totalPx: number; totalPy: number;   // 系统总动量
  totalEk: number;                     // 系统总动能
  totalEpGravity: number;              // 系统总重力势能
  totalEpSpring: number;               // 系统总弹性势能
  totalE: number;                      // 系统总机械能
}
```

### 5.3 采样与内存策略

- 仿真步进 60fps，但图表记录可降采样（如每 2 帧记 1 次 = 30fps）
- 环形缓冲区，最多保留最近 N 秒数据（如 60 秒 × 30 = 1800 帧）
- 超过上限时丢弃最早的数据
- 暂停仿真时停止记录，恢复时继续

## 六、各分析视角详细设计

### 6.1 运动分析视角

**画布叠加**：
- 轨迹点阵（可选）：物体颜色，透明度渐变
- 无速度/加速度矢量箭头（图表已充分表达）

**图表标签页**：
- v-t：纵轴=速度大小（标量），横轴=时间
- a-t：纵轴=加速度大小，横轴=时间
- x-t：纵轴=位移（相对初始位置），横轴=时间

**教学价值**：
- 自由落体 → v-t 直线（验证匀加速）
- 斜面滑动 → a-t 水平线（匀加速）
- 单摆 → v-t 正弦波

### 6.2 能量分析视角

**画布叠加**：
- 无矢量渲染（能量是标量，不适合画箭头）
- 可选：在物体旁显示小型 mini 能量条（Ek|Ep）

**图表标签页**：
- E-t（时间序列）：多条曲线 — Ek（红）、Ep_重（蓝）、Ep_弹（绿）、E总（灰虚线）
  - 有摩擦时 E总 递减，教师可直观看到能量耗散
  - 子系统勾选时显示系统各能量分量
- E柱状（关键时刻对比）：
  - 横轴=关键时刻（如"初始"、"碰撞时"、"最终"），纵轴=各能量分量堆叠柱状
  - 或实时柱状：显示当前时刻各能量分量的高度

**颜色方案**（参考 PhET + 需求文档）：
- Ek 动能 → 红色 #F44336
- Ep 重力势能 → 蓝色 #2196F3
- Ep 弹性势能 → 绿色 #4CAF50
- E总 → 灰色 #9E9E9E
- Q 摩擦热 → 橙色 #FF9800

### 6.3 动量分析视角

**画布叠加**：
- 无矢量渲染（动量方向=速度方向，与速度箭头信息重复，图表已充分表达）

**图表标签页**：
- p-t（时间序列）：纵轴=动量（可选标量|p|或分量px/py），横轴=时间
  - 碰撞场景：两条曲线在碰撞时刻突变，但系统总动量曲线保持水平
- p柱状（碰撞对比）：
  - 碰前/碰后 各物体动量 + 系统总动量
  - 直观验证 Σp_before = Σp_after

**碰撞事件检测**：
- 利用已有的 post-solve 事件自动标记碰撞时刻
- 在图表上用竖线标注碰撞发生的时间点
- 柱状图自动捕获碰撞前后的数据快照

## 七、视角控制 UI

### 7.1 画布底部区域（地面以下）

视角 checkbox 放在画布内部、地面线以下的空白区域（该区域目前无内容）：

```
┌──────┬──────────────────────────────────────┬────────┐
│ 物体 │            Canvas 画布               │ 属性   │
│ 面板 │     ┌───┐  ○                         │ 面板   │
│      │     │物块│  球    (地面以上)           │        │
│      │─────┴───┴──○────────────────地面线───│        │
│      │                                      │        │
│      │  ☑受力  ☐运动  ☐能量  ☐动量          │        │
│      │          (地面以下空白区)              │        │
│      ├──────────────────────────────────────┤        │
│      │  [数据源] │ [v-t][a-t][x-t] 图表区   │        │
│      │          │  ~~~~~~~~~~~~~~~~~~       │        │
└──────┴──────────────────────────────────────┴────────┘
```

- checkbox 渲染在 Canvas 内部（地面以下固定位置，不随画布平移/缩放）
- 每个 checkbox 独立控制
- 勾选运动/能量/动量中任一项时，底部图表面板自动展开
- 受力分析 checkbox 仅控制力箭头显隐（不影响图表面板）
- 全部取消勾选（运动/能量/动量）时，底部面板自动折叠

### 7.2 底部面板标签页联动

勾选的视角决定底部面板可用的标签页。例如：
- 只勾选"运动" → 标签页：v-t | a-t | x-t
- 勾选"运动"+"能量" → 标签页：v-t | a-t | x-t | E-t | E柱状
- 全勾选 → 全部标签页可用
