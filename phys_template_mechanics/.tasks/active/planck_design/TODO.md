## 约束系统增强

### 完整滑轮组（多轮多绳段）
- **来源**：阶段5讨论（2026-03-25）
- **描述**：支持定滑轮组 + 动滑轮组组合，多段绳在多个滑轮间穿绕，动滑轮随重物上下移动
- **挑战**：
  - 物理层：需多个 PulleyJoint 级联，或自定义约束求解
  - 渲染层：绳路由算法（计算绳在多个轮间的穿绕路径）
  - 交互层：教师逐步选择多个滑轮和连接点的 UI 流程
  - 动滑轮随动：动滑轮 body 需为 dynamic，随绳长变化上下移动
- **前置条件**：阶段5单滑轮（PulleyJoint）完成
- **预估工作量**：独立阶段，3-5天
- **优先级**：中（初高中物理常考，但 MVP 已有单滑轮覆盖基础场景）

### 绳碰撞（链式模拟）
- **来源**：5.2 绳约束用户反馈（2026-03-25）
- **描述**：当前 RopeJoint 只是两端距离约束，绳线不参与碰撞检测。绳应能搭在斜面角上而非穿透过去。
- **实现方案**：链式模拟——将绳分成多段小刚体用 RevoluteJoint 串联，每段有碰撞体积
- **挑战**：
  - 性能：段数越多越真实但计算量增大
  - 创建/销毁：需动态管理多段 body + joint
  - 渲染：从简单贝塞尔改为多段折线渲染
  - 编辑态锚点/属性面板：需适配链式结构
- **决策**：暂不实现，优先推进其他约束类型（杆、弹簧、滑轮）
- **优先级**：低（教学演示场景中绳穿透可接受，真实绳模拟复杂度高）

### 锚点位置优化（表面连接）
- **来源**：5.2 绳约束用户反馈（2026-03-25）
- **现状**：锚点 = 物体中心（localAnchor = {0,0}）
- **问题**：绳从物体中心连出，不够真实（实际应从表面连接）
- **已探索方案**：
  - "朝向对方的表面点"：适用大多数场景，但圆形物体旋转时锚点跟着晃动
  - "固定顶部"：仅适合悬挂场景，不适合斜面+滑轮等水平连接
  - "混合策略"（圆形用中心，矩形用表面）：物理正确但实现复杂
- **决策**：暂保持中心锚点，等教师用户反馈后再确定最佳策略
- **优先级**：低（待用户反馈驱动）

### 设备模拟/触摸环境拖拽创建一致性
- **来源**：阶段 8.6.5 用户反馈（2026-04-01）
- **现状**：在浏览器 DevTools 设备工具栏（触摸语义）下，左侧物体库原生 `drag & drop` 不稳定；当前已有“点击创建”兜底可用。
- **问题**：桌面模式支持拖拽跟手预览，触摸/设备模拟仅点击创建，交互不一致。
- **建议方案**：补一套基于 Pointer Events 的自定义拖拽链路（不依赖 HTML5 `dataTransfer`），统一桌面与触摸的创建体验。
- **优先级**：低（当前有点击创建可用，属于体验增强项）
