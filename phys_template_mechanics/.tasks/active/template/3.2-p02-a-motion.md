## 执行方式
1. 先完成已存在样例 `MOT-001` 的 T3 复核闭环，再推进剩余 6 个模板。
2. 优先处理共享构型（地面匀速、斜面下滑、抛体）以复用命令片段。
3. 每个模板必须在工作台与编辑器两端完成实际点击验证。

## 单模板标准流程（强制）
1. 在 `src/templates/presets/p02-motion/*.json` 定义模板指令与教学元信息。
2. 运行 `pnpm run templates:generate-json` 生成场景 JSON。
3. 在 `src/templates/catalog-data/templates.json` 注册 `ready + sceneJsonPath + sceneSource`。
4. 使用 `loadTemplateById` 验证加载与 sanity 链路（必要时查看日志摘要）。
5. 工作台进入编辑器做拖拽/改参/仿真检查。

## 风险与回滚
- 风险：抛体/斜面模板可能因初始位姿与参数联动导致“可加载但现象偏离”。
- 控制：保持命令定义与 JSON 同步生成，出现偏差优先从命令修正后再生成 JSON。
- 回滚：单模板异常时下线该模板 ID，保留模块其余模板可用。
