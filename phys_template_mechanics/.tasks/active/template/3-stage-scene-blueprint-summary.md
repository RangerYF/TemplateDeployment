# T3 阶段场景母版整理文档

## 1. 任务定位
- 任务 ID：`04-02-15-34-template-scene-blueprint-summary`
- 规则来源：`用户显式指令 > 风险分级自动流转 > 当前模式默认路径`
- 所属任务：`04-01-16-12-template-library-overall`
- 风险等级：`L2`
- 文档目的：
  - 不再按需求文档模块硬拆模板
  - 改为按“场景母版”整理 T3 阶段可复用的模板资产
  - 让后续实现时以场景为单位搭建，以分析视角为单位出教学入口

## 2. 核心结论

### 2.1 更合理的模板定义
当前编辑器能力已经说明，后续更适合采用下面的定义：

`教学模板 = 场景母版 + 分析视角 + 参数预设`

而不是：

`教学模板 = 需求文档中的某一条模块条目`

### 2.2 为什么要这么改
- 你们的编辑器先构建场景，再在仿真中查看：
  - 受力
  - 运动
  - 能量
  - 动量
  - 回溯过程
- 这意味着很多需求文档里的“模板”，在工程上并不是不同场景，而是：
  - 同一场景
  - 不同参数
  - 不同分析目标
  - 不同讲解入口

### 2.3 模块在新模型中的角色
- `P-01`：受力分析视角
- `P-02`：运动过程视角
- `P-05`：振动/周期视角
- `P-12`：动量/冲量视角
- `P-14`：能量转化视角

结论：
- 模块适合作为“教学分类”或“入口标签”
- 不适合作为“场景资产”的唯一组织方式

## 3. 推荐的资产抽象

### 3.1 场景母版层
负责保存真正需要搭建的物体和关系。

建议最少包含：
- `sceneMasterId`
- `name`
- `coreBodies`
- `coreRelations`
- `defaultScene`
- `editableParams`
- `supportedAnalyses`
- `mappedDemandIds`

### 3.2 分析视角层
负责定义进入这个场景后优先看什么。

可统一收敛为：
- `force`
- `motion`
- `oscillation`
- `momentum`
- `energy`
- `replay`

### 3.3 教学预设层
负责定义某次课堂讲解想讲哪一个具体现象。

建议最少包含：
- `presetId`
- `sceneMasterId`
- `analysisFocus`
- `mappedDemandIds`
- `teachingObjective`
- `keyParams`
- `initialStateNotes`

## 4. T3 场景母版清单

## 4.1 S01 水平单体平面
- 母版定义：
  - 一个物体在水平面上运动或静止。
- 核心物体：
  - `ground`
  - `block`
- 核心关系：
  - `block snapTo ground`
  - 可选外力作用于 `block`
  - 可选摩擦
- 可分析视角：
  - `force`
  - `motion`
  - `energy`
  - `replay`
- 对应需求编号：
  - `FM-001`
  - `FM-002`
  - `MOT-001`
  - `MOT-002`
- 适合承载的教学预设：
  - 光滑水平面静止
  - 粗糙水平面静止
  - 匀速直线运动
  - 匀加速直线运动
  - 初速度存在但受摩擦减速
  - 斜向拉力/斜向推力改变支持力
- 物体关系要点：
  - 本质是“单接触面 + 单主体”
  - 所有变体都不需要新几何，只需要改：
    - `friction`
    - `initialVelocity`
    - `force`
- 教学价值：
  - 这是最基础的单体模板母版
  - 适合作为受力、运动图像、能量入门三类模板的共同底座
- 构建建议：
  - 只保留一个主体物体，保证力箭头和图表结论都清晰
  - 这是最值得先做成高复用命令片段的母版

## 4.2 S02 斜面单体
- 母版定义：
  - 一个物体在斜面上静止、下滑、上滑或做能量转化。
- 核心物体：
  - `ground`
  - `slope`
  - `block`
- 核心关系：
  - `slope snapTo ground`
  - `block snapTo slope`
  - 可选摩擦
  - 可选沿斜面初速度或外力
- 可分析视角：
  - `force`
  - `motion`
  - `energy`
  - `replay`
- 对应需求编号：
  - `FM-011`
  - `MOT-011`
  - `MOT-012`
  - `ENE-021`
  - `ENE-022`
- 适合承载的教学预设：
  - 光滑斜面下滑
  - 粗糙斜面静止
  - 匀加速下滑
  - 上滑减速后回滑
  - 光滑斜面机械能守恒
  - 有摩擦斜面机械能衰减
- 物体关系要点：
  - 几何结构完全相同
  - 差别主要来自：
    - 斜面角度
    - 摩擦系数
    - 初速度
    - 是否显示能量视角
- 教学价值：
  - 这是“同一场景，多学科分析”的典型代表
  - 受力、运动、能量三个模块其实都在复用同一结构
- 构建建议：
  - 斜面模板必须作为母版资产维护，不要分散复制
  - 后续只新增预设，不新增独立场景

## 4.3 S03 单点悬挂与摆
- 母版定义：
  - 一个主体通过单一连接件挂在固定点上。
- 核心物体：
  - `anchor`
  - `ball` 或 `block`
- 核心关系：
  - `anchor -> rope/rod -> body`
  - 可由竖直静止状态进入，也可由偏角释放
- 可分析视角：
  - `force`
  - `oscillation`
  - `energy`
  - `replay`
- 对应需求编号：
  - `FM-021`
  - `FM-052`
  - `SHM-003`
  - `ENE-051`
- 适合承载的教学预设：
  - 单绳静止悬挂
  - 杆连接受力
  - 小角度单摆
  - 单摆能量转化
- 物体关系要点：
  - 固定点 + 单连接件 + 单主体
  - 若连接件从 `rope` 切到 `rod`，分析语义会变化，但几何骨架不变
- 教学价值：
  - 同一个场景既能讲张力，也能讲周期，也能讲能量转换
- 构建建议：
  - 连接件类型应作为预设参数，而不是强行拆成不同场景文件
  - `rope` 与 `rod` 需要不同的教学说明，但可共用一套母版

## 4.4 S04 多支悬挂平衡
- 母版定义：
  - 一个主体由两条不同方向的连接件共同支撑。
- 核心物体：
  - `anchor(left)`
  - `anchor(right)`
  - `body`
- 核心关系：
  - 两条连接件汇聚到同一主体
  - 连接件组合可为：
    - `rope + rope`
    - `rope + rod`
- 可分析视角：
  - `force`
  - `replay`
- 对应需求编号：
  - `FM-022`
  - `FM-023`
- 适合承载的教学预设：
  - 双绳悬挂平衡
  - 绳杆组合悬挂
- 物体关系要点：
  - 本质是二维平衡问题
  - 水平分量相互抵消，竖直分量平衡重力
- 教学价值：
  - 特别适合讲力分解和多力平衡
- 构建建议：
  - 这类模板的核心不是仿真轨迹，而是受力箭头布局
  - 应保证两条连接件在视觉上清晰分开

## 4.5 S05 双体连接系统
- 母版定义：
  - 两个物体通过绳或滑轮形成耦合运动。
- 核心物体：
  - `block/ball A`
  - `block/ball B`
  - 可选 `ground`
  - 可选 `slope`
  - 可选 `anchor/pulley-mount`
- 核心关系：
  - `A <-> rope <-> B`
  - 可包含转折点或滑轮
  - 两物体共享约束运动
- 可分析视角：
  - `force`
  - `motion`
  - `energy`
  - `replay`
- 对应需求编号：
  - `FM-031`
  - `FM-032`
  - `ENE-011`
- 适合承载的教学预设：
  - 水平面双物块绳连
  - 斜面-悬挂连接体
  - 阿特伍德机
- 物体关系要点：
  - 这三类题虽然外观略不同，但核心都是“两个主体通过同一约束耦合”
  - 真正变化的是：
    - 一端在地面
    - 一端在斜面
    - 一端上下悬挂
- 教学价值：
  - 适合整体法、隔离法、系统能量分析
- 构建建议：
  - 建议把“双体耦合”做成统一母版族
  - 再通过布局模式切换成水平版、斜面版、阿特伍德版

## 4.6 S06 弹簧系统
- 母版定义：
  - 场景中有弹簧，弹簧形变量是主导物理量。
- 核心物体：
  - `anchor`
  - `block/ball`
  - 可选第二个主体
- 核心关系：
  - `anchor -> spring -> body`
  - 或 `bodyA <-> spring <-> bodyB`
- 可分析视角：
  - `force`
  - `oscillation`
  - `momentum`
  - `energy`
  - `replay`
- 对应需求编号：
  - `FM-033`
  - `FM-051`
  - `SHM-001`
  - `SHM-002`
  - `MOM-012`
  - `ENE-041`
- 适合承载的教学预设：
  - 弹簧受力分析
  - 竖直弹簧振子
  - 水平弹簧振子
  - 弹簧压缩后释放
  - 弹簧系统能量转化
- 物体关系要点：
  - 这是 T3 里复用价值最高的母版之一
  - 一个弹簧系统同时能服务：
    - 力学受力
    - 简谐运动
    - 动量守恒
    - 机械能守恒
- 教学价值：
  - 最典型的“同一场景，不同分析面”
- 构建建议：
  - 应把“弹簧方向、是否固定一端、是否双体、是否有阻尼”做成参数化开关
  - 不要把 `FM-033/SHM-001/SHM-002/ENE-041` 当成四个完全独立场景

## 4.7 S07 竖直圆周
- 母版定义：
  - 一个主体绕固定中心在竖直面内做圆周运动。
- 核心物体：
  - `anchor`
  - `ball`
- 核心关系：
  - `anchor -> rope/rod -> ball`
  - 通过最低点初速度进入圆周
- 可分析视角：
  - `force`
  - `motion`
  - `energy`
  - `replay`
- 对应需求编号：
  - `FM-043`
  - `FM-044`
  - `MOT-033`
- 适合承载的教学预设：
  - 绳模型竖直圆周
  - 杆模型竖直圆周
  - 速度临界条件分析
- 物体关系要点：
  - 场景母版相同
  - 只需要切换连接件类型和初速度策略
- 教学价值：
  - 能同时讲向心力、临界速度、不同连接件语义
- 构建建议：
  - 场景层统一维护半径和锚点
  - 预设层决定：
    - `rope` 还是 `rod`
    - 临界速度附近还是高于临界速度

## 4.8 S08 曲面与槽道约束
- 母版定义：
  - 主体在曲面、半球面或凹槽中受约束运动或受力。
- 核心物体：
  - `ground`
  - `hemisphere` 或 `groove`
  - `ball`
- 核心关系：
  - `support snapTo ground`
  - `ball snapTo support`
- 可分析视角：
  - `force`
  - `motion`
  - `momentum`
  - `replay`
- 对应需求编号：
  - `FM-053`
  - `FM-055`
  - `MOM-032`
- 适合承载的教学预设：
  - 半球面顶部受力分析
  - 凹槽内双支持力分析
  - 槽道内约束运动
- 物体关系要点：
  - 核心都是“支持面不是平面”
  - 重点在法向约束和几何方向变化
- 教学价值：
  - 适合展示非平面接触问题
- 构建建议：
  - 建议统一成“曲面支撑体母版”
  - 通过支撑体类型切换半球、凹槽等几何

## 4.9 S09 运动工作面与相对运动
- 母版定义：
  - 一个主体在可运动的工作面或承载体上，与参考物之间存在相对运动。
- 核心物体：
  - `conveyor` 或大尺寸 `block/platform`
  - `block/person`
- 核心关系：
  - 主体与工作面接触
  - 工作面本身可动或有给定速度
- 可分析视角：
  - `force`
  - `motion`
  - `momentum`
  - `replay`
- 对应需求编号：
  - `FM-054`
  - `MOM-031`
- 适合承载的教学预设：
  - 传送带模型
  - 人船模型
- 物体关系要点：
  - 这两类看起来不同，但都在讲“相对运动 + 系统整体响应”
  - 一个是传送带给速度边界条件
  - 一个是系统内部位移引起反向位移
- 教学价值：
  - 适合讲相对运动、参考系、系统响应
- 构建建议：
  - 建议定义“承载体 + 主体”的通用母版
  - 再决定承载体是主动运动还是被动反冲

## 4.10 S10 抛落体
- 母版定义：
  - 主体离开支撑后只受重力做飞行或竖直运动。
- 核心物体：
  - `ball`
  - 可选 `ground`
  - 可选 `slope`
  - 可选 `launch platform`
- 核心关系：
  - 释放或抛出后进入自由飞行
- 可分析视角：
  - `motion`
  - `energy`
  - `replay`
- 对应需求编号：
  - `MOT-021`
  - `MOT-022`
  - `ENE-001`
  - `ENE-002`
- 适合承载的教学预设：
  - 自由落体
  - 竖直上抛
  - 标准平抛
  - 斜面上平抛
- 物体关系要点：
  - 本质不是接触关系，而是“脱离支撑后的初始条件”
  - 初始高度、初速度方向、落点参考面决定具体现象
- 教学价值：
  - 这类场景最适合结合轨迹和能量一起讲
- 构建建议：
  - 应把“抛射初态”做成统一母版
  - 再按 `vertical/horizontal/slope-target` 切预设

## 4.11 S11 一维碰撞与碰后滑行
- 母版定义：
  - 两个主体沿近似同一直线发生碰撞，碰后速度变化是核心现象。
- 核心物体：
  - `ground`
  - `body A`
  - `body B`
- 核心关系：
  - 两主体在同一运动轴上接近
  - 可带 `restitution`
  - 可带碰后摩擦滑行
- 可分析视角：
  - `momentum`
  - `energy`
  - `motion`
  - `replay`
- 对应需求编号：
  - `MOM-001`
  - `MOM-003`
  - `MOM-013`
- 适合承载的教学预设：
  - 完全弹性碰撞
  - 部分弹性碰撞
  - 碰后带摩擦滑行
- 物体关系要点：
  - 几何结构几乎相同
  - 差别主要来自：
    - 恢复系数
    - 碰撞前速度
    - 碰后是否继续受摩擦
- 教学价值：
  - 这类模板最适合用回溯看“碰前/碰后”关键帧
- 构建建议：
  - “碰撞阶段”和“碰后阶段”应作为同一模板内的两个分析片段，而不是两个场景

## 4.12 S12 冲坡与类碰撞斜面系统
- 母版定义：
  - 一个主体带速度进入斜面或与斜面系统发生能量和动量交换。
- 核心物体：
  - `ground`
  - `slope`
  - `block`
- 核心关系：
  - 主体冲上斜面
  - 可考虑斜面整体参与响应
- 可分析视角：
  - `momentum`
  - `energy`
  - `motion`
  - `replay`
- 对应需求编号：
  - `MOM-011`
- 适合承载的教学预设：
  - 滑块冲坡
  - 到达最高点与斜面共速
- 物体关系要点：
  - 虽然只有一个需求编号，但它是“斜面单体母版”的重要扩展
  - 与普通斜面模板的差别在于：这里讲的是系统交换，而不是单体下滑
- 教学价值：
  - 适合作为后续高阶场景扩展点
- 构建建议：
  - 可先作为 `S02` 的扩展预设存在
  - 若后续需要可单独升级为“可动斜面母版”

## 6. 对模板工程结构的直接建议

### 6.1 不再按模块存“重复场景”
更合理的做法：

```text
src/templates/
  scene-masters/
    s01-horizontal-single/
    s02-single-slope/
    s03-single-hang-pendulum/
    ...
  presets/
    force/
    motion/
    oscillation/
    momentum/
    energy/
```

### 6.2 一个母版下挂多个教学预设
例如 `S02 斜面单体` 可以直接挂这些预设：
- `force/fm-011-static-friction`
- `motion/mot-011-down-slide`
- `motion/mot-012-up-then-down`
- `energy/ene-021-smooth`
- `energy/ene-022-friction`

### 6.3 工作台入口建议
工作台不一定只按模块展示，还可以支持两种入口：
- 教学入口：
  - 按 `P-01/P-02/P-05/P-12/P-14`
- 资产入口：
  - 按 `场景母版`

这样：
- 教师找课题方便
- 开发和维护模板也方便

## 7. 哪些场景应该合并，哪些不该强合并

### 7.1 适合合并为同一母版的情况
- 几何结构相同
- 物体关系相同
- 只改参数或分析视角

典型例子：
- `FM-011 + MOT-011 + ENE-021/022`
- `SHM-003 + ENE-051`
- `FM-043 + FM-044 + MOT-033`
- `FM-033 + SHM-001/002 + ENE-041`

### 7.2 不宜强合并成同一教学预设的情况
- 连接件语义已经变化
  - 如 `rope` 和 `rod`
- 课堂目标不同
  - 同样是单摆，一个讲周期，一个讲能量
- 初始条件决定了不同现象
  - 如“静止悬挂”和“圆周临界通过”

结论：
- 场景层可以合并
- 教学预设层仍然要分开
