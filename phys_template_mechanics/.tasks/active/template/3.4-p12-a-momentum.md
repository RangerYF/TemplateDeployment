## 执行方式
1. 先完成单次碰撞模板，再推进双体/多阶段碰撞模板。
2. 模板参数需显式覆盖质量、初速度、恢复系数与接触关系。
3. 每个模板都要完成碰前/碰后关键量观察验证。

## 单模板标准流程（强制）
1. 在 `src/templates/presets/p12-momentum/*.json` 定义碰撞场景命令与初始参数。
2. 执行 `pnpm run templates:generate-json` 生成对应 JSON。
3. 在 `src/templates/catalog-data/templates.json` 上架模板并绑定 `sceneJsonPath + sceneSource`。
4. 通过生成阶段一致性门禁（命令产物 vs JSON）与运行态 sanity 校验。
5. 在编辑器中运行仿真，核对碰撞行为与可编辑性。

## 风险与回滚
- 风险：碰撞模板对初始速度和接触位置敏感，易出现“未碰撞/穿透/现象不明显”。
- 控制：模板验收时固定仿真步长下做多次回放，必要时微调初始位置与速度。
- 回滚：失败模板下线并记录问题，避免影响模块整体发布节奏。
