# Checkpoint

默认入口。目标是以最低必要成本完成一次可追溯收尾。

## 模式说明

### `checkpoint`（智能入口）
- 先做 preflight，再自动选择 `fast` 或 `full`
- 适合绝大多数日常收尾

### `checkpoint-fast`
- 做关键输入归档 + 任务文档链同步
- 会更新：当前子任务文档、父级阶段文档、总计划文档
- 不调用 `update-knowledge`

### `checkpoint-full`
- 用于阶段完成、验收闭环、上下文将结束、或明确要做知识沉淀时
- 会同步任务文档链，并在 preflight 命中时调用 `update-knowledge`

## Step 0：Preflight

先用最小上下文判断本轮该走哪一档，避免每次都跑完整流程。

### 需要快速确认的 4 个问题
1. 本轮是否存在**关键用户输入**
2. 本轮是否发生**阶段/状态切换**
3. 本轮是否形成**稳定可复用知识**
4. 用户是否显式要求**完整收尾**

### 选择规则

命中以下任一条件，走 `full`：
- 用户显式要求“完整 checkpoint / full / 更新知识库 / update-knowledge”
- 当前任务完成验收闭环，或阶段状态发生变化
- 需要同步父级阶段文档或总计划文档
- 已形成稳定知识信号：
  - 新的架构模式/模块边界
  - 新的设计规范
  - 新的产品边界决策
  - 已验证的通用踩坑/根因修复
  - 可复用的实施流程（至少 3 步）
- 上下文接近上限，需要做阶段性归档

若以上均不命中，走 `fast`。

### 空转保护

满足以下全部条件时，必须直接走 `fast`，不要升级到 `full`：
- 无关键用户输入
- 无阶段/状态切换
- 无稳定知识信号
- 本轮目标只是子任务收尾、局部续作收口、或中途留痕

## Fast 流程

### Step 1：提取关键用户输入

回顾本轮对话，仅提取以下类型：
- `[决策]`
- `[Bug]`
- `[调整]`
- `[反馈]`
- `[设计]`

不记录以下内容：
- 确认型提问
- 操作指令（如“开始执行”“继续”“$checkpoint”）
- 进度询问
- 对 AI 工作方式的纠正

若没有关键输入，输出“无关键输入”，后续跳过 `user-inputs.md` 写入。

### Step 2：写入原始层

若 Step 1 非空，则追加到 `.knowledge/user-inputs.md`。

### Step 3：同步任务文档链

按需更新以下内容：
- 当前子任务文档
- 父级阶段文档
- 总计划文档
- 与当前任务直接相关、且本轮确有变化的文档

重点不是写很多，而是保证后续子任务能沿着文档链继续执行。

### Step 3.1：补“后续承接信息”

若本轮产出会影响后续子任务，至少记录以下内容中的命中项：
- 新约束
- 依赖变更
- 必看文件
- 可直接复用的结论
- 未决风险

这些内容优先写在当前子任务文档，并在阶段文档/总计划文档中做简短汇总。

### Step 4：输出摘要

```
## Checkpoint 完成
- mode: fast
- user-inputs.md: +{N} 条输入归档
- 任务文档：{更新了哪些文档}
- update-knowledge: 跳过 — fast 模式不做知识沉淀
```

## Full 流程

### Step 1：执行 Fast 的 Step 1-3.1

先完成关键输入归档和最小文档同步。

### Step 2：决定是否调用 `update-knowledge`

只有命中以下任一条件才调用：
- Preflight 已识别出稳定知识信号
- 用户显式要求更新知识库
- 本轮是阶段完成/验收闭环，且知识有沉淀价值

若未命中，直接跳过，并在摘要中说明“full 模式但无知识信号”。

### Step 3：输出摘要

```
## Checkpoint 完成
- mode: full
- user-inputs.md: +{N} 条输入归档
- 任务文档：{更新了哪些文档}
- update-knowledge: [已调用/跳过] — {原因或结果摘要}
- 知识库变更：引用 `update-knowledge` 的输出摘要；若跳过则写“无”
```

## 执行原则

- `checkpoint` 的职责是“收尾编排”，不是每次都做重型知识评估
- 优先选择最低必要档位
- 子任务完成时，`fast` 仍必须保证任务文档链连续
- 只有在确实形成稳定知识时，才把成本转移给 `update-knowledge`
- 若本轮只是局部收口，宁可保守地走 `fast`
