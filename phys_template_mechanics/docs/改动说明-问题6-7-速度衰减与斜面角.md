# 问题6/7改动说明：速度衰减、斜面角与重力统一

## 1. 背景

本说明对应 2026-04-08 问题清单中的第 6、7 项：

- 问题 6：部分实验运行中存在速度衰减，典型样本为竖直圆周运动。
- 问题 7：力学中的斜面模型不能直接调节斜面角，且重力加速度希望统一按 `g = 10` 处理。

本次文档基于已落地实现整理，核心实现已在 2026-04-09 合入提交 `8afa26f feat: 优化力学模板观测与 joint 耗散表现`。

## 2. 改动摘要

本轮已完成三类工作：

1. 修复竖直圆周等 rope/rod 约束场景中的额外数值耗散，降低“逐圈越来越慢”的非教学性误差。
2. 给斜面模型补上“斜面倾角（°）”可调入口，并把力学场景默认重力基线统一到 `g = 10`。
3. 补齐观察手段，让后续其他实验能在当前基线上逐个复核，而不是继续把所有异常都归到同一个原因上。

## 3. 问题6：速度衰减修复说明

### 3.1 现象与结论

- 竖直圆周场景中，“物体从底点上升到顶点时变慢”本身属于正常物理现象，因为重力势能增加。
- 已确认的问题不在这里，而在于 rope/rod 约束场景存在额外的逐圈耗散，表现为每次回到底点时峰值速度继续下降。
- 真实归因已经确认：主因是 `rope / rod joint + 1/60 固定步长` 下的数值耗散，不是模板加载错误，也不是图表误读。

### 3.2 实现改动

#### 引擎层

- [src/engine/PhysicsBridge.ts](/root/phys_template_mechanics/src/engine/PhysicsBridge.ts)
  - 新增 `stepFrame(scene, dt)`，把“整帧推进 + 连续外力施加 + 子步进”统一到一个入口。
  - 对 joint 场景启用 scene-aware substepping：
    - rope 场景：`8x`
    - rod 场景：`4x`
    - 其他场景：`1x`

#### 仿真循环

- [src/components/Canvas.tsx](/root/phys_template_mechanics/src/components/Canvas.tsx)
  - 仿真主循环改为直接调用 `physicsBridge.stepFrame(currentScene, dt)`。
  - 避免 `Canvas` 外层再手动拆分 `applyExternalForces / beginForceFrame / step`，保证主循环与探测逻辑一致。

#### 诊断与观测层

- [src/engine/AnalysisRecorder.ts](/root/phys_template_mechanics/src/engine/AnalysisRecorder.ts)
  - 新增诊断字段：
    - `eMech`
    - `jointError`
    - `ropeSlack`
    - `bottomPeakSpeed`
    - `topPeakSpeed`
- [src/store/analysisStore.ts](/root/phys_template_mechanics/src/store/analysisStore.ts)
  - 新增“诊断”标签组，支持 `E机-t / ΔL-t / slack-t / v底-t / v顶-t`。
- [src/components/charts/TimeSeriesChart.tsx](/root/phys_template_mechanics/src/components/charts/TimeSeriesChart.tsx)
  - 补齐系统级诊断量展示，便于把“正常减速”和“额外耗散”区分开看。

#### 验收脚本

- [scripts/check-joint-dissipation.mjs](/root/phys_template_mechanics/scripts/check-joint-dissipation.mjs)
  - 新增 joint 耗散验收脚本。
  - 以 `FM-043 / FM-044 / MOT-033` 为固定样本，输出：
    - `endDriftPct`
    - `maxAbsJointError`
    - `slackFrameRatioPct`
    - `maxSlackDepth`
    - `bottomRetentionPct`

### 3.3 模板同步

为配合修复与教学预期，以下竖直圆周模板同步调整：

- [src/templates/presets/p01-force-analysis/fm-043.json](/root/phys_template_mechanics/src/templates/presets/p01-force-analysis/fm-043.json)
- [src/templates/presets/p01-force-analysis/fm-044.json](/root/phys_template_mechanics/src/templates/presets/p01-force-analysis/fm-044.json)
- [src/templates/presets/p02-motion/mot-033.json](/root/phys_template_mechanics/src/templates/presets/p02-motion/mot-033.json)

同步内容包括：

- 重力从 `9.8` 统一调整为 `10`
- 竖直圆周初速度按教学阈值重新校准，减少“起始条件过紧导致现象放大”的问题

### 3.4 当前结果

以 `pnpm run check:joint-dissipation` 的结果为准，当前三组样本均已明显收敛：

| 样本 | 结果摘要 |
| --- | --- |
| `FM-043` | `endDriftPct=-6.37%`，`slackFrameRatioPct=0%`，`bottomRetentionPct=97.76%` |
| `FM-044` | `endDriftPct=-5.26%`，`slackFrameRatioPct=0%`，`bottomRetentionPct=96.67%` |
| `MOT-033` | `endDriftPct=-6.11%`，`slackFrameRatioPct=0%`，`bottomRetentionPct=97.89%` |

结论：

- “逐圈越来越慢”的异常衰减已从原先的高漂移状态显著压低。
- rope 场景的 `slack` 已压到 `0%`。
- 当前仍不是“数学上零耗散”，但已经达到教学演示可接受范围。

## 4. 问题7：斜面角与重力统一说明

### 4.1 斜面角可调

- [src/components/panels/PropertyPanel.tsx](/root/phys_template_mechanics/src/components/panels/PropertyPanel.tsx)
  - 当选中 `slope` 物体时，新增 `斜面倾角 (°)` 输入项。
  - 角度范围限制为 `1° ~ 85°`。
  - UI 层按教学语义暴露“角度”，底层仍保持现有几何模型：
    - `baseLength`
    - `slopeHeight`
  - 具体换算为：`slopeHeight = baseLength * tan(theta)`。

- [src/models/bodyTypes/slope.tsx](/root/phys_template_mechanics/src/models/bodyTypes/slope.tsx)
  - 斜面底层几何模型仍然沿用 `baseLength + slopeHeight`。
  - 这意味着新增角度控制后，不需要重写斜面渲染、碰撞、吸附与翻转逻辑。

这样处理的好处是：

- 对教师/学生来说，面板可以直接调“斜面角”。
- 对现有代码来说，不破坏已经稳定的三角形斜面实现。

### 4.2 重力默认取 10

当前策略不是“禁用重力配置”，而是把默认值和模板基线统一到 `g = 10`：

- [src/store/editorStore.ts](/root/phys_template_mechanics/src/store/editorStore.ts)
  - 编辑器默认重力改为 `{ x: 0, y: -10 }`
- [src/store/sceneStore.ts](/root/phys_template_mechanics/src/store/sceneStore.ts)
  - 默认新场景重力改为 `{ x: 0, y: -10 }`
- [src/templates/commands/executor.ts](/root/phys_template_mechanics/src/templates/commands/executor.ts)
  - 命令生成场景时的默认重力也改为 `{ x: 0, y: -10 }`
- [src/components/panels/PropertyPanel.tsx](/root/phys_template_mechanics/src/components/panels/PropertyPanel.tsx)
  - 未选中物体时仍可在环境配置面板修改 `g`，只是系统基线改为 `10`

## 5. 对其他实验的意义

这两项修复完成后，其他实验的观测前提已经更稳定：

- 竖直圆周类样本不再因为明显的 joint 数值耗散，把“本来正常的上升减速”误判成 bug。
- 斜面类样本现在可以直接调角度，教学上更接近教材参数表达。
- 重力默认值统一为 `10` 后，模板之间的口径更一致。

但这不等于“其他实验已经全部验收完成”。

当前更合理的做法是：

1. 基于这次修复后的基线，按模板逐个观测。
2. 对高速过程，优先使用已有的慢放能力（如 `0.1x / 0.25x`）再观察。
3. 对碰撞类问题，优先结合以下诊断量看：
   - `E机-t`
   - `ΔL-t`
   - `slack-t`
   - `v底-t / v顶-t`

例如：有些碰撞过程只有在速度慢下来后，才能明确区分“碰撞本身的问题”还是“前置运动过程就已经有误差积累”。

## 7. 结论

问题 6 和问题 7 的核心诉求目前都已落地：

- 问题 6：竖直圆周等场景的额外速度衰减已定位并显著压低，同时补上了可用于继续观察的诊断手段。
- 问题 7：斜面模型已经可以直接调节斜面角，系统默认重力和模板资产也已统一到 `g = 10`。

后续工作重点不再是重复怀疑这两处基线，而是在此基础上，把其余实验按场景逐个复核，尤其是碰撞、高速运动和多约束联动场景。
