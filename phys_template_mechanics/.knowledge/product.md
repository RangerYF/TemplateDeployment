# 产品定位与边界

## 目标用户
中国初高中物理教师

## 产品定位
力学实验模板编辑器——教学辅助工具。教师创建力学实验场景模板，学生在模板上填写受力分析。

## 核心使用流程
1. 教师在编辑模式下搭建力学场景（放置物体、设置约束）
2. 点击仿真预览物理行为
3. 导出为模板供学生使用
4. 学生在模板上标注力的方向和大小

## 模板交付门禁（2026-04-01）

- 模板上线采用“单模板纵向闭环”原则：每个模板都必须同时满足工作台可见、可加载、可编辑、可仿真、可验收。
- 在模板指令系统（T2）完成前，不继续扩展内置模板场景数据；`catalog` 仅保留元信息，避免以手工估值场景进入教学链路。
- 后续高阶段任务文档按阶段开始时创建，避免在需求未稳定时提前铺设执行文档。

## 模板可读性与初始条件门禁（2026-04-01，T3.1-C1）

- 固定锚点模板的最低可视基线：`anchor.position.y > 9`。
- 静止悬挂链路（anchor-dynamic 的 rope/rod）最小几何基线：`锚点距 d > 4`。
- 初速度默认策略：除模板显式声明外，动态体 `initialVelocity` 默认为零。
- 尺寸默认策略：除模板显式声明外，`block/ball` 使用默认尺寸基线。
- 决策理由：用户验收明确指出“画面过于紧凑会影响力箭头展示与教学可读性”，因此将其上升为模板上线门禁而非个别模板调参。

## 模板配置与工作台信息密度决策（2026-04-01，T3.1-C2）

- 模板配置采用“每模板独立文件 + 按模块目录组织”，作为后续扩展与人工维护的唯一入口。
- 模板教学元信息先收敛为最小集：`teachingObjective + constructionSteps`。
  - 其中 `constructionSteps` 以命令序列为准，避免人工文案与真实行为不一致。
- 工作台卡片信息收敛为：`名称 + ID + 状态`，不承载长文本教学说明。

## C3 预置模板回写能力边界（2026-04-02）

- C3 已恢复执行并落地首版：允许“加载预置模板 -> 人工调整 -> 保存回预置”。
- 回写必须双写：
  - `src/templates/presets/<module>/<template-id>.json`
  - `public/templates/scenes/<TEMPLATE_ID>.json`
- 模板来源分为两类：
  - `sceneSource=command`：命令可重生成模板（默认工程化路径）
  - `sceneSource=manual`：人工微调模板（以 scene JSON 为真值）
- 交互边界：`保存预置/导出JSON` 只在本地开发环境可见，不进入生产教学界面。
- 目的：为模板研发提供“人工微调闭环”，同时保持正式教学使用场景的稳定性与安全边界。

## 做什么
- 常见力学实验场景的快速搭建（斜面、滑轮、传送带、弹簧等）
- 2D 刚体物理仿真预览
- 物体的直观拖拽编辑（移动、缩放、旋转）
- 智能吸附与对齐辅助
- 受力分析视角（阶段6进行中）：力箭头可视化、正交分解、力列表面板

## 不做什么
- 不做通用物理沙盒/游戏引擎
- 不做 3D
- 不做流体、热力学、电磁学模拟
- 不做实时多人协作
- 不做精确数值仿真（教学演示精度即可）

## 物体分类体系

| 分类 | 说明 | 包含类型 |
|------|------|---------|
| basic | 基础刚体 | Block（方块）、Ball（球） |
| support | 支撑结构与安装件（当前实现暂并组） | Wall（挡板）、Slope（斜面）、Ground（地面）、Anchor（固定点）、PulleyMount（滑轮座） |
| surface | 特殊表面与工作面 | Conveyor（传送带）、Hemisphere（球槽）、HalfSphere（半球）、Groove（V形槽） |

### 当前分组决策
- 代码接口仍保留 `mechanism` 类别扩展位，但当前产品和物体库 UI 先保持 3 组：`basic / support / surface`
- `anchor / pulley-mount` 暂归入 `support`
- `conveyor` 暂归入 `surface`
- 用户已明确："先保持这样吧"

### 默认材料语义
- 所有物体默认 `restitution = 0`
- 理由：当前产品服务于中国大陆初高中力学教学，默认场景更强调受力和运动分析，而不是反弹碰撞专题；只有在专门讨论碰撞/恢复系数时才手动调高

## 约束系统交互决策

### 绳约束编辑行为
| 场景 | 行为 | 理由 |
|------|------|------|
| 拖拽物体超过绳长，另一端是固定点 | 钳制，不能拖过去 | 绳长是硬约束 |
| 拖拽物体超过绳长，另一端是动态物体 | 联动拖拽，另一个物体跟着移动 | 符合物理直觉 |
| 缩短绳长，当前距离超过新绳长 | 自动调整物体位置（static不动，dynamic端拉近） | 编辑所见即所得 |
| 增长绳长，当前距离不变 | 绳变松弛下垂 | 绳长 > 距离 = 松弛 |

### 绳的物理语义
- 属性面板显示"绳长"而非"最大长度"（底层仍映射 Planck.js RopeJoint.maxLength）
- 绳长是固定值，不是弹性属性；弹性连接用弹簧（DistanceJoint）
- 创建时默认绳长 = 两物体间距

## 力系统产品决策

### SceneForce 只存用户外力
- 系统力（重力/支持力/摩擦力/张力/弹簧力）由引擎每帧计算，不持久化到 Scene
- 用户外力（ForceTool 创建）存入 SceneForce，可编辑、可撤销、可序列化
- 理由：系统力是物理引擎的输出，不应让用户手动设置；外力是教师意图的表达

### 视角模式
- 力面板始终可见（双 Tab：属性 + 力），无全局视角切换
- 分析图表面板默认展开（底部，运动/能量/动量标签页）
- 可分析物体 = 非 static 物体（用户决策：动态斜面也可分析，判定标准是 isStatic 而非物体类型）

## 仿真播放与回溯语义（8.5）

- 顶部“选择/约束”入口移除，播放控制独立为时间回溯中心交互区。
- 播放控制语义固定为：
  - 播放：开始/继续仿真
  - 暂停：暂停仿真，后续可继续
  - 停止：结束当前播放并停在当前帧，允许回溯
  - 退出（重置语义）：清空仿真过程并返回编辑态
- 时间轨道支持历史回溯与输入跳转；回溯是教学演示的一等能力，不是调试附属功能。

## 物体交互约束决策表

| 物体类型 | canMove | canResize | canRotate | canDelete | 特殊约束 |
|---------|---------|----------|----------|-----------|---------|
| Block/Ball/Bar 等动态体 | true | true | true | true | 无 |
| Slope/Wall/Conveyor | true | true | true | true | 无 |
| Ground | vertical-only | false | false | false | X 锁定，仅可上下调整高度 |
| Anchor/PulleyMount | 取决于 isStatic | false | false | false | 无 |
