# 交互设计规范

## 视觉编码

### 颜色系统
- 选中态：蓝色 `#3b82f6`
- Hover 态：蓝色虚线
- 吸附目标面：蓝色加粗线段
- 对齐辅助线：蓝色虚线 `#3b82f6`，lineWidth=1，setLineDash([4,4])

### 8.6.3 视觉统一 Token
- 统一交互主色：`#3B82F6`（selected/hover/snap/guide 统一来源）
- 顶栏与圆形控制按钮：高度 `56px`，按钮尺寸 `32px`，边框 `1.5px`，阴影使用 `SHADOWS.sm`
- 面板标题：`11px` + `0.08em` 字距；面板内容基线内边距 `12px`
- 反馈参数：`selectedFill=rgba(59,130,246,0.10)`，`selectedBorder=rgba(59,130,246,0.30)`
- 虚线参数：outline `[6,3]`，guide `[4,4]`
- 线宽参数：base `1.5px`，hover `2px`，selected `2.5px`，snap `2.5px`

### 选中状态
- **普通物体选中**：蓝色边框 + 4 个角手柄（白底蓝框空心方块）+ 左上角外侧旋转弧形箭头
- **普通物体 hover**：蓝色虚线边框
- **Ground 选中**：蓝色实线全宽高亮
- **Ground hover**：蓝色虚线全宽 + `ns-resize` 光标

### Ground 样式
物理课本风格：实线 + 下方 45° 斜线阴影（hatching），12px 高，8px 间距

### 勾选框（全局规范）
所有勾选框统一使用自定义样式，禁止使用原生 checkbox 或黑色勾：
- 尺寸：14×14px，圆角 3px
- 选中态：背景 `primary`（绿色），边框 `1.5px solid primary`，白色 ✓（10px）
- 未选中态：背景 `bg`（白色），边框 `1.5px solid borderStrong`（灰色）
- 设计依据：用户反馈"勾选框里面怎么能用黑色的勾呢？不符合整个设计风格呀"

## 手柄设计

### 缩放手柄
- 4 个角手柄（NW/NE/SW/SE）：非等比缩放，对角锚点固定
- 边框 hover 检测（5px 阈值）：单轴缩放（N/S 改高度，E/W 改宽度）
- 不使用边中点手柄（简洁优先）

### 旋转手柄
- 位置：左上角外侧
- 样式：双向弧形箭头图标（SVG）
- 三态：灰色(默认) → 黑色(hover) → 隐藏(拖拽中)
- 光标：自定义 SVG data URI 旋转光标

### 光标映射
```
NW/SE 角 → nwse-resize
NE/SW 角 → nesw-resize
N/S 边   → ns-resize
E/W 边   → ew-resize
旋转柄   → 自定义旋转光标
Ground    → ns-resize
```

## 拖拽交互

### 三种拖拽模式
1. **Move**：点击物体拖拽移动，支持吸附和对齐
2. **Resize**：拖拽角手柄/边框缩放，基于 dragStartBody 快照计算
3. **Rotate**：拖拽旋转柄旋转，使用相对旋转（offset）防止跳变

### 处理优先级
```
onMouseDown:
  hitTestHandle? → resize/rotate 模式
  hitTestBody?   → move 模式

onMouseMove (move 模式):
  1. 计算新位置
  2. SnapEngine.computeSnap()（表面吸附）
  3. AlignEngine.computeGuides()（对齐辅助线）
  → 先吸附后对齐，两者叠加
```

### 手柄命中检测优先级
旋转柄 > 角手柄 > 边框 hover

## 吸附与对齐

### 吸附优先级
1. 表面吸附（SnapEngine）— 修正 position + angle
2. 对齐辅助线（AlignEngine）— 微调 x 或 y
3. Alt 键按住时禁用吸附

### 吸附反馈
- 吸附时：目标表面蓝色加粗高亮
- 对齐时：蓝色虚线穿越视口

### 地面接触行为
- 物体接触地面后继续向下拖动，应始终保持与地面接触（不穿透、不脱离）
- 实现：普通 snap 失败时用 `threshold=Infinity` 强制 snap 到地面
- 水平面吸附时自动将物体旋转角度归零（从斜面拖回地面时回正）
- 球体不重置角度（旋转对球无视觉影响）

### 地面联动
- 拖拽地面上移时，地面上的物体一起上移（递归：斜面上的物块也跟随）
- 地面上移碰到悬空物体时，物体自动纳入联动
- PropertyPanel 编辑地面 Y 时同样触发联动

### 物体翻转（自然拖拽行为）
- 翻转不是独立功能，而是 resize 拖拽越过对面边界时的自然结果
- 不需要快捷键、不需要翻转按钮、不需要 PropertyPanel 设置
- 用户反馈："我只是要当我能拖动他的时候，他的形状能够变成另一个方向"
- 当前支持：slope（斜面），其他非对称物体可后续扩展

## 物体库面板

### 布局
- CSS Grid 3 列卡片，分 3 组：基础物体(2) / 支撑与约束(4) / 特殊表面(4)
- ground 不出现在物体库中（场景自带）
- 卡片：缩略图（自适应列宽，`w-full aspect-square`）+ 物体名称，hover 高亮
- 物体卡片 Tooltip：`拖入画布可创建物体`
- 当前先保持 3 组，不恢复 `mechanism` 独立分组

### 缩略图
- 预生成静态 PNG（`public/thumbnails/`），不在面板中实时渲染
- 用户决策："在列表里面实时渲染，感觉不太对"
- 生成方式：dev 工具页 (`#thumbnails`) 实时预览并调参后导出
- 连接件缩略图也走同一流程，文件命名：`joint-rope.png` / `joint-rod.png` / `joint-spring.png` / `joint-pulley.png`

### 连接件入口与引导（8.4）
- 左侧面板新增“连接件”分组：绳 / 杆 / 弹簧 / 滑轮绳
- 连接件支持两种入口：
  - 拖拽到画布（drag-and-drop）
  - 点击卡片后，鼠标进入画布即激活创建流程
- 拖拽连接件进入画布时（光标加号阶段），就显示步骤提示，不等待 drop
- 引导浮层固定包含 `Esc 退出` 提示
- 快捷键决策：不新增连接件快捷键，仅保留左侧入口（方案 C）
- 连接件卡片 Tooltip 显示：`拖入画布后并选择要连接的物体可创建xx`（`xx` 为连接件名）
- 引导浮层视觉：浅色背景 + 灰边 + 轻阴影（与项目设计 token 一致）

### 物体默认尺寸
- 以中国初高中物理教材器材比例为依据，物块(0.8×0.6m)为基准
- 物块占斜面底长约 1/4，球径≈物块短边，杆长粗比 33:1
- 详见 stage-8.2-object-library.md 中完整尺寸对照表
- 8.3.4 收尾默认尺寸：
  - 斜面：`7.5 × 4.33`
  - 挡板高度：`4.33`（与斜面一致）
  - 球槽外轮廓宽度：约 `7.5`
  - 半球宽度：`7.5`
  - 传送带：`11.25 × 1.2`
  - V形槽：`6.5 × 4.33`，默认作为稳定支撑槽处理

### 拖放预览
- 从面板拖入 Canvas 时显示半透明真实物体预览（`globalAlpha=0.5` + `renderEdit()`）
- 实时 snap，松手位置 = 预览位置（零跳变）
- 用户决策："松手时实际创建的物体跟预览的也要位置完全一样"
- 需隐藏浏览器默认 ghost image（`setDragImage(emptyImg)`）
- 拖回面板区域松手不创建物体（HTML5 DnD 原生行为）

### 物体渲染风格（教材标准）
- **固定端标记**：45° 斜线阴影（hatching），通用函数 `src/renderer/hatching.ts`，默认 COLORS.dark + alpha 0.85
- **锚点**：教材「固定铰支座」画法 — 固定面粗线 + 斜线阴影 + 连接杆 + 空心铰链圆 + 中心点
- **滑轮座**：固定面粗线 + 斜线阴影 + 支架杆 + 轮环 + 中心轴点
- **球槽**（原半球面）：长方体+半球挖槽 — 矩形块体(margin=0.35r, blockDepth=0.55r) + 弧形凹槽(加粗3px) + 底边斜线阴影，填充极淡(0.08)。物理：chain(弧面) + 3 polygon(左壁/右壁/底板)。默认非 static
- **半球**：白底黑线的倒扣半球，顶部承载面可供其他物体放置
- **斜面**：三角形 + 底边下方斜线阴影（isStatic 时）
- **传送带**：虚线分隔段 + 三角箭头在皮带面内 + 十字轴心滚轮
- **V形槽**：壁面半透明填充 + 稳定底座 + 底部斜线阴影，默认作为固定支撑槽
- **颜色原则**：所有物体统一采用教材式白底黑线；对象本体、轮廓、阴影线、传送带细节统一用深色线稿，避免品牌色和彩色填充干扰分析

## 约束渲染

### 绳（rope）
- **绷紧**（距离 ≥ 绳长 × 98%）：直线
- **松弛**（距离 < 绳长 × 98%）：二次贝塞尔曲线向下垂，sag = √(L²-d²)/2（V形精确公式）
  - d=L 时 sag=0（绷直），d→0 时 sag→L/2（完全垂下）
- 线宽：默认 `1.5px`，hover `2px`，选中 `2.5px`
- 颜色：默认 `#555`，选中 `#3B82F6`，hover `rgba(59,130,246,0.55)`
- 锚点标记：3px 圆点，与线同色
- 渲染层级：在物体之下（先绘制约束，再绘制物体）

### 杆（rod）
- **主体**：圆端矩形条（半宽 3px），浅灰填充 `#d4d4d4` + 深色描边 `#555`
- **铰链**：两端空心圆圈（外径 4.5px 白底 + 描边，内径 2px 实心圆心点），表示可转动连接
- **选中态**：描边 `#3B82F6` + 填充 `#dbeafe`，线宽 `2.5px`
- **hover 态**：描边 `rgba(59,130,246,0.55)` + 填充 `#e8f0fe`，线宽 `2px`
- **设计依据**：用户反馈"跟物理课本不太一样"，课本杆为有宽度的条状物+两端铰链

### 弹簧（spring）
- **渲染**：锯齿线（zigzag），coilCount=8 固定，amplitude=6px，两端各留 10% 直线段（leadIn）
- **颜色**：默认 `#555`，选中 `#3B82F6`，hover `rgba(59,130,246,0.55)`
- **线宽**：默认 `1.5px`，hover `2px`，选中 `2.5px`
- **锚点**：3.5px 实心圆点（与线同色）
- **仿真时**：锯齿间距随弹簧伸缩动态变化（coilCount 固定，间距 = 可用长度 / coilCount）
- **属性面板**：提供"恢复自然长度"按钮，调整物体位置使间距等于 springLength
  - 设计依据：用户反馈"构建场景时不知道怎样让弹簧处在自然长度"
  - 移动策略：static 端不动，dynamic 端移动；两端都 dynamic 各移一半

### 滑轮绳（pulley）
- **渲染**：A→切点A 直线 + 切点A→切点B 圆弧（绕过滑轮顶部）+ 切点B→B 直线
- **切点**：真实切点计算（`θ ± acos(r/d)`），随锚点位置动态变化
- **侧分配**：创建时确定 `sideA: 'left'|'right'`（A.x < B.x → left），持久化到 SceneJoint，渲染时只读取不重算
- **颜色**：默认 `#555`，选中 `#3B82F6`，hover `rgba(59,130,246,0.55)`（同绳）
- **线宽**：默认 `1.5px`，hover `2px`，选中 `2.5px`
- **属性面板**：总绳长（totalLength）可编辑，调整时物体沿到滑轮顶方向按比例移动；ratio 不暴露（MVP 固定为 1）
- **拖拽约束**：totalLength = distA + distB 守恒，拖动一端另一端沿滑轮方向联动
- **设计依据**：用户反馈"绳没有绕着滑轮的感觉"，需要圆弧绕过效果

### 约束选中态
- 与物体共用蓝色选中/hover 编码
- 点击检测：点到线段距离 < 5px/scale

## 播放控制区（8.5）

### 布局原则
- 顶栏三段布局：左（元信息/快捷操作）、中（播放控制区）、右（面板占位），中间段宽度与画布列一致。
- 播放控制区单行结构：图标按钮组 + 时间轴 + 时间读数/跳转输入。
- 时间轴优先填满中间剩余宽度，右侧读数和跳转区固定不被挤压。

### 控件与语义
- 按钮采用纯图标圆形按钮：播放、暂停、停止、退出（重置语义）。
- 播放区圆形按钮与画布左下角撤销/重做按钮共用同一视觉规格（32px、1.5px 边框、sm 阴影）。
- Tooltip 统一使用 `Tip` 组件，位置在按钮下方，文案按状态语义区分（如“开始仿真/继续仿真”）。
- 停止：停在当前帧并允许回溯；退出：清空仿真过程并返回编辑态。

### 时间轴交互
- 支持拖动回溯到任意历史时刻，拖动过程不破坏历史。
- 仅在“从历史点继续播放”时裁剪未来分支。
- 每秒刻度 + 近秒吸附（阈值约 0.12s），`0s` 标签保留、`0s` 刻度线隐藏。
- 支持输入跳转（Enter/按钮触发，自动做 0~maxTime 范围钳制）。

## 视口默认
- offset.y = 100，地面在图表面板分界线上方 100px（地面与分界线间隔被斜线填充）
- 图表面板折叠时通过 `pan(0, panelHeight)` 补偿 offset.y，展开时撤销，确保地面不跳动

## 响应式适配（8.6.4）

### 目标分辨率
- 主适配：`1920×1080`
- 兼容下限：`1280×720`（核心编辑/播放/分析可用，布局不破碎）

### 顶层三列宽度策略
- 左侧物体库列：`clamp(12rem, 16vw, 15rem)`
- 右侧属性列：`clamp(14rem, 19vw, 17.5rem)`
- 设计意图：窄屏时优先保证中心画布和底部分析区可交互，宽屏时保留足够信息密度。

### 顶栏播放区窄屏降级
- 时间轴容器最小宽度分档：`180px`（base）/ `220px`（sm）/ `260px`（xl）
- 状态文字（如“播放中”）仅在 `2xl` 显示，窄屏隐藏
- “跳转到时间”输入区仅在 `xl` 及以上显示，窄屏保留时间读数 + 时间轴

### 分析面板短屏策略
- 面板高度上限：`min(500px, 50vh)`；默认高度：`min(280px, 38vh)`，且不低于 `150px`
- 标签组头部支持横向滚动（`overflow-x: auto`），避免 `1280×720` 下标签被挤压截断

## 力的视觉编码

### 力箭头颜色系统
| 力类型 | 颜色 | 色值 | 课本标签 |
|--------|------|------|---------|
| 重力 | 黑色 | `COLORS.dark` | G |
| 支持力 | 黑色 | `COLORS.dark` | N |
| 摩擦力 | 黑色 | `COLORS.dark` | f |
| 张力/弹簧力 | 黑色 | `COLORS.dark` | T |
| 外力（用户） | 黑色 | `COLORS.dark` | F |
| 合力 | 黑色 | `COLORS.dark` | F合（虚线箭头） |
| 分量 | 黑色 | `COLORS.dark` | Fx / Fy（虚线箭头） |
| 选中态 | 黑色 | `COLORS.dark` | 与教材统一，不再用彩色编码 |

### 力箭头样式
- 起点：从物体边缘直接出发（非质心），无间距无圆点（教材标准画法）
- 长度映射：对数缩放 `MIN + (MAX-MIN) × log(1+mag)/log(1+BASE)`，MIN=30px，MAX=180px，BASE=100
- 线宽：2px（默认），2.5px（hover/选中）
- 箭头头部：实心三角形，长 10px，底宽 8px
- 设计依据：来自 p01 实践——线性映射导致小力不可见，从质心出发会被物体遮挡

### 力标签格式
- 格式：「字母符号 中文名 = 数值N」（如 `G 重力 = 19.6N`）
- 水平力标签在箭头尖端外侧，竖直力标签在箭头中点侧面
- 张力/弹簧力标签额外偏移 6px（远离连接件）
- 设计依据：用户反馈"力的中文名也应该显示出来"

### 力箭头防重叠（三层策略）
- 层1·力-力共线：点积 > 0.87（cos30°）→ 垂直偏移 10px，递增轨道
- 层2·力-连接件共线：力方向与绳/杆方向共线 → 偏移到远离连接件一侧
- 层3·合力偏移：合力偏移到负方向轨道，14px（与独立力方向相反）
- 合力冗余检测：合力≈某独立力时不渲染
- 标签布局：候选位置 + 贪心选择（局部物体内防重叠，不用全局障碍物系统）
- 设计依据：p01 经验——分层独立处理比统一扇形展开更实用
- 交互回归补充（2026-04-02）：
  - 双连接件张力避让使用 `sourceId->joint` 绑定 + `abs(dot)>0.6` 同线判定；
  - 偏移方向按“物体中心→锚点”向量自动选外偏侧，避免单侧仍贴绳。
  - 张力连接件避让偏移建议值：14px（普通力-连接件保留 10px）。

### 重力箭头层级规范（2026-04-02）
- 重力语义：起点必须是物体质心。
- 层级语义：物体主体视觉必须在箭头之上。
- 实现建议：不使用 `destination-over`；对重力箭头使用“物体轮廓外部裁剪（evenodd）”，只显示物体外部段。

### ForceTool 方向智能吸附
- 候选方向自动生成：全局 4 方向 + 接触面 4 方向 + 约束 4 方向
- 吸附范围：±12°，Alt 禁用，Shift 锁定 15° 网格
- 设计依据：用户反馈"物体在斜面上就往斜面方向吸附，就是当前物体常用的外力方向"

### 分解动画（四阶段渐进）
- 总时长 0.8s 渐入，0.3s 渐出
- 阶段：坐标轴参考线(0-30%) → 分量箭头+引导虚线(30-60%) → 直角标记(50-80%) → 分量标签(60-100%)
- 设计依据：p01 实践——一次性出现所有元素会造成视觉混乱

### 正交分解渲染参数（参考 p01 优化）
- 分量箭头：虚线 [6,4]，线宽 1.8px，透明度 70%，黑色
- 引导虚线：[3,3]，透明度 35%，黑色
- 坐标轴：虚线 [6,4]，黑色，透明度 30%
- 直角标记：8px 大小，透明度 60%，放在力尖端（非起点）
- 分量标签：11px 字号，下标 ₁₂ 符号，垂直于分量方向偏移 12px
- 分量起点：从物体中心沿分量方向到边缘（`getEdgeStart`，与独立力一致）
- 共线避让：同向（dot > 0.87）时垂直偏移 10px，偏移方向与合力偏移相反
- 分解轴：有接触力时从 contactNormal 推导（沿接触面），否则默认水平/竖直
- 分解状态持久化：`decomposedForceKeys` 在 store 中，仿真时保持 + 实时更新

### 正交分解面板入口
- 力列表每行（非合力）右侧有 Split 图标按钮
- 默认弱化文本色，激活时使用统一蓝色反馈；保持 `cursor-pointer` 与 Tip 提示
- Tip 组件即时提示（"正交分解"/"取消分解"）
- 设计依据：用户反馈"如果一个力可以正交分解的话，可以在力的列表里面给它出现一个勾选框"

### 力面板（原受力分析面板）
- 双 Tab 设计：「属性」Tab + 「力」Tab，选中物体时**始终显示**
- Tab 入口放在顶栏右侧占位区，右侧面板内容区仅展示当前激活 Tab 内容
- ~~默认视角只显示属性 Tab~~ → 已改为始终双 Tab（用户反馈 6.4："顶部不需要统一的受力视角开关"）
- 力 Tab 显示该物体受到的**所有力**（重力+接触+约束+外力），编辑模式通过 probeForces 获取
- 力排序：外力(0) > 重力(1) > 支持力(2) > 摩擦力(3) > 张力(4) > 合力(5)
- 每行：Eye/EyeOff 显隐按钮 + 颜色点 + 标签 + 中文名 + 数值 + 分解按钮 + 删除按钮（仅外力）
- 行选中反馈使用统一蓝色 token（浅蓝底 + 蓝色边框）；批量按钮 hover 使用统一反馈底色
- 批量"全部显示/全部隐藏"按钮
- 外力编辑区**默认展开**（大小 NumberInput + 方向 Slider+Input 同行）
- 外力方向：Slider ±180°，**0° = 物体当前角度**，拖拽中用 local state 防闪烁
- 外力标签：多个外力自动编号 F1/F2/F3（最大已有序号+1）
- 合力行：分隔线后显示，可独立显隐，合力始终基于全部力计算（不受其他力显隐影响）
- 力选中独立于物体选中：面板/画布点击力只高亮力箭头，不影响物体选中状态
- 非分析目标物体力默认隐藏，可手动覆盖（`_manualOverrides`）
- 当前默认隐藏范围：全部 `support` 类物体 + `conveyor / hemisphere / half-sphere / groove`
- 仿真模式力箭头显隐跟随面板设置（forceDisplayStore.hiddenForceKeys）
- ~~Toolbar 力按钮~~ → 已移除，创建外力统一从面板操作

### 属性面板空态（8.6.1）
- 当右侧面板“无选中物体”时，不显示空白提示，改为显示“环境配置”面板。
- 当前最小可配项为重力加速度 `g`，采用输入式配置（不提供快捷按钮）。
- 输入校验：

  - 范围 `0-50`
  - 非法输入显示错误提示，且不写入场景状态

### 画布空态引导层（8.6.5）
- 触发条件：仅编辑态且场景无用户内容（仅有默认地面）时显示。
- 文案布局：标题/说明/按钮整体居中，说明文字控制最大宽度，避免长行压迫感。
- 操作入口：提供“一键添加方块/小球”快捷动作，文案保持短句，不叠加额外教学说明。

  - 有速度时显示 `v=...`
  - 有初始加速度时追加 `a=...`

## 模板工作台（T0）视觉与导航规范

### 扁平化视觉原则
- 背景使用纯色页面底（`COLORS.bgPage`），不叠加大面积渐变氛围。
- 模块分组不使用卡片阴影堆叠；优先通过留白与标题层级区分结构。
- 模块间不使用分割线，统一由垂直间距（`space-y`）构建节奏。

### 模板卡片样式
- 卡片保持单层边框（`1px border`），默认无阴影、无位移动画。
- Hover 仅做颜色反馈：边框改为交互蓝（`FEEDBACK_VISUAL.selectedColor`），背景改为浅蓝填充（`FEEDBACK_VISUAL.selectedFill`）。
- 状态表达使用“色条 + 文本”而非胶囊标签，降低装饰密度。
- 卡片信息密度控制：仅显示 `模板名 + ID + 状态`，不展示长段说明（教学目标/步骤等）以保持浏览效率。

### 编辑器返回入口
- 顶栏左侧提供“返回工作台”icon。
- 图标样式：`ChevronLeft`（不带横线），无背景、无边框，仅保留点击热区与 tooltip。
- 交互语义：纯导航行为（返回 `#workbench`），不隐式修改当前模板数据。

### 模板开发动作入口（T3.1-C3）
- 适用场景：仅模板研发/调参阶段，不属于正式教学交互。
- 可见性：仅 `import.meta.env.DEV` 显示，生产环境隐藏。
- 布局位置：编辑器顶栏右侧区域，按钮使用绝对定位覆盖层，不参与顶栏元素布局流。
- 定位约束：右侧贴边、顶部贴齐（“贴顶”），避免挤压 Tab 与其他导航元素。
- 动作集合：
  - `保存预置`：将当前场景回写到模板 preset 与场景 JSON。
  - `导出JSON`：下载当前场景 JSON 供人工构建模板数据。
