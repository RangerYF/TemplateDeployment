# V0.4 演示清单

v0.1~v0.3 已演示过的功能不再重复。本清单仅覆盖 v0.4 新增/增强的功能。

**标记说明**
- 【反馈】= 教师第二轮反馈采纳项
- 【发散】= 开发自行发散或 V0.3 延期功能

---

## 一、Bug修复与视觉优化

1. 【反馈】**参数滑块修复** — 所有几何体的参数滑块均可流畅拖动
   - 来源：教师1反馈"边长有些情况下没办法调节滑轮"
   - 演示：选棱锥 → 将底面边长拉到较大值 → 侧棱长滑块仍可正常拖动；拖动 sides 滑块实时切换底面边数
2. 【反馈】**外接球视觉优化** — 从密集网格线改为3条正交大圆，更清爽
   - 来源：教师2反馈"外接球球面上的线段过于多"
   - 演示：开启外接球 → 对比优化前后效果（3条大圆线，不遮挡几何体）

---

## 二、直线方程显示

3. 【反馈】**选中线段查看方程** — 建系后选中线段，Inspector 中显示对称式方程和方向向量
   - 来源：教师1反馈"建系之后点击直线无法显示直线方程"
   - 演示：建坐标系 → 选中一条棱 → 查看右侧 Inspector 中的直线方程（对称式 + 方向向量）
4. 【反馈】**无坐标系提示** — 未建系时选中线段，提示"创建坐标系后显示方程"
   - 演示：未建系时选中线段 → 显示灰色提示文案

---

## 三、文本指令输入

5. 【反馈】**画线段工具文本输入** — 切到画线工具时，出现输入框，输入"AB"回车直接创建线段
   - 来源：教师1反馈"增加打字AB画线段的方式"
   - 演示：切到画线工具 → 输入框出现 → 输入"AB" → 回车 → 线段AB创建成功
6. 【反馈】**截面工具文本输入** — 切到截面工具时，输入"ABC"回车直接创建截面
   - 演示：切到截面工具 → 输入"ABC" → 回车 → 截面创建成功
7. 【发散】**下标点名支持** — 输入"A1B1"自动匹配 A₁ 和 B₁
   - 演示：输入"A1B1" → 正确创建线段 A₁B₁
8. 【发散】**错误提示** — 输入不存在的点名，显示友好错误提示
   - 演示：输入"XY" → 显示"未找到点 X"

## 四、动点功能

9. 【发散】**棱上动点参数控制** — 选中棱上的点，通过 t 参数滑块控制位置
   - 来源：V0.3 延期 + 教师多次提及"增加动点功能"
   - 演示：在棱 AA₁ 上取点 P → 选中 P → 拖动 t 滑块 → 点沿棱移动
10. 【发散】**动点 + 度量联动** — 动点移动时，关联的角度/距离度量实时更新
    - 演示：创建距离度量 BP → 拖动 P 的 t 滑块 → 距离值实时变化
11. 【发散】**动画播放** — 点击播放按钮，动点自动沿棱往返运动
    - 演示：点击播放 → 点 P 自动运动 → 距离值持续变化 → 点击暂停
12. 【发散】**动画速度控制** — 速度滑块调节动画快慢
    - 演示：调节速度滑块 → 动画加速/减速
13. 【发散】**动画重置** — 点击重置按钮，点回到初始位置
    - 演示：播放一段后 → 点击重置 → 回到初始 t 值

14. 【发散】**点拖拽体验优化** — 拖拽点时不再带动视图旋转，拖拽更流畅
    - 演示：拖拽棱上的点 → 视图不跟着动，拖拽丝滑无卡顿

## 综合演示建议（串联场景）

**场景A：文本指令快速建模（2分钟）**

1. 选择正方体 ABCD-A₁B₁C₁D₁
2. 切到画线工具 → 输入"AC" → 回车 → 对角线创建
3. 输入"A1C" → 回车 → 另一条辅助线创建
4. 切到截面工具 → 输入"A1BC1" → 回车 → 截面创建
5. 展示：比传统鼠标点选快很多

**场景B：动点+度量演示（3分钟）**

1. 选择正方体 → 在棱 AA₁ 上取点 P
2. 创建距离度量：P 到点 B、P 到点 D₁
3. 拖动 P 的 t 滑块 → 观察两个距离值实时变化
4. 点击播放 → 观察动点往返运动 + 距离值动态变化
5. 找到 BP+PD₁ 最小值的位置 → 暂停
6. 建坐标系 → 选中线段 → 查看直线方程

**场景C：外接球+参数调节（1分钟）**

1. 选择正四面体 → 开启外接球 → 展示清爽的3条大圆线
2. 调节棱长滑块 → 观察外接球同步变化 → 滑块流畅

## 需要产品经理确认的发散功能汇总

以下功能不在教师反馈采纳项中，是开发自行发散的：

**1. 动点系统**
- 棱上动点 + t 参数滑块 + 动画播放/暂停/重置/速度控制
- 发散理由：教师在两轮反馈中均提及"增加动点功能"，是高考动点题型的核心演示需求

**2. 下标点名匹配**
- 文本指令输入时 A1 自动匹配 A₁
- 发散理由：几何体上底面点标签带下标（A₁/B₁等），用户输入习惯是打 A1 而非 A₁

**3. 点拖拽体验优化**
- RAF 节流 + transient 状态 + Three.js 对象缓存
- 发散理由：开发过程中发现拖拽卡顿和视图跟随问题，顺手优化
