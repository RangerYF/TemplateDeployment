# 4.1.2 截面生成逻辑重设计

## 任务类型 & 风险评估

- **任务类型**：功能修复 + 逻辑重设计
- **风险等级**：L2（涉及数据实体创建链路、跨模块联动）
- **流程路径**：MODE 0 → MODE 1（研究）→ MODE 3（计划）→ MODE 4（执行）→ MODE 5（审查）→ MODE 6

## 问题描述

### 问题1：交点重复创建

**现象**：选3个已有顶点创建截面时，计算出的交点如果恰好在已有顶点位置（t=0 或 t=1），仍然创建了新的 Point Entity，导致：
- 同一位置出现两个点（原始顶点 + 新交点）
- 新交点显示为自定义点样式（红色），而非复用已有的 builtIn 顶点
- 新交点有新名字（P1、P2...），与原始顶点标签不一致

**用户期望**：如果交点位置与已有点重合（顶点或自定义点），应直接复用已有点，不重复创建。

### 问题2：截面应生成两个面

**现象**：当前截面只生成一个 Face（由交点组成的多边形）。

**用户期望**：
1. 3个定义点之间应形成一个面（三角形），且定义点之间应有连线（segment，已有的不重复创建）
2. 截面平面与立方体相交的完整多边形是另一个面
3. 总共应生成**两个面**

### 需要研究定义的问题

截面生成的完整逻辑需要重新设计。核心问题：

1. **点复用策略**：交点与已有点重合时如何判定？用什么容差？
2. **面的构成**：截面结果应生成几个面？每个面由哪些点组成？
3. **线段创建**：截面多边形的每条边是否需要创建 Segment Entity？哪些可复用已有棱线？
4. **定义点面 vs 截面面**：两个面的区别？source 如何标记？

## MODE 1：研究分析

### 当前实现链路

```
用户点击3个点(definingPointIds)
    ↓
crossSectionTool.tryCreateCrossSection()
    ↓
1. 解析3个定义点坐标
2. fitPlane() 拟合平面
3. computeEdgeIntersections() 遍历所有棱计算交点
4. sortIntersectionsByAngle() 按环绕顺序排序
    ↓
CreateCrossSectionCommand 执行:
  - 为每个交点创建新 Point Entity (constraint: edge)
  - 创建 1 个 Face Entity (pointIds = 新交点IDs, source: crossSection)
```

### 当前代码中的问题分析

**`computeEdgeIntersections()` 的行为**：
- 遍历所有棱 `result.edges`
- 如果棱的某个端点在平面上（t≈0 或 t≈1），会产生交点
- 这些端点交点与定义点中的顶点重合，但仍然被记录为 `EdgeIntersection`
- `CreateCrossSectionCommand` 对每个 intersection 都创建新 Point，导致重复

**旧版 `crossSection.ts`（engine 计算层）的做法**：
- 使用 `addUniquePoint()` 做坐标级去重（EPSILON=1e-6）
- 选定点本身先加入交点集合，然后棱线交点通过去重自动跳过重合点
- 但旧版只返回坐标数组，不区分"复用已有点"vs"新建交点"

### 截面几何学分析

以正方体为例，选3个顶点 A(上前左)、B(上前右)、C(下后左) 定义截面：

```
A ─────── B         选定点: A, B, C
│╲        │         平面: 过 A, B, C
│  ╲      │
│    ╲    │         截面与棱的交点:
│      ╲  │         - 棱AB上: A (t=0), B (t=1) → 复用已有
│        ╲│         - 棱AC上: A (t=0), C (t=1) → 复用已有
D ─────── E         - 其他棱可能有新交点 P1, P2...

截面多边形 = [A, B, P1, C, P2, ...] （按环绕排序）
定义点三角形 = [A, B, C]
```

### 正确的截面生成逻辑（提案）

#### 步骤1：计算截面多边形顶点

1. 将定义点坐标加入候选集合
2. 遍历所有棱计算与平面的交点
3. **去重**：如果交点与已有点（顶点或自定义点）坐标重合（容差 1e-6），标记为复用，不创建新点
4. 按平面内角度排序，形成截面多边形

#### 步骤2：区分复用点和新交点

对截面多边形中的每个点：
- **复用点**：已有 Point Entity（builtIn 顶点或自定义点），直接使用其 ID
- **新交点**：在棱内部（0 < t < 1）且不与已有自定义点重合，创建新 Point Entity

#### 步骤3：创建面

- **截面面**（Face, source: crossSection）：pointIds = 截面多边形所有点的 ID（含复用+新建）
- **定义点面**（Face, source: crossSection-defining 或新的 source type？）：pointIds = 定义点 ID

> **待讨论**：是否真的需要两个面？还是一个面即可，但确保定义点之间有连线？

#### 步骤4：创建连线

截面多边形相邻点之间，如果没有已有 Segment 连接，创建新 Segment。
定义点之间也需要连线（如果没有已有 Segment）。

### 设计原则（已确认）

**实体统一原则**：所有点、线、面都是一等公民 Entity，享有统一交互能力。区别仅在 properties 元数据（builtIn / source / constraint），决定级联行为、可编辑性、显示样式。

- 截面产生的交点 → Point Entity（constraint: edge）
- 截面多边形的边 → Segment Entity（关联到截面面，级联删除）
- 截面 → Face Entity（source: crossSection）

**连线策略（已确认）**：截面多边形的每条边都创建 Segment Entity。已有的棱线复用，不重复创建。

### 待用户确认的设计决策

**面的数量**：

用具体例子说明。正方体选 A(上前左)、B(上前右)、C(下后右)，截面还产生交点 P1：

```
    A ─────── B
   /│ ╲      /│
  / │   ╲   / │
 /  │    ╲ /  │
    │     P1  │
    │      ╲  │
    D ─────── C
```

- **方案A**（1个面）：只生成截面多边形面 [A, B, C, P1]
- **方案B**（2个面）：定义点三角形面 [A, B, C] + 截面多边形面 [A, B, C, P1]
  - 这两个面会有重叠区域（三角形 ABC 是多边形 ABCP1 的子集）
- **方案C**（2个面，无重叠）：定义点三角形面 [A, B, C] + 延伸部分三角形面 [A, C, P1]

**决策：方案C** — 2个无重叠的面，组合成完整截面。每个面可独立删除。

## MODE 3：执行计划

### 概述

重写 `tryCreateCrossSection()` 和 `CreateCrossSectionCommand`，实现：
1. 点复用（交点与已有点重合时不创建新点）
2. 方案C面生成（多个无重叠三角形/多边形组成截面）
3. 截面边创建为 Segment Entity（复用已有棱线）

### 算法设计

#### 截面多边形构建

```
输入: definingPointIds (3个定义点), builderResult (几何体)
输出: 截面多边形顶点列表 (pointId[]), 每个点标记为"复用"或"新建"

1. 解析定义点坐标，拟合平面
2. 遍历所有棱计算交点 (edgeStart, edgeEnd, t)
3. 去重合并：
   - t ≈ 0 → 复用 edgeStart 对应的已有 Point Entity
   - t ≈ 1 → 复用 edgeEnd 对应的已有 Point Entity
   - 0 < t < 1 → 检查是否与已有自定义点重合，重合则复用，否则新建
4. 按平面内角度排序形成多边形
```

#### 面分割（方案C）

```
截面多边形 = [A, B, C, P1]  (定义点 A,B,C + 新交点 P1)

分割策略：以定义点为分割线，将大多边形拆成无重叠的子面
- 定义点在多边形中的索引: iA=0, iB=1, iC=2
- 定义点三角形: Face [A, B, C]
- 延伸部分: 每段"定义点之间被新交点扩展"的区域，形成三角形扇

简化实现：
- 如果截面多边形 == 定义点（无额外交点）→ 只生成 1 个面
- 如果有额外交点 → 用定义点三角形作为基础面，
  其余部分按相邻定义点+新交点组成额外三角形/多边形
```

### 详细步骤

#### S1. entityStore 新增点查找方法

// 根据 constraint 查找已有 Point Entity

#### S2. 重写 crossSectionTool.tryCreateCrossSection()

核心流程：
1. 拟合平面（不变）
2. 计算边交点（不变）
3. **新增**：对每个交点做复用判定
   - t ≈ 0/1 → 查找已有顶点 Point（`findPointAtVertex`）
   - 0 < t < 1 → 查找已有棱上点（`findPointOnEdge`）
4. 按角度排序形成截面多边形（不变）
5. **新增**：面分割算法
   - 找到定义点在多边形中的索引
   - 按定义点分割多边形为子面
6. 构建 Command 参数（复用点ID列表、新建点参数列表、子面列表）

#### S3. 重写 CreateCrossSectionCommand

输入参数变更：
```typescript
constructor(
  geometryId: string,
  definingPointIds: string[],
  // 截面多边形信息（已排序）
  polygonPoints: Array<
    | { type: 'reuse'; pointId: string }              // 复用已有点
    | { type: 'create'; edgeStart: number; edgeEnd: number; t: number; label: string }  // 新建交点
  >,
  // 子面拆分
  subFaces: Array<{ pointIndices: number[] }>,  // 每个子面引用 polygonPoints 的索引
)
```

execute 逻辑：
1. 创建新交点 Point Entity（仅 type='create' 的）
2. 组装每个子面的 pointIds（复用 + 新建）
3. 为每个子面创建 Face Entity（source: crossSection）
4. 为截面多边形相邻点之间创建 Segment Entity（已有的跳过）
5. 全部包裹在 BatchCommand 中
