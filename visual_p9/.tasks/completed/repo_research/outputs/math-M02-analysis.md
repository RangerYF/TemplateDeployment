# M-02 函数图像实验室 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：初中/高中数学老师
- **解决的教学痛点**：函数是贯穿初高中的"主线任务"，老师在黑板上无法展示参数连续变化对图像的影响（如 a 从 1 变到 0.5 时抛物线如何变宽），也无法动态演示切线旋转、导数几何意义等概念。学生只能看静态的两三张对比图，无法建立"连续变化"的直觉。
- **核心产品价值**：一个实时交互的函数绘图工具，老师输入表达式或选预设，拖滑块即可实时看到图像变化，同时自动标注特征点、画切线、叠加导函数，把抽象的函数分析变成直观的视觉体验。

## 2. 用户实际使用方式

### 教师典型操作流程

- **课前准备**：从预设库选函数（或输入自定义表达式），调好参数范围和初始值，可能预设多条对比曲线（如 y=2^x 和 y=log₂x），保存/截图作为课件素材
- **课中演示**：
  - 实时拖动参数滑块，学生看到图像连续变化（如 a 从正变负看开口翻转）
  - 开启"导数可视化"，拖切点沿曲线移动，切线跟着旋转，f'(x) 图上的点同步联动
  - 播放参数动画，完整呈现变化过程（如 a 从 -3 到 3）
  - 点击变换按钮做平移/对称动画演示
- **讲题**：输入特定函数，用特征点标注验证手算结果，用定积分面积可视化辅助解题

### 怎样算"好用"

- 输入表达式要接近数学写法（`x^2 + 2x - 1`），不需要编程语法
- 滑块拖动时图像逐帧跟随，无可感知延迟
- 多条曲线颜色在投影仪上可清晰区分
- 特征点自动标注，精确值用分数/根号表示

### 交互频率和深度

- **高频模块**：函数是初高中数学核心，从初二到高三持续使用，覆盖一次函数、二次函数、反比例函数、指数/对数/幂函数、三角函数、导数等几乎全部函数章节
- **交互深度**：中等偏深——核心操作是滑块调参（浅），但多函数对比、导数联动、变换动画等属于深度交互

## 3. 核心功能点

### 功能清单（按重要性排序）

1. **函数表达式输入与解析** — 支持数学写法输入，自动解析绘图
2. **预设函数一键加载** — 按学段/类型分类的函数库（一次、二次、反比例、指数、对数、幂、三角、绝对值、复合）
3. **参数滑块实时调节** — 每个可调参数自动生成滑块，逐帧联动
4. **多函数对比** — 同一坐标系最多 6 条曲线，自动分配颜色+图例
5. **特征点自动标注** — 零点、y 轴交点、极值点、拐点，支持精确值（分数/根号）
6. **切线工具** — 点击曲线上的点画切线，沿曲线拖动实时更新斜率
7. **导数可视化** — 自动计算并绘制 f'(x) 图像，切点与 f'(x) 联动
8. **定积分面积可视化** — 区间着色，显示面积值，可选黎曼和矩形近似
9. **函数变换动画** — 7 种变换（平移/对称/伸缩）的平滑动画演示
10. **分段函数** — 分段输入，自动处理连续性（实心/空心点）
11. **参数动画播放** — 参数自动从起始到终止连续播放，支持暂停/调速/循环
12. **坐标读取与网格** — 鼠标跟随坐标、点击取精确值、滚轮缩放、拖动平移

### 需要实体管理的功能

- 多函数曲线管理（创建/选中/删除/编辑/隐藏/显示）
- 特征点标注（开关控制）
- 切线对象（创建/拖动/删除）
- 变换结果保留对比（原图虚线 + 新图实线）

### 需要 undo/redo 的功能

- 函数添加/删除/修改
- 参数值修改
- 变换操作
- 需求明确性：PRD 未专门提及 undo/redo，但作为编辑器型工具应支持

### 参数驱动的功能（调参 → 画面实时变化）

- **核心模式**：参数滑块驱动是本模块的主要交互方式
- 所有预设函数的可调参数（a、b、c、k、ω、φ 等）
- 坐标系范围、网格间距调节
- 定积分区间 [a, b] 拖动

### 模拟/动画驱动的功能

- 参数动画播放（参数自动连续变化）
- 函数变换动画（0.5-1 秒平滑过渡）
- 切线沿曲线拖动时的实时旋转（准动画）

## 4. 核心依赖识别

### 技术依赖

**渲染方式**：2D Canvas 或 SVG
- Canvas 更适合：高频重绘（滑块拖动逐帧更新）、大量采样点绘制（每单位 500 点）、多曲线叠加
- SVG 可考虑用于：坐标轴标注、图例等 DOM 交互元素
- **推荐**：Canvas 主绑图 + DOM/SVG 辅助 UI，或纯 Canvas

**数学/物理计算需求**：
- 表达式解析器（将用户输入的数学表达式解析为可计算函数）— **核心技术难点**
- 数值求解：零点（牛顿法/二分法）、极值（导数为零的点）
- 符号微分或数值微分（切线斜率、导函数图像）
- 精确值表达（分数、根号形式的特征点坐标）
- 定积分数值计算（黎曼和/辛普森法则）
- 函数变换的数学表达

**公式渲染需求**：
- 函数表达式显示（图例区、标注区）
- 特征点坐标的精确值显示
- 建议使用 KaTeX 进行公式渲染

### 交互依赖

**交互模式**：**参数驱动型为主 + 编辑器型辅助**
- 主模式：滑块拖动 → 图像实时变化（参数驱动）
- 辅模式：多函数管理、切线创建/拖动、区间选择（编辑器型）

**关键交互**：
- 滑块（参数调节 — 最核心）
- 输入框（表达式输入、精确参数值输入）
- 点选/拖拽（切线切点沿曲线拖动、积分区间端点拖动）
- 滚轮缩放 + 拖动平移（坐标系导航）
- 鼠标悬停（坐标读取、特征点精确值）
- 按钮/切换（变换、导数显示、标注开关、动画播放控制）

**是否需要实体系统**：需要轻量级实体管理
- 多条函数曲线的增删改查和可见性管理
- 切线、标注点等辅助对象
- 不需要像 M-01（立体几何）那样的完整 Entity-Component 系统

**是否需要命令系统（undo/redo）**：建议支持
- 函数增删、参数修改等操作可回退
- 可用简单的操作栈实现，不需要复杂的 Command 模式

## 5. 竞品分析

### Desmos（desmos.com）

- **产品形态**：Web 应用（免费），是目前最主流的在线函数绘图工具
- **技术实现**：
  - 自研表达式解析引擎（支持隐函数、参数方程、不等式）
  - Canvas 渲染
  - 自研数学排版（类似 MathQuill 的所见即所得输入）
- **做得好的地方**：
  - 表达式输入体验极佳：所见即所得的数学排版输入（分数线、根号自动渲染）
  - 滑块响应极其灵敏，60fps 实时更新
  - 支持极其丰富的函数类型（隐函数、参数方程、极坐标、不等式着色）
  - 免费、无需登录即可使用
  - 教师可创建"活动"分享给学生
- **不足的地方**：
  - 界面全英文，中国教师使用有门槛
  - 没有针对中国教材的预设模板和教学要点
  - 没有"五点法"等中国高考专属的教学功能
  - 导数可视化不够直观（没有切线沿曲线滑动的联动）

### GeoGebra（geogebra.org）

- **产品形态**：Web + 桌面 + 移动端，开源（GPLv3）
- **技术实现**：
  - Java 起家，后用 GWT 编译为 JS
  - 自研 CAS（计算机代数系统）
  - Canvas 渲染
- **做得好的地方**：
  - 功能极其全面：函数、几何、3D、概率统计、CAS 全合一
  - 有中文版和大量中文社区教学资源
  - 支持符号计算（精确值求解）
  - 有"步骤构建"功能，教师可设计分步教学
- **不足的地方**：
  - 界面复杂，学习成本高，教师上手慢
  - 性能偏重（加载慢，尤其是 3D 模块）
  - 交互不够"现代"，UI 风格偏老旧
  - 功能过于通用，缺乏针对性的教学引导

### 其他竞品

- **Mathway / Wolfram Alpha**：计算导向，非可视化教学工具
- **几何画板（GSP）**：桌面软件，老牌几何工具，函数绘图能力一般
- **网易有道数学**：面向学生做题，非教学演示

### 对我们的启示

1. **表达式输入是核心体验**：Desmos 的所见即所得输入是标杆，我们至少要做到"接近数学写法"的文本输入 + 实时预览
2. **性能是底线**：滑块拖动必须 60fps 无延迟，这是教学场景的刚需
3. **差异化在"教学场景化"**：预设函数按中国教材分类、教学要点提示、中文界面 — 这是 Desmos/GeoGebra 做不到的
4. **功能收敛**：不要试图做成 GeoGebra 那样的全能工具，聚焦"函数图像教学"这一个场景做到极致
5. **底层组件可复用**：表达式解析器、2D 坐标系渲染、滑块联动机制可被 M-03（解析几何）、M-04（三角函数）复用

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：HTML5 Canvas 2D
  - 理由：高频重绘场景（滑块拖动逐帧更新、动画播放）Canvas 性能远优于 SVG；多曲线叠加+采样点密集（每单位 500 点）适合 Canvas 立即模式渲染
- **UI 层**：React DOM
  - 理由：输入框、滑块、按钮、图例等控件用 DOM 实现，交互性和可维护性更好
- **公式渲染**：KaTeX
  - 理由：轻量、渲染速度快、支持服务端渲染，适合教学场景的数学公式显示

### 推荐的架构模式

- **参数驱动型架构**（区别于 M-01 的编辑器型 Entity-Command 架构）
  - 核心数据流：`参数状态 → 数学计算 → Canvas 渲染`
  - 状态管理：React 状态（或 Zustand）管理函数列表、参数值、视图状态
  - 渲染层：Canvas 绑图引擎，支持坐标系变换、曲线绘制、标注绘制
  - 计算层：表达式解析器 + 数值分析模块（求零点、极值、导数等）

### 关键技术风险或难点

1. **表达式解析器**（难度：高，风险：高）
   - 需要将用户输入的文本（如 `x^2 + 2x - 1`、`ln(x+1)`、`A*sin(w*x+p)+k`）解析为可高效求值的函数
   - 需支持：运算符优先级、括号嵌套、内置函数（sin/cos/tan/ln/log/sqrt/abs）、常量（pi/e）、参数变量
   - 需良好的错误处理和提示
   - **建议方案**：使用成熟库（如 math.js 的 expression parser）或自研递归下降解析器
   - math.js 的 `compile()` 可将表达式编译为可重复求值的函数，性能足够

2. **数值分析精度与性能**（难度：中，风险：中）
   - 零点求解：需处理多零点、重零点情况
   - 导函数计算：符号微分（准确但实现复杂）vs 数值微分（简单但有精度损失）
   - 精确值表达：将数值结果转换为分数/根号形式（如 `(-1+√5)/2` 而非 `0.618`）较复杂
   - 不连续函数处理：如 tan(x) 在渐近线处不应画连线

3. **滑块实时渲染性能**（难度：中，风险：低）
   - 单曲线 500 采样点/单位 × 20 个可见单位 = 10000 点，6 条曲线 = 60000 点
   - 加上特征点计算、标注渲染，需保证 60fps
   - Canvas 绑图本身性能足够，关键是表达式求值效率（编译后的表达式求值通常在微秒级）

4. **与 M-04 三角函数演示台的功能重叠**（风险：低，需设计时注意）
   - 重叠点：三角函数图像绘制（y = A sin(ωx + φ) + k 的参数调节）
   - M-02 定位：通用函数绘图工具，三角函数只是其中一种函数类型
   - M-04 定位：三角函数专题教学，核心是单位圆↔函数图像联动、五点法作图、解三角形
   - **复用策略**：2D 坐标系渲染引擎、表达式解析器、参数滑块组件可在两者间共用
