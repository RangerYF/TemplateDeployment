# C-03 化学方程式配平器与纠错系统 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
初中/高中化学教师，核心用户画像是需要在课堂上讲解化学方程式配平方法、纠正学生常见方程式书写错误的化学教师。不要求有编程或专业软件操作基础，打开浏览器即用。

### 解决什么教学痛点
- **配平过程难以逐步展示**：板书配平只能写出最终结果，中间每一步的思考过程（找变价元素、计算电子转移、确定系数）难以动态呈现，学生看不到"为什么这样配"
- **多种配平方法对比困难**：同一方程式可用观察法、最小公倍数法、化合价升降法等不同方法配平，板书难以在多种方法间切换演示
- **学生方程式错误类型多样且纠错耗时**：学生常见错误包括反应不存在（如 Cu + HCl）、产物写错（如 Fe + Cl₂ → FeCl₂）、条件缺失、配平错误、化学式书写错误等7大类，教师逐一讲解耗费大量时间
- **离子方程式拆分规则复杂**：哪些强电解质拆分、哪些不拆（如不溶性强电解质 BaSO₄、可溶性弱电解质 HgCl₂），规则有例外，纯靠记忆容易出错
- **缺乏系统化的配平练习工具**：现有题目分散在习题册中，教师无法按学段、反应类型、难度等维度快速筛选练习题

### 核心产品价值
**一句话**：教师打开浏览器，输入任意化学方程式即可获得逐步配平演示和方程式纠错反馈，支持7种配平方法按学段分类、覆盖100+题库的分步练习模式——让"方程式怎么配、哪里写错了"一目了然。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 从题库中按学段/反应类型筛选方程式 → 预览配平演示效果 → 确认步骤说明文字是否清晰准确 → 准备课上要展示的方程式列表 |
| **课中演示-配平方法** | 投影仪打开 → 输入或从题库选择一个未配平方程式（如 P + O₂ → P₂O₅）→ 选择配平方法（最小公倍数法）→ 逐步播放配平过程（找元素差异 → 求LCM → 定系数 → 检查）→ 每步点"下一步"控制节奏 → 切换为化合价升降法重新演示同一方程式 |
| **课中演示-纠错** | 输入一个学生常见的错误方程式（如 Cu + HCl → CuCl₂ + H₂）→ 系统显示红色纠错卡片"铜活动性排在氢之后，不与稀盐酸反应" → 讲解为什么错、正确写法是什么 |
| **课中演示-离子方程式** | 进入离子方程式书写引导 → 选择分子方程式 → 系统分步引导：识别溶解性 → 判断拆分 → 删去旁观离子 → 检查守恒 → 每步可回退讲解 |
| **课中练习** | 切换练习模式 → 随机抽取未配平方程式 → 学生尝试填写系数 → 系统判定对错并给出分步解析 |

### 怎么用才是"好用"的
- **输入零门槛**：输入 `H2O` 自动格式化为 H₂O，输入 `SO4^2-` 自动显示为 SO₄²⁻，无需学习特殊语法
- **逐步可控**：配平演示每一步有"下一步"按钮，教师完全掌控讲解节奏，任何一步可回退重讲
- **方法可切换**：同一方程式可用不同配平方法重新演示，方便对比教学
- **纠错即时且有教育价值**：不仅指出"错了"，还解释"为什么错"和"正确写法"，带有教学引导性
- **题库筛选高效**：按学段（初中/高中必修/高中选修）、反应类型（化合/分解/置换/复分解/氧化还原/离子反应）、难度（1-5）三个维度快速定位

### 用户与产品的交互频率和深度
- **交互频率**：每周化学课2-3次，每次使用10-20分钟（演示+练习），是常规课堂工具
- **交互深度**：中等偏深——涉及方程式输入、配平方法选择、逐步控制、纠错查看、练习判定等多种操作，但每个操作都很直观

## 3. 核心功能点

### 功能清单（按重要性排序）

| 优先级 | 功能 | 类型 |
|--------|------|------|
| P0 | 方程式输入与自动格式化（上下标、反应条件） | 输入组件 |
| P0 | 配平逐步演示（7种方法按学段分类，动画流程） | 模拟/动画驱动 |
| P0 | 方程式纠错系统（反应可行性验证+物质选取错误+7种错误类型） | 规则引擎 |
| P0 | 守恒验证模块（原子守恒+电荷守恒+电子守恒） | 参数驱动 |
| P1 | 离子方程式书写引导（分步引导+拆分规则引擎） | 模拟/动画驱动 |
| P1 | 题库与筛选（100+方程式，按学段/类型/难度） | 数据检索 |
| P1 | 练习模式（抽题→填写→判定→解析） | 交互流程 |

### 哪些功能需要实体管理
- **不需要传统意义的实体系统**（无几何对象的创建/选中/删除）
- 方程式可视为一个"文档对象"：包含反应物列表、产物列表、系数、条件、状态标记
- 题库中的方程式是静态数据实体（CRUD 仅限管理后台，前台只读+检索）

### 哪些功能需要 undo/redo
- 配平演示的步骤回退（每一步均可回退，类似时间轴回放而非传统 undo）
- 离子方程式书写引导的步骤回退
- 练习模式中学生可清除已填系数重新输入
- **不需要复杂的命令模式 undo/redo 系统**，步骤回退用简单的步骤索引即可实现

### 哪些功能是参数驱动的
- 守恒验证表格：修改系数 → 表格实时更新各元素原子数/电荷数对比
- 输入框的自动格式化：键入 → 实时渲染化学式

### 哪些功能是模拟/动画驱动的
- 配平逐步演示：步骤1→步骤2→...→步骤N，每步有动画过渡（高亮变价元素、显示化合价标注、箭头标注电子转移等）
- 离子方程式书写引导：四步引导流程，每步有视觉过渡
- 电子守恒可视化：电子转移过程的动画展示

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式：DOM + SVG 混合
- **主体UI**：纯 DOM 渲染（输入框、按钮、卡片、表格、步骤指示器）
- **化学式渲染**：DOM 上下标渲染（`<sub>` / `<sup>` 标签），或轻量自定义组件
- **配平步骤动画**：DOM 动画 + CSS Transition/Animation（元素高亮、系数变化、箭头出现）
- **电子转移箭头/化合价标注**：少量 SVG 绘制（弧形箭头、标注线）
- **不需要 Canvas 或 WebGL**

#### 数学/物理计算需求
- **化学式解析器**：将字符串 `Fe₂O₃` 解析为结构化数据 `{Fe: 2, O: 3}`，支持括号嵌套如 `Ca(OH)₂`、带电离子如 `SO₄²⁻`
- **配平算法**：
  - 待定系数法：构建线性方程组，高斯消元求整数解（矩阵运算）
  - 氧化还原法：需要元素化合价数据库，计算化合价变化、电子转移数、最小公倍数
- **纠错规则引擎**：基于知识库的规则匹配（金属活动性顺序、反应条件必要性、常见错误模式）
- 数学复杂度不高，核心是线性代数中的整数方程组求解

#### 公式渲染需求
- 化学方程式渲染（上下标、反应条件标注、状态符号 ↑↓）
- 不需要 LaTeX 级数学公式渲染
- 可自建轻量化学式渲染组件，或使用 KaTeX 的子集（mhchem 扩展）

### 交互依赖

#### 交互模式：混合型（输入驱动 + 步骤演示）
- 不是传统编辑器型（无画布、无拖拽）
- 不是纯参数驱动型（不是调滑块看变化）
- 核心是**输入驱动 + 规则反馈 + 逐步演示**的组合模式

#### 关键交互
- **输入框**：化学式输入，带自动格式化和智能补全
- **按钮/步进控制**："下一步""上一步""重新演示""切换方法"
- **下拉选择/筛选器**：学段选择、反应类型、难度、配平方法
- **卡片展示**：纠错结果卡片（红色=错误/绿色=正确）
- **表格**：守恒验证对比表
- **点选**：题库列表中选择方程式
- 无拖拽、无滑块、无时间轴（步骤控制用按钮而非时间轴）

#### 是否需要实体系统
**不需要**。没有需要统一管理的几何对象或可视化实体。方程式数据用普通 TypeScript 接口即可描述。

#### 是否需要命令系统（undo/redo）
**不需要完整命令系统**。步骤回退通过维护一个 `currentStep` 索引即可实现，本质是"步骤导航"而非"操作撤销"。

## 5. 竞品分析

### 主要竞品

| 竞品 | 形态 | 特点 |
|------|------|------|
| **ChemicalAid Balancer** (chemicalaid.com) | Web | 输入方程式 → 直接给出配平结果，支持大量方程式 |
| **Webqc.org Chemical Equation Balancer** | Web | 经典在线配平器，矩阵法配平，显示结果但无过程 |
| **PhET Balancing Chemical Equations** | Web (HTML5) | 交互式配平练习，拖拽系数到方程式上，有天平可视化验证 |
| **OKPALETTE 化学方程式配平** | Web/小程序 | 中文配平工具，面向中国初高中学生 |
| **Wolfram Alpha** | Web | 输入方程式自然语言，给出配平结果+反应细节 |

### 竞品技术实现方式判断
- **ChemicalAid / Webqc**：后端计算（矩阵法），前端仅展示结果，纯文本/HTML 渲染
- **PhET**：前端 HTML5 Canvas 或 SVG，交互式拖拽，有动画效果，教育导向设计
- **Wolfram Alpha**：强大的后端 CAS（计算机代数系统），前端展示

### 做得好的地方
- **PhET**：交互设计极其优秀，天平隐喻直观，学生可以"试错"并立即看到反馈，教育价值最高
- **ChemicalAid**：方程式数据库极大，几乎涵盖所有常见反应，搜索方便
- **Webqc**：简洁实用，无需注册，输入即用

### 不足的地方
- **所有竞品都缺乏逐步配平过程展示**：只给结果不给过程，教学价值有限
- **没有纠错功能**：竞品只处理"正确的方程式如何配平"，不处理"错误的方程式哪里错了"
- **不区分配平方法**：竞品通常只用一种算法（矩阵法），不按教学方法分类演示
- **不按中国课标学段分类**：没有初中/高中必修/高中选修的分层
- **PhET 虽好但场景单一**：只有配平练习，不支持方程式输入、纠错、离子方程式等

### 对我们的启示
1. **核心差异化在"过程"而非"结果"**：竞品都能给配平结果，我们的价值在于逐步展示配平思路
2. **纠错系统是独特卖点**：目前无竞品提供方程式纠错功能，这是教师明确要求的功能
3. **PhET 的交互设计值得借鉴**：天平隐喻、即时反馈、试错学习的设计理念
4. **按中国课标分层是刚需**：7种配平方法严格按学段分类，这是面向中国教师的核心适配
5. **化学式输入体验是门槛**：输入 `H2O` 自动变 H₂O 的体验必须丝滑，否则教师会回到板书

## 6. 初步技术判断

### 推荐的渲染技术
- **主体**：React + DOM 渲染，纯组件化开发
- **化学式展示**：自建轻量化学式渲染组件（解析化学式字符串 → 输出带 `<sub>`/`<sup>` 的 React 元素）；若需更复杂公式可引入 KaTeX + mhchem 扩展
- **配平步骤动画**：CSS Transition + React 状态驱动（步骤切换触发 DOM 动画）
- **电子转移箭头等标注**：内联 SVG（仅在氧化还原配平步骤中使用，量不大）
- **不需要 Canvas / WebGL**

### 推荐的架构模式


**架构特点**：
- **纯前端架构**：所有计算在浏览器端完成，无需后端服务，符合"单HTML文件浏览器打开即用"的产品形态
- **逻辑与UI分离**：化学式解析器、配平引擎、纠错引擎是纯函数/类，独立于 React 组件，便于测试
- **数据静态内嵌**：题库（100+方程式）和知识库（金属活动性、溶解性表等）作为 JSON/TypeScript 常量内嵌
- **相对独立**：与其他化学模块（C-04 元素周期表等）无强耦合，架构上可作为独立模块开发

### 关键技术风险或难点

| 风险点 | 级别 | 说明 | 应对策略 |
|--------|------|------|----------|
| **化学式解析器的鲁棒性** | 中 | 需处理括号嵌套 `Ca(OH)₂`、带电离子 `SO₄²⁻`、有机式 `CH₃COOH`、含水合物 `CuSO₄·5H₂O` 等复杂格式 | 设计完善的 tokenizer + 递归下降解析器，参考现有开源化学式解析库 |
| **配平算法的完备性** | 中 | 7种配平方法各有适用场景，部分方程式（如 EQ-049 Cu₂S + HNO₃）极其复杂 | 待定系数法（矩阵法）作为兜底通用算法；其他方法按规则生成步骤，复杂方程式回退到矩阵法 |
| **纠错知识库的覆盖度** | 中 | 需内置足够的化学知识（金属活动性、反应条件、常见错误模式）来支持开放输入的纠错 | 先覆盖题库中的18个纠错示例和5.1/5.2规则，后续迭代扩展；对未覆盖的输入给出"暂无法判断"而非错误结论 |
| **化学式输入体验** | 中低 | 自动格式化（H2O→H₂O）的时机和交互需要打磨，避免输入过程中频繁跳动 | 采用"失焦后格式化"或"实时预览区"策略，输入区保留原始文本，预览区实时渲染 |
| **步骤动画的教学准确性** | 低 | 每种配平方法的步骤生成必须符合教学规范，不能有化学知识错误 | 步骤模板由数据驱动（参考题库中的步骤演示数据），非通用算法自动生成 |
| **离子方程式拆分的边界情况** | 中 | 可溶性弱电解质（HgCl₂、醋酸铅等）不拆分，需要精确的电离程度判断 | 内置完整的强弱电解质+溶解性数据表，规则引擎基于"溶解性+电离类型"双维度判断 |
