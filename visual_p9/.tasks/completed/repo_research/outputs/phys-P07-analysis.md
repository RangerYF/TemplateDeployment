# P-07 热力学与气体分子模拟器 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中物理老师，不要求编程基础
- **解决的教学痛点**：热力学是高中物理中"看不见摸不着"的典型板块。压强、温度、体积三个宏观量的变化与微观分子运动之间的因果关系，学生很难建立直觉；液柱密封气体模型和气缸活塞模型是高考计算题高频考点，但黑板上无法动态展示升温后气柱长度变化、活塞移动方向等过程；布朗运动更是"只能看课本照片"，无法感受颗粒大小、温度对运动剧烈程度的影响。
- **核心产品价值**：一个从微观粒子到宏观热力学过程的全链路可视化工具——微观层面用粒子动画展示分子热运动和布朗运动，宏观层面用参数驱动的示意图 + 实时计算演示三种气体变化过程、液柱模型和气缸模型，让抽象的热力学概念和高考计算题模型都变得可见可感。

## 2. 用户实际使用方式

### 教师典型操作流程

- **课前准备**：选择要演示的模块（如"液柱模型-竖直管开口向上"），预设好初始温度、气柱长度、液柱长度等参数，保存为一个教学场景
- **课中演示**：
  - 讲分子热运动：打开微观模拟，调高温度，让学生看到分子加速、碰壁频率增大、颜色从蓝变红；缩小容器体积，看到压强增大
  - 讲三种变化过程：三个容器并排，分别演示等温/等压/等容，对应的 p-V、V-T、p-T 图实时绘制，学生一眼看出三种变化的图像特征
  - 讲布朗运动：展示大颗粒的无规则轨迹，调大颗粒半径看运动变规则，强调"布朗运动不是分子运动"
- **讲题时**：液柱模型和气缸模型是高考计算题高频模型。老师设置好题目条件（管的方向、液柱长度、温度变化），系统实时计算并动画展示气柱变化，学生可以对照公式验证答案

### 怎样算"好用"

- 微观模拟流畅——500 个粒子同屏运动不卡顿，温度变化时速度响应即时
- 液柱/气缸模型的计算结果精确——与手算公式完全一致，数值误差 < 1%
- 参数调节实时生效——拖动温度滑块，气柱长度、压强等动画同步变化
- 投影仪友好——试管、气缸等示意图在 1080p 下清晰可辨

### 交互频率和深度

- **中频模块**：热力学在高中物理选修中占一个完整单元（选择性必修第三册），使用频率中等
- **交互深度**：中等——微观模拟部分是"调参数看动画"的浅交互；液柱模型和气缸模型涉及多参数联动计算和受力分析，属于中深度交互

## 3. 核心功能点

### 功能清单（按重要性排序）

1. **液柱密封气柱模型**（高考计算题核心）— 竖直管（开口向上/向下）、水平管、倾斜管、U 型管、两端密封管；设置初始条件，系统计算压强平衡 + 状态方程，动画展示气柱变化
2. **气缸/活塞模型**（高考计算题核心）— 单活塞（竖直/水平）、双活塞；温度变化后活塞移动方向和最终位置，压强平衡 + 受力分析
3. **三种气体变化过程对比** — 等温/等压/等容三容器并排演示 + p-V/V-T/p-T 图实时绘制
4. **气体分子微观动画** — 容器内粒子随机运动，温度影响速率，体积影响碰壁频率，实时显示 p/V/T 并验证 pV/T = 常数
5. **布朗运动模拟** — 大颗粒被气体分子撞击的无规则运动，轨迹留存，颗粒大小和温度影响运动剧烈程度
6. **麦克斯韦速率分布曲线** — 温度变化时分布曲线峰值移动和展宽的实时展示

### 需要实体管理的功能

- **无明显实体管理需求**：本模块不涉及用户创建/删除对象的编辑器模式。所有场景都是预设模型 + 参数调节，不需要实体系统

### 需要 undo/redo 的功能

- **不需要 undo/redo**：所有操作都是参数调节和动画播放，不存在需要撤回的编辑操作

### 参数驱动的功能（调参数 → 画面实时变化）

- 液柱模型：温度、气柱长度、液柱长度、管方向、大气压 → 气柱长度和压强变化
- 气缸模型：温度、活塞质量、气缸方向 → 活塞位置和压强变化
- 三种变化过程：p/V/T 中的自变量 → 图像实时绘制
- 微观模拟：温度、容器体积 → 粒子速度和碰壁频率
- 布朗运动：颗粒半径、温度 → 运动轨迹的剧烈程度

### 模拟/动画驱动的功能（播放/暂停/逐步）

- 气体分子微观动画：粒子持续运动，需要播放/暂停控制
- 布朗运动轨迹：持续模拟 + 轨迹采样记录
- 三种变化过程的缓慢演示：从初态到末态的动画过渡

## 4. 核心依赖识别

### 技术依赖

- **渲染方式**：
  - 微观粒子模拟（分子热运动 + 布朗运动）：**2D Canvas** — 需要高频重绘大量小圆点（最多 500 个粒子），Canvas 的即时模式渲染比 SVG/DOM 更适合粒子系统
  - 液柱模型 / 气缸模型示意图：**SVG 或 Canvas** — 试管、液柱、活塞等静态结构件适合 SVG 的结构化描述；但如果追求统一渲染管线，Canvas 也完全可以胜任
  - p-V / V-T / p-T 图表：**Chart.js** 或 Canvas 自绘 — 实时绘制二维函数曲线
  - 麦克斯韦速率分布曲线：Canvas 自绘或 Chart.js

- **数学/物理计算需求**：
  - 理想气体状态方程：pV = nRT, p1V1/T1 = p2V2/T2
  - 压强平衡方程：液柱模型中 p = p0 ± ρgh（不同管方向不同公式）
  - 受力分析：气缸模型中活塞受力平衡 p = p0 + Mg/S
  - 麦克斯韦速率分布：f(v) = 4π(m/2πkT)^(3/2) v² exp(-mv²/2kT)
  - 分子平均动能：Ek = (3/2)kBT
  - 粒子碰撞：弹性碰撞的速度更新（粒子-粒子、粒子-壁面）
  - U 型管和两端密封管需要联立方程组求解

- **公式渲染需求**：KaTeX — 显示 pV=nRT、pV/T=C 等状态方程和推导过程

### 交互依赖

- **交互模式**：**参数驱动型 + 模拟演示型混合**
  - 液柱/气缸模型：纯参数驱动 — 设置参数 → 系统计算 → 动画展示结果
  - 微观粒子/布朗运动：模拟演示型 — 持续运行的粒子系统 + 参数实时调节
  - 三种变化过程：参数驱动 + 图表联动

- **关键交互**：
  - 滑块：温度、体积、液柱长度、倾斜角等连续参数调节
  - 输入框：精确数值输入（配合题目条件）
  - 下拉/按钮组：模块选择（5 个子模块）、管方向选择（竖直/水平/倾斜）、气缸方向选择
  - 时间轴控制：微观动画的播放/暂停/调速
  - 可能的拖拽：拖动活塞改变体积（微观模拟中）

- **是否需要实体系统**：不需要。所有场景都是预设模型，用户只调参数不创建对象
- **是否需要命令系统（undo/redo）**：不需要。纯参数调节 + 动画播放，不涉及可撤回的编辑操作

## 5. 竞品分析

### 竞品 1：PhET Gas Properties（科罗拉多大学）

- **产品形态**：Web（HTML5），免费开源（CC-BY），可离线运行
- **网址**：https://phet.colorado.edu/en/simulations/gas-properties
- **核心功能**：向容器内泵入气体分子，调节体积（拖动壁面）、温度（加热/冷却），观察压强变化；显示动能和速率直方图；支持轻/重两种粒子的扩散对比
- **技术实现**：基于 PhET 自研框架（Scenery 渲染引擎），HTML5 Canvas + SVG 混合渲染
- **做得好的地方**：
  - 微观模拟极其流畅，粒子动画与宏观量（p/V/T）实时联动
  - 速率分布直方图实时更新，学生能直观看到温度变化对分布的影响
  - 可拖拽容器壁改变体积，交互直觉性强
  - 多语言支持，全球教育界广泛采用
- **不足的地方**：
  - 没有液柱密封模型和气缸活塞模型——这两个是中国高考的核心计算题模型
  - 没有三种变化过程（等温/等压/等容）的并排对比
  - 没有布朗运动模块
  - 不提供数值计算和公式验证功能
- **对我们的启示**：微观粒子模拟的交互设计可以借鉴（泵入分子、拖动壁面），但我们的核心差异化在于液柱/气缸计算题模型和中国高考场景适配

- **核心功能**：模拟灰尘颗粒在空气分子撞击下的布朗运动，可显示/隐藏分子，温度滑块调节，轨迹留存
- **技术实现**：纯 HTML5 Canvas + JavaScript，轻量单页
- **做得好的地方**：轨迹留存效果清晰，温度调节对运动剧烈程度的影响直观
- **不足的地方**：功能单一，无法调节颗粒大小对比，无与其他热力学概念的联动
- **对我们的启示**：布朗运动的轨迹留存设计可以参考；我们需要增加颗粒大小对比功能（颗粒越大运动越规则）来覆盖教学要点

## 6. 初步技术判断

### 推荐的渲染技术

- **粒子系统（微观模拟 + 布朗运动）**：**2D Canvas**
  - 理由：需要高频重绘 20~500 个粒子，Canvas 的即时模式渲染在粒子系统场景下性能远优于 SVG/DOM；与 M-05 概率统计模拟器的蒙特卡罗投点有技术相似性，可复用粒子渲染管线
  - 性能预估：500 粒子 @ 60fps 在 Canvas 上无压力，PhET 的 Gas Properties 实现了更大数量级的粒子

- **液柱/气缸示意图**：**SVG 或 Canvas**
  - SVG 优势：试管、液柱、活塞等结构化图形用 SVG 描述更语义化，便于交互（点击、拖拽活塞）
  - Canvas 优势：与粒子系统统一渲染管线，减少技术栈碎片化
  - 推荐：如果项目统一采用 Canvas 方案则用 Canvas；如果希望示意图部分更易维护和交互则用 SVG

- **图表（p-V/V-T/p-T 图 + 速率分布曲线）**：**Chart.js 或 Canvas 自绘**
  - 图表数据量不大，Chart.js 足够胜任实时更新

- **公式渲染**：**KaTeX**

### 推荐的架构模式

- **参数驱动 + 模拟引擎混合架构**：
  - 整体采用**模块选择器 + 参数面板 + 演示区 + 图表区**的四区布局
  - 5 个子模块共享统一的 UI 壳（模块切换、参数面板框架、图表区框架）
  - 每个子模块实现自己的：
    - 物理计算引擎（状态方程求解、压强平衡计算、粒子运动更新）
    - 渲染器（粒子绘制、示意图绘制、图表绘制）
    - 参数配置 schema（哪些参数可调、范围、步进）

- **状态管理**：响应式状态（如 React useState/Zustand）
  - 参数变化 → 触发物理计算 → 更新渲染
  - 微观模拟部分需要 requestAnimationFrame 循环驱动

- **物理计算层与渲染层分离**：
  - 计算层：纯函数，输入参数输出结果（便于测试和复用）
  - 渲染层：根据计算结果绘制画面

### 关键技术风险或难点

1. **粒子系统性能**（中风险）
   - 500 个粒子的碰撞检测（粒子-粒子 + 粒子-壁面）每帧需要 O(N²) 的朴素算法，或 O(N log N) 的空间分区算法
   - 缓解措施：对于教学演示级别，N=500 时朴素算法也可接受（250,000 次距离比较/帧 @ 60fps）；如需优化可用网格分区
   - PhET 的实现证明了浏览器 Canvas 完全能胜任此规模

2. **液柱/气缸模型的计算正确性**（高风险）
   - 特别是 U 型管和两端密封管，需要联立两段气体的状态方程求解
   - 双活塞模型的压强平衡也需要受力分析 + 状态方程联立
   - 缓解措施：建立完善的物理计算单元测试，用手算值验证；PRD 数据文档已提供了详细的公式和典型数据

3. **麦克斯韦速率分布的真实模拟 vs 视觉近似**（中风险）
   - 真实模拟需要从麦克斯韦分布采样初速度并通过碰撞演化
   - 视觉近似可以直接根据温度参数绘制分布曲线
   - 推荐：粒子速度从分布采样，分布曲线从实际粒子速度统计得出（更真实、更有教学价值）

4. **5 个子模块的 UI 统一性**（低风险）
   - 微观模拟（粒子动画）和宏观模型（示意图）的视觉风格差异较大
   - 缓解措施：统一的布局框架 + 配色方案 + 参数面板组件

5. **倾斜管和 U 型管的示意图绘制**（低风险）
   - 需要根据倾斜角度旋转试管，液柱高度随角度变化
   - 缓解措施：使用 Canvas/SVG 的 transform 旋转
