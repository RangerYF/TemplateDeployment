## 分析要求

按照 `PROGRESSIVE-PLAN.md` 中的**综合分析模板**产出 `outputs/phys-summary.md`。

### 第一步：模块间相似度评估

从以下维度构建对比矩阵，识别相似聚类：

| 维度 | 说明 |
|------|------|
| 交互模式 | 编辑器型 / 参数驱动型 / 模拟演示型 / 混合型 |
| 渲染技术 | 2D Canvas / SVG / 3D WebGL / 混合 |
| 实体系统需求 | 完整（多种实体 + undo/redo）/ 轻量 / 无 |
| undo/redo 需求 | 完整命令系统 / 不需要 |
| 核心计算引擎 | 力学求解 / 运动方程 / 光线追迹 / 电路分析(MNA) / ODE数值积分 / 粒子碰撞 / 场计算 / 状态方程 |
| 时间轴控制需求 | 精确回放型（播放/暂停/单步/倒放）/ 持续模拟型 / 不需要 |
| 用户交互深度 | 深度编辑（创建/连线/删除）/ 参数调节+动画观察 / 纯演示 |

### 第二步：暂分组并分析

基于对比矩阵将相似模块暂分组。已知的初步线索（从分析卡片得出）：

- **P-04 独立性极强**：唯一的编辑器型模块，需要完整实体系统 + undo/redo + MNA 电路分析引擎，架构模式与其他 11 个模块完全不同
- **P-02 / P-05 / P-06 / P-09 / P-12 / P-13 / P-14 可能一组**：都是"参数+时间轴+动画"模式的力学/运动模拟，共享时间轴控制器、Canvas 坐标系、图像联动的需求
- **P-01 / P-03 可能一组**：纯参数驱动，无时间轴控制需求，"调参 → 看结果"的静态展示
- **P-07 的微观模拟部分**有独特的粒子系统需求，但宏观模型部分（液柱/气缸）是参数驱动的
- **P-08 的场线追迹**有独特的算法需求，但粒子偏转部分与 P-02 的运动模拟有相似性
- **P-13 电磁感应**跨越力学 + 电磁学 + 电路，可能与 P-04 有电路层面的复用

**但以上仅是初步线索，需要在分析中验证或推翻。**

### 第三步：分组验证（核心步骤）

对每个暂定分组回答三个问题：

1. **共享核心层的接口能否清晰定义？**
   - 能用一两句话说清楚这个核心层做什么吗？
   - 接口边界在哪里？

2. **核心层能支撑各模块的哪些功能？**
   - 逐模块列出：哪些功能走核心层，哪些功能是模块专属
   - 核心层覆盖率高 → 分组合理
   - 核心层覆盖率低（大部分功能是模块专属）→ 分组可能勉强

3. **各模块的专属能力是什么？**
   - 专属能力的实现复杂度如何
   - 专属能力之间是否有冲突

### 第四步：产出仓库划分与架构方案

对每个最终确定的仓库：

- 包含哪些模块，划分依据
- 核心架构模式
- 核心技术依赖
- 通用层 vs 模块专属层的边界定义
- 项目结构概览

### 特别注意

- **P-04 电路搭建器的独立性**：这是物理产品线中唯一的编辑器型模块，需要实体系统、命令系统、连线算法、MNA 电路分析等独特能力。是否独立成仓，还是与其他模块共享某些基础设施，需要仔细评估
- **时间轴控制的通用性**：P-02 的时间轴控制器可能被 P-05/P-06/P-09/P-12/P-13/P-14 共享，这是一个重要的共享核心能力
- **数值积分器的共享**：P-09（开普勒方程/RK4）、P-13（ODE 耦合系统）、P-05（大摆角精确解）都需要数值积分，是否值得抽象统一
- **2D Canvas 坐标系渲染的通用性**：所有 12 个模块都使用 2D Canvas，坐标系网格/缩放/平移的渲染是高度可共享的
- **P-13 与 P-04 的电路复用**：P-13 电磁感应中有感应电路的分析需求，是否复用 P-04 的电路分析引擎
- **粒子系统**：P-07（气体分子）、P-08（带电粒子偏转）都有粒子渲染需求，但物理模型差异较大
- 不要为了归组而归组：如果某个模块确实独立性强，单独成仓是合理的
- 架构设计要考虑实际开发效率：共享层太薄不如不共享，共享层太厚则模块灵活性受损
