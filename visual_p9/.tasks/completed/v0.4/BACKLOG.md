## P0 — 不修复则影响基础使用

### F01 参数滑轮边长调节Bug修复
- **来源**：教师1反馈第1条（采纳）
- **当前问题**：参数设置中的边长在某些情况下无法通过拖动滑轮调整
- **目标行为**：
  1. 排查所有几何体类型的参数滑块，确保均可正常拖动调节
  2. 检查滑块范围是否合理（min/max/step）
  3. 特殊情况处理：联动参数（如棱锥的高/侧棱长）在极端值时的滑块行为
- **涉及模块**：`components/panels/ParameterPanel.tsx`、各几何体的参数定义
- **风险等级**：L0（单文件Bug修复）

### F02 外接球视觉优化
- **来源**：教师2反馈第3条（采纳）
- **当前问题**：外接球球面上的经纬线过多，视觉杂乱，遮挡几何体
- **目标行为**：
  1. 减少球面经纬线数量或降低线条密度
  2. 经纬线颜色调淡（更高透明度或更浅颜色）
  3. 确保球面仍可被识别为球体，不丢失空间感
- **涉及模块**：`components/scene/renderers/CircumSphereRenderer.tsx`
- **风险等级**：L0（单文件UI微调）

### F04 直线方程显示
- **来源**：教师1反馈第2条（采纳）
- **描述**：建立坐标系后，选中直线/线段可查看其方程
- **目标行为**：
  1. 建系后，选中一条线段/直线，右键菜单中显示"显示方程"选项
  2. 点击后在 Inspector 面板或弹窗中显示直线方程
  3. 方程格式：
     - 参数方程形式：(x, y, z) = P₀ + t·d
     - 或对称式（两面相交形式）：(x-x₀)/a = (y-y₀)/b = (z-z₀)/c
  4. 方程中的坐标值使用建系后的坐标
- **涉及模块**：
  - `SegmentInspector` 增加方程显示区域
  - `ContextMenu3D.tsx` 增加"显示方程"菜单项
  - 新增 `engine/math/lineEquation.ts`（直线方程计算）
- **前置条件**：坐标系已建立
- **风险等级**：L1（2-3文件改动）

### F05 文本指令输入
- **来源**：教师1反馈第5条（采纳）
- **描述**：通过文本输入快速创建几何元素，降低操作步骤
- **目标行为**：
  1. 提供输入框（工具栏或快捷键唤出），支持文本指令
  2. 输入 "AB" → 自动在点A和点B之间画线段
  3. 输入 "ABC" → 自动用点A、B、C三点定义一个平面（高亮显示或创建面实体）
  4. 支持点名自动匹配（根据已有 Point Entity 的 label 查找）
  5. 输入无效名称时给出友好提示（如"未找到点X"）
  6. 可逐步扩展更多指令（如 "⊥AH" 做高等）
- **涉及模块**：
  - 新增 `components/panels/CommandInput.tsx`（文本指令输入组件）
  - 新增 `editor/commandParser.ts`（指令解析器）
  - 复用 EntityStore 的点查找和 CreateEntityCommand
- **风险等级**：L1（新增独立模块，不影响现有功能）

## P2 — 做了产品更完整

### F06 动点功能
- **来源**：V0.3 P3 延期 + 教师1反馈第6条（round1 & round2均提及）
- **描述**：在棱或面上设置可动点，拖动滑块时点在约束路径上运动，相关度量值实时变化
- **目标行为**：
  1. 棱上点增加"设为动点"选项，启用后出现参数滑块（t 值 0~1）
  2. 面上点增加"设为动点"选项，启用后出现 (u, v) 参数滑块
  3. 拖动滑块时点的位置实时更新
  4. 与该点关联的所有度量（距离、角度）实时重算并更新显示
  5. 支持动画播放：点击播放按钮，点自动沿路径运动
- **涉及模块**：
  - `PointInspector` 增加动点控制区域
  - `PointEntityRenderer` 支持参数化位置更新
  - 度量系统的响应式更新（已有 useMemo 机制可复用）
- **风险等级**：L2（跨模块联动，需确保度量实时更新性能）
