# V0.4 教学反馈迭代 — 渐进式开发计划

## 📌 版本目标

基于第二轮教师反馈，修复参数滑块Bug、优化外接球视觉、新增直线方程显示、实现文本指令快速建模（输入"AB"画线段），并引入动点功能，使产品更贴合一线教学操作习惯。

## 🔄 开发流程总览

```
阶段1: Bug修复 + 视觉优化（滑轮Bug + 外接球线条优化）                      ✅
  → 阶段2: 直线方程显示（建系后选中线段查看方程）                            ✅
    → 阶段3: 文本指令输入（输入"AB"画线段、"ABC"定义平面）                   ✅
      → 阶段4: 动点功能（棱上/面上动点 + 参数滑块 + 度量联动）              ✅
```

## 📋 串行执行阶段

---

### 第1阶段：Bug修复 + 视觉优化 ⏱️ 1-2天

**目标**：修复参数滑轮无法调节的Bug，优化外接球视觉效果，消除第二轮反馈中的基础体验问题。

**主要任务**：
1. 排查所有13种几何体的参数滑块，定位"某些情况下无法拖动调节"的具体场景和原因（F01）
2. 修复滑块问题：检查 min/max/step 范围、联动参数（棱锥高/侧棱长）在极端值时的行为（F01）
3. 优化外接球渲染：减少经纬线数量或降低密度，调淡线条颜色/透明度，减少对几何体的视觉遮挡（F02）

**涉及文件范围**：
- `src/components/panels/ParameterPanel.tsx` — 滑块组件逻辑
- `src/types/geometry.ts` — 各几何体参数范围定义
- `src/components/scene/renderers/CircumSphereRenderer.tsx` — 外接球渲染样式

**验收标准**：
✅ 13种几何体的所有参数滑块均可正常拖动调节，无卡死或无响应情况
✅ 联动参数（棱锥高/侧棱长）在极端值附近滑块行为正确
✅ 外接球经纬线视觉更清爽，不遮挡几何体主体
✅ 外接球仍可被识别为球体，空间感保留
✅ `pnpm lint && pnpm tsc --noEmit` 通过

**本阶段产出**：
基础体验问题全部修复，为后续功能开发提供干净的基线。

**注意**：具体实现细节将在执行本阶段时另建子任务文档详细规划

### 第2阶段：直线方程显示 ⏱️ 1-2天

**目标**：建立坐标系后，选中任意线段可查看其在坐标系下的直线方程，支持参数方程和对称式两种格式显示。

**主要任务**：
1. 实现直线方程计算：根据线段两端点在坐标系中的坐标，计算直线的方向向量和参数方程（F04）
2. 方程格式化输出：支持参数方程形式和对称式形式，坐标值使用分数/根号精确表示（F04）
3. SegmentInspector 扩展：建系后选中线段时，Inspector 中显示直线方程区域（F04）
4. 无坐标系时的处理：未建系时方程区域不显示或显示"请先建立坐标系"提示（F04）

**涉及文件范围**：
- `src/engine/math/` — 新增 lineEquation.ts（直线方程计算与格式化）
- `src/components/panels/inspectors/SegmentInspector.tsx` — 增加方程显示区域

**验收标准**：
✅ 建系后选中线段，Inspector 中显示直线方程
✅ 方程中坐标值正确（与坐标系原点和轴方向一致）
✅ 支持参数方程形式展示
✅ 支持对称式形式展示（方向向量分量为0时正确处理）
✅ 未建系时方程区域不显示或给出提示
✅ 棱线、用户线段等所有类型线段均可显示方程
✅ `pnpm lint && pnpm tsc --noEmit` 通过

**本阶段产出**：
直线方程显示功能，完善坐标系建立后的教学演示链路（建系→查看坐标→查看方程→解析法解题）。

**注意**：具体实现细节将在执行本阶段时另建子任务文档详细规划

### 第3阶段：文本指令输入 ⏱️ 2-3天

**目标**：实现文本指令输入框，教师可通过输入"AB"快速画线段、输入"ABC"快速定义平面，减少鼠标操作步骤，提升建模效率。

**主要任务**：
1. 设计指令解析器：解析文本输入，识别点名序列，匹配已有 Point Entity 的 label（F05）
2. 实现指令类型识别与执行（F05）：
   - 两个点名（如"AB"）→ 创建线段
   - 三个点名（如"ABC"）→ 高亮/选中对应面，或创建平面
   - 无效点名 → 友好错误提示
3. 实现 CommandInput 输入组件：输入框 UI、快捷键唤出（如 `/` 或 `Ctrl+K`）、输入历史、自动补全提示（F05）
4. 集成到编辑器布局：输入框定位（工具栏区域或浮动面板）、与现有工具系统的协调（F05）
5. 可扩展设计：预留指令扩展接口，未来可添加更多指令类型（F05）

**涉及文件范围**：
- `src/editor/` — 新增 commandParser.ts（指令解析器）
- `src/components/panels/` — 新增 CommandInput.tsx（输入组件）
- `src/components/layout/` — 布局调整，集成输入框
- `src/editor/store/entityStore.ts` — 可能需要增加按 label 查找点的辅助方法

**验收标准**：
✅ 通过快捷键或按钮唤出文本输入框
✅ 输入"AB"后回车，自动在点A和点B之间创建线段
✅ 输入"ABC"后回车，自动选中/高亮由A、B、C构成的面（或创建面实体）
✅ 输入不存在的点名时，显示友好错误提示（如"未找到点X"）
✅ 创建的线段通过 Command 执行，可 Ctrl+Z 撤销
✅ 输入框不影响现有键盘快捷键（Escape关闭输入框、非输入状态快捷键正常）
✅ `pnpm lint && pnpm tsc --noEmit` 通过

**本阶段产出**：
文本指令输入系统，教师可通过键盘快速建模。指令解析器的可扩展设计为后续添加更多指令类型（如做高、度量等）奠定基础。

**注意**：具体实现细节将在执行本阶段时另建子任务文档详细规划

### 第4阶段：动点功能 ⏱️ 3-4天

**目标**：实现棱上点和面上点的动点功能，通过参数滑块控制点的位置，拖动时所有关联度量（距离、角度）实时更新，支持动画播放。

**主要任务**：
1. Point Entity 扩展：新增 `animated` 标记和参数化位置属性，棱上点用 t 值（0~1）参数化，面上点用 (u, v) 参数化（F06）
2. PointInspector 动点控制区域：增加"设为动点"开关、参数滑块（t 或 u/v）、播放/暂停按钮（F06）
3. 参数驱动位置更新：滑块拖动时，通过 UpdatePropertiesCommand 更新点的约束参数（t 值），触发位置重算（F06）
4. 度量联动：确保与动点关联的角度度量、距离度量实体在点位置变化时自动重算（验证现有 useMemo 响应链路是否覆盖）（F06）
5. 动画播放：点击播放按钮后，t 值按固定步长自动递增，点沿棱/面运动，支持暂停/重置（F06）
6. 性能验证：多个度量同时关联动点时，确保拖动/动画流畅无卡顿（F06）

**涉及文件范围**：
- `src/editor/entities/types.ts` — Point properties 扩展 animated 相关字段
- `src/components/panels/inspectors/PointInspector.tsx` — 动点控制 UI
- `src/components/scene/renderers/PointEntityRenderer.tsx` — 参数化位置计算
- `src/editor/commands/` — 动点参数更新 Command
- `src/components/scene/renderers/AngleMeasurementRenderer.tsx` — 验证度量联动
- `src/components/scene/renderers/DistanceMeasurementRenderer.tsx` — 验证度量联动

**验收标准**：
✅ 选中棱上点后，Inspector 出现"设为动点"开关
✅ 开启动点后出现 t 参数滑块（0~1），拖动时点沿棱移动
✅ 面上点开启动点后出现 (u, v) 参数滑块，拖动时点在面内移动
✅ 动点移动时，关联的距离度量值实时更新
✅ 动点移动时，关联的角度度量值实时更新
✅ 点击播放按钮后，点自动沿路径运动，度量值同步变化
✅ 动画可暂停、可重置
✅ 多个度量关联动点时，拖动/动画流畅（无明显卡顿）
✅ 动点参数变更可 Ctrl+Z 撤销
✅ `pnpm lint && pnpm tsc --noEmit` 通过

**本阶段产出**：
完整的动点系统，教师可演示"点在棱上运动时，某个角度/距离如何变化"的动态过程，覆盖高考动点类题型的教学演示需求。
