## 子任务清单（串行执行）

### T2B.1 实现棱锥 Builder（多面体）

**目标**：输入底面边数(3~8)、底面边长、高，输出正棱锥的多面体结构。

**产出文件**：
- `src/engine/builders/pyramid.ts`

**技术要点**：
- 底面为正 n 边形，中心在 XZ 原点，底面在 y=0 平面
- 底面 n 个顶点 + 顶部 1 个顶点（共 n+1 个）
- 底面顶点标签：A、B、C、D、E、F、G、H（按边数截取）
- 顶部顶点标签：P
- 面：1 个底面（n 边形）+ n 个三角形侧面
- 棱：n 条底边 + n 条侧棱（共 2n 条）
- 底面正 n 边形顶点坐标公式：
  ```
  x_i = sideLength / (2 * sin(π/n)) * cos(2πi/n - π/2)
  z_i = sideLength / (2 * sin(π/n)) * sin(2πi/n - π/2)
  y_i = 0
  ```
  其中外接圆半径 R = sideLength / (2 * sin(π/n))
- 顶部顶点坐标：[0, height, 0]

**验收**：
- 3 棱锥（正三棱锥）有 4 个顶点、4 面、6 棱
- 4 棱锥（正四棱锥）有 5 个顶点、5 面、8 棱
- 切换边数后模型正确更新

### T2B.3 实现圆柱 Builder（曲面体）

**目标**：输入底面半径和高，输出圆柱的曲面体结构。

**产出文件**：
- `src/engine/builders/cylinder.ts`

**技术要点**：
- 输出类型：`SurfaceResult`
- `geometryType: 'cylinder'`
- `geometryArgs: [radius, radius, height, 64]`（CylinderGeometry 参数：radiusTop, radiusBottom, height, segments）
- `positionOffset: [0, height/2, 0]`（Three.js CylinderGeometry 默认中心在原点，偏移使底面对齐 y=0）
- 特征点：
  - O₁（顶面圆心）：[0, height, 0]
  - O（底面圆心）：[0, 0, 0]
- 线条：
  - 4 条母线（generatrix），90° 均匀分布，从底面到顶面
  - 1 条底面圆轮廓线（baseCircle），64 段
  - 1 条顶面圆轮廓线（topCircle），64 段

**验收**：
- 圆柱顶底面平行，母线等长
- 特征点标签 O₁、O 位置正确

### T2B.4 实现球 Builder（曲面体）

**目标**：输入半径，输出球的曲面体结构。

**产出文件**：
- `src/engine/builders/sphere.ts`

**技术要点**：
- 输出类型：`SurfaceResult`
- `geometryType: 'sphere'`
- `geometryArgs: [radius, 64, 32]`（SphereGeometry 参数：radius, widthSegments, heightSegments）
- `positionOffset: [0, radius, 0]`（使球底部对齐 y=0）
- 特征点：
  - O（球心）：[0, radius, 0]
- 线条：
  - 3 条经线（meridian）：XY 平面、YZ 平面、XZ 旋转 45° 平面各一条，64 段
  - 1 条赤道线（equator）：y=radius 平面上的大圆，64 段
  - 注意：所有线条坐标需加上 positionOffset 偏移

**验收**：
- 球体渲染为半透明球面
- 经线和赤道线清晰可见
- 球心标签 O 位于球体中心

## 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 正方体实现 | 复用长方体 Builder | 正方体是长方体的特例（l=w=h），避免重复代码 |
| 棱锥底面 | 正 n 边形 | 教学场景需要正棱锥，不规则棱锥暂不考虑 |
| 球的线条 | 3 经线 + 1 赤道 | 兼顾清晰度和简洁性，避免线条过密 |
| 圆柱母线数量 | 4 条（与圆锥一致） | 保持曲面体渲染风格统一 |
| 顶点标签约定 | 棱锥：P+A~H；圆柱：O₁+O；球：O | 符合高中数学教学惯例 |

## 风险与注意事项

1. **棱锥三角化**：底面为 n 边形（n=3~8），渲染层的扇形三角化法已可处理，但需验证 n>4 时法线计算是否正确。
2. **球的位置偏移**：球的 positionOffset 使底部对齐 y=0，但经纬线坐标也需要对应偏移——builder 输出的线条坐标应已包含偏移。
3. **圆柱双圆轮廓**：圆柱有顶底两条圆，圆锥只有底面一条，需确认 SurfaceRenderer 能正确渲染多条同类型线。
4. **正方体复用**：若直接调用 `buildCuboid({ length: s, width: s, height: s })`，类型系统需确认 `CubeParams.sideLength` 到 `CuboidParams` 的映射无问题。
