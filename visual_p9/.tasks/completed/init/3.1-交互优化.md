## 问题背景

当前取点交互为：先点击两个顶点选中棱线 → 在标注面板点"取中点"。此流程存在以下局限：

1. **曲面体无法取点**：圆锥/圆柱的底面圆、圆锥侧面没有棱线概念，当前交互模型完全无法覆盖
2. **操作链路长**：必须先选中两个顶点（且它们必须构成棱线），再到面板点按钮
3. **不支持线段/面选中**：只能通过两个顶点间接选中棱线，无法直接点击线段或面

## 任务目标

引入**右键上下文菜单**作为核心交互入口，支持直接对棱线、母线、面、曲面进行操作（取点、命名等）。同时扩展取点能力到曲面体。

## 核心设计思路

### 交互模型变更

| 操作 | 当前方式 | 优化后 |
|------|----------|--------|
| 棱线取点 | 先选两顶点 → 面板按钮 | 右键点击棱线 → 菜单"取中点/n等分" |
| 棱线命名 | 先选两顶点 → 输入框 | 右键点击棱线 → 菜单"命名线段" |
| 圆上取点 | 不支持 | 右键点击底圆线 → 菜单"取圆上点" |
| 母线取点 | 不支持 | 右键点击母线 → 菜单"取中点" |
| 面取点 | 不支持 | 右键点击面 → 菜单"在面上取点"（可选，可后续迭代） |

### 右键菜单系统

- 右键点击 3D 场景中的几何元素 → 根据元素类型弹出上下文菜单
- 菜单项根据上下文动态生成（棱线：取中点/等分/命名；曲线：取圆上点/命名）
- 点击菜单外区域 / ESC 关闭菜单
- 使用 HTML overlay（与标签类似），定位在鼠标点击位置

### 曲面体取点扩展

需要支持的取点场景：

| 几何体 | 可取点线条 | 取点方式 |
|--------|-----------|----------|
| 圆锥 | 底圆、母线 | 圆上按角度取点；母线按 t 参数取点 |
| 圆柱 | 底圆、顶圆、母线 | 同上 |
| 球 | 经线、赤道 | 圆上按角度取点 |

#### 曲线取点数据模型

当前 `CustomPoint` 仅支持线性插值（`edgeStart` + `edgeEnd` + `t`）。需要扩展以支持曲线上的点：

```typescript
// 扩展 CustomPoint，支持曲线上的点
interface CustomPoint {
  id: string;
  label: string;

  // ─── 线性边上的点（多面体棱线、母线）───
  type: 'edge';
  edgeStart: number;
  edgeEnd: number;
  t: number;            // 0~1 参数

  // ─── 或：曲线上的点 ───
  type: 'curve';
  lineIndex: number;     // SurfaceResult.lines 中的索引
  t: number;             // 0~1 参数，映射到 line.points 的弧长参数
}
// 用判别联合表达
```

## 风险与注意事项

1. **Raycasting 精度**：`<Line>` 组件默认不参与 Three.js raycast，需要用不可见的圆柱体或 tube 做 hitbox
2. **OrbitControls 右键冲突**：OrbitControls 默认右键拖拽是平移，需要改为仅中键平移或 Shift+右键平移
3. **曲线插值精度**：`SurfaceLine.points` 是离散点序列（64段），取点时需要在分段之间插值
4. **CustomPoint 类型变更**：扩展为判别联合后，所有使用处需适配
5. **性能**：曲线 hitbox 数量多（底圆64段），需注意渲染性能

### 自定义线段/面功能定位（2026-03-05 讨论）

**背景**：用户提出当前只能在初始几何体提供的实体上操作，缺少"连线"和"创建新面"的能力。

**决策**：
- 自定义线段（任意两点连线）→ **推迟到阶段5A** 实现
- 自定义面（多点围成面）→ **推迟到阶段5** 实现
- 3.1 任务保持原定范围（右键菜单 + 曲面体取点），不扩展

**未来交互方案**（阶段5参考）：
- 采用**工具栏模式切换**：左侧面板切换到"画线"模式 → 依次点击两点自动连线；"画面"模式 → 点击多点后确认

  }

### 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 线段点击检测 | 不可见 mesh hitbox | drei Line (Line2) raycast 不可靠，需手动设 raycaster.camera 且有分辨率/正交相机问题 |
| 直线段 hitbox | CylinderGeometry | 两端点确定，计算简单，性能好 |
| 曲线 hitbox | TubeGeometry (CurvePath) | 比 64 个独立 Cylinder 少大量 draw call |
| 曲线 t 参数 | 归一化弧长 | 即使离散精度变化，已保存点位置不偏移 |
| 右键菜单位置 | Canvas 外层 HTML DOM | 不受 R3F 渲染管线限制，定位精确 |
| OrbitControls 冲突 | 禁用右键，中键平移 | 最简洁的冲突解决方案 |
| contextMenu 状态 | Store 管理 | 多组件需读写（hitbox 触发、菜单渲染、pointerMissed 关闭） |
