## 子任务清单

### T5A-2.1 展开图算法 — 长方体

**产出文件**：`src/engine/unfolding/cuboidUnfold.ts`

**内容**：
- 输入：CuboidParams (length, width, height)
- 输出：6 个矩形的 2D 坐标 + 顶点标签映射
- 展开布局：十字形（底面居中，前后左右上5个面环绕）
- 每个面输出：4 个 2D 顶点坐标 + 对应的 3D 顶点标签
- 需要定义 `UnfoldResult` 类型

**手算验证**：长方体 (3,2,2) → 十字形总宽 = 2+3+2+3 = 10，总高 = 2+2+2 = 6

### T5A-2.2 展开图算法 — 圆锥

**产出文件**：`src/engine/unfolding/coneUnfold.ts`

**内容**：
- 输入：ConeParams (radius, height)
- 母线长 l = √(r² + h²)
- 侧面展开为扇形：半径 = l，圆心角 θ = 2πr/l（弧度）
- 底面展开为圆：半径 = r，位置在扇形下方
- 输出：扇形参数 + 圆参数 + 特征点标注

**手算验证**：圆锥 (r=1.5, h=2) → l=2.5, θ = 2π×1.5/2.5 = 1.2π (216°)

### T5A-2.3 展开图注册表

## 技术决策点（待研究阶段确认）

1. **2D 渲染方案**：Canvas 2D vs SVG vs 独立小 R3F Canvas（正交相机）
   - Canvas 2D：性能好，导出方便，但交互开发成本略高
   - SVG：天然支持缩放不失真，但复杂图形性能一般
   - 独立 R3F Canvas：与主场景技术栈统一，但引入第二个 Canvas
   - 初步倾向 **SVG**（展开图是矢量图形，缩放不失真，元素数量少性能无压力）

2. **展开图布局**：是否允许用户拖拽面来调整展开方式？
   - V1 不做：固定布局即可

3. **面板展示方式**：Dialog 弹窗 vs 右侧面板 vs 底部抽屉
   - 初步倾向 **Dialog 弹窗**（不影响主布局，使用已有 dialog.tsx 组件）

## 风险与注意事项

1. **展开图顶点标签对应**：3D 顶点标签被用户改名后（labelOverrides），展开图上也应显示新名字
2. **圆锥扇形精度**：扇形弧需要用足够多段的折线近似
3. **导出分辨率**：PNG 导出需考虑 DPI（投影仪使用场景需高分辨率）
4. **展开图面板 z-index**：如使用 Dialog，需注意与 CalcStepsModal 等的层叠关系
