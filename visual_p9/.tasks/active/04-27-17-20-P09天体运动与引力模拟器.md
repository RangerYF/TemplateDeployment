## MODE 1 研究记录
- 项目现状：`visual_p09` 当前没有 `src/`、`package.json`、Vite 配置和可构建前端源码；需要按 `visual_template` 创建新仓库实现。
- 模板架构：`visual_template` 使用 React 19 + TypeScript + Vite + Tailwind + Zustand + Three/R3F；主结构为 `TopBar + 左/中/右三栏 + 折叠面板 + token/ui 组件`。
- 可复用风格：`visual_template/src/styles/*`、`components/ui/*`、`components/layout/*` 的 token、按钮、滑块、输入、TopBar、右侧面板风格；`visual_chem02` 的 `TopBar`、`AppLayout`、`PanelSection` 体现了单产品页面布局。
- `visual_chem02` 注意项：部分文件存在 git 合并冲突标记，因此只参考布局/视觉，不直接复制冲突内容。
- P09 核心需求：单星圆/椭圆轨道、开普勒三定律/面积定律、高轨低速大周期、霍曼转移、三宇宙速度、双星系统、不同轨道追及、播放控制、参数联动、数值面板。
- 推荐技术路径：`.tasks/completed/repo_research/outputs/phys-P09-analysis.md` 建议 2D Canvas + 参数 Store + 物理纯函数 + requestAnimationFrame，避免使用完整 3D 引擎。
- 数据结构样本（来源于 P09 模型数据文档）：`{ id: string, name_cn: string, category: 'kepler'|'orbit_change'|'binary'|'chase', params: object, formulas: array, animations: object, teaching_points: array }`。
- 关键字段映射：`CEL-001` 圆轨道使用 `centralMassKg/orbitRadiusM`；`CEL-002` 椭圆轨道使用 `semiMajorAxisKm/eccentricity`；`CEL-011` 霍曼转移使用 `lowOrbitRadiusM/highOrbitRadiusM/earthMassKg`；`CEL-012` 使用三宇宙速度 `7.9/11.2/16.7 km/s`；`CEL-021` 使用 `m1Kg/m2Kg/separationKm`；`CEL-031` 使用 `innerRadiusM/outerRadiusM/initialAngleDeg`。
- 权威补充数据：NASA Earth/Moon/Sun facts 与 NASA Earth satellite orbit 页面用于核验地球/月球/太阳参数和地球同步轨道半径；NIST CODATA 用于核验万有引力常量 G。
- 2026-04-27 续作：滑块白色圆形 thumb 无法直接拖动。真实结构样本：`Slider` 由轨道 `div`、透明原生 `input[type=range]`、白色 thumb 装饰 `div` 组成；thumb 装饰层位于 input 后方渲染且覆盖在 input 上，没有 `pointer-events: none`，导致点击白色圆点时事件被装饰层拦截。
- 2026-04-27 续作：用户反馈“霍曼转移那里，能不能不固定点火位置和椭圆轨迹，用户点击点火按钮之后再生成对应位置相切的椭圆轨迹？另外，‘面积定律扇形’似乎无法点击，也没显示在画面上。”
- 续作结构样本：`buildHohmannFrame` 当前固定点火点 `{x:r1,y:0}` / `{x:-r2,y:0}`，固定转移椭圆 `ellipsePoints(..., -transferBPx)`；`fireHohmann` 仅切换 `low/transfer/high`，未保存点火角；`ControlPanel` 对非 `CEL-002` 禁用面积扇形开关；`drawSectors` 只填充低透明度扇形且无标签。
- 2026-04-27 续作：用户反馈“三宇宙速度速度为 3 时没有落回地面，也没有见到圆轨道；天体追及内轨半径不应大于外轨半径；所有模型公式与关系希望用 LaTeX 渲染。”
- 续作结构样本：`buildEscapeFrame` 当前将 `speed < 7.9` 绘制为普通抛物线片段，轨迹不保证返回地球表面；`speed === 7.9` 进入椭圆分支，没有圆轨道特例；`setParam`/`ParameterPanel` 未对 `CEL-031` 的 `innerRadiusM < outerRadiusM` 做动态约束；`FormulaPanel` 当前使用 `<code>` 直接显示普通字符串，`pnpm-lock.yaml` 已保留 `katex@0.16.33` 与 `@types/katex@0.16.8` 包记录。
- 2026-04-28 续作：用户要求左右两个栏目边缘可拖拽，在一定范围内调整三栏比例。当前 `AppLayout` 直接渲染固定宽度左侧 `ModelListPanel` 与固定宽度右侧 `aside`。
- 2026-04-28 续作：用户要求撰写仓库功能演示清单，对照 P09 需求文档和当前代码，使用【需求】标注需求内功能、【扩展】标注额外完成能力，并参考 `visual_chem02/docs/功能演示清单示例.md` 的格式。
