v0.1 已有的基础功能不再重复（12种模拟类型、SVG图表渲染、参数面板、撤销/重做、单次/多次模式切换、动画播放控制）。本清单仅覆盖 v0.2 新增/增强/修复的内容。

标记说明
- 【修复】= v0.1 反馈中明确指出的 bug
- 【优化】= v0.1 反馈中提出的体验改进
- 【增强】= 基于反馈发散的功能增强

## 一、Bug 修复

### 1. 【修复】摸球实验 n 参数不生效（反馈 #3 按键无法点击）
- **问题**：摸球实验（ballDraw）的试验次数 n 被硬编码为 1000，参数面板无法调整
- **修改**：
  - `src/types/simulation.ts`：BallDrawParams 新增 `n: number` 字段
  - `src/editor/commands/runSimulation.ts`：ballDraw 分支改用 `p.n` 替代硬编码 1000
  - `src/components/panels/inspectors/BallDrawInspector.tsx`：新增 n 参数的 ScaleNSlider（数量级选择器 + 滑块 + 无上限输入）
- **演示**：选择「摸球实验」→ 右侧参数面板拖动「试验次数」滑块 → 点击「运行模拟」→ 确认图表数据量与设定 n 一致

### 2. 【修复】约会相遇问题动画缺失（反馈 #3 + 教师反馈 #3）
- **问题**：约会相遇问题（meetingProblem）运行后无法使用「动画播放」，点击无响应
- **修改**：
  - `src/engine/singleStep.ts`：将 meetingProblem 加入 MultiAnimatableType
  - `src/components/scene/renderers/MeetingProblemRenderer.tsx`：新增 `displayN` 参数，支持逐步显示散点
  - `src/components/scene/SimulationCanvas.tsx`：向 MeetingProblemRenderer 传递 displayN
  - `src/components/panels/ControlPanel.tsx`：handleStartAnim 新增 meetingProblem 分支（从 `result.points.length` 获取 total）
- **演示**：选择「约会相遇问题」→ 运行模拟 → 点击「动画播放」→ 观察散点从 0 逐步增长到最终结果，支持暂停/继续/停止/重播

## 二、实验次数取消上限 + 数量级选择器（反馈 #1）

### 3. 【优化】实验次数参数取消上限，新增数量级选择器
- **问题**：原 v0.1 滑块上限过低（如硬币投掷仅 10,000），输入框也有上限，不足以展示大数定律收敛效果
- **修改**：新建通用组件 `src/components/ui/scale-n-slider.tsx`（ScaleNSlider），替换全部 8 个 Inspector 的 n 参数控件：
  - **输入框无上限**：点击数字可直接输入任意正整数（技术上限 999,999,999）
  - **数量级选择器**：滑块下方提供「千 / 万 / 十万 / 百万 / 千万」五档按钮，点击切换滑块的最大范围
  - **滑块自适应**：在选定数量级范围内滑动，步进随数量级自动调整（千→步进10，万→步进100，十万→步进1000，百万→步进10000，千万→步进100000）
  - **自动升档**：若输入框输入的值超出当前数量级，数量级自动升档匹配

| 模拟类型 | 涉及文件 | 默认数量级 |
|---------|---------|-----------|
| 硬币投掷 | CoinFlipInspector.tsx | 千（1,000） |
| 骰子投掷 | DiceRollInspector.tsx | 千（1,000） |
| 两骰之和 | TwoDiceSumInspector.tsx | 千（1,000） |
| 摸球实验 | BallDrawInspector.tsx | 千（1,000） |
| 蒙特卡洛求π | MonteCarloPiInspector.tsx | 万（10,000） |
| 布丰投针 | BuffonsNeedleInspector.tsx | 千（1,000） |
| 约会相遇 | MeetingProblemInspector.tsx | 千（1,000） |
| 大数定律 | LawOfLargeNumbersInspector.tsx | 千（1,000） |

- **演示**：
  1. 选择任一模拟 → 观察滑块下方的数量级选择器（千/万/十万/百万/千万）
  2. 点击「万」→ 滑块范围变为 10~10,000，拖动滑块
  3. 点击「千万」→ 滑块范围变为 10~10,000,000
  4. 点击输入框直接输入 5000000 → 确认数量级自动切换到「千万」
  5. 运行模拟 → 确认图表正常渲染

## 三、教学说明增强（反馈 #2 + #4 + #6）

### 4. 【增强】所有模拟新增详细教学说明
- **问题**：v0.1 参数面板仅有参数滑块，缺乏对实验原理的说明，学生不理解模拟目的
- **修改**：`src/components/panels/inspectors/InspectorPanel.tsx` 新增 `SIM_DETAILS` 对象，为全部 14 种模拟（含直方图、二项分布）提供：
  - **标题**：实验名称
  - **详细说明**：实验原理的教学描述（2-4 句话）
  - **现实案例**（部分模拟）：与生活实际结合的应用场景描述
- **涉及的模拟**（含现实案例的标注 ★）：
  - ★ 硬币投掷 — 体育比赛开球权
  - ★ 骰子投掷 — 桌游设计
  - ★ 两骰之和 — 大富翁骰子概率
  - ★ 摸球实验 — 质检抽样
  - ★ 蒙特卡洛求π — 随机模拟在工程中的应用
  - ★ 约会相遇问题 — 等公交概率
  - ★ 布丰投针 — 几何概率与π的关系
  - 大数定律、直方图、二项分布、正态分布、超几何分布、线性回归
- **UI 交互**：可折叠面板，带 Info 图标，默认展开，点击可收起节省空间
- **演示**：切换不同模拟类型 → 观察参数面板顶部的说明区域 → 点击收起/展开 → 阅读「蒙特卡洛求π」的现实案例说明

## 四、SVG 图表文字重叠修复（教师反馈 #2）

### 5. 🔍【修复】正态分布 σ 区域标签重叠
- **问题**：正态分布图中 68.27%、95.45%、99.73% 三行标签在同一 y 坐标上重叠
- **修改**：`NormalDistRenderer.tsx` — 三组标签分层错开显示：
  - ±1σ (68.27%) 位于图表下方
  - ±2σ (95.45%) 位于图表中部
  - ±3σ (99.73%) 位于图表上部
- **演示**：选择「正态分布」→ 运行 → 确认三行百分比标签无重叠，清晰可读
- **⚠ 需手动确认**：不同屏幕分辨率下标签是否仍然清晰不重叠

### 6. 🔍【修复】频率直方图底部统计信息溢出 + X轴刻度过密
- **问题**：直方图底部统计信息（n/均值/标准差/范围/组距）一行放不下导致文字截断；组数多时X轴数字重叠
- **修改**：`HistogramRenderer.tsx` —
  - 底部统计信息拆为两行显示
  - X轴刻度根据总数自动跳步（≤12全显示，≤24隔一个，>24按 ceil(n/10) 跳步）
- **演示**：选择「频率直方图」→ 选择不同数据集（尤其数据点多的）→ 确认底部统计完整显示，X轴刻度不重叠
- **⚠ 需手动确认**：多数据集下底部两行是否都正常显示

### 7. 🔍【修复】二项分布统计面板遮挡图表
- **问题**：二项分布图右侧统计面板文字与柱状图重叠
- **修改**：`BinomialDistRenderer.tsx` — 统计面板增加背景矩形，字号从 12→11 调整间距
- **演示**：选择「二项分布」→ 设置较大 n 值 → 运行 → 确认统计面板背景清晰，不遮挡柱状图
- **⚠ 需手动确认**：不同参数组合下统计面板位置是否合理

### 8. 🔍【修复】超几何分布统计信息溢出
- **问题**：超几何分布底部统计（E(X)/D(X)/σ/模式）一行显示超出 SVG 宽度
- **修改**：`HypergeometricDistRenderer.tsx` — 统计信息拆为两行
- **演示**：选择「超几何分布」→ 运行 → 确认底部统计信息两行完整显示
- **⚠ 需手动确认**：极端参数下是否仍然整齐

### 9. 🔍【修复】骰子投掷事件统计面板行距不足
- **问题**：骰子投掷图表右侧事件统计面板各行文字间距过紧，视觉拥挤
- **修改**：`DiceRollRenderer.tsx` — 面板高度 110→120，行间距 20px→24px
- **演示**：选择「骰子投掷」→ 运行 → 查看右侧事件统计面板各行间距是否舒适
- **⚠ 需手动确认**：多骰子模式下事件行数增多时是否仍然不溢出

## 五、UI 一致性优化（教师反馈 #1 + design-align.md）

### 10. 【优化】按钮禁用态统一
- **问题**：禁用按钮样式不一致，部分按钮禁用后视觉无变化
