# 单步模拟专属视觉动画技术调研报告

> 作者：Claude Sonnet 4.6 / 2026-03-13
> 范围：掷硬币（SIM-001）、掷骰子（SIM-002）在单次模拟模式下的专属视觉动画

---

## 1. 背景与范围

### 1.1 已有的效果（不在本文讨论范围）

- **柱状图数值更新**：每次 `handleSingleStep` 调用 `setResult` 后，React 重渲染使柱高即时跳变到新值，功能已覆盖。
- **多步模拟动画**：`useAnimationEngine` + `animStep` 驱动的逐帧播放，已完整实现。

### 1.2 本文目标

为单步模式增加**硬币翻转**和**骰子滚动**的专属视觉动画，让用户感受到"随机过程正在发生"，而不是结果瞬间出现。

**期望效果：**
- 掷硬币：一枚硬币在空中旋转翻转，落定后正面/反面亮起
- 掷骰子：骰子快速滚动（点数随机闪烁），逐渐减速，最终停在目标点数

### 1.3 现有技术栈约束

- React 19 + Zustand 5，纯 SVG 渲染
- 无 Framer Motion、GSAP 等动画库
- 当前单步触发链路**完全同步**：点击 → 生成结果 → 立即更新 UI，无任何延迟
- TypeScript strict 模式，tsc 0 错误门禁

## 2. 竞品分析（同类教育工具）

### 3.1 GeoGebra 概率模拟器

- **实现方式**：HTML5 Canvas + JavaScript，骰子用 3D CSS transform 旋转动画
- **效果**：骰子旋转 600ms 后停止并显示点数，同时柱状图平滑更新
- **优点**：视觉冲击力强
- **缺点**：3D 旋转成本较高，需要额外的骰子素材图

### 3.2 Khan Academy 概率课

- **实现方式**：SVG + CSS `transition`
- **效果**：硬币用 CSS `rotateY` 翻转（500ms），柱子高度用 CSS `transition: height 300ms`
- **优点**：实现简单，纯 CSS 无需额外依赖
- **缺点**：CSS 动画与 SVG 的兼容性在部分浏览器上不稳定（`height` 在 SVG 中无法直接 transition）

### 3.3 Seeing Theory（布朗大学）

- **实现方式**：D3.js + SVG transition
- **效果**：柱状图使用 `d3.transition().duration(300)` 平滑过渡，硬币用 CSS `animation`
- **优点**：D3 的 transition 系统非常成熟
- **缺点**：引入 D3 体积约 +70KB，与现有纯 SVG 架构存在重叠

### 3.4 Wolfram Demonstrations

- **实现方式**：Mathematica 生成 GIF / WebGL
- **效果**：高质量 3D 动画，但交互性差
- **缺点**：不适合实时交互教学场景

### 3.5 小结

多数竞品采用 **CSS 动画（硬币/骰子视觉效果）+ CSS transition 或 requestAnimationFrame（图表过渡）** 的组合。核心诉求是：
1. 用户看到"随机过程"而非"瞬间结果"
2. 图表更新不突兀，有平滑感

## 4. 技术路线对比

### 方案 A：CSS `animation` + SVG `transform`（推荐，成本最低）

**核心思路**

在 `singleLastDisplay` 更新时，给显示结果的 SVG 元素（或 HTML div）加一个短暂的 CSS keyframe animation，视觉上模拟"翻转/滚动"效果，然后静止显示结果。

**硬币动画**

```css
@keyframes coin-flip {
  0%   { transform: rotateY(0deg) scaleX(1); }
  40%  { transform: rotateY(90deg) scaleX(0); }   /* 侧面消失 */
  50%  { transform: rotateY(90deg) scaleX(0); }   /* 切换正反 */
  60%  { transform: rotateY(90deg) scaleX(0); }
  100% { transform: rotateY(0deg) scaleX(1); }    /* 落定显示 */
}
```

用一个 `<div>` 或 SVG `<text>` 显示 H/T，绑定 `animation` 类，每次 `singleLastDisplay` 变化时通过 `key` 属性强制重触发：

```tsx
<div key={animKey} className="coin-animate">
  {result === 'H' ? '正面' : '反面'}
</div>
```

**骰子动画**

用 SVG 绘制一个简单骰子图（6面 `<rect>` + 点阵），加 `rotateZ` 的 keyframe：

```css
@keyframes dice-roll {
  0%   { transform: rotate(0deg); }
  25%  { transform: rotate(90deg); }
  50%  { transform: rotate(180deg); }
  75%  { transform: rotate(270deg); }
  100% { transform: rotate(360deg); }
}
```

动画时长约 400–600ms，结束后静止显示最终点数。

**图表柱高过渡**

SVG `<rect>` 的 `height` 和 `y` 属性可以用 CSS transition（在现代浏览器中 SVG 属性支持 transition）：

```css
.bar-rect {
  transition: height 250ms ease-out, y 250ms ease-out;
}
```

**优点**
- 零依赖，不增加 bundle 体积
- 与现有 SVG 架构完全兼容
- 实现成本低（1–2天）

**缺点**
- SVG 属性 transition 在部分旧 Safari 有兼容性问题（需测试）
- 复杂3D效果受限（`rotateY` 在 SVG 中需借助 `perspective`）
- CSS keyframe 与 React state 结合需要 `key` trick 或 `useEffect` 控制

**与现有框架的结合点**

在 `handleSingleStep()` 完成后，将 `animKey` 自增：

```typescript
// animationStore 新增字段
singleAnimKey: number;  // 每次单步 +1，驱动 CSS 动画重触发

// pushSingleTrial 内部
set(s => ({ ..., singleAnimKey: s.singleAnimKey + 1 }));
```

渲染器中：
```tsx
// CoinFlipRenderer 新增一个动画覆盖层
<g key={singleAnimKey} style={{ animation: 'coin-flip 500ms ease-out' }}>
  <text x={cx} y={cy}>{lastResult === 'H' ? 'H' : 'T'}</text>
</g>
```

### 方案 B：`requestAnimationFrame` 手写补间（灵活，成本中等）

**核心思路**

不依赖 CSS animation，用 `useRef` + `requestAnimationFrame` 手写一个简单的补间引擎，每帧更新 SVG 元素的位置/透明度/旋转角。

**示例：骰子点数"滚动"效果**

```typescript
function useDiceRollAnim(targetFace: number) {
  const [displayFace, setDisplayFace] = useState(targetFace);
  const rafRef = useRef<number>();
  const startRef = useRef<number>();

  useEffect(() => {
    const duration = 500; // ms
    startRef.current = performance.now();

    const tick = (now: number) => {
      const elapsed = now - (startRef.current ?? now);
      const progress = Math.min(elapsed / duration, 1);

      // 在动画过程中随机切换显示的点数，产生"滚动"感
      if (progress < 0.9) {
        setDisplayFace(Math.floor(Math.random() * 6) + 1);
      } else {
        setDisplayFace(targetFace); // 落定
      }

      if (progress < 1) rafRef.current = requestAnimationFrame(tick);
    };

    rafRef.current = requestAnimationFrame(tick);
    return () => cancelAnimationFrame(rafRef.current!);
  }, [targetFace]);

  return displayFace;
}
```

**优点**
- 完全控制动画逻辑（缓动函数、时序）
- 与 SVG 原生兼容，无第三方依赖
- 可实现"随机数字快速滚动后停止"的经典效果

**缺点**
- 代码量较大（每种模拟需单独写补间逻辑）
- 需要管理 `useEffect` 清理，防止组件卸载后 RAF 泄漏

**与现有框架的结合点**

新增 `hooks/useSingleStepAnim.ts`，在 `ControlPanel` 的 `handleSingleStep` 触发后激活动画：

```typescript
// animationStore 新增
singleAnimating: boolean;  // 动画进行中（禁用再次点击）

// handleSingleStep 流程改造
animStore.setSingleAnimating(true);
// ... 运行试验、更新结果 ...
// 动画完成回调
setTimeout(() => animStore.setSingleAnimating(false), 550);
```

### 方案 C：引入 Framer Motion（效果最佳，但有依赖）

**核心思路**

使用 Framer Motion 的 `<motion.div>` 和 `animate` API，对结果展示层做动画。

```tsx
import { motion, AnimatePresence } from 'framer-motion';

<AnimatePresence mode="wait">
  <motion.div
    key={singleAnimKey}
    initial={{ rotateY: 90, opacity: 0 }}
    animate={{ rotateY: 0, opacity: 1 }}
    exit={{ rotateY: -90, opacity: 0 }}
    transition={{ duration: 0.4 }}
  >
    {lastResult === 'H' ? '正面' : '反面'}
  </motion.div>
</AnimatePresence>
```

**优点**
- 动画质量高，3D 翻转效果自然
- `AnimatePresence` 完美处理进出场动画
- API 声明式，与 React 融合最佳

**缺点**
- **bundle 体积 +52KB**（gzip 后约 +17KB）
- Framer Motion 主要针对 HTML DOM 元素，SVG `<rect>` 等直接使用需要额外处理
- 项目当前无此依赖，引入需用户确认

### 方案 D：纯 SVG SMIL 动画（无 JS，最简单）

**核心思路**

SVG 原生支持 `<animate>` 和 `<animateTransform>` 元素，无需 JS：

```svg
<rect x="40" y="100" width="60" height="0" fill="#00C06B">
  <animate attributeName="height" from="0" to="80" dur="0.3s" fill="freeze" />
  <animate attributeName="y" from="100" to="20" dur="0.3s" fill="freeze" />
</rect>
```

**缺点**
- 每次 React 重渲染时，SMIL 动画会重置（与 React 声明式更新冲突严重）
- 现代浏览器对 SMIL 支持参差不齐（Chrome 曾多次考虑废弃）
- **不推荐**在 React 项目中使用

## 5. 推荐方案与实施路径

### 推荐：方案 A（CSS 动画）+ 方案 B 局部使用

**分两个阶段实施：**

#### 阶段一：图表柱高平滑过渡（1–2小时）

在 `CoinFlipRenderer` 和 `DiceRollRenderer` 的 `<rect>` 柱子元素上添加 CSS transition，柱高变化时自动平滑：

```tsx
// svgChartUtils 新增工具
// 或直接在渲染器 rect 上加 style
<rect
  style={{ transition: 'height 250ms ease-out, y 250ms ease-out' }}
  x={bx} y={by} width={barW} height={bh}
/>
```

**注意**：SVG `height`/`y` 属性 transition 需要用 `style` 写法而非 SVG attribute，现代浏览器均支持。

#### 阶段二：结果呈现动画（0.5–1天）

在 ControlPanel 的单步结果显示区域（`singleLastDisplay` badge）外层包一个带 `key` 的容器，利用 React 的 reconciliation 机制在 key 变化时重新挂载并触发 CSS 进入动画：

```tsx
// animationStore 新增
singleAnimKey: 0,

// pushSingleTrial 内：set(s => ({ singleAnimKey: s.singleAnimKey + 1, ... }))
```

```css
/* index.css 新增 */
@keyframes pop-in {
  0%   { transform: scale(0.5) translateY(-8px); opacity: 0; }
  60%  { transform: scale(1.08) translateY(0); opacity: 1; }
  100% { transform: scale(1) translateY(0); opacity: 1; }
}

.result-pop {
  animation: pop-in 350ms cubic-bezier(0.34, 1.56, 0.64, 1);
}
```

```tsx
// ControlPanel 中
<div key={singleAnimKey} className="result-pop">
  {singleLastDisplay}
</div>
```

#### 可选阶段三：硬币/骰子专属视觉（1天）

在 SimulationCanvas 的单步结果叠加层（目前的 `singleLastDisplay` overlay）替换为：
- **硬币**：用 SVG 绘制圆形 + 文字，加 `rotateY` keyframe（需 `perspective` wrapper）
- **骰子**：用 `useDiceRollAnim` hook 实现随机数字滚动效果（方案 B 的 RAF 方式）

### 与现有框架结合点汇总

| 修改位置 | 改动内容 | 影响范围 |
|---------|---------|---------|
| `animationStore.ts` | 新增 `singleAnimKey: number` 字段，`pushSingleTrial` 时自增 | 小，只新增字段 |
| `CoinFlipRenderer.tsx` | `<rect>` 加 CSS transition style | 仅渲染器 |
| `DiceRollRenderer.tsx` | `<rect>` 加 CSS transition style | 仅渲染器 |
| `ControlPanel.tsx` | `singleLastDisplay` badge 加 `key={singleAnimKey}` + CSS 类 | 小，只加 key 和类名 |
| `SimulationCanvas.tsx` | 单步结果 overlay 替换为带动画的组件（可选） | 中 |
| `src/styles/` 或 `index.css` | 新增 `@keyframes` 定义 | 独立，无侵入 |

### 风险评估

| 风险 | 等级 | 应对 |
|------|------|------|
| SVG `height` transition 在旧 Safari 不生效 | 低 | 降级无动画，核心功能不受影响 |
| CSS keyframe 与 React `key` 冲突 | 低 | 使用 `key` trick 是 React 官方支持的模式 |
| 动画期间用户连续点击导致状态错乱 | 中 | 动画进行中禁用按钮（`singleAnimating` 标志） |
| 新增字段破坏 TSC | 低 | `singleAnimKey` 初始为0，类型简单 |

## 6. 结论

| 方案 | 推荐度 | 新增依赖 | 实施成本 | 效果 |
|------|--------|---------|---------|------|
| A CSS 动画 | ⭐⭐⭐⭐⭐ | 无 | 低（半天） | 中 |
| B RAF 手写 | ⭐⭐⭐⭐ | 无 | 中（1天） | 中高 |
| C Framer Motion | ⭐⭐⭐ | +52KB | 低（半天） | 高 |
| D SMIL | ⭐ | 无 | 低 | 差（兼容差） |

**最终建议：** 先以方案 A 实现"柱高平滑过渡 + 结果 pop-in"，零成本、零风险、可立即上线。若后续需要更精细的硬币翻转/骰子滚动效果，再用方案 B 的 RAF hook 补充，两者可独立开发互不影响。
