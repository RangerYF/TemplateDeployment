生成今日工作日报。

## 执行步骤

### 第一步：获取今天的 git 提交记录
- 用 `git log --since="today 00:00:00" --format="%h %ai %s" --reverse` 查看今天所有提交
- 用 `git log --since="today 00:00:00" --format="%h %s" --reverse` 配合 `git diff --shortstat` 逐提交统计代码行数
- 用 `git diff --shortstat $(第一个提交的parent)..$(最后一个提交)` 统计今日总变更行数

### 第二步：提取今天的 Claude Code 对话记录
- 对话记录目录：`~/.claude/projects/` 下与当前项目路径对应的目录（路径中 `/` 替换为 `-`）
- 筛选今天修改的 `.jsonl` 文件
- 每个 JSONL 文件是一个会话，从中提取 `type=="user"` 的消息（`message.content` 字段，可能是字符串或数组）
- 用 python3 脚本批量提取每个会话中用户发送的所有完整消息文本（不截断）

### 第三步：综合分析并输出日报
结合 git 提交记录和对话记录，从**具体功能**的角度总结今天的工作。

## 输出格式要求

用中文输出，格式如下：

```
## [项目名] — M月D日工作日报

### 今日完成
（用 3-6 个要点，从用户可感知的功能角度描述，不要用内部阶段编号）
（每个要点 1-2 句话，简明扼要）

### 代码统计
| 提交 | 新增 | 删除 | 净增 |
|------|------|------|------|
| 提交说明1 | +xxx | -xxx | +xxx |
| ... | ... | ... | ... |
| **合计** | **+xxx** | **-xxx** | **+xxx** |

### 当前状态
（一句话说明项目当前进展到哪里，下一步要做什么）
```

## 注意事项
- 面向非技术同事，不要出现内部阶段编号、模块名、文件路径
- 用"功能语言"描述，比如"支持了XX功能"、"修复了XX问题"
- 简要、概括、好理解
- 如果今天没有提交记录，如实说明
