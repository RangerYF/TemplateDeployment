## 任务目标

优化"画线段"和"截面"功能的进入/退出/操作交互体验，使老师能更直观地理解当前处于什么模式、如何操作、如何退出。

---

## 当前问题分析

### 问题1：进入模式不够醒目
- 画线/截面按钮很小（`px-2 py-0.5 text-xs`），混在辅助功能面板中
- 进入模式后仅按钮自身变色（红/蓝），3D 场景区域无任何视觉提示
- 老师可能不知道自己已经进入了"画线模式"

### 问题2：退出模式不便
- 只能通过再次点击按钮退出，无 Escape 键支持
- 画线完成一条后自动停留在画线模式（继续等待下一条起点），无明确的"完成"出口
- 截面模式无"确认完成选点"机制

### 问题3：起点反馈缺失
- 画线模式选中第一个点后，仅有文字提示"点击第二个点"
- 被选中的起点顶点无视觉高亮，用户不确定选没选上

### 问题4：截面可用性判断不完整
- `crossSectionAvailable = result?.kind === 'polyhedron'` 排除了球体
- 但球体截面已在 5B 实现（`sphereCrossSection.ts`），按钮却仍禁用
- 球体用户看到"仅多面体可用"会困惑

### 问题5：几何体切换未清理绘制状态
- 切换几何体类型时，drawingMode/crossSectionPoints/customSegments 未重置
- 可能导致：线段引用旧顶点 → resolvePointRef 返回 null → 线段消失但残留数据

## 验收标准

- [ ] 进入画线/截面模式时，3D 场景上方出现模式提示条（含退出按钮）
- [ ] 按 Escape 键可退出当前绘制模式
- [ ] 画线模式下，已选起点有视觉高亮标记
- [ ] 球体截面按钮可用（修复 crossSectionAvailable 条件）
- [ ] 切换几何体类型时自动退出绘制模式并清理截面点
- [ ] 自定义线段在几何体切换时自动清理（因 PointRef 可能失效）
- [ ] 三视图和展开图默认关闭，点击开关才展开
- [ ] `pnpm lint && pnpm tsc --noEmit` 通过

### T6.4.4 修复截面按钮对球体的可用性

**修改文件**：`src/components/panels/AuxiliaryTools.tsx`

**问题代码**：
```typescript
const crossSectionAvailable = result?.kind === 'polyhedron';
```

**修改为**：
```typescript
const crossSectionAvailable =
  result?.kind === 'polyhedron' ||
  (result?.kind === 'surface' && result?.geometryType === 'sphere');
```

**同时更新**不可用提示文本：
```
"仅多面体可用" → "仅多面体和球体可用"
```

---

### T6.4.5 几何体切换时清理绘制状态

**修改文件**：`src/store/useGeometryStore.ts`

**动作**：
- 在 `setType` action 中，切换几何体类型时重置绘制相关状态：
  - `drawingMode: 'none'`
  - `drawingStartRef: null`
  - `crossSectionPoints: []`
  - `crossSectionEnabled: false`
  - `customSegments: []`（PointRef 引用旧几何体顶点，不再有效）
- 同时重置坐标系和外接相关状态（coordinateOriginIndex 等指向旧顶点）

## 技术要点

- **Escape 键监听**：需在 window 级别（`document.addEventListener`），不能依赖 Canvas focus。注意与其他组件的键盘事件不冲突（如顶点标签编辑中的 Escape）
- **提示条定位**：3D Canvas 区域是 `<Canvas>` 组件，需要在 Canvas 外部用绝对定位覆盖，或使用 R3F 的 `<Html>` 组件
- **起点高亮**：VertexLabels 中已有 `drawingStartRef` 可从 store 读取，与顶点 index 对比即可
