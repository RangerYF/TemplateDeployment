### T1.5 定义几何体类型与 Zustand Store

**产出文件**：
- `src/types/geometry.ts`
- `src/store/useGeometryStore.ts`

**验收**：TypeScript 类型无报错，store 可在组件中正常 hook 调用

### T1.7 集成 3D Canvas 与 OrbitControls

**目标**：在中央区域渲染 R3F 3D 画布，显示占位立方体线框，支持鼠标旋转/缩放/平移。

**动作**：
1. 创建 `src/components/scene/Scene3D.tsx`：
   - 使用 `@react-three/fiber` 的 `<Canvas>` 组件
   - 设置相机初始位置、背景色
   - 集成 `@react-three/drei` 的 `<OrbitControls>`（enablePan、enableZoom）
   - 添加基础光照（ambientLight + directionalLight）
2. 渲染占位立方体：
   - 使用 `<mesh>` + `<boxGeometry>` + `<meshBasicMaterial wireframe>`
   - 或使用 `<Edges>` 组件显示线框
   - 立方体位于场景中心
3. 在 AppLayout 中央区域嵌入 `<Scene3D />`
4. 确保 Canvas 自适应父容器尺寸（`style={{ width: '100%', height: '100%' }}`）

**产出文件**：
- `src/components/scene/Scene3D.tsx`
- `src/App.tsx`（更新，或通过 AppLayout 集成）

**验收**：
- 中央区域显示 3D 立方体线框
- 鼠标左键拖动旋转
- 滚轮缩放
- 右键平移
- 窗口缩放时 Canvas 自适应

## 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 包管理器 | pnpm | 用户项目约定 |
| 路径别名 | `@/` → `src/` | 与 design_guid 组件导入路径一致 |
| 状态管理 | Zustand | 主计划指定，轻量且适合 |
| 3D 渲染 | R3F + drei | 主计划指定，React 生态集成最佳 |
| UI 组件来源 | design_guid/ 移植 | 复用已有设计系统，保持视觉一致 |
| Tailwind 版本 | v3（配合 postcss） | 稳定版，design_guid 配置兼容 |

## 风险与注意事项

1. **两套色彩 token 并存**：`tokens.ts` 主色 `#00C06B`（UI 组件 inline style），`colors.ts` 主色 `#32D583`（Tailwind 类名）。主计划明确以 `#00C06B` 为准，后续可能需统一。
2. **UI 组件 "use client" 指令**：部分组件有 Next.js 的 `"use client"` 指令，在 Vite 项目中无影响但可选清理。
3. **@tailwindcss/typography 插件**：本阶段不直接使用，但 tailwind 配置中保留以备后续 Markdown 渲染。
