## 数据架构设计

### 核心数据模型

标注系统需要在 Builder 输出的只读几何数据之上，维护一层**用户编辑叠加层**（overlay），存储在 Store 中：

```typescript
// 标注叠加数据（每次切换几何体类型或参数变化时重置）
interface AnnotationState {
  /** 自定义顶点标签覆盖，key = 顶点索引, value = 新标签 */
  labelOverrides: Record<number, string>;

  /** 选中状态 */
  selectedVertices: number[];       // 选中的顶点索引
  selectedEdge: [number, number] | null;  // 选中的棱线 [startIdx, endIdx]

  /** 自定义点（中点/边上自由点） */
  customPoints: CustomPoint[];

  /** 线段命名 */
  edgeLabels: Record<string, string>;  // key = "startIdx-endIdx", value = 名字
}

interface CustomPoint {
  id: string;
  /** 所在边的两端顶点索引 */
  edgeStart: number;
  edgeEnd: number;
  /** 在边上的参数 t (0~1)，0.5 为中点 */
  t: number;
  /** 标签 */
  label: string;
}
```

### 统一点引用方案

多面体和曲面体的点来源不同：
- **多面体**：`result.vertices[index]` — 用索引引用
- **曲面体**：`result.featurePoints[index]` — 用索引引用

两者都是 `LabeledPoint[]` 结构，标注系统通过统一的索引访问，无需区分几何体类型。

### 顶点拖拽约束

- **多面体**：拖拽直接修改顶点坐标（存储为 `positionOverrides`），面和棱自动跟随（因为都是索引引用）
- **曲面体**：特征点拖拽暂不支持（曲面由参数定义，拖拽特征点语义不清晰）。阶段3仅实现多面体顶点拖拽。

### T3.3 实现选中状态与高亮

**目标**：点击顶点/棱线可选中，选中元素有视觉高亮反馈。

**修改文件**：
- `src/components/scene/GeometryRenderer.tsx` — 顶点/棱线根据选中状态切换样式

**产出文件**：
- `src/hooks/useInteraction.ts` — 新建：交互逻辑 hook（点击选中）

**技术要点**：
- 顶点选中：点击标签或顶点位置的不可见 sphere（hitbox）
- 棱线选中：点击两个顶点依次选中 → 若两点构成一条棱线则选中该棱线
- 选中顶点高亮：标签背景变为主色、字色变白
- 选中棱线高亮：棱线颜色变为橙色/加粗
- 再次点击已选中元素 → 取消选中
- 点击空白区域 → 清除全部选中

**交互流程**：
```
点击顶点 A → selectedVertices = [0]
点击顶点 B → selectedVertices = [0, 1]
  若 A-B 构成棱线 → selectedEdge = [0, 1]，触发线段命名输入
```

**验收**：
- 点击长方体顶点 A，标签高亮
- 再点击 B，若 A-B 是棱线则棱线高亮
- 点击空白区取消选中

### T3.6 实现顶点拖拽（仅多面体）

**目标**：按住多面体顶点拖动，关联面和棱自动跟随更新。

**修改文件**：
- `src/hooks/useInteraction.ts` — 扩展：拖拽逻辑
- `src/components/scene/GeometryRenderer.tsx` — 拖拽交互绑定

**技术要点**：
- 仅多面体（`kind === 'polyhedron'`）支持拖拽
- 拖拽使用 R3F 的 pointer events（onPointerDown → onPointerMove → onPointerUp）
- 拖拽时需禁用 OrbitControls（避免冲突）
- 拖拽映射：将屏幕 2D 移动映射到 3D 空间的平面移动（使用与相机视角垂直的平面）
- 拖拽结果存入 `store.positionOverrides[index]`
- 渲染时：`finalPosition = positionOverrides[index] ?? vertices[index].position`
- 拖拽时面和棱使用 finalPosition 渲染，自动跟随

**约束**：
- 拖拽不改变 Builder 的原始数据（overlay 模式）
- 修改参数后 positionOverrides 清空（几何体重建）

**验收**：
- 长方体：按住顶点 A 拖动，关联的 3 条棱和 3 个面跟随变形
- 松开后顶点停留在新位置
- 修改参数后恢复原始形状

## 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 标注数据存储 | Store overlay 模式 | 不修改 Builder 只读数据，标注层独立于几何构建 |
| 顶点拖拽范围 | 仅多面体 | 曲面体由参数定义，拖拽特征点语义不清晰 |
| 点击检测 | 不可见 hitbox sphere | 比 raycasting 到线条更可靠，点击面积适中 |
| 拖拽映射 | 相机垂直平面投影 | 符合直觉，避免深度方向误操作 |
| 标签编辑 | Html + input 原生输入框 | 比 3D 文本输入更可靠、支持 IME、无字体问题 |
| 参数变化重置 | 自动清空标注 | 几何体重建后旧标注索引可能失效 |
| 自定义点坐标 | t 参数 + 实时计算 | 参数变化后端点移动，t 值不变保持相对位置 |

---

## 风险与注意事项

1. **OrbitControls 与拖拽冲突**：拖拽顶点时必须临时禁用 OrbitControls，否则会同时旋转。需通过 ref 控制 `enabled` 属性。
2. **Raycasting 精度**：棱线很细难以点击，使用不可见 sphere hitbox 提升体验。sphere 半径需适配缩放级别。
3. **曲面体交互有限**：球只有 1 个特征点（O），无棱线，交互功能受限。需确保 UI 状态正确（无棱线时禁用"取中点"按钮）。
4. **标签重叠**：多个标签靠近时可能重叠遮挡。本阶段暂不处理（后续阶段6打磨时优化）。
5. **性能**：拖拽时每帧重新计算面和棱线坐标。长方体（8顶点12棱）无压力，但需确认棱锥（最多9顶点16棱）也流畅。
