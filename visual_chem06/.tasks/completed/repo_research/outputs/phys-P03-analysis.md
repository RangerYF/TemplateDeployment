# P-03 光学实验台 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中物理教师（不要求编程基础），部分透镜成像内容面向初中教师
- **解决的教学痛点**：
  - 折射、全反射的光路靠手绘容易出错，无法动态调整入射角观察临界角变化
  - 干涉、衍射现象极度抽象，粉笔图无法展示条纹随参数的实时变化
  - 薄膜干涉（牛顿环、楔形膜）的图样形状难以手绘，学生缺乏直观感知
  - 透镜成像的五种情况（u > 2f / u = 2f / f < u < 2f / u = f / u < f）切换不直观
- **核心产品价值一句话概括**：提供可交互的光路追迹与波动光学演示环境，让折射/全反射光路和干涉/衍射图样实时可见可调，替代无法在课堂完成的光学实验

## 2. 用户实际使用方式

- **课前准备**：教师预设好实验场景（如选定"半球形玻璃砖"实验，设好折射率和入射角），保存为预设状态，上课直接打开
- **课中演示**：
  - 几何光学：投影仪上实时拖拽入射角滑块，学生观察折射角变化，逐步逼近临界角时看到全反射发生（反射光线高亮闪烁）
  - 透镜成像：拖拽物体位置，实时看到像的位置、大小、虚实变化，对应成像规律五种情况
  - 波动光学：调节双缝间距/波长/屏距，实时看到干涉条纹疏密变化；切换不同薄膜模型看图样形态
- **讲题辅助**：遇到全反射临界角计算题时，用工具验证 sin θc = n2/n1 的计算结果
- **"好用"的标准**：参数调节后画面响应延迟 < 100ms；角度标注、法线、公式验证等辅助信息可一键开关；光线颜色对应真实波长
- **交互频率与深度**：
  - 几何光学部分交互较深——需要选择不同形状的光学元件（矩形/半球/光导纤维），调整多个参数
  - 波动光学部分交互较浅——主要是参数驱动演示，调缝间距/波长看图样变化

## 3. 核心功能点

### 功能清单（按重要性排序）

| 序号 | 功能 | 类型 | 重要性 |
|------|------|------|--------|
| 1 | 折射与全反射光路追迹（平行界面、矩形玻璃砖、半球形玻璃砖、光导纤维） | 参数驱动 + 实时渲染 | P0 |
| 2 | 双缝干涉条纹实时演示 | 参数驱动演示 | P0 |
| 3 | 单缝/圆孔衍射图样 | 参数驱动演示 | P0 |
| 4 | 凸/凹透镜成像（初中内容） | 参数驱动 + 拖拽 | P0 |
| 5 | 薄膜干涉图样（肥皂泡、楔形膜、牛顿环） | 参数驱动演示 | P1 |
| 6 | 全反射临界角标注与公式验证 | 辅助显示 | P1 |
| 7 | 光线颜色映射（波长 → RGB） | 视觉增强 | P1 |
| 8 | 强度分布曲线图表 | 图表联动 | P1 |
| 9 | 角度标注、法线显示 | 辅助标注 | P2 |

### 功能分类分析

- **需要实体管理的功能**：几何光学部分的光学元件（玻璃砖形状选择、光源位置）、透镜成像的物体和透镜——但实体种类有限且固定，不需要完整的创建/删除/自由组合系统，更接近"场景预设 + 参数调节"模式
- **需要 undo/redo 的功能**：基本不需要。本模块不是编辑器型产品，用户操作主要是调参数和切换实验模式，不存在需要撤销的复杂编辑链
- **参数驱动型功能**（调参数 → 画面实时变化）：**几乎全部功能**都是参数驱动型
  - 折射模块：入射角、折射率、玻璃砖厚度/半径
  - 干涉模块：缝间距 d、屏距 L、波长 λ
  - 衍射模块：缝宽 a / 孔径 D、波长 λ
  - 薄膜模块：楔角 α、膜折射率 n、波长 λ、曲率半径 R
  - 透镜模块：焦距 f、物距 u
- **模拟/动画驱动型功能**：本模块**不需要**时间轴控制（播放/暂停/逐步），所有现象都是静态光路或静态图样，参数变化后立即重新计算和渲染

## 4. 核心依赖识别

### 技术依赖

- **渲染方式**：
  - 几何光学（光路追迹）：**2D Canvas**。光线是直线段的几何计算与绘制，需要亚像素级精度的线段渲染，Canvas 是最佳选择
  - 干涉/衍射图样：**2D Canvas**。条纹图样是像素级的强度分布渲染（逐像素或逐行计算光强并映射为颜色），Canvas 的 ImageData API 最适合
  - 薄膜干涉图样（牛顿环等）：**2D Canvas**。同心圆环等图样需要 2D 像素级渲染
  - 透镜成像：**2D Canvas** 或 **SVG**。光线追迹用 Canvas，物/像标注可用 SVG overlay
  - 强度分布曲线：**Chart.js** 或自研 Canvas 图表
  - 公式渲染：**KaTeX**（斯涅尔定律、透镜公式、干涉条纹公式等）

- **数学/物理计算需求**：
  - 斯涅尔定律：n1 sin θ1 = n2 sin θ2
  - 全反射临界角：sin θc = n2/n1
  - 矩形玻璃砖侧向位移：Δ = d sin(θ1 - θ2) / cos θ2
  - 半球形玻璃砖光路：需要射线-圆弧交点计算 + 曲面法线计算 + 折射方向计算
  - 光导纤维：多次全反射光路递归计算
  - 透镜成像公式：1/u + 1/v = 1/f
  - 双缝干涉强度分布：I = I0 cos²(πd sin θ / λ)
  - 单缝衍射强度分布：I = I0 [sin(πa sin θ / λ) / (πa sin θ / λ)]²
  - 圆孔衍射（艾里斑）：涉及贝塞尔函数 J1
  - 薄膜干涉：2nd cos θ = kλ，牛顿环半径 r_k = sqrt(kRλ)
  - 波长 → RGB 颜色映射函数
  - 整体计算量**中等**，不需要数值积分或微分方程求解，均为解析公式直接计算

- **公式渲染需求**：中等。需要显示斯涅尔定律、透镜公式、干涉衍射条件等，KaTeX 足够

### 交互依赖

- **交互模式**：**参数驱动型为主，轻度混合型**
  - 核心交互是滑块/数值输入调参 → 画面实时更新
  - 几何光学部分有少量拖拽（如拖拽入射角、PRD 提到"拖拽摆放光学元件"但实际更接近选择形状 + 调参数）
  - 透镜成像部分物距可拖拽调节
  - 波动光学部分纯参数驱动

- **关键交互**：
  - 滑块（最核心的交互方式，所有参数均通过滑块调节）
  - 数值输入框（精确设定参数值）
  - 实验模块选择按钮（5 个顶部按钮切换模块）
  - 显示选项开关（角度标注、公式验证、颜色显示）
  - 材料预设选择（折射率快速选择：空气/水/玻璃/金刚石等）

- **是否需要实体系统**：**不需要**。光学元件种类固定、数量少（每个实验场景 1-2 个元件），不存在自由创建/删除/组合的需求。用"场景配置对象"即可描述
- **是否需要命令系统（undo/redo）**：**不需要**。用户操作全部是参数调节，参数有明确的默认值可以重置

## 5. 竞品分析

### 竞品 1：Ray Optics Simulation（phydemo.app/ray-optics）

- **产品形态**：免费开源 Web 应用，纯浏览器运行
- **技术实现**：HTML5 Canvas 多层渲染（ctxLight / ctxBelowLight / ctxAboveLight / ctxGrid / ctxVirtual 等分层），ES6 + Webpack 打包，原生 JavaScript
- **覆盖范围**：几何光学为主（光源、镜面、透镜、衍射光栅），支持自定义方程曲面
- **做得好的地方**：
  - 编辑器型交互：可自由拖拽放置任意光学元件，组合创建复杂光学系统
  - 多种可视化模式（光线/延长线/全部像点/观察者视角）
  - 模块化组件系统，支持自定义参数
  - 10 年开源积累，100+ 贡献者，20 种语言翻译
  - 可作为 npm 模块嵌入其他项目
- **不足的地方**：
  - 没有物理光学（干涉、衍射条纹图样、薄膜干涉）功能
  - 交互复杂度较高，对不懂技术的中学教师来说学习成本偏大
  - 没有教学向的预设场景和教学要点标注
- **对我们的启示**：
  - Canvas 多层渲染是几何光学的成熟方案
  - 我们的产品面向教师，应该做"预设场景 + 参数调节"模式而非完全自由编辑器
  - 波动光学（干涉/衍射/薄膜）是差异化切入点

### 竞品 2：PhET Geometric Optics（phet.colorado.edu）

- **产品形态**：免费 Web 应用（HTML5），科罗拉多大学出品
- **技术实现**：HTML5 + 自研 SceneryJS 渲染框架（Canvas + SVG 混合），TypeScript
- **覆盖范围**：几何光学（透镜成像为主），有 Basics 简化版和完整版
- **做得好的地方**：
  - 极致的教育设计：5 种可拖拽物体（铅笔/企鹅/星星等），直观有趣
  - 可调折射率和曲率半径，参数化程度高
  - 教育研究支撑：基于大量教育学研究设计交互
  - 开源（GitHub phetsims/geometric-optics）
- **不足的地方**：
  - 只覆盖透镜成像，不包含折射/全反射、干涉/衍射
  - Wave Interference 是另一个独立 simulation，与 Geometric Optics 不在同一工具中
  - 单个 PhET simulation 功能范围窄，无法在一个工具中完成光学全部教学
- **对我们的启示**：
  - 教育向产品需要精心设计预设和引导，而非暴露过多底层参数
  - 将几何光学和物理光学**整合到一个工具**中是我们的产品优势

### 竞品 3：PhET Wave Interference（phet.colorado.edu）

- **产品形态**：免费 Web 应用（HTML5）
- **覆盖范围**：波的干涉（水波/声波/光波），含单缝衍射和双缝干涉
- **做得好的地方**：
  - 多种波源类型（水龙头/扬声器/激光），直观类比
  - 可视化波前扩展过程，干涉图样动态形成
  - 屏障设置灵活，支持单缝/双缝
- **不足的地方**：
  - 不包含薄膜干涉（牛顿环、楔形膜）
  - 不包含折射/全反射
  - 参数范围和精度面向概念理解，不适合高中定量计算验证

### 竞品 4：Falstad Ripple Tank（falstad.com/ripple）

- **产品形态**：免费 Web 应用，从 Java Applet 迁移而来
- **技术实现**：早期 Java Applet → 现代 WebGL 版本（ripplegl），2D 波动模拟
- **覆盖范围**：二维波动现象（干涉、衍射、折射、共振、多普勒效应）
- **做得好的地方**：
  - 真正的波动模拟（非公式计算），视觉效果震撼
  - 可鼠标绘制障碍物，自由度极高
  - 预设场景丰富
- **不足的地方**：
  - 界面老旧，操作不直观
  - 没有教学标注（公式、教学要点）
  - 不包含几何光学（折射/透镜）
  - 性能依赖 WebGL，低端设备可能卡顿

### 竞品 5：oPhysics（ophysics.com）

- **产品形态**：免费 Web 应用，单页面 simulation 集合
- **覆盖范围**：双缝衍射与干涉（l4.html / l5.html 等独立页面）
- **做得好的地方**：
  - 简洁直观，参数滑块 + 实时图样
  - 面向教学，标注明暗纹条件
- **不足的地方**：
  - 每个现象是独立页面，无法在一个工具中完成光学全部教学
  - 功能单一，无几何光学
  - 界面简陋，视觉效果一般

### 竞品总结

| 维度 | Ray Optics Sim | PhET Geometric | PhET Wave | Falstad | oPhysics | **我们（P-03）** |
|------|---------------|----------------|-----------|---------|----------|----------------|
| 几何光学 | 强 | 透镜为主 | 无 | 无 | 无 | **完整覆盖** |

| 薄膜干涉 | 无 | 无 | 无 | 无 | 无 | **独有** |

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染引擎**：**HTML5 Canvas 2D**
  - 几何光学光路：Canvas 直线段/弧线绘制，支持亚像素精度
  - 干涉/衍射图样：Canvas ImageData API 逐像素计算光强并映射颜色，适合生成条纹和环形图案
  - 薄膜干涉图样：同上，使用 2D 像素级渲染
  - 透镜成像光路：Canvas 线段 + 虚线
  - 参考 Ray Optics Simulation 的多层 Canvas 架构（背景层 / 光路层 / 标注层 / 交互层）

- **辅助渲染**：
  - 强度分布曲线：Chart.js 或自研 Canvas 小图表
  - 公式显示：KaTeX 内联渲染
  - 参数面板 UI：DOM（HTML/CSS），不需要 Canvas 渲染 UI
  - 角度标注/法线等辅助图形：Canvas 叠加层

### 推荐的架构模式

- **"实验模块选择 + 参数驱动渲染"架构**（非编辑器架构）

```
┌─────────────────────────────────────────────────┐
│ ExperimentSelector（5 个模块按钮）               │
├──────────────┬──────────────────────────────────┤
│ ParamPanel   │ CanvasRenderer                   │
│ (DOM 滑块)   │ ├─ BackgroundLayer               │
│              │ ├─ OpticsLayer（光路/图样）       │
│              │ ├─ AnnotationLayer（标注/法线）   │
│              │ └─ InteractionLayer（拖拽热区）   │
│              ├──────────────────────────────────┤
│              │ FormulaPanel (KaTeX)              │
│              │ IntensityChart (Chart.js)         │
└──────────────┴──────────────────────────────────┘
```

- **状态管理**：简单的响应式状态（类似 Zustand 或 vanilla reactive store），参数变化触发重新计算和渲染
- **计算层与渲染层分离**：
  - `OpticsCalculator`：纯函数，输入参数 → 输出光路点序列 / 光强分布数组
  - `OpticsRenderer`：读取计算结果，绘制到 Canvas
- **实验模块抽象**：每个实验模块（折射/透镜/干涉/衍射/薄膜）实现统一接口：
  - `getParams()`: 返回参数定义（名称、范围、步进、默认值）
  - `calculate(params)`: 输入参数，返回可视化数据
  - `render(ctx, data)`: 将计算结果绘制到 Canvas

### 关键技术风险或难点

1. **半球形玻璃砖光路计算**：射线与圆弧的交点计算、曲面法线方向判断、多次折射/全反射的递归处理，需要精确的几何算法
2. **光导纤维多次全反射**：弯曲光纤中光线的多次反射路径计算，需要处理曲面法线和多步迭代
3. **干涉/衍射图样的实时渲染性能**：800px 宽的干涉图样需要逐像素计算光强，双缝干涉相对简单，但圆孔衍射涉及贝塞尔函数（J1），需要高效的近似算法或查表法
4. **波长 → RGB 颜色映射**：需要实现准确的可见光谱到 sRGB 的映射函数，处理色域外截断和 gamma 校正
5. **白光干涉的多波长叠加渲染**：白光干涉需要对可见光全波段（380-780nm）做积分叠加，计算量较大，可能需要采样 + 插值优化
6. **牛顿环/薄膜干涉的 2D 图样渲染**：牛顿环需要在 2D 平面上逐像素计算干涉级次并映射颜色，图像尺寸较大时性能是挑战

### 与其他物理模块的关系

- P-03 与其他物理模块**无直接功能耦合**
- 但与 P-06（波动与振动演示台）在波动光学原理上有概念重叠（干涉/衍射），技术实现上可复用波长→颜色映射、强度分布计算等工具函数
- 架构模式与 P-05（简谐运动）、P-07（热力学）等参数驱动型模块高度相似，可共享"实验模块选择 + 参数面板 + Canvas 渲染"的基础框架
