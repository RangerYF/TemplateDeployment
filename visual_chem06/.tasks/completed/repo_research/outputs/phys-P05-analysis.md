# P-05 简谐运动与弹簧振子 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程基础。核心用户画像是高中物理教师，需要在课堂上讲解简谐运动（弹簧振子、单摆）以及振动与波的关系。

### 解决什么教学痛点
- **运动图像与运动本身难以对应**：学生看到 x-t / v-t / a-t 图，很难将图上的点与振子实际所处的位置对应起来
- **平衡位置概念混淆**：竖直弹簧振子的平衡位置 ≠ 弹簧自然长度位置（重力造成偏移），学生容易搞混
- **对称性理解困难**：最高点和最低点关于平衡位置的"位移对称"和"加速度对称"，黑板上静态图无法动态展示
- **小角度近似的直觉缺失**：单摆的小角度近似与精确解差异有多大，学生没有直观感受
- **波的形成不可见**：从单个振子的简谐运动到一列振子形成行波的过程，粉笔画不清楚

### 核心产品价值
**一句话**：老师选一个振动模型、设几个参数，系统实时同步展示振子运动动画和 x-t / v-t / a-t 图像，自动标注平衡位置和对称性，直观呈现简谐运动的核心规律。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选振动模型（竖直弹簧振子）→ 设参数（k=100, m=0.5, x0=0.05）→ 开启对称性标注 → 记住配置 |
| **课中演示** | 投影仪打开 → 播放动画 → 指出平衡位置虚线 → 暂停在最高/最低点讲对称性 → 切换不同质量观察平衡位置变化 |
| **讲题** | 选水平弹簧振子 → 打开三图联动 → 暂停在某时刻 → 指出"此时位移为正、速度为负、加速度为负"→ 与选择题选项对应 |
| **引入波** | 切到"波的形成预览"→ 演示 20 个振子排列 → 展示行波形成 → 过渡到 P-06 波动演示台 |

### 怎么用才是"好用"的
- **参数即改即看**：调整弹簧系数或质量后，动画和图像立刻更新，不需要重新启动
- **三图联动直观**：x-t、v-t、a-t 三图与动画用竖线同步标记当前时刻，一目了然
- **标注自动生成**：平衡位置虚线、最高/最低点标注、对称性双向箭头均由系统自动生成，老师只需勾选显示/隐藏
- **投影清晰**：在 1080p 投影仪上弹簧动画和图像文字清晰可辨

### 交互频率和深度
- **频率**：每节课使用 1-2 次，每次 5-15 分钟
- **深度**：中等偏浅 — 主要是参数调整 + 动画控制 + 显示选项切换，不涉及复杂建模或对象编辑

### 哪些功能需要实体管理
- **不需要**。本模块没有"创建/选中/删除对象"的需求。所有元素（弹簧、物块、摆绳、摆球）都是由模型选择自动生成的，用户不手动创建或编辑实体。

### 哪些功能需要 undo/redo
- **不需要**。本模块是纯参数驱动 + 动画播放型，不存在需要撤销的编辑操作。参数修改直接覆盖。

### 哪些功能是参数驱动的
- 振动模型选择、所有物理参数（k/m/x0/L/θ0/g/b）、显示选项（平衡位置/对称标注/速度箭头/加速度箭头）
- 参数修改后画面和图像实时变化，这是本模块的核心交互模式

### 哪些功能是模拟/动画驱动的
- 弹簧振子运动动画、单摆摆动动画、三图实时绘制、速度/加速度箭头动画、小角度vs精确解对比动画、多摆耦合动画、波的形成预览动画
- 需要时间轴控制系统（播放/暂停/单步/倒放/调速 0.1x~2x）

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式：2D Canvas（推荐）
- 弹簧锯齿线绘制需要按帧更新（圈数随伸长变化），Canvas 逐帧绘制最合适
- 三图联动需要每帧重绘曲线和当前时刻竖线
- 单摆绳+球、速度/加速度箭头均为简单 2D 图元
- 不需要 3D，不需要 SVG 的 DOM 事件

#### 数学/物理计算需求
- **简谐运动方程**：x(t) = A cos(ωt + φ)，v(t) = -Aω sin(ωt + φ)，a(t) = -ω²x
- **角频率与周期**：ω = sqrt(k/m)，T = 2π sqrt(m/k)（弹簧振子）；ω = sqrt(g/L)，T = 2π sqrt(L/g)（单摆）
- **平衡位置计算**：竖直弹簧 x_eq = mg/k
- **能量计算**：Ek = ½mv²，Ep = ½kx²，E = ½kA²
- **大摆角精确解**：需要数值积分（椭圆积分近似或 RK4 数值积分）
- **阻尼振动**：x(t) = Ae^(-bt/2m) cos(ω't + φ)，ω' = sqrt(ω² - (b/2m)²)
- **耦合振动**：双摆耦合微分方程组，需要数值求解器
- 计算复杂度中等，三角函数/指数函数/数值积分均为标准运算，Math.js 或原生 JS 即可胜任

#### 公式渲染需求
- 较轻。主要是参数面板上显示公式（T = 2π√(m/k)等），KaTeX 即可。不需要大量公式排版。

### 交互依赖

#### 交互模式：参数驱动型 + 模拟演示型（混合）
本模块是典型的"参数面板 + 动画播放器"组合模式：
- **参数驱动**：调参数 → 场景实时变化（振子、平衡位置、图像全部更新）
- **模拟演示**：播放/暂停/逐步/调速控制时间轴
- **不是编辑器型**：无需拖拽创建对象、无需选中/删除操作

#### 关键交互
- **滑块 + 数值输入框**：调节 k/m/x0/L/θ0/g/b 等参数
- **按钮组**：模型选择（4 种振动模型）
- **时间轴控制栏**：播放/暂停/单步前进/单步后退/倒放/调速滑块
- **勾选框**：显示/隐藏平衡位置线、对称标注、速度箭头、加速度箭头、各图选择性显示
- **无拖拽交互**：不需要拖拽物体或创建连线

#### 是否需要实体系统
**不需要**。所有可视元素由模型和参数唯一决定，不存在用户自定义对象。

#### 是否需要命令系统（undo/redo）
**不需要**。纯参数调节 + 动画播放，参数修改即覆盖，无编辑历史需求。

## 5. 竞品分析

### 5.1 PhET — Masses and Springs

- **产品形态**：Web (HTML5)，免费开源，由科罗拉多大学博尔德分校开发
- **核心功能**：
  - 弹簧挂重物，调节劲度系数和阻尼
  - 支持不同星球重力环境（有趣但非教学核心）
  - 可显示动能/势能/热能的能量条形图
  - 支持秒表测周期
  - "Basics"版本可对比两组弹簧振子
- **技术实现**：HTML5 Canvas，PhET 自研框架 Scenery（类似 Scene Graph）
- **做得好的地方**：
  - 交互非常自然，可直接拖拽重物挂到弹簧上
  - 能量条形图实时更新，直观
  - 教育研究驱动设计，学生探究式学习效果好
- **不足之处**：
  - **没有 x-t / v-t / a-t 三图联动**，这正是中国高中物理教学的核心需求
  - **没有平衡位置动态标注**，不能直观展示不同质量下平衡位置的变化
  - **没有对称性标注**（位移对称、加速度对称），这是中国高考重要考点
  - **没有单摆模块**（单摆在另一个独立 Sim 中）
  - **没有波的形成预览**
- **对我们的启示**：PhET 的拖拽交互和能量可视化做得好，但其设计面向西方探究式教学，不聚焦中国高中物理考试要求的"图像联动"和"对称性分析"。我们的核心差异化在三图联动和对称性自动标注。

### 5.2 PhET — Pendulum Lab

- **产品形态**：Web (HTML5)，免费
- **核心功能**：
  - 1-2 个单摆，可调绳长、质量、重力
  - 可显示速度/加速度箭头
  - 支持不同星球重力
  - 可显示能量条形图和周期测量
- **做得好的地方**：
  - 双摆对比功能直观
  - 速度/加速度箭头在摆球上实时显示
- **不足之处**：
  - 没有 θ-t 图像联动
  - 没有小角度 vs 精确解的对比模式
  - 弹簧振子和单摆分散在不同 Sim 中，无法统一对比
- **对我们的启示**：将弹簧振子和单摆统一在一个工具中，统一的时间轴和图像联动，比分散的工具体验更好。

### 5.3 oPhysics — Simple Harmonic Motion: Mass on a Spring

- **产品形态**：Web，免费，教育资源网站
- **核心功能**：
  - 弹簧振子动画 + x-t / v-t / a-t 图像同步显示
  - 可调初始位移、质量、弹簧系数
  - 图上有当前时刻标记线
- **技术实现**：Canvas + JavaScript
- **做得好的地方**：
  - **这是最接近我们需求的竞品**，已经实现了三图联动
  - 界面简洁，功能聚焦
- **不足之处**：
  - 没有平衡位置动态标注
  - 没有对称性标注（位移对称、加速度对称）
  - 只有水平弹簧振子，没有竖直弹簧振子和单摆
  - 没有波的形成预览
  - 界面较为简陋，不适合投影仪课堂展示
  - 没有时间轴精细控制（单步/倒放/调速）
- **对我们的启示**：oPhysics 证明了三图联动的技术可行性和教学价值。我们需要在此基础上增加竖直弹簧振子（含平衡位置）、对称性标注、单摆模式、以及更精细的时间轴控制。

### 5.4 GeoGebra — SHM 系列小程序

- **产品形态**：Web，免费，用户社区贡献
- **核心功能**：
  - 各种 SHM 相关小程序（弹簧、单摆、圆周运动与 SHM 关系等）
  - 支持滑块调参
  - 部分有图像联动
- **技术实现**：GeoGebra 引擎（基于 Java/JS）
- **做得好的地方**：
  - 社区资源丰富，各种角度的演示都有
  - GeoGebra 的数学精确性好
- **不足之处**：
  - 分散在不同小程序中，没有统一体验
  - GeoGebra 界面对非数学教师不够友好
  - 动画流畅度和视觉效果不如 Canvas 原生渲染
  - 没有针对中国高考考点的定制化设计
- **对我们的启示**：统一整合 > 分散小工具。我们应将所有振动模型整合在一个统一界面中。

### 5.5 Open Source Physics @ Singapore — 振动专题合集

- **产品形态**：Web (HTML5/JS)，免费
- **核心功能**：
  - 多个 SHM 相关 JavaScript 小程序
  - 涵盖弹簧振子、单摆、圆周运动与 SHM 关系、能量图等
- **做得好的地方**：
  - 专业性强，物理内容准确
  - 部分小程序有较好的图像联动
- **不足之处**：
  - 每个功能点是独立的小程序，缺乏统一的产品体验
  - 视觉设计较为学术化，不适合课堂投影
- **对我们的启示**：确认了这类工具的教学需求是真实存在的。我们的产品化优势在于统一界面、投影优化、中国教学考点定制。

## 6. 初步技术判断

### 推荐的渲染技术
**2D Canvas（单一 Canvas 或分区 Canvas）**
- 主动画区使用一块 Canvas 绘制弹簧/物块/摆绳/摆球/箭头/标注
- 图像区使用独立 Canvas 或 Chart.js 绘制 x-t / v-t / a-t 三图
- 60fps 刷新率要求下，Canvas 逐帧绘制最高效
- 弹簧锯齿线的圈数随伸长量动态变化，Canvas 路径绘制最灵活

### 推荐的架构模式
**参数驱动 + 时间轴动画 的双环架构**

```
参数变更环：
  用户调参 → 更新物理模型参数 → 重算平衡位置/频率/振幅 → 重置动画起点

时间轴动画环：
  requestAnimationFrame → 推进 t → 根据运动方程算 x(t)/v(t)/a(t)
  → 更新动画区（弹簧位置/物块位置/箭头）
  → 更新图像区（三图曲线 + 当前时刻竖线）
```

**模块内部分层**：
```
┌─ UI 层 ─────────────────────┐
│ 参数面板 / 模型选择 / 时间轴 │
├─ 动画层 ────────────────────┤
│ 主 Canvas: 振子渲染器        │
│ 图 Canvas: 三图渲染器        │
├─ 物理层 ────────────────────┤
│ SHM 求解器（解析解 + 数值解）│
│ 平衡位置计算 / 能量计算      │
├─ 模型层 ────────────────────┤
│ 振动模型定义（参数 + 方程）  │
│ 模型工厂（4种模型）          │
└─────────────────────────────┘
```

### 与其他模块的关系
- **与 P-02 运动模拟器高度相似**：两者都是"时间轴 + 动画 + 图像联动"模式。时间轴控制组件、图像绘制组件可以共享或抽象为通用组件。
- **与 P-06 波动演示台的联动**：P-05 的"波的形成预览"功能是 P-06 的前置，两者可能共享振动模型定义。单个振子的简谐运动模型应设计为可被 P-06 复用。
- **与 P-14 机械能守恒的重叠**：P-14 有"弹簧振子能量"模型，与 P-05 的能量关系展示有重叠。建议 P-05 专注运动学（x/v/a 图像），P-14 专注能量转化（能量条形图），两者通过共享的 SHM 物理模型避免重复实现。

### 关键技术风险或难点

| 风险点 | 说明 | 应对策略 |
|--------|------|---------|
| 大摆角精确解 | 需要数值积分（椭圆积分），计算量比解析解大 | 使用 RK4 数值积分；预计算一个周期缓存结果；或使用椭圆积分的级数展开近似 |
| 三图联动的帧同步 | 主 Canvas 和图 Canvas 必须严格同步到同一个 t | 使用统一的 t 变量驱动两区渲染，在同一个 rAF 回调中更新 |
| 弹簧锯齿线绘制 | 圈数需要随伸长量动态变化，且要"看起来像弹簧" | 使用正弦波路径绘制，波数 = 固定值，波幅随伸长量缩放 |
| 耦合振动求解 | 双摆耦合是二阶 ODE 组，需要数值求解 | RK4 求解器；保持时间步长稳定（固定 dt = 1/60s） |
| 阻尼模式下的长时间动画 | 阻尼振动振幅趋零，长时间后可能有浮点精度问题 | 当振幅小于阈值时自动停止动画 |
