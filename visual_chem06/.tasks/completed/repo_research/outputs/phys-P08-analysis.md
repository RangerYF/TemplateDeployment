# P-08 电场与磁场可视化器 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程基础。核心用户画像是需要讲授电磁学（静电场、磁场、带电粒子运动、复合场模型）的高中物理教师。

### 解决什么教学痛点
- **场的概念极度抽象**：电场线、磁感线、等势面无法在课堂中实物展示，学生只能靠想象理解"看不见的力场"
- **场的叠加无法静态表达**：双点电荷叠加后的合场线分布，黑板上很难准确画出
- **粒子偏转轨迹难以手工绘制**：电容器偏转的抛物线、洛伦兹力圆周运动的圆弧，特别是各类边界模型（直线边界、圆形边界、磁聚焦/磁发散），手绘耗时且不准确
- **复合场模型高度综合**：速度选择器、回旋加速器涉及电场与磁场的叠加效果，纯口述或静态图片根本讲不清
- **公式验证不直观**：r = mv/(qB)、T = 2πm/(qB)、v = E/B 等关键公式需要动态演示才能让学生感受其物理含义

### 核心产品价值
**一句话**：老师选一个电磁场模型、调几个参数，系统实时绘制场线/等势面或粒子运动轨迹，覆盖高中电磁场全部核心考点（静电场、磁场、洛伦兹力边界模型、复合场），让"看不见的场"变得可见可交互。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选择场景模块（如"静电场"）→ 选择子场景（如"双点电荷场"）→ 设参数（Q1=+2μC, Q2=-1μC, 间距10cm）→ 确认场线与等势面显示效果 → 记录参数 |
| **课中演示（场线教学）** | 投影仪打开 → 拖动点电荷改变位置，实时看场线变化 → 切换等势面显示 → 指出电场线与等势面垂直关系 → 切换预设配置（等量异号→等量同号）对比 |
| **课中演示（粒子运动）** | 选择"洛伦兹力圆周运动" → 设置磁场方向、粒子速度、荷质比 → 播放动画观察圆周轨迹 → 调节速度看半径变化 → 切换边界模型（直线→圆形→磁聚焦） |
| **讲题** | 选择"速度选择器"模型 → 设置 E 和 B → 调节粒子速度，演示只有 v=E/B 的粒子能直线通过 → 切换到回旋加速器，演示螺旋加速轨迹 |

### 怎么用才是"好用"的
- **场景切换极快**：5 个主模块（静电场/电场粒子/磁场/磁场粒子/复合场）+ 子场景一键切换，无需重新配置
- **参数实时生效**：拖动滑块或拖拽电荷位置，场线/等势面/轨迹立即重绘
- **预设配置丰富**：双点电荷的 4 种典型配置（等量异号/等量同号正/等量同号负/不等量异号）、直线边界的 4 种入射角（90°/30°/60°/0°）等预设直接可用
- **关键公式同步显示**：界面上显示当前场景的关键公式和实时计算值（如 r=___、T=___、偏转量=___）
- **投影清晰**：场线颜色区分明确（电场线橙色、等势面绿色虚线、磁感线紫色、粒子轨迹黄色），1080p 投影仪上清晰可辨

### 交互频率和深度
- **频率**：电磁学教学中每节课使用 1-2 次，每次 10-20 分钟；复习课可能整节课使用
- **深度**：中高 — 主要是场景选择 + 参数调节 + 预设切换 + 粒子运动播放控制；静电场模块有拖拽放置电荷的编辑器特征

### 需要实体管理的功能
- **静电场模块**：点电荷（可创建/拖拽/删除/修改电荷量和极性）、平行板电容器（可设置电压/间距/板长）
- 其他模块偏向"预设场景 + 参数调节"，不需要自由创建/删除实体

### 需要 Undo/Redo 的功能
- 仅静电场模块中点电荷的放置/移动/删除/参数修改可能需要 Undo
- 其他模块参数调节通过滑块实时操作，不强需 Undo（优先级 P2）

### 参数驱动的功能
- 所有场景的参数滑块：电荷量、电流、磁感应强度、粒子速度、荷质比、入射角、板间电压等 → 场线/轨迹实时更新
- 预设配置切换 → 参数组整体切换
- 场线密度调节

### 模拟/动画驱动的功能
- 带电粒子在电场中运动：电容器偏转（抛物线）、加速电场
- 带电粒子在磁场中运动：洛伦兹力圆周、各类边界模型、磁聚焦/磁发散
- 复合场模型：速度选择器粒子运动、回旋加速器螺旋加速
- 需要时间轴控制：播放/暂停/单步/调速

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：2D Canvas — 场线绘制、等势面绘制、粒子轨迹动画、矢量场可视化
  - 场线追迹：从电荷出发沿场强方向的数值积分曲线
  - 等势面：等高线算法（Marching Squares 或类似）
  - 粒子轨迹：实时积分运动方程绘制轨迹曲线
- **备选**：SVG 可用于静态场线/标注，但动态重绘性能不如 Canvas
- **不需要 3D WebGL**：所有场景均为 2D 投影（磁场向里/向外用 ×/· 符号表示）

#### 数学/物理计算需求
- **重度 — 矢量场计算**：
  - 库仑定律场强叠加 E = kQ/r² 矢量求和（任意多电荷叠加）
  - 电势标量叠加 V = Σ(kQi/ri)
  - 磁场公式 B = μ₀I/(2πr)、B = μ₀I/(2R)、B = μ₀nI
- **重度 — 场线追迹算法**：
  - 从场源出发，沿场强方向以小步长数值积分（RK4 或 Euler），绘制场线
  - 需要合理的终止条件（到达边界、接近异号电荷、场强过小等）
  - 场线密度控制（从每个电荷均匀角度出发若干条线）
- **重度 — 粒子运动数值积分**：
  - 电场中匀加速运动（解析解：抛物线）
  - 磁场中匀速圆周运动（解析解：圆弧 r=mv/qB）
  - 复合场中运动（可能需要数值积分，如回旋加速器中的螺旋轨迹）
- **中度 — 几何计算**：
  - 直线边界/圆形边界与圆弧轨迹的交点计算（入射点/出射点）
  - 磁聚焦：多条不同方向圆弧的汇聚点
  - 回旋加速器：每半圈加速后半径增大的螺旋轨迹

#### 公式渲染需求
- 关键公式在界面上展示（KaTeX）：E = kQ/r²、r = mv/(qB)、T = 2πm/(qB)、v = E/B 等
- 实时计算值展示：偏转量 y = qUL²/(4mdv₀²)、出口偏转角 tanθ = qUL/(2mdv₀²) 等

### 交互依赖

#### 交互模式
**混合型**：参数驱动型（主导）+ 模拟演示型（粒子运动）+ 编辑器型（静电场电荷放置）

本模块是三种交互模式的混合体：
- **参数驱动**：大部分场景通过滑块/输入框调节参数，画面实时更新
- **模拟演示**：粒子运动需要播放/暂停/时间轴控制
- **编辑器**：静电场模块允许拖拽放置/移动点电荷（有限的编辑能力）

#### 关键交互

| 交互类型 | 具体场景 |
|---------|---------|
| 拖拽放置 | 将正/负点电荷拖到画布上（静电场模块） |
| 拖拽移动 | 移动已放置的点电荷位置 |
| 滑块调节 | 电荷量、电流、磁感应强度、速度、荷质比、入射角等参数 |
| 数字输入框 | 精确参数设置 |
| 场景/预设选择 | 顶部 5 个模块按钮、预设配置下拉 |
| 播放控制 | 播放/暂停/单步/调速（粒子运动场景） |
| 显示开关 | 等势面、场线密度、轨迹留存、公式标注 |
| 粒子参数预设 | 一键选择电子/质子/α粒子 |

#### 是否需要实体系统
**有限需要** — 仅静电场模块中的点电荷需要创建/选中/删除/修改的实体管理。其他模块的场源和粒子通过参数面板配置即可，不需要复杂的实体系统。

可采用轻量化的"场景对象"管理，而非 M-01 级别的完整 Entity + Registry 系统。

#### 是否需要命令系统
**低优先级** — 核心场景是"选预设 → 调参数 → 看效果"，Undo/Redo 不是强需求。静电场模块中电荷的放置/移动可简单支持。

## 5. 竞品分析

### 5.1 PhET Charges and Fields

- **产品形态**：Web（HTML5，纯浏览器运行）
- **网址**：[phet.colorado.edu/en/simulations/charges-and-fields](https://phet.colorado.edu/en/simulations/charges-and-fields)
- **技术实现**：HTML5 Canvas + 自研物理引擎；由科罗拉多大学开发；开源（CC-BY 许可）
- **核心功能**：
  - 拖拽放置正/负点电荷
  - 实时显示电场矢量（箭头网格）
  - 显示等势面线
  - 可放置电压传感器测量电势差
  - 支持电偶极子、电容器等配置
- **优势**：
  - 交互极为直观，拖拽即用
  - 完全免费开源，全球广泛使用
  - 场线/等势面可视化效果优秀
  - 有配套教学活动设计
- **不足**：
  - **仅覆盖静电场**，没有磁场、粒子运动、洛伦兹力、复合场模型
  - 无粒子运动模拟
  - 无场线追迹（用箭头网格而非连续场线）
  - 无预设教学场景
- **启示**：交互设计标杆（拖拽放置电荷），但功能范围远不够覆盖高中电磁学全部考点

### 5.2 Falstad 电磁场模拟系列

- **产品形态**：Web（JavaScript，从早期 Java Applet 迁移）
- **网址**：[falstad.com/emstatic](https://www.falstad.com/emstatic/)（静电场）、[falstad.com/vector3de](https://www.falstad.com/vector3de/)（3D 电场）、[falstad.com/vector3dm](https://www.falstad.com/vector3dm/)（3D 磁场）
- **技术实现**：原生 JavaScript + Canvas，从 Java 移植
- **核心功能**：
  - 2D/3D 静电场：可拖拽点电荷，显示场线、电势面、粒子在场中运动
  - 3D 磁场：长直导线、环形线圈、螺线管等磁场可视化
  - 粒子在场中运动演示
  - 丰富的预设实验配置
- **优势**：
  - 覆盖面广（电场+磁场+电磁波，数十个子模块）
  - 完全免费
  - 3D 可视化能力
  - 有粒子运动模拟
- **不足**：
  - UI 极度简陋（90 年代风格），完全不适合课堂投影
  - 各模拟分散在不同页面，无统一入口
  - 交互体验差（小字、密集控件）
  - 不贴合中国高中物理教材考点（无洛伦兹力边界模型、无速度选择器、无回旋加速器）
  - 无中文
- **启示**：功能覆盖面的参考，但 UI/UX 完全不可用于教学投影场景；P-08 的差异化在于"教学场景化 + 优质 UI"

### 5.3 oPhysics — Charged Particle in Magnetic Field

- **产品形态**：Web
- **网址**：[ophysics.com/em7.html](https://ophysics.com/em7.html)
- **技术实现**：JavaScript + Canvas，简洁的物理模拟
- **核心功能**：
  - 带电粒子在磁场中运动
  - 可调质量、电荷、速度、磁场强度
  - 显示圆周轨迹半径
- **优势**：界面简洁、直接展示核心物理关系
- **不足**：功能极为单一，仅覆盖一个场景（磁场中圆周运动）；无边界模型、无复合场
- **启示**：证明了简单的"一个场景+几个滑块"模式在教学中有需求

### 5.4 Explerify — 洛伦兹力/回旋加速器模拟器

- **产品形态**：Web
- **网址**：[explerify.com/simulator-lorentz-force](https://explerify.com/simulator-lorentz-force/)、[explerify.com/simulator-cyclotron](https://explerify.com/simulator-cyclotron/)
- **技术实现**：Web 前端，交互式模拟
- **核心功能**：
  - 洛伦兹力模拟器：可调参数观察粒子轨迹变化
  - 回旋加速器模拟器：展示 D 形盒内的螺旋加速过程
- **优势**：专注于特定模型、交互直观
- **不足**：每个模型独立页面，无统一教学工具体验；功能有限

### 5.5 GeoGebra 电磁场相关

- **产品形态**：Web / Desktop / Mobile
- **网址**：[geogebra.org](https://www.geogebra.org/)（社区资源）
- **技术实现**：GWT 编译 JS + WebGL
- **核心功能**：

### 5.8 市场空白与 P-08 定位

**核心发现**：**系统化覆盖高中电磁学全部考点的 Web 教学工具不存在**。

- PhET 仅覆盖静电场，不涉及磁场和粒子运动
- Falstad 覆盖面广但 UI 完全不适合教学投影
- oPhysics/Explerify 仅覆盖单一场景
- 没有任何产品覆盖"洛伦兹力边界模型"（直线/圆形边界、磁聚焦/磁发散），而这是中国高中物理的核心考点
- 没有任何产品将静电场、磁场、粒子运动、复合场整合到统一的教学界面中

P-08 的差异化定位：**唯一将静电场可视化 + 磁场可视化 + 粒子偏转模拟 + 洛伦兹力边界模型 + 复合场模型整合到单一 Web 工具中的高中电磁学教具**，且专门贴合中国高中物理教材考点体系。

## 6. 初步技术判断

### 推荐的渲染技术

**2D Canvas（主渲染）+ SVG（辅助标注）**

- **Canvas**：场线追迹、等势面等高线、粒子轨迹动画的核心渲染，性能满足实时重绘需求
- **SVG/DOM**：参数面板、公式标注、数值显示等 UI 层
- **KaTeX**：公式渲染（E = kQ/r²、r = mv/(qB) 等）
- **不需要 3D WebGL**：所有场景均为 2D 表达（磁场方向用符号 ×/· 表示垂直屏幕分量）
- **可选 D3.js**：用于矢量场箭头网格渲染，但自研 Canvas 绑定同样可行

### 推荐的架构模式

**场景驱动 + 参数响应式 + 动画引擎**

```
用户选择场景 → 加载场景配置 → 参数面板渲染 → 参数变更 → 物理计算引擎 → Canvas 重绘
                                                         ↓
                                                   动画引擎（粒子运动场景）
                                                         ↓
                                                   帧循环 → 数值积分 → Canvas 重绘
```

| 子系统 | 职责 | 设计模式 |
|--------|------|---------|
| SceneManager | 管理 5 大模块 × N 个子场景的配置和切换 | 策略模式 |
| PhysicsEngine | 电场/磁场计算、场线追迹、粒子运动积分 | 纯函数/无副作用 |
| FieldRenderer | 场线/等势面/矢量场箭头的 Canvas 绘制 | 渲染策略 |
| ParticleSimulator | 粒子运动的帧循环、时间轴控制（播放/暂停/调速） | 状态机 + requestAnimationFrame |
| ParameterStore | 当前场景的全部参数管理和响应式更新 | Zustand / 响应式 Store |
| ChargeManager | 静电场模块的点电荷实体管理（放置/移动/删除） | 轻量实体管理 |

### 与 M-01 架构的关系

P-08 与 M-01（立体几何展示台）的架构差异显著：

| 维度 | M-01 | P-08 |
|------|------|------|
| 渲染 | 3D WebGL (Three.js) | 2D Canvas |
| 交互模式 | 编辑器型为主 | 参数驱动 + 模拟演示为主 |
| 实体系统 | 9 种实体类型，完整 Registry | 轻量（仅静电场模块的电荷） |
| Command 系统 | 完整 Undo/Redo（11 种 Command） | 低需求 |
| 动画系统 | 无（纯静态） | 核心需求（粒子运动帧循环） |
| 计算引擎 | 几何计算（顶点/面/体积） | 场计算 + 运动方程积分 |

**可复用资产**：
- Zustand Store 模式
- UI 组件库（滑块、输入框、按钮组、参数面板布局）
- KaTeX 公式渲染集成
- Signal/事件系统
- 场景导出/导入 JSON 模式

**需新建能力**：
- 2D Canvas 场线/等势面渲染引擎
- 粒子运动动画引擎（帧循环 + 时间轴控制）
- 场线追迹算法（RK4 数值积分）
- 洛伦兹力边界模型的几何解算（圆弧与直线/圆的交点）

### 关键技术风险或难点

1. **场线追迹算法的鲁棒性**：多电荷叠加场中的场线追迹，终止条件判断容易出 Bug（如场线在电荷附近发散、场线不正确地穿过电荷）。需要精心处理电荷附近的奇点、合理的步长自适应。

2. **等势面绘制性能**：Marching Squares 等高线算法在每次电荷位置变化时需要对整个网格重新计算电势并提取等值线，可能成为实时交互的性能瓶颈。优化方案：降低网格分辨率 + Web Worker 异步计算。

3. **洛伦兹力边界模型的几何精确性**：直线/圆形边界与圆弧轨迹的交点计算涉及求解二次方程和选取正确的根（入射/出射方向判断），磁聚焦需要多条不同方向圆弧的精确汇聚，浮点误差可能导致视觉上的不汇聚。

4. **回旋加速器渲染复杂度**：粒子每经过一次加速间隙半径增大，螺旋轨迹由多段半径递增的半圆弧拼接。需要精确对齐每段弧的起止点，否则轨迹出现断裂或跳变。

5. **场线密度控制**：场线数量需要反映场强大小（强场区域线密、弱场区域线疏），但数量过多影响性能和可读性。需要设计合理的场线生成策略。

6. **多模块整合的状态管理**：5 个主模块 × 多个子场景，参数集各不相同，场景切换时需要干净地重置状态。设计良好的 SceneConfig 模式是关键。
