# C-02 分子结构查看器 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：初中及高中化学教师，覆盖初中基础化学到高中选修（物质结构与性质）全学段。
- **解决的教学痛点**：
  - 分子空间结构（VSEPR 构型、键角、杂化）是抽象概念，传统教学只能用平面图或实物球棍模型，角度不可调、数据不可标注、无法快速切换对比。
  - 同分子式多结构（如 C₂H₆O 对应乙醇和甲醚）的辨析在传统教学中缺乏直观手段。
  - 键长键角变化偏移的分析题（高考重点）需要定量数据支撑，教师备课查阅数据成本高。
- **核心产品价值**：一句话——**让教师在课堂上 3 秒内调出任意中学化学分子的 3D 模型，配合精确键长键角数据和 VSEPR 构型可视化，替代实物球棍模型和静态图片。**

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 搜索目标分子（化学式/中文名/IUPAC名）→ 选择显示模式（球棍/空间填充）→ 调整视角 → 截图或记录分子参数用于 PPT 或学案 |
| **课中演示** | 投屏打开查看器 → 输入分子 → 旋转/缩放展示空间构型 → 点击原子/键显示标注数据 → 切换显示模式对比（如球棍 vs 空间填充）→ 使用分子对比功能并排展示两个分子（如 CH₄ vs NH₃） |
| **讲题/复习** | 输入题目涉及的分子 → 标注键角键长 → 利用 VSEPR 速查解释构型差异 → 同分子式多结构选择界面帮学生辨析异构体 |

### 怎么用才是"好用"的？

- **快**：输入分子名后 < 1秒出模型，不需要等加载。分子库内置本地数据，非远程查询。
- **准**：键长键角数据精确到 pm/°，来源标注 NIST/CRC Handbook，教师可直接引用。
- **直观**：3D 旋转流畅（≥30fps），键角偏移有注释气泡解释原因（如"孤对电子排斥使键角减小"）。
- **对比方便**：分栏对比不需要开两个窗口，对比表格自动生成。

### 交互频率和深度

- **频率**：高频工具。分子结构贯穿高中化学必修+选修全程，几乎每章都会用到。
- **深度**：中等深度。单次使用通常 2-5 分钟（选分子 → 查看 → 标注 → 对比），不涉及长时间编辑或复杂流程。

## 3. 核心功能点

### 功能清单（按重要性排序）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 分子检索与输入 | 化学式、IUPAC 命名、中文名三种输入方式；同分子式多结构列表选择 |
| P0 | 3D 球棍模型渲染 | 默认显示模式，原子球体+化学键圆柱体，CPK 配色方案 |
| P0 | 3D 交互操作 | 鼠标拖拽旋转、滚轮缩放、右键平移、双击原子/键显示信息 |
| P0 | 键长键角标注 | 点选 2 原子显示键长、3 原子显示键角，附带偏移注释 |
| P1 | 显示模式切换 | 球棍模型 / 空间填充模型 / 结构简式 / 电子式 / 键线式 / 电子云模型 |
| P1 | 分子对比 | 左右分栏 3D 展示 + 下方参数对比表格（杂化、键角、键长、极性、VSEPR 等） |
| P1 | VSEPR 构型可视化 | 孤电子对显示、电子域构型与分子构型联动 |
| P1 | 命名互译面板 | 同时显示 IUPAC 名、中文名、化学式、结构简式 |
| P2 | 电子云模型 | 分子轨道等值面渲染，支持点击单键显示成键轨道 |

### 需要实体管理的功能

- **有限的实体管理**：本模块不是编辑器型产品。用户不创建/删除分子对象，而是从分子库中选择查看。但"分子对比"功能需要管理"当前查看的分子"和"对比槽位"（左/右两个）。
- 不需要完整的实体系统（无拖拽创建、无自由组合）。

### 需要 undo/redo 的功能

- **不需要**。本模块是查看器（Viewer），不是编辑器。用户操作可逆性通过"重新选择分子"或"重置视角"实现，无需命令系统。

### 参数驱动的功能（调参数 → 画面实时变化）

- **显示模式切换**：切换球棍/空间填充/电子式等，3D 场景实时重渲染。
- **视角控制**：旋转/缩放/平移参数驱动相机变换。
- 无滑块式连续参数调整需求。

### 模拟/动画驱动的功能

- **无**。本模块是静态结构查看器，不涉及动画播放/暂停/逐步。

## 4. 核心依赖识别

### 技术依赖

| 维度 | 判断 |
|------|------|
| **渲染方式** | **3D WebGL（Three.js）**。球棍模型、空间填充模型、电子云模型均需 3D 渲染。结构简式/电子式/键线式可用 2D Canvas 或 SVG 叠加层。 |
| **数学计算需求** | 中等。需要：(1) 从键长/键角/二面角数据计算原子 3D 坐标；(2) VSEPR 构型到 3D 空间布局的映射；(3) 空间填充模型的范德华半径缩放；(4) 电子云模型的轨道函数等值面计算（P2 拓展功能）。 |
| **公式渲染需求** | 轻度。分子式中的下标（H₂O）、离子上标（SO₄²⁻）需要渲染，但复杂度低于数学公式，可用 HTML/CSS 或简单文本处理。 |

### 交互依赖

| 维度 | 判断 |
|------|------|
| **交互模式** | **参数驱动型 + 查看器型**（非编辑器型）。用户选择分子 → 系统渲染；用户切换模式 → 系统重渲染。无自由创建/编辑对象的需求。 |
| **关键交互** | 搜索输入框（分子检索）、下拉选择（同分子式多结构）、3D 鼠标交互（旋转/缩放/平移）、点选（原子/键高亮与信息展示）、模式切换按钮/选项卡、分栏对比布局 |
| **是否需要实体系统** | **不需要**。分子数据来自预定义分子库（80+ 条静态数据），非用户动态创建。 |
| **是否需要命令系统** | **不需要**。查看器无编辑操作，无 undo/redo 需求。 |

## 5. 竞品分析

### MolView（molview.org）

- **产品形态**：Web 应用，免费开源
- **技术实现**：基于 GLmol（Three.js 封装）进行 3D 渲染；2D 结构编辑器基于 Ketcher/JME；分子数据来自 PubChem/RCSB/COD 远程 API
- **优点**：界面简洁，2D 编辑器 + 3D 查看器联动，支持从 PubChem 搜索海量分子，支持球棍/空间填充/线框等多种渲染模式
- **不足**：面向专业用户，缺乏教学场景适配（无学段标注、无 VSEPR 可视化、无键角偏移解释、无分子对比功能）；依赖远程 API，离线不可用；无中文支持
- **启示**：2D+3D 联动的交互范式值得参考；分子检索应支持本地库优先以保证课堂响应速度

### 3Dmol.js

- **产品形态**：JavaScript 库（非独立产品），常嵌入其他 Web 应用
- **技术实现**：纯 WebGL 自研渲染引擎（非 Three.js），支持 PDB/SDF/MOL2 等格式
- **优点**：渲染性能好，API 灵活，支持大分子（蛋白质级别），嵌入性强
- **不足**：是底层库不是成品应用，需大量二次开发；无教学辅助功能；文档面向开发者
- **启示**：可作为技术参考但不建议直接使用——本项目已选择 Three.js 技术栈，且中学分子规模小（< 100 原子），无需专用分子渲染库的性能优化

### Jmol / JSmol

- **产品形态**：桌面应用（Java Applet）/ Web 版（JSmol，纯 JS）
- **技术实现**：Java 原生 / JavaScript Canvas 渲染，历史悠久（2000 年代起）
- **优点**：功能极其丰富，支持晶体、蛋白质、脚本控制，学术界广泛使用
- **不足**：界面陈旧，交互体验差；Java Applet 已被现代浏览器淘汰；JSmol 渲染性能低于 WebGL；学习成本高
- **启示**：功能全面但 UX 差是反面教材，说明教育工具的易用性与视觉质量同样重要

### PubChem 3D Viewer

- **产品形态**：Web，嵌入 PubChem 数据库页面
- **技术实现**：基于 3Dmol.js 或 Mol* 渲染引擎
- **优点**：数据权威（NIST/PubChem），分子覆盖面极广
- **不足**：查看功能单一（仅旋转缩放），无教学辅助（无键角标注、无 VSEPR、无对比）；需联网；无中文
- **启示**：数据源权威性可参考，但产品形态太工具化，缺乏教学场景设计

### 竞品总结对我们的启示

1. **教学场景是差异化关键**：所有竞品都是面向科研/通用场景，无一提供学段标注、VSEPR 可视化、键角偏移解释、分子对比表格等教学功能——这是我们的核心差异化。
2. **本地数据库保证速度**：课堂场景不能依赖远程 API，80+ 分子的预定义库足以覆盖中学教学。
3. **Three.js 是正确选择**：主流竞品要么用 Three.js 封装，要么用自研 WebGL——Three.js 生态成熟且与 C-05 共享技术栈。
4. **简洁 > 功能堆砌**：Jmol 的教训说明功能多但交互差不可取，教师需要的是"3 秒出模型"而非"无限可配置"。

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：Three.js（WebGL 2.0），与 C-05 晶体结构查看器共享 3D 技术栈
- **原子渲染**：`SphereGeometry`（球棍模型用小球）或 `InstancedMesh`（同元素原子实例化渲染，提升性能）
- **化学键渲染**：`CylinderGeometry`（单键单圆柱、双键双平行圆柱、三键三角排列圆柱），苯环离域键用半透明环
- **2D 叠加层**：结构简式/电子式/键线式可用 HTML Canvas 或 SVG overlay，不进入 3D 场景
- **信息标注**：CSS2DRenderer（Three.js 内置）用于原子/键的浮动标签，避免 3D 文字渲染的性能开销

### 推荐的架构模式

- **参数驱动型查看器架构**（非编辑器架构）：
  - **数据层**：静态分子库 JSON（80+ 分子，含键长/键角/VSEPR/CPK 颜色等全部参数，来源于 `C02_分子结构数据.md`）
  - **模型构建层**：分子数据 → 3D 坐标计算 → Three.js 场景对象生成。核心算法：根据键长、键角、二面角递推计算各原子的笛卡尔坐标
  - **渲染层**：Three.js 场景管理、相机控制（OrbitControls）、光照、显示模式切换
  - **交互层**：Raycaster 射线拾取（原子/键点选高亮）、信息面板联动
  - **UI 层**：React 组件（搜索栏、模式切换、信息面板、对比表格）
- **与 C-05 的共享策略**：
  - 共享：Three.js 场景初始化、OrbitControls 相机控制、Raycaster 拾取逻辑、CPK 配色系统、光照方案
  - 不共享：分子坐标计算（C-02 用 VSEPR/键角推导）vs 晶体坐标计算（C-05 用晶胞参数+空间群）

### 关键技术风险或难点

| 风险/难点 | 严重性 | 说明 |
|-----------|--------|------|
| **分子 3D 坐标计算** | 高 | 分子库提供的是键长+键角，需要算法将其转换为 3D 笛卡尔坐标。简单分子（如 CH₄、H₂O）可用 VSEPR 几何模板直接放置；复杂有机分子（如正丁烷、乙酸乙酯）需要递推坐标算法（Z-matrix → Cartesian 转换），二面角处理是难点。 |
| **同分子式多结构的数据建模** | 中 | 化学式 → 分子不是一对一映射（如 C₂H₆O → 乙醇 or 甲醚）。数据层需要支持"一个 formula 对应多个 molecule"的索引结构。分子库已按独立 ID（MOL-XXX）编号，化学式作为搜索索引而非唯一键即可。 |
| **电子云模型渲染** | 中（P2 拓展） | 需要计算分子轨道的等值面并渲染为半透明 mesh。可使用 Marching Cubes 算法生成等值面几何体。由于标注为拓展功能，可后期实现。 |
| **双键/三键/离域键的 3D 渲染** | 中 | 双键需要两根平行圆柱体，必须正确计算平行偏移方向（垂直于键轴且在分子平面内）。苯环离域键需要特殊处理（半透明环面）。 |
| **移动端触摸适配** | 低 | PRD 要求支持触摸手势（双指缩放、单指旋转），Three.js OrbitControls 原生支持，但需测试平板端点选精度。 |
| **分子对比的双视口同步** | 低 | 左右分栏各一个 Three.js 渲染器，需处理两个独立场景的性能开销和视口尺寸响应。方案成熟，风险低。 |
