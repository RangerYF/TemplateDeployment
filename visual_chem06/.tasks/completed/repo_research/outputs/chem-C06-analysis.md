# C-06 电化学演示台 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中化学教师（主要为高中必修和选修"化学反应原理"模块授课教师），次要用户为化学竞赛辅导教师。教师明确评价"这个模块很难做，虽然是每年高考必考点"，说明教学需求强烈但对产品质量期望极高。

### 解决什么教学痛点
- **电子/离子流向抽象难懂**：原电池和电解池的工作原理涉及外电路电子流向、内电路离子迁移、盐桥离子交换等多层信息同步运动，黑板板书完全无法动态呈现
- **正负极/阴阳极混淆**：原电池的正极负极与电解池的阴极阳极概念容易混淆，需要在同一装置上对比标注
- **充放电方向反转**：铅蓄电池、锂离子电池的充放电过程中电极反应方向反转，是高考核心考点但静态图无法直观展示
- **电极反应方程式与装置图脱节**：学生难以将半反应方程式与实际装置中的物质变化对应起来
- **反应环境影响产物形式**：不同pH环境下产物存在形式不同（如CO2在碱性中变为CO3^2-），是常见易错点
- **反应竞争难以可视化**：电解食盐水中Cl^-和OH^-在阳极的放电竞争、电极电势比较等概念抽象
- **实验条件限制**：电化学实验器材昂贵、耗材消耗大，许多学校无法开齐13种电池/电解池实验

### 核心产品价值
**一句话**：教师从13个预设的电池/电解池模型中选择目标模型，即可获得完整的电化学装置可视化——包含外电路电子流动、内电路离子迁移、电极反应方程式同步标注、电极表面物质变化动画——支持播放/暂停/逐步/调速控制，覆盖高中电化学核心教学需求。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 从模型库选择目标电池（如 Zn-Cu 双液原电池）→ 确认动画效果 → 调整播放速度到适合课堂讲解的节奏 |
| **课中演示 — 原电池原理** | 选择 Zn-Cu 单液原电池 → 播放动画 → 逐步讲解：①Zn电极溶解（负极）②电子沿导线流向Cu电极（外电路）③Cu^2+在Cu电极上析出（正极）④溶液中离子迁移（内电路）→ 暂停在关键帧讲解电极反应方程式 |
| **课中演示 — 充放电对比** | 选择铅蓄电池模型 → 先播放放电过程 → 暂停 → 切换到充电模式 → 播放充电过程 → 对比两种方向的电极反应方程式变化 |
| **课中演示 — 电解池** | 选择电解饱和食盐水模型 → 播放动画 → 展示阴极H+放电产生H2、阳极Cl-放电产生Cl2 → 标注反应竞争（Cl-优先于OH-放电） |
| **讲题/纠错** | 进入练习模式 → 展示电池装置图 → 学生判断正负极/阴阳极 → 选择电极材料和电解质 → 系统给出纠正和解释 |
| **拓展教学** | 选择浓差电池模型 → 展示两侧浓度随时间变化 → 定性展示电动势变化趋势 → 标注与生物渗透压的跨学科联系 |

### 怎么用才是"好用"的
- **选模型即可演示**：从模型库选择后，装置图 + 动画 + 方程式一站式呈现，无需教师自行组装
- **动画可精确控制**：播放/暂停/逐步前进/调速（0.5x/1x/2x），教师可在任意关键帧暂停讲解
- **方程式与动画同步**：电极反应方程式随动画进度实时显示，学生能将"宏观现象"与"微观方程式"直接对应
- **充放电一键切换**：铅蓄电池、锂离子电池可直接切换充放电模式，对比展示
- **投影清晰**：装置示意图、粒子动画、方程式标注在1080p教室投影仪上清晰可辨

### 交互频率和深度
- **频率**：高中"化学反应原理"模块集中使用约6-10课时，每课时使用2-4次，每次5-10分钟
- **深度**：中度 — 核心操作是选模型+播放动画+暂停讲解（浅），但充放电对比、练习模式、拓展模型需要较深交互

### 需要实体管理的功能
本模块**不是编辑器型**应用，不需要用户自由创建/删除实体。所有电池模型均为预设数据驱动。但需要管理以下状态：
- 当前选中的电池/电解池模型
- 动画播放状态（播放/暂停/当前帧）
- 播放速度
- 充放电模式切换状态（仅铅蓄电池、锂离子电池）
- 练习模式下的学生选择状态与判定结果

### 需要 Undo/Redo 的功能
**不需要** — 本模块是演示器型应用，用户操作均为非破坏性（选模型、播放/暂停、切换模式），无需撤销/重做。

### 参数驱动的功能
- 模型选择 → 整个装置图和动画内容切换
- 播放速度调节（0.5x/1x/2x）→ 动画速率实时变化
- 充放电模式切换 → 电极反应方向反转、动画方向反转
- 浓差电池浓度参数 → 电动势趋势变化（定性）

### 模拟/动画驱动的功能
- 外电路电子流向动画（小球沿导线路径移动）
- 内电路离子迁移动画（阳离子向阴极、阴离子向阳极移动）
- 盐桥离子交换动画
- 隔膜选择性透过动画
- 电极表面物质变化动画（溶解/析出/气泡）
- 溶液颜色渐变动画
- 充放电过程全流程动画

以上所有动画均需支持：播放、暂停、逐步前进、速度调节（0.5x/1x/2x），且关键帧配有文字标注。

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：2D Canvas 或 SVG — 电池装置示意图（电极、容器、导线、开关/电源、盐桥）本质上是2D示意图，不需要3D渲染
- **粒子动画层**：Canvas 2D — 电子小球沿导线移动、离子在溶液中迁移的粒子动画，使用 `requestAnimationFrame` 驱动
- **可选方案**：纯 SVG + CSS Animation 也可实现，但粒子数量较多时 Canvas 性能更优
- **不建议使用 3D WebGL**：电化学装置是平面示意图，引入 Three.js 增加复杂度但无实质收益
- **信息面板**：DOM/HTML — 方程式展示、参数面板、控制按钮

#### 数学/物理计算需求
- **轻度**：无复杂数学计算。核心是动画路径计算（粒子沿预设路径移动）
- **轻度**：浓差电池的电动势定性趋势展示（不要求能斯特方程精确计算，PRD明确说明）
- **路径计算**：电子沿导线路径、离子在溶液中的迁移路径，需要为每个模型预定义路径曲线

#### 公式渲染需求
- **中度偏重**：每个模型需渲染2个半反应方程式 + 1个总反应方程式
- 化学方程式含上下标（如 Cu^2+、SO4^2-）、反应条件标注、电子转移标注
- 需要支持化学式格式化渲染（上下标、箭头、条件标注）
- **方案选择**：可用 HTML/CSS 实现化学式渲染（`<sup>`/`<sub>` 标签），复杂度低于 LaTeX；也可用轻量公式库（如 KaTeX）
- 反应环境标注（酸性/碱性/中性/熔盐）

### 交互依赖

#### 交互模式
**模拟演示型**（主导）+ **参数驱动型**（辅助）

这是一个动画演示器型应用：
- 用户从预设模型库选择电池/电解池模型
- 核心交互是动画播放控制（播放/暂停/逐步/调速）
- 充放电模式切换属于参数驱动
- 练习模式（P2）增加少量交互判定

#### 关键交互

| 交互类型 | 具体场景 |
|---------|---------|
| 下拉列表/卡片选择 | 从13个模型中选择目标电池/电解池 |
| 播放/暂停按钮 | 控制动画播放状态 |
| 逐步前进按钮 | 按关键帧逐步推进动画 |
| 速度滑块/按钮 | 调节动画速度（0.5x/1x/2x） |
| 切换按钮 | 铅蓄电池/锂离子电池的充放电模式切换 |
| 点选（练习模式） | 学生点击判断正负极/阴阳极 |
| 选择器（练习模式） | 学生选择电极材料和电解质 |
| 悬停/点击 | 查看方程式详细解释、错误提示 |

#### 是否需要实体系统
**不需要** — 无需 Entity Registry 或对象创建/删除/编辑。所有电池模型和动画参数由预设数据驱动。

#### 是否需要命令系统
**不需要** — 无破坏性编辑操作，无需 Undo/Redo。

## 5. 竞品分析

### 5.1 PhET Interactive Simulations — Battery-Resistor Circuit / Electrolysis

- **形态**：Web（HTML5），桌面（Java legacy），免费开源
- **技术**：HTML5 Canvas + 自研渲染框架 Scenery，JavaScript/TypeScript
- **优势**：
  - 物理直觉极强的粒子动画（电子作为蓝色小球沿导线流动，直观易懂）
  - 交互设计面向教学，操作简洁
  - 全球教育机构广泛使用，教师信任度高
  - 开源可参考实现细节
- **不足**：
  - PhET 的电化学模块较基础，主要是电路模拟（Battery-Resistor Circuit），不覆盖高中化学的电极反应方程式、离子迁移、充放电对比等核心需求
  - 无中文化学方程式支持
  - 没有"有限预设模型"的概念，偏向物理电路通用模拟
  - 不涉及反应竞争、环境对产物形式影响等化学专有知识点
- **启示**：PhET 的电子粒子动画风格（小球沿路径流动）是成熟且直观的方案，C-06 可采用类似的可视化语言；但 C-06 需在化学专业性上远超 PhET

### 5.2 虚拟化学实验室类产品（NovaStar 仿真实验/学科网虚拟实验等）

- **形态**：Web/桌面，多为商用付费
- **技术**：Unity WebGL 或 Flash（旧版本），部分为 HTML5
- **优势**：
  - 模拟真实实验场景（烧杯、导线、电极、万用表等），视觉拟真度高
  - 部分产品覆盖了原电池和电解池实验
- **不足**：
  - 过度追求"实验操作模拟"（拖拽连线、组装装置），忽略了原理可视化，与 PRD 中教师意见"点击拖拽图标拼装置不是真正的交互性"一致
  - 电子/离子流向动画通常非常粗糙或缺失
  - 电极反应方程式不与动画同步
  - 充放电对比功能缺失
  - 使用 Unity WebGL 导致加载时间长、包体积大
- **启示**：C-06 应避免"操作模拟"路线，聚焦"原理演示"——装置图是示意图而非拟真图，动画重点在微观粒子运动而非宏观操作

### 5.3 教学动画/课件类产品

- **形态**：PPT 内嵌 GIF/视频、Flash 动画（已淘汰）、H5 课件
- **现状**：国内高中化学教师最常用的电化学教学辅助方式是 PPT 中嵌入预录制动画或静态示意图
- **不足**：
  - 不可交互（无法暂停在特定帧、无法调速、无法切换充放电模式）
  - 每个电池模型需单独制作动画，工作量巨大且无法复用
  - 方程式与动画不同步
  - 无练习/纠错功能
- **启示**：C-06 的核心优势在于"可交互"和"模型库复用"——一套引擎 + 13 个数据模型，远优于逐个制作静态动画

### 5.5 市场空白与 C-06 定位

**核心发现**：**面向中国高中化学教学的可交互电化学原理演示工具不存在**。

- PhET 是交互模拟标杆但偏物理电路，不覆盖化学电极反应
- 虚拟实验室偏"操作模拟"而非"原理演示"，与教师真实需求错位
- 教师普遍使用 PPT/GIF 教学，不可交互且制作成本高
- **充放电对比**、**反应竞争可视化**、**方程式同步标注**在所有竞品中均为空白

C-06 的差异化定位：**唯一面向中国高中化学课纲、以动画驱动的13个预设电池/电解池模型、支持电子离子流向动画+方程式同步标注+充放电对比+播放控制的电化学原理演示器**。

## 6. 初步技术判断

### 推荐的渲染技术

**分层渲染方案**：

| 层 | 技术 | 职责 |
|----|------|------|
| 装置示意图层 | SVG | 电极、容器、导线、开关/电源、盐桥/隔膜等静态装置元素的渲染，支持缩放无损 |
| 粒子动画层 | Canvas 2D + requestAnimationFrame | 电子小球沿导线路径移动、离子在溶液中迁移的粒子动画，粒子数量可能较多(数十个)时 Canvas 性能优于 SVG |
| 物质变化层 | SVG/CSS 动画 | 电极溶解/析出的形变动画、溶液颜色渐变、气泡上升动画 |
| UI/信息层 | DOM/HTML | 方程式渲染、参数面板、播放控制栏、关键帧文字标注 |

**不推荐 3D WebGL**：电化学装置图本质上是2D示意图，3D引入不必要的复杂度。教师评价"这个模块很难做"已预示了技术工作量大，应选择最直接的技术栈。

### 推荐的架构模式

**数据驱动动画引擎 + 模型配置**

```
模型配置 JSON（13个模型）
  ├─ 装置布局定义（电极位置、容器形状、导线路径）
  ├─ 动画序列定义（电子路径、离子路径、关键帧）
  ├─ 化学数据（半反应方程式、总反应、环境、产物）
  └─ 教学标注（关键帧说明文字、易错提示）
       ↓
动画引擎（通用）
  ├─ 粒子系统（电子/离子沿预设路径移动）
  ├─ 时间轴控制器（播放/暂停/逐步/调速/关键帧）
  ├─ 物质变化渲染器（电极形变/颜色渐变/气泡）
  └─ 方程式同步器（根据动画进度显示对应方程式）
       ↓
渲染输出（SVG + Canvas + DOM）
```

| 层 | 职责 | 说明 |
|----|------|------|
| 数据层 | 模型配置 JSON | 13个模型的装置布局、动画参数、化学数据、教学标注 |
| 引擎层 | 通用动画引擎 | 粒子系统 + 时间轴控制器 + 物质变化渲染 — 与模型数据解耦，一套引擎驱动所有模型 |
| 视图层 | SVG + Canvas + DOM | 装置图 + 粒子动画 + 信息面板 |
| 控制层 | 播放控制 + 状态管理 | 播放/暂停/逐步/调速/模型切换/充放电切换 |

**关键设计原则**：引擎与数据分离。13个模型的差异全部由配置 JSON 描述（装置布局、粒子路径、方程式、关键帧），动画引擎是通用的。新增模型只需添加配置数据，无需修改引擎代码。

### 关键技术风险或难点

1. **模型配置的设计复杂度（高风险）**：13个模型的装置布局、粒子路径、关键帧序列各不相同。需要设计一套足够灵活的配置 Schema，既能描述简单的 Zn-Cu 单液电池，也能描述复杂的氯碱工业电解池（含隔膜、多种离子、反应竞争）。这是架构层面最核心的设计难点。

2. **粒子动画的视觉表达（中高风险）**：电子小球沿导线的流动动画容易实现，但溶液中离子迁移的动画更复杂——需要表达"阳离子向阴极移动、阴离子向阳极移动"且路径不碰撞、视觉上清晰可辨（不同离子用不同颜色/标签区分）。粒子数量、速度、路径的平衡需要反复调试。

3. **动画与方程式的同步（中等风险）**：关键帧系统需要精确同步——动画播放到"Zn失去电子"这一帧时，方程式面板需同步高亮 `Zn - 2e⁻ → Zn²⁺`。逐步模式下，每按一次"下一步"，动画推进一个关键帧同时方程式更新。时间轴的精确控制是技术实现重点。

4. **充放电方向反转（中等风险）**：铅蓄电池和锂离子电池需要支持充放电两个方向的动画。所有粒子流向反转、电极反应方程式反转、正负极标注变化，需要在同一个装置图上无缝切换。

5. **化学方程式渲染（中低风险）**：化学式含上下标、离子电荷、反应条件等格式化需求。使用 HTML/CSS 的 `<sup>`/`<sub>` 可覆盖大部分需求，但若方程式复杂度高（如电子转移标注、反应条件多行）可能需要引入 KaTeX 或自定义渲染组件。

6. **13个模型的数据准确性（质量风险）**：每个模型的电极反应、电子转移数、离子种类和迁移方向、反应产物均需经过专业审核。错误的化学内容会严重损害产品信任度。这不是技术风险而是内容质量管控要求。

7. **装置图视觉设计工作量（项目风险）**：13个模型的装置示意图设计工作量可观——虽然共享通用元素（电极、导线、容器），但燃料电池、氯碱工业、质子交换膜电解水等模型的装置结构差异较大，需要分别设计。建议抽象出可复用的装置组件库（电极组件、容器组件、连接线组件、盐桥组件、隔膜组件等），通过组合降低设计成本。
