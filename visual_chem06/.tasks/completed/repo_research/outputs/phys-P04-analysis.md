# P-04 电路搭建器 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师（兼顾初中），不要求有任何编程或软件操作基础。核心用户画像是高中物理电学部分授课教师，需要在课堂上讲解电路搭建、动态电路分析、五大电学实验以及故障排查。

### 解决什么教学痛点
- **实物电路成本高、操作慢**：每次实验课要准备大量器材，接线耗时长，一节课只能做一两个电路，且器材易损坏
- **动态电路分析靠口述**：调节滑动变阻器时各支路电流电压如何变化，黑板上只能定性画箭头，无法给出定量实时变化
- **内外接法误差理解困难**：学生搞不清为什么内接法测量值偏大、外接法偏小，缺乏直观对比工具
- **电路故障排查难以演示**：断路/短路的现象和检测方法只能口头描述，无法实际操作演示
- **电路图与实物图对应困难**：初中考察重点，学生经常画不出实物连线图，缺乏电路图↔实物图对照训练工具
- **五大电学实验覆盖不完整**：伏安法测电阻、半偏法、测电源EMF和内阻、欧姆表原理、电桥法——每个实验都需要单独搭电路，课堂时间远远不够

### 核心产品价值
**一句话**：老师从元器件库拖拽元件到画布、连线搭建电路，系统自动完成电路分析并实时显示各支路电流电压；调参数即刻看变化，五大实验模板一键加载，覆盖高中电学 90% 的课堂教学和实验演示需求。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选择"实验模板模式" → 点选"测电源EMF和内阻" → 模板电路自动加载 → 微调元件参数（电源EMF=6V, 内阻=2Ω）→ 试跑一遍确认数据正确 → 保存场景 |
| **课中演示（动态电路）** | 选择"自由搭建模式" → 从元器件库拖出电源、开关、滑动变阻器、定值电阻、电流表、电压表 → 逐个连线 → 闭合开关 → 拖动滑动变阻器滑片 → 投影仪上实时展示各支路电流电压变化 → 引导学生归纳"R增大→I减小→U端增大"规律 |
| **讲题（内外接法对比）** | 加载"伏安法测电阻"模板 → 一键切换内接法/外接法 → 对比两种接法下电流表电压表读数差异 → 指出测量值与真实值的偏差方向和原因 → 引导学生记忆"大内小外"选择判据 |
| **实验故障排查教学** | 搭好一个完整电路 → 点击"故障模拟" → 设置某元件断路 → 系统高亮异常支路 → 演示"用电压表并联检测断路位置"的方法 |
| **初中实物连线训练** | 搭好电路图 → 一键切换到"实物图"视图 → 让学生观察电路图与实物图的对应关系 → 再切回电路图加深理解 |

### 怎么用才是"好用"的
- **5分钟搭好一个完整电路**：拖拽+点击连线，不需要精确对齐，系统自动吸附接线柱
- **参数调节即时响应**：拖动滑动变阻器滑片的同时，各表读数实时更新，无延迟感
- **实验模板一键加载**：不需要手动搭建五大实验的标准电路，选模板直接开始讲解
- **错误操作有提示**：超量程表盘变红、短路有警告、连线不合法有提示
- **投影清晰**：1080p投影仪上元件符号、数值、连线清晰可辨

### 交互频率和深度
- **频率**：电学章节教学期间每节课使用，每次 15-30 分钟（电路搭建+调参演示时间较长）
- **深度**：高 —— 涉及元件拖拽放置、连线、参数调节、模板切换、故障模拟、视图切换等多层交互，是所有物理模块中交互最复杂的

### 功能清单（按重要性排序）

### 哪些功能需要实体管理（创建/选中/删除/编辑对象）
- **所有元件**：电源、电阻、电容、开关、电流表、电压表、灯泡、电阻箱、电动机 —— 都是独立实体，可创建、选中、删除、移动、编辑参数
- **导线**：连接两个接线柱的导线也是独立实体，可选中删除
- **接线柱**：每个元件上的接线柱是连接点，需要作为实体的端口建模

### 哪些功能需要 undo/redo
- **必须有**：拖拽放置元件、连线、删除元件/导线、修改元件参数 —— 都需要undo/redo
- 这是典型的编辑器场景，undo/redo是刚需
- 预计需要完整的命令模式（Command Pattern）支撑

### 哪些功能是参数驱动的（调参数 → 画面实时变化）
- 滑动变阻器滑片拖动 → 阻值变化 → 全电路电流电压重算
- 电源EMF/内阻调节 → 全电路重算
- 定值电阻阻值调节 → 全电路重算
- 开关断开/闭合 → 电路拓扑变化 → 全电路重算
- 电流表/电压表量程切换 → 内阻变化 → 全电路重算（微调）
- 所有参数变化都要求**实时**重新求解电路方程并更新显示

### 哪些功能是模拟/动画驱动的（播放/暂停/逐步）
- 电容充放电过程（指数衰减动画，需要时间轴控制）
- 电流方向动画（移动点/箭头表示电流流向）
- 故障模拟中的逐步排查演示
- 注意：本模块的核心并非时间轴驱动的动画，而是**参数驱动的实时计算**，这一点与运动模拟器等模块有本质区别

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **推荐 2D Canvas + SVG 混合**：
  - Canvas：电路画布主体（元件渲染、导线绘制、电流动画），性能好，适合大量元件和连线
  - SVG：仪表盘（电流表/电压表表盘）、元件符号（标准电路符号需要精确矢量图形），缩放不失真
  - DOM：参数面板、数值显示面板、工具栏等UI部分
- **不需要 3D WebGL**：电路搭建是纯2D场景

#### 数学/物理计算需求
- **电路求解算法（核心难点）**：
  - 改进节点电压法（Modified Nodal Analysis, MNA）：这是电路仿真的工业标准方法，SPICE就使用此算法
  - 需要构建节点电导矩阵（G矩阵）和源向量，求解线性方程组 Gv = i
  - 串联/并联电阻自动等效简化
  - 支持含电压源、电流源的混合电路
  - 理想元件模型（电流表内阻=0或很小，电压表内阻=∞或很大）
- **线性代数求解器**：
  - 高斯消元法或LU分解求解线性方程组
  - 电路规模有限（教学电路通常<20个节点），性能不是瓶颈
  - 可用 Math.js 的线性代数功能，也可自研轻量求解器
- **电容充放电**：指数衰减方程 q(t) = Q₀·e^(-t/RC)，需要数值积分或解析解
- **灯泡非线性电阻**：电阻随温度/功率变化的非线性模型，需要迭代求解（牛顿法）

#### 公式渲染需求
- KaTeX渲染：欧姆定律 U=IR、闭合电路欧姆定律 ε=I(R+r)、功率公式 P=UI 等
- 误差分析公式（内接法偏大原因、外接法偏小原因）
- U-I图的线性拟合公式和结果

### 交互依赖

#### 交互模式
**编辑器型为主，参数驱动为辅** —— 这是物理产品线中最典型的编辑器型模块：
- **编辑器型**（主要）：拖拽放置元件、连线、选中/删除/移动元件、undo/redo
- **参数驱动型**（辅助）：调节元件参数 → 电路实时重算 → 数值实时更新
- **模拟演示型**（少量）：电容充放电动画、电流方向动画

#### 关键交互
| 交互方式 | 应用场景 |
|---------|---------|
| **拖拽（Drag）** | 从元器件库拖元件到画布；画布上移动元件位置；拖动滑动变阻器滑片 |
| **点击连线** | 从接线柱A点击拖拽到接线柱B，创建导线 |
| **点选** | 选中元件弹出参数面板；选中导线高亮 |
| **右键/长按菜单** | 删除元件、设置故障（断路/短路） |
| **滑块** | 参数面板中的元件参数调节 |
| **输入框** | 精确输入参数值 |
| **按钮** | 开关闭合/断开、量程切换、视图切换、模板选择 |

#### 是否需要实体系统（统一的对象模型）
**必须需要，且是所有物理模块中最复杂的实体系统**：
- **元件实体**：每种元件都是实体，有唯一ID、类型、位置、参数、接线柱列表、状态（正常/断路/短路）
- **导线实体**：连接两个接线柱的导线，有起点和终点接线柱引用
- **接线柱（端口）模型**：每个元件有固定数量的接线柱（电源2个、电流表3个含量程选择、滑动变阻器4个），每个接线柱有位置和连接状态
- **电路拓扑图**：由元件和导线构成的图数据结构，用于电路分析算法的输入
- **实验模板**：预定义的电路拓扑和元件参数集合

#### 是否需要命令系统（undo/redo）
**必须需要**：
- 添加元件（AddComponentCommand）
- 删除元件（RemoveComponentCommand）
- 移动元件（MoveComponentCommand）
- 添加导线（AddWireCommand）
- 删除导线（RemoveWireCommand）
- 修改参数（ChangeParamCommand）
- 切换开关状态（ToggleSwitchCommand）
- 设置故障（SetFaultCommand）

## 5. 竞品分析

### 竞品1：PhET Circuit Construction Kit (科罗拉多大学)

- **产品形态**：Web（HTML5），免费开源，教育领域最知名的电路模拟器
- **版本**：DC版、DC Virtual Lab版、AC版、AC+DC版
- **技术实现**：早期Java Applet，现已全面迁移到HTML5；PhET自研的仿真框架（scenery渲染引擎 + 自研物理引擎）
- **做得好的地方**：
  - 拖拽交互极其流畅，接线柱自动吸附做得非常好
  - 电流动画（蓝色移动点）直观表示电流方向和大小
  - 元件种类合理（电池、电阻、灯泡、开关、保险丝、电流表、电压表）
  - 支持"探索日常物品是导体还是绝缘体"等趣味功能
  - 国际化做得好，支持多语言
  - 完全免费，教育友好
- **不足的地方**：
  - 不支持实验模板一键加载（必须手动搭建）
  - 没有故障模拟功能
  - 没有内/外接法对比功能（面向通用教育，不针对中国高考考点）
  - 没有电路图↔实物图切换
  - 元器件种类有限（没有滑动变阻器专项组件、没有电阻箱）
  - AC版缺乏高中阶段需要的特定实验模型
- **对我们的启示**：PhET的拖拽交互体验是行业标杆，我们需要达到或接近这个水平；但PhET是通用教育工具，我们的差异化在于针对中国高中物理课纲的五大实验模板和特定教学功能

### 竞品2：Falstad Circuit Simulator (CircuitJS1)

- **产品形态**：Web，开源（GPL），在线访问 falstad.com/circuit/
- **技术实现**：原始Java代码通过GWT（Google Web Toolkit）交叉编译为JavaScript运行在浏览器中；使用自研电路分析引擎
- **做得好的地方**：
  - 电路分析引擎非常强大，支持DC/AC/瞬态分析
  - 电压用颜色编码（绿=正，灰=地，红=负），电流用黄色移动点表示，视觉效果好
  - 元件库极其丰富（远超教学需求）
  - 支持JavaScript API外部控制
  - 完全开源，可以参考其电路分析算法实现
  - 社区活跃，持续维护
- **不足的地方**：
  - 界面偏工程师风格，对教师用户不够友好
  - 没有针对教学场景的实验模板
  - 没有实物图视图
  - 交互方式偏传统（菜单驱动），不如PhET的拖拽直观
  - 缺乏中文支持
  - 元件过多反而增加了教师的选择困难
- **对我们的启示**：Falstad的电路分析引擎实现（特别是MNA方法）是极好的技术参考；但UI/UX需要完全面向教师重新设计，大幅简化元件库

### 竞品3：EveryCircuit

- **产品形态**：Web + iOS + Android，商业产品（$14.99一次性购买或$1/月订阅）
- **技术实现**：自研仿真引擎，针对移动端交互优化；支持DC/AC/瞬态分析
- **做得好的地方**：
  - 实时动画效果最佳：导线上显示mini波形图，直流显示电压数值，数字信号颜色编码
  - 交互设计极其直观：手指拖拽元件、捏合缩放、摇晃启动振荡器
  - 运行中可实时调参（拨动开关、旋转电位器、调节电压）
  - 自动布线功能优秀
  - 社区电路库（290万+电路设计）
  - 内置示波器
- **不足的地方**：
  - 付费产品，不适合大规模教育部署
  - 面向电子工程师，不面向中学物理教学
  - 没有教学实验模板
  - 元件库偏专业（MOSFET、运放等），缺乏教学专用元件（滑动变阻器、电阻箱）
  - 没有中文支持
  - 没有误差分析等教学辅助功能
- **对我们的启示**：EveryCircuit的实时动画和交互设计是最佳实践；mini波形和颜色编码的信号可视化方式值得借鉴；自动布线算法可以参考

### 竞品4：Tinkercad Circuits (Autodesk)

- **产品形态**：Web，免费，Autodesk旗下教育平台
- **技术实现**：Web前端 + 云端仿真引擎
- **做得好的地方**：
  - 面向教育设计，UI极其友好
  - 支持虚拟面包板+元件的实物化呈现，学生容易理解
  - 集成Arduino编程，支持代码+电路联动
  - 班级管理功能（教师创建班级、布置作业、监控进度）
  - 完全免费，跨平台
- **不足的地方**：
  - 偏向maker/STEM教育，不针对中国高中物理课纲
  - 电路分析偏简单，不支持复杂的动态电路分析
  - 没有伏安法、半偏法等特定实验模板
  - 需要网络连接（云端仿真）
  - 响应速度受网络影响
- **对我们的启示**：Tinkercad的教育友好度和实物化呈现思路值得参考；面包板视图可以作为"实物图"视图的设计参考

### 竞品5：Circuit Sandbox (Spinning Numbers)

- **产品形态**：Web，开源，教育专用
- **技术实现**：纯JavaScript编写的电路仿真引擎（非SPICE衍生），支持DC/AC/瞬态分析
- **做得好的地方**：
  - 纯JavaScript实现，轻量级，适合嵌入网页
  - 面向教学设计，功能精简
  - 支持原理图捕获和仿真一体化
- **不足的地方**：
  - 界面较原始，交互体验一般
  - 功能有限，不支持复杂教学场景
  - 缺乏维护和社区
- **对我们的启示**：纯JavaScript电路引擎的可行性验证；说明教学级电路仿真不需要SPICE级别的复杂度

**核心差异化**：所有现有竞品都没有针对中国高中物理课纲设计的实验模板、误差分析、故障模拟和电路图↔实物图切换。这四个功能是我们产品的核心竞争壁垒。

## 6. 初步技术判断

### 推荐的渲染技术

**Canvas（主画布）+ SVG（精密元件符号）+ DOM（UI面板）三层混合架构**：

| 层级 | 技术 | 负责内容 |
|------|------|---------|
| 底层画布 | Canvas 2D | 导线绘制、电流动画（移动点/箭头）、网格背景、选区框 |
| 元件层 | SVG 或 Canvas预渲染 | 10种标准电路符号（需要矢量精度，缩放不失真）；实物图模式下的元件实物图 |
| UI层 | DOM + CSS | 元器件库面板、参数调节面板、数值显示面板、工具栏、实验模板选择 |
| 图表层 | Canvas / Chart.js | U-I图绘制、电容充放电曲线等 |

选择理由：
- Canvas性能好，适合频繁重绘（滑动变阻器拖动时需要60fps重绘电流动画）
- SVG精度高，适合电路符号（电阻的锯齿线、电容的平行板、电流表的圈A等）需要在任意缩放下保持清晰
- 实物图模式可能需要加载预制的元件图片资源

### 推荐的架构模式

**编辑器架构（类似立体几何展示台的实体-命令系统）+ 电路分析引擎**：

```
┌─────────────────────────────────────────────────┐
│                   视图层 (View)                    │
│  Canvas画布 + SVG元件 + DOM面板                    │
├─────────────────────────────────────────────────┤
│               交互控制层 (Controller)               │
│  工具系统(选择/放置/连线/删除) + 事件分发           │
├─────────────────────────────────────────────────┤
│                 实体层 (Model)                      │
│  元件实体 + 导线实体 + 电路拓扑图                   │
│  命令系统(undo/redo) + 实体Store                   │
├─────────────────────────────────────────────────┤
│              电路分析引擎 (Engine)                   │
│  MNA矩阵构建 → 线性方程组求解 → 结果分发           │
│  故障注入 → 拓扑变换 → 重新求解                    │
├─────────────────────────────────────────────────┤
│              实验模板系统 (Templates)                │
│  电路拓扑JSON + 元件参数预设 + 操作步骤描述         │
└─────────────────────────────────────────────────┘
```

核心架构要素：
1. **实体系统**：统一的元件/导线对象模型，所有元件继承自BaseComponent基类，包含id、type、position、params、ports等属性
2. **命令系统**：完整的undo/redo支持，每个用户操作封装为Command对象
3. **电路拓扑图**：图数据结构（邻接表），节点=接线柱/导线交叉点，边=元件/导线段
4. **MNA电路分析引擎**：核心算法模块，输入电路拓扑和元件参数，输出各节点电压和各支路电流
5. **响应式数据流**：参数变化 → 触发拓扑更新 → 触发MNA求解 → 触发视图更新，全链路响应式
6. **模板系统**：JSON描述实验电路的拓扑和参数，加载后实例化为实体

### 关键技术风险或难点

#### 风险1（高）：MNA电路分析引擎的正确性
- **描述**：改进节点电压法（Modified Nodal Analysis）是电路仿真的核心，需要正确处理：理想电压源（导致矩阵扩展）、电流表（近似零内阻的电压源）、电压表（极大内阻的电阻）、开关（零阻/断路切换）、短路检测、浮空节点检测
- **难度**：中高。算法本身有成熟理论，但实现细节容易出bug（矩阵奇异、数值精度、边界条件）
- **缓解措施**：参考Falstad CircuitJS1的开源实现（GPL协议需注意）；构建全面的单元测试用例库；先实现纯电阻电路，再逐步支持电容/电感

#### 风险2（高）：拖拽连线的交互体验
- **描述**：从元器件库拖拽元件到画布、元件的放置/移动/对齐、接线柱之间的连线绘制 —— 这些交互的流畅度直接决定产品可用性
- **难度**：中高。需要处理碰撞检测、磁性吸附、导线自动路由（避开元件）、连线合法性检查
- **缓解措施**：参考PhET的交互设计；接线柱吸附半径设大一些（容错）；导线先用折线方案（水平+垂直段），暂不追求曲线美化

#### 风险3（中）：实物图渲染
- **描述**：需要将标准电路图一键转换为实物连线图，元件从"电路符号"变为"实物外观"，导线从"示意线"变为"模拟实物导线"
- **难度**：中。需要两套渲染器，元件位置可能需要重新布局（电路图紧凑，实物图更分散）
- **缓解措施**：P2优先级，可在MVP后再实现；实物图布局可预定义（每种实验模板提供实物图布局模板）

#### 风险4（中）：非线性元件求解
- **描述**：灯泡电阻随温度变化（非线性）、电动机有反电动势 —— 这些非线性元件需要迭代求解
- **难度**：中。需要牛顿-拉夫逊迭代法
- **缓解措施**：先用线性近似（灯泡用固定电阻），非线性模型作为增强功能后续迭代

#### 风险5（低-中）：电容充放电的时域仿真
- **描述**：电容充放电是时域过程，需要时间步进的数值积分（如梯形法/后向欧拉法），与其他元件的稳态DC分析不同
- **难度**：中。需要在MNA框架中加入动态元件的伴随模型
- **缓解措施**：P2优先级；初期可用解析解（RC电路有闭合解）；复杂电路的瞬态分析后续再做

#### 整体复杂度评估
**这是物理产品线中复杂度最高的模块**，原因：
1. 是唯一的"编辑器型"模块（其他都是参数驱动或模拟演示型）
2. 需要完整的实体系统和命令系统
3. 核心算法（MNA）有非平凡的数学复杂度
4. 两套渲染模式（电路图 + 实物图）
5. 交互模式最复杂（拖拽、连线、参数调节、模板加载、故障注入的组合）
6. 10种不同类型的元件，每种有独特的参数和行为模型

建议分阶段交付：
- **阶段1**：核心搭建（元件拖拽+连线+纯电阻电路MNA求解+数值显示）
- **阶段2**：完整元件（电容/灯泡/电动机）+ 五大实验模板 + 故障模拟
- **阶段3**：实物图视图 + 电容充放电动画 + U-I图绘制 + 非线性元件
