# C-04 元素周期表交互平台 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
初中/高中化学教师，核心用户画像是高中化学教师，需要在课堂上讲解元素性质、周期律、电子排布等知识点。不要求有编程或专业软件操作基础。

### 解决什么教学痛点
- **元素数据查阅不便**：教师讲解某元素时需要翻书翻表，数据分散在不同资料中，课堂上无法快速获取权威数据
- **周期律趋势难以直观展示**：原子半径、电负性、电离能等属性随周期/族的变化趋势，仅靠板书或静态图表难以让学生形成直觉
- **电子排布理解困难**：电子层排布（初中）、亚层排布式（高中必修）、轨道表示式（高中选修）三个层次的电子排布，板书难以动态展示填充过程
- **元素分类与对比**：118 个元素的分类（碱金属、过渡金属、非金属等）及同族/同周期性质递变，需要一个全局视图来辅助理解
- **数据权威性焦虑**：教师担心非权威来源的数据误导学生，需要明确标注数据来源（IUPAC、NIST、CRC Handbook）

### 核心产品价值
**一句话**：教师打开浏览器，在可交互的完整元素周期表上点击任意元素即可查看权威详情，切换属性染色模式即可直观展示周期律趋势，播放电子排布动画即可分层讲解电子结构——覆盖初高中化学元素教学的核心场景。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 元素周期表主界面 → 点击要讲的元素查看详情 → 确认数据准确 → 切换到属性染色模式预览趋势图效果 |
| **课中演示-元素性质** | 投影仪打开 → 点击目标元素（如 Na）→ 展示详情卡片（基本信息、物理/化学性质、发现史、应用）→ 翻转故事卡看趣味事实 |
| **课中演示-周期律** | 切换属性染色模式（如"电负性"）→ 全表渐变着色，直观看出同周期从左到右递增 → 勾选同族元素生成折线图对比 |
| **课中演示-电子排布** | 点击元素 → 进入电子排布动画 → 选学段模式（初中/高中必修/高中选修）→ 播放逐层/逐轨道填充动画 |
| **讲题-金属活动性** | 展示周期表下方金属活动性顺序条 → 点击顺序中的金属 → 高亮其在周期表中的位置 → 对比讨论 |

### 怎么用才是"好用"的
- **即点即看**：点击元素 → 立即显示详情，无需等待加载
- **数据权威可信**：每个页面底部有明确数据来源声明，教师可以放心在课堂使用
- **分层适配学段**：初中模式 / 高中必修模式 / 高中选修模式 切换，不同学段看到不同深度的内容
- **投影清晰**：118 个元素格在 1080p 投影仪上清晰可辨，元素符号和原子序数字体足够大
- **趋势一目了然**：属性染色模式下，颜色深浅直接映射数值大小，无需读数就能看出规律

### 交互频率和深度
- **频率**：每节元素化学课使用 1-2 次，每次 5-20 分钟
- **深度**：中等偏浅 — 主要是浏览查看和视觉对比，没有复杂的建模或编辑操作；电子排布动画是最深的交互

### 需要实体管理的功能
本模块**不需要传统实体管理**（创建/选中/删除/编辑对象）。118 个元素是预定义的静态数据集，用户不会创建或删除元素。唯一的"选择"操作是：
- 点击选中一个元素查看详情
- 勾选多个元素进行属性对比
- 这些都是**数据浏览型选择**，不是编辑器型实体管理

### 需要 Undo/Redo 的功能
**不需要**。本模块是数据浏览和可视化工具，用户不产生可编辑的变更状态。

### 参数驱动的功能
- 属性染色模式：选择不同属性 → 全表配色实时变化
- 折线图对比：勾选/取消元素 + 切换属性 → 图表实时更新
- 学段切换：初中/高中必修/高中选修 → 详情内容和电子排布动画深度切换

### 模拟/动画驱动的功能
- 电子排布动画：逐层/逐轨道填充电子的动画播放
- 元素故事卡翻转动画（轻量 CSS 动画）

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **周期表主界面**：推荐 **DOM + CSS Grid**。标准 18 列周期表是规则的网格布局，CSS Grid 天然适配；118 个 DOM 元素的性能完全可控；便于实现 hover、点击、配色切换等交互；响应式适配也更容易
- **属性染色模式**：纯 CSS — 动态计算每个元素格的 `background-color`，基于属性值线性映射到渐变色阶
- **折线图对比**：需要图表库 — 推荐 **ECharts** 或 **Chart.js**（折线图、数据点 hover、响应式）
- **电子排布动画**：推荐 **SVG + CSS Animation / GSAP**
  - 初中模式（圆形电子层 + 小球）：SVG 圆环 + 动画定位
  - 高中选修模式（轨道表示式 + 箭头填充）：SVG 绘制轨道能级图 + 箭头动画
  - Canvas 也可行但 SVG 更适合这种结构化、可交互的图形
- **元素故事卡翻转**：纯 CSS 3D Transform（`rotateY`），无需额外渲染库

#### 数学/物理计算需求
- **无复杂计算**：属性染色只需线性归一化（min-max 映射到色阶），折线图是纯数据绑定
- 电子排布动画需要预定义的电子填充规则（构造原理、泡利不相容、洪特规则），但这是**规则引擎而非实时计算**

#### 公式渲染需求
- 电子排布式需要上下标渲染（如 `1s²2s²2p⁶3s¹`），可用 Unicode 上标字符或轻量 KaTeX
- 化学式渲染（如 `H₂O`、`Na⁺`）需要下标和上标，同理可用 Unicode 或 KaTeX

#### 数据管理需求（核心）
- **118 元素 × 15+ 字段的结构化数据**：这是本模块最大的技术特征
- 数据字段包括：原子序数、符号、中英文名、相对原子质量、周期、族、分类、电子构型、密度、熔点、沸点、主要化合价、电负性、常温状态、化学性质文本、发现史文本、应用文本、趣味事实文本、教学关联文本
- 数据来源：IUPAC 2021、NIST、CRC Handbook（97th Ed.）
- 需要 `version` 和 `lastUpdated` 版本控制字段
- 数据量评估：5 份 MD 文档，每份覆盖一个周期范围（1-18、19-36、37-54、55-86、87-118），需转换为结构化 JSON

### 交互依赖

#### 交互模式
**数据浏览型 + 参数驱动型（混合）**
- 核心是数据浏览：点击元素 → 查看详情，是典型的信息检索模式
- 属性可视化是参数驱动：选择属性 → 全表配色变化 / 图表生成
- 电子排布动画是模拟演示型（轻量）：播放/暂停动画

#### 关键交互
- **点击（Click）**：点击元素格 → 打开详情页/故事卡
- **悬停（Hover）**：鼠标悬停元素格 → 显示简要信息浮窗
- **下拉选择（Select）**：选择属性染色类型、学段模式
- **复选勾选（Checkbox）**：勾选多个元素进行折线图对比
- **播放控制（Play/Pause）**：电子排布动画的播放/暂停/逐步
- **联动点击**：金属活动性顺序条点击 → 高亮周期表对应元素

#### 是否需要实体系统
**不需要**。所有元素是只读预定义数据，不存在用户创建/删除/编辑对象的场景。

#### 是否需要命令系统（Undo/Redo）
**不需要**。用户操作是纯浏览和视图切换，不产生可撤销的状态变更。

## 5. 竞品分析

### Ptable.com
- **产品形态**：Web 应用（免费）
- **技术实现**：纯 DOM + CSS，元素格用 `<td>` 表格布局；属性可视化用渐变背景色；无 Canvas/WebGL
- **做得好的地方**：
  - 信息密度极高，单页面展示海量数据
  - 属性叠加层（Orbital、Property、Compound）切换流畅
  - 支持多种属性的热力图着色
  - 同位素、化合物、轨道形状等高级功能丰富
- **不足**：
  - 界面较为老旧，视觉设计偏学术风，对中学教师不够友好
  - 无电子排布动画
  - 中文支持不佳
  - 信息过载，初中/高中不需要那么多字段
- **启示**：DOM + CSS 完全可以胜任元素周期表渲染，无需 Canvas；属性染色模式的 UX 可参考其切换方式

### Royal Society of Chemistry (RSC) Periodic Table
- **产品形态**：Web + 移动端 App
- **技术实现**：Web 端 DOM 渲染，响应式设计；移动端原生 App
- **做得好的地方**：
  - 视觉设计精良，配色专业
  - 元素详情页信息组织清晰，分板块展示
  - 每个元素配有播客音频和视频
  - 趋势图（Trends）做得直观
- **不足**：
  - 无电子排布动画
  - 趋势图只能查看预设的（无自由选择元素对比）
  - 无中文版，无课标关联
- **启示**：详情页的板块组织和视觉设计值得参考；趋势图应支持更灵活的自定义对比

### WebElements
- **产品形态**：Web 应用（学术级参考工具）
- **技术实现**：传统 HTML + JavaScript
- **做得好的地方**：
  - 数据最全面、最权威（马克·温特教授维护 30 年）
  - 轨道形状的 3D 渲染
  - 晶体结构展示
- **不足**：
  - 界面非常学术化，对中学教师来说过于复杂
  - 导航层级深，找一个数据需要多次点击
  - 移动端体验差
- **启示**：数据全面性的标杆，但我们的目标是"中学教师友好"，需要大幅简化信息层级

### 竞品共性总结与差异化机会
- **共性**：都采用 DOM 渲染周期表（无人用 Canvas），都有属性可视化，都有详情页
- **差异化机会**：
  1. **课标分层**：初中/高中必修/高中选修 三档内容适配，竞品均无此功能
  2. **电子排布动画**：分学段的逐步填充动画，竞品中缺失
  3. **中文本土化**：中文元素名、中文介绍文本、课标关联标注
  4. **信息密度控制**：不追求学术完整性，聚焦中学教学核心字段
  5. **故事卡**：趣味化呈现（发现史+应用+趣味事实），增强课堂生动性

## 6. 初步技术判断

### 推荐的渲染技术
- **主界面（周期表网格）**：DOM + CSS Grid — 性能无忧（仅 118 个格子），开发效率最高，交互（hover/click/配色切换）实现最简单
- **属性染色**：CSS 动态样式 — 通过 JavaScript 计算归一化值，设置内联 `background-color`
- **折线图**：ECharts 或 Chart.js — 成熟图表库，折线图+hover+响应式
- **电子排布动画**：SVG + 动画库（GSAP 或 Framer Motion）— 圆形电子层、轨道能级图、箭头填充均适合 SVG 路径动画
- **故事卡翻转**：CSS 3D Transform — 纯 CSS 实现，无额外依赖
- **公式渲染**：Unicode 上下标优先（`¹²³⁴⁵⁶⁷⁸⁹⁰`、`₁₂₃`），复杂公式用 KaTeX

### 推荐的架构模式
- **数据驱动型单页应用**（非编辑器型）
  - 核心是一个大型静态数据集（118 元素 JSON）+ 多种视图（表格视图、详情视图、图表视图、动画视图）
  - 状态管理简单：当前选中元素、当前属性、当前学段、当前勾选的对比元素列表
  - 无需实体系统、命令系统、undo/redo
  - 推荐 React + Zustand（或 Context）足矣
- **数据层设计**：
  - 118 元素数据从独立 JSON 文件加载（非硬编码），支持版本化更新
  - JSON Schema 定义数据契约，确保字段一致性
  - 数据预处理脚本：将 5 份 MD 源文档转换为结构化 JSON
- **视图层设计**：
  - `PeriodicTableView`：主网格（CSS Grid 18 列，特殊处理镧系/锕系的两行偏移）
  - `ElementDetailPanel`：详情面板（侧边栏或模态框，8 个板块 Tab/折叠）
  - `PropertyHeatmapLayer`：属性染色叠加层
  - `ComparisonChart`：折线图对比面板
  - `ElectronConfigAnimation`：电子排布动画组件（按学段切换 3 种渲染模式）
  - `StoryCard`：故事卡翻转组件
  - `ActivitySeriesBar`：金属活动性顺序条

### 关键技术风险或难点

| 风险点 | 等级 | 说明 | 缓解策略 |
|--------|------|------|---------|
| 数据录入与质量保证 | **高** | 118 元素 × 15+ 字段，数据量大且要求权威准确；5 份 MD 文档需人工/脚本转换为 JSON，转换过程易出错 | 编写自动化转换脚本 + JSON Schema 校验 + 抽样人工复核 |
| 电子排布动画复杂度 | **中高** | 3 种学段模式（圆形层模型/亚层排布/轨道表示式）需要 3 套不同的 SVG 动画方案；高中选修模式涉及构造原理、洪特规则等填充逻辑 | MVP 仅做前 36 号元素，降低工作量；先实现初中模式（最简单），再逐步添加 |
| 周期表布局的特殊性 | **低** | 标准长式周期表不是规则矩形网格——镧系/锕系需要单独两行，H/He 位置特殊，某些族列需要留空 | CSS Grid 的 `grid-column` / `grid-row` 精确控制每个元素格的位置，预定义位置映射表 |
| 属性数据缺失处理 | **低** | 部分超重元素（如 Og）的物理性质为估算值或缺失 | 明确标注"估算"或"暂无数据"，UI 上用灰色/虚线表示 |
| 响应式与投影适配 | **低** | 118 个格子在小屏上可能拥挤 | 设定最小宽度（如 1024px），支持缩放；课堂场景以桌面/投影为主，移动端可降级 |
