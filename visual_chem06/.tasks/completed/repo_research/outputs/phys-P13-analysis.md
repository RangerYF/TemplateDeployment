# P-13 电磁感应 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程或软件操作基础。核心用户画像是需要讲解电磁感应综合题型的高中物理教师——电磁感应是高考压轴题高频考点，涉及力学、电学、电路的综合分析。

### 解决什么教学痛点
- **楞次定律判断过程抽象**：磁通量变化→感应电流方向→安培力方向，这一多步推理链条在黑板上只能静态画箭头，学生难以建立"阻碍变化"的直觉
- **单棒/双棒运动与电路耦合难以动态展示**：导体棒的运动速度影响感应电动势，电动势影响电流，电流又通过安培力反作用于运动——这个闭环因果链条是高中物理最复杂的动态过程之一，黑板无法演示
- **终态分析缺乏直观过程**：学生只看到"最终匀速"的结论，但看不到从初始状态到终态的完整演化过程（指数衰减、趋近等行为）
- **含电容/电源的特殊电路难以理解**：电容充电过程中电流如何变化、电源如何改变终态速度，口述难以解释清楚
- **v-t、i-t 图像与运动过程的对应关系**：学生经常在考试中画错这些图像的形状（指数型 vs 线性型）

### 核心产品价值
**一句话**：老师选择电磁感应模型（楞次定律/单棒/双棒/竖直导轨），设好参数，系统实时模拟导体棒运动+电路响应的完整耦合过程，配合 v-t/i-t 图联动和逐步受力分析动画，让高考最难的综合分析题变得可见可感。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选择模型（如"单棒+纯电阻"）→ 设参数（B=0.5T, L=0.5m, v0=5m/s, R=2Ω）→ 试运行确认终态正确 → 准备好暂停点 |
| **课中演示（楞次定律）** | 选"磁棒-线圈"模型 → 拖动磁棒插入线圈 → 系统逐步动画：磁通量方向→变化方向→感应电流方向→安培力方向 → 切换四种情况对比 |
| **课中演示（单棒终态）** | 选"单棒基础"模型 → 播放模拟 → 观察导体棒减速过程 → 同步看 v-t 图的指数衰减曲线 → 暂停某时刻点"分析受力" → 系统逐步展示安培力分析流程 |
| **讲题（含电源终态）** | 选"单棒+含电源" → 设电源电动势 → 播放观察导体棒最终匀速 → 验证 v终=ε0/(BL) → 改参数看终速度变化 |
| **讲题（双棒共速）** | 选"双棒基础" → 设两棒质量和初速 → 播放观察两棒最终共速 → 验证动量守恒 → 对比 v-t 图上两条曲线的交汇 |

### 怎么用才是"好用"的
- **一看就懂**：导体棒运动方向、电流方向、安培力方向的箭头颜色区分清晰（蓝=速度，橙=电流，红=安培力）
- **逐步分析**：暂停后点击"分析受力"，系统按步骤（速度方向→EMF方向→电流方向→安培力方向）逐步展示，每步有文字说明
- **图像联动**：v-t 和 i-t 图与动画同步，当前时刻在图上有竖线标记
- **终态高亮**：终态数值（终速度、终电流）用黄色高亮显示，方便课堂讲解

### 交互频率和深度
- **频率**：电磁感应章节集中使用，约 4-6 节课，每节课使用 2-3 次，每次 5-15 分钟
- **深度**：中高——主要是参数调节 + 模拟播放 + 暂停分析受力，但模型种类多（6 类以上）、参数组合丰富

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式：2D Canvas（主）
- 导轨、导体棒、磁场区域（×/· 符号）、箭头标注：2D Canvas 绑定即可
- 楞次定律的磁棒-线圈模型：2D 侧视图足够
- v-t / i-t 实时图表：Canvas 自绘或 Chart.js
- 不需要 3D 渲染——所有模型都可以在 2D 平面内清晰表达

#### 数学/物理计算需求（核心且复杂）
- **ODE 数值积分**：单棒运动方程 `m·dv/dt = -B²L²v/(R+R_棒) - μmg`（含指数衰减），需要 RK4 或类似数值方法
- **双棒耦合 ODE**：两棒的运动方程耦合（棒1的速度影响棒2的安培力，反之亦然），需要同时求解两个耦合微分方程
- **含电容电路的 ODE**：电容充电过程引入额外的微分方程（`C·dU/dt = i`），耦合度更高
- **终态解析解**：部分模型有解析终态公式（v终 = mg(R+R_棒)/(B²L²) 等），用于验证数值结果
- **动量守恒验证**：双棒模型需要实时验证总动量守恒

#### 公式渲染需求
- KaTeX 渲染公式：ε = BLv、F = BIL、v(t) = v0·exp(...)  等
- 终态公式和安培力分析步骤中的公式标注

### 交互依赖

#### 交互模式：模拟演示型为主，混合参数驱动
- **主要模式**：选模型 → 调参数 → 播放模拟 → 暂停分析
- **参数驱动**：调参后可重新运行模拟，终态数值实时更新
- **编辑器型**（仅自由组装模式）：拖拽放置电路元件

#### 关键交互
- **滑块 + 数值输入框**：参数设置（B、L、m、R、v0、μ 等）
- **时间轴控制**：播放/暂停/单步/调速（与 P-02 运动模拟器一致）
- **拖拽**：楞次定律中拖动磁棒；自由组装模式中拖拽元件
- **点击**：暂停后点击"分析受力"触发逐步分析
- **模型切换按钮**：顶部 6 个模型选择按钮

#### 是否需要实体系统
- 预设模型模式：**不需要**统一的实体系统，对象固定（导轨+棒+电路元件）
- 自由组装模式（P2）：需要简单的实体模型（导轨、棒、电阻/电容/电源的节点连接关系）

#### 是否需要命令系统（undo/redo）
- 预设模型模式：**不需要**
- 自由组装模式（P2）：可选

## 5. 竞品分析

### 5.1 PhET Faraday's Law / Faraday's Electromagnetic Lab

- **产品形态**：Web（HTML5），免费，支持 PC / iPad / Chromebook
- **技术实现**：基于 PhET 自研框架（Scenery 渲染引擎），HTML5 Canvas
- **核心功能**：
  - 拖动磁棒靠近/远离线圈，观察感应电流（灯泡亮度变化）
  - 可调线圈匝数、磁场强度
  - 电压表显示感应电动势
- **做得好的地方**：
  - 交互极其直观——直接拖磁棒
  - 实时反馈（灯泡亮度、电压表偏转）让因果关系一目了然
  - 跨平台兼容性强
- **不足之处**：
  - **没有导体棒/导轨模型**——只有磁棒+线圈，无法演示高考核心的单棒/双棒模型
  - **没有终态分析**——没有 v-t/i-t 图，无法展示运动-电路耦合的动态过程
  - **没有安培力分析流程**——缺乏逐步推理展示
  - 没有定量参数设置（如磁感应强度 B 的具体数值）
- **对我们的启示**：拖拽磁棒的交互方式值得借鉴（用于楞次定律模块）；但我们需要覆盖的模型范围远超 PhET

来源：[PhET Faraday's Law](https://phet.colorado.edu/en/simulations/faradays-law) | [PhET Faraday's Electromagnetic Lab](https://phet.colorado.edu/en/simulations/faradays-electromagnetic-lab)

### 5.2 oPhysics 电磁感应模拟

- **产品形态**：Web（HTML5/JS），免费
- **技术实现**：基于 GlowScript/VPython 或自研 JS
- **核心功能**：
  - 磁棒穿过线圈的电磁感应演示
  - 显示感应电流方向
- **做得好的地方**：
  - 简洁、加载快
  - 物理过程清晰
- **不足之处**：
  - 功能单一，只有磁棒-线圈模型
  - 没有导轨模型、终态分析、图像联动
  - 交互有限
- **对我们的启示**：简洁 UI 值得参考，但功能深度不够

来源：[oPhysics Electromagnetic Induction](https://ophysics.com/em11.html)

### 5.3 GeoGebra 电磁感应资源

- **产品形态**：Web，免费，用户可自建 Applet
- **技术实现**：GeoGebra 引擎（Java → HTML5）
- **核心功能**：
  - 法拉第定律可视化（磁通量变化 → 感应电动势）
  - 参数可调的几何模型
- **做得好的地方**：
  - 数学公式与图形的联动是 GeoGebra 的强项
  - 教师社区贡献了大量 Applet
- **不足之处**：
  - 各 Applet 质量参差不齐，缺乏统一体验
  - 没有专门的导体棒动力学模拟（GeoGebra 不擅长 ODE 求解动画）
  - UI 不够现代，不适合投影仪演示
- **对我们的启示**：数学公式与可视化联动的思路可借鉴

来源：[GeoGebra Faraday's Law](https://www.geogebra.org/m/Pdv2AJhB) | [GeoGebra Magnetic Induction](https://www.geogebra.org/m/JPFxyhtA)

### 5.4 Explerify Lenz's Law Simulator

- **产品形态**：Web，免费
- **核心功能**：
  - 楞次定律的可视化展示
  - 磁通量变化与感应电流方向的关系
- **做得好的地方**：
  - 聚焦楞次定律这一个知识点，讲解清晰
- **不足之处**：
  - 仅覆盖楞次定律，没有定量计算和运动模拟
- **对我们的启示**：楞次定律的逐步展示方式（磁通量变化→电流方向→安培力方向）可参考其分步设计

来源：[Explerify Lenz's Law Simulator](https://explerify.com/simulator-lenzs-law/)

### 5.5 Physics 3D Interactive

- **产品形态**：Web（3D），免费开源
- **技术实现**：Three.js / WebGL
- **核心功能**：
  - 电磁感应 3D 可视化
- **做得好的地方**：
  - 3D 展示磁场和线圈关系更直观
- **不足之处**：

- **对我们的启示**：楞次定律的磁棒-线圈模型可考虑 2.5D 或 3D 视角增强直观性，但导轨模型用 2D 更清晰

## 6. 初步技术判断

### 推荐的渲染技术
- **2D Canvas**：所有模型均可在 2D 平面内清晰表达
  - 导轨俯视图：两条平行线 + 棒 + 磁场×/·符号
  - 竖直导轨侧视图
  - 楞次定律磁棒-线圈侧视图
  - 箭头标注（速度/电流/安培力）
- **Chart.js 或 Canvas 自绘图表**：v-t / i-t 实时联动图
- **KaTeX**：公式渲染（ε=BLv、终态公式等）
- **SVG 可选**：电路拓扑图的显示可用 SVG，便于线条清晰缩放

### 推荐的架构模式
- **模拟演示型架构**（非编辑器型）：
  - 核心是"物理引擎"——ODE 数值积分器驱动状态演进
  - 状态模型：`{ t, v1, v2, i, Uc, ... }` 随时间演进
  - 渲染层订阅状态变化，实时绘制画面和图表
  - 时间轴控制器管理播放/暂停/步进

- **模块内部分层**：
  ```
  ┌─────────────────────────────────────────────┐
  │  UI 层（参数面板 + 模型选择 + 控制栏）       │
  ├─────────────────────────────────────────────┤
  │  渲染层（Canvas 导轨动画 + 箭头 + 图表）     │
  ├─────────────────────────────────────────────┤
  │  仿真引擎层（ODE 求解器 + 状态管理）          │
  │  ├── 楞次定律模块（离散状态机）              │
  │  ├── 单棒模型（1阶 ODE / 含电容2阶 ODE）    │
  │  ├── 双棒模型（耦合 ODE）                    │
  │  └── 竖直导轨模型（含重力 ODE）              │
  ├─────────────────────────────────────────────┤
  │  物理公式库（BLv, F=BIL, 终态公式等）        │
  └─────────────────────────────────────────────┘
  ```

### 关键技术风险或难点

1. **ODE 数值积分精度与性能**（高风险）
   - 单棒指数衰减还算简单，但双棒耦合+含电容的 ODE 需要高精度积分
   - 含电容时微分方程阶数更高，数值稳定性需注意
   - 需要选择合适的积分步长和方法（推荐自适应步长 RK45）

2. **运动-电路双向耦合的实时模拟**（高风险）
   - 运动产生 EMF → EMF 决定电流 → 电流产生安培力 → 安培力改变运动
   - 这个耦合环必须在每个时间步内自洽求解，而非顺序近似

3. **终态检测算法**（中风险）
   - 何时判断"已到终态"？需要检测速度变化率低于阈值
   - 含电容模型的终态判定更复杂（电流趋零+速度趋稳）

4. **与 P-04 电路搭建器的关系**（中风险）
   - 自由组装模式（P2 优先级）如果需要复用 P-04 的电路分析能力（KVL/KCL 求解），则需要定义清晰的接口
   - 预设模型可以硬编码电路方程，不需要通用电路求解器

5. **与 P-08 电磁场可视化器的关系**（低风险）
   - P-13 的磁场是均匀外加磁场（用×/·符号表示即可），不需要 P-08 那样的磁场线计算和绘制
   - 两模块可以完全独立实现

6. **楞次定律逐步动画的状态机设计**（低风险）
   - 需要设计一个 5 步状态机（原磁通量→变化方向→电流方向→感应磁场→安培力）
   - 四种情况的数据可以硬编码为查找表
