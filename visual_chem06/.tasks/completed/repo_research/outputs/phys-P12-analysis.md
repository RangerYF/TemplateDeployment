# P-12 动量定理及动量守恒 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中物理教师，无编程基础要求；高中学生（自主预习/课后复习）。
- **教学痛点**：
  - 动量是矢量概念，碰撞前后的速度变化、动量转移过程抽象，板书很难表达"连续变化"的过程。
  - 碰撞类型（弹性/非弹性/完全非弹性）之间的差异只靠公式推导，学生缺乏直觉。
  - 类碰撞模型（爬坡、弹簧释放）和人船模型中的位移关系是高考混淆重灾区，静态图示无法展示动态过程。
  - 动量定理的"缓冲原理"（相同冲量下力与时间的反比关系）缺少对比演示手段。
- **核心产品价值**：通过可交互的碰撞动画 + 动量矢量可视化 + 实时数值面板，让学生"看见"碰撞全程的动量分配与守恒验证，覆盖碰撞、反冲、人船等全部经典高考模型。

## 2. 用户实际使用方式

### 教师典型操作流程

1. **课前准备**：选择本节要演示的模型类别（碰撞/类碰撞/反冲/人船），预设好质量、初速度等参数，保存为教学预设。
2. **课中演示**：
   - **碰撞教学**：先用完全弹性碰撞（等质量）演示速度交换，再拖动恢复系数滑块从 1→0，让学生观察碰撞结果从弹性到完全非弹性的连续变化，同时观察动能损失数值。
   - **人船模型**：设置人/船质量，播放动画，学生看到人向前走→船向后退，数值面板实时显示 m人·Δx人 + m船·Δx船 = 0。
   - **动量定理**：演示相同冲量下硬着陆 vs 缓冲的力-时间对比，解释安全气囊原理。
3. **讲题**：调整参数为题目数值，先让学生预测，再播放验证。

### 怎么用才是"好用"的

- 恢复系数滑块一拖即可切换碰撞类型，无需重新建模。
- 动量箭头与数值面板同步更新，视觉信息和数值信息互补。
- 碰撞前后状态可对比（前后快照或分屏）。
- 动画可暂停/逐帧/慢速播放，便于讲解碰撞瞬间。

### 交互频率和深度

- **频率**：动量章节教学期间高频使用（约3-5课时），考前复习时再次使用。
- **深度**：主要是参数调整 + 播放控制，不涉及复杂编辑操作；但参数组合多（5大类10+子模型），需要便捷的模型切换。

## 3. 核心功能点

### 功能清单（按重要性排序）

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 碰撞模型演示 | 完全弹性/完全非弹性/部分弹性碰撞，恢复系数连续调节 |
| P0 | 动量可视化 | 每个物体的动量矢量箭头 + 系统总动量箭头 + 守恒验证 |
| P0 | 碰撞动画播放 | 播放/暂停/重置/速度控制，碰撞瞬间闪光动画 |
| P0 | 数值面板 | 碰前碰后动量、速度、动能损失的实时数值对比 |
| P1 | 人船模型 | 人在船上行走动画，位移关系实时标注 |
| P1 | 类碰撞模型 | 爬坡模型、弹簧压缩释放、滑块模型 |
| P1 | 反冲模型 | 火箭推进、爆炸分离 |
| P1 | 动量定理/冲量演示 | 施加冲量 → 动量变化，缓冲对比 |
| P2 | 速度-时间图联动 | 碰撞动画与v-t图同步显示 |
| P2 | 动能条形图 | 可视化碰撞前后动能对比及损失量 |
| P2 | 预设管理 | 保存/加载参数预设，教学场景一键切换 |

### 功能分类

- **需要实体管理的功能**：无显著实体管理需求。模型中的物体（小球/滑块/船/人）数量固定，由模型类型决定，不需要用户自由创建/删除实体。
- **需要 undo/redo 的功能**：需求较弱。参数调整可直接重置；无画布编辑操作。undo/redo 可作为 P2 需求。
- **参数驱动的功能**（调参数 → 画面实时变化）：
  - 质量 m1/m2、初速度 v1/v2、恢复系数 e — 核心参数
  - 弹簧劲度系数 k、压缩量 x0 — 弹簧模型
  - 人/船质量、行走距离 — 人船模型
  - 火箭总质量、喷出质量/速度 — 反冲模型
  - 力 F、作用时间 t — 冲量演示
- **模拟/动画驱动的功能**（播放/暂停/逐步）：
  - 碰撞过程动画（含碰撞前接近、碰撞瞬间、碰撞后分离三阶段）
  - 人船行走动画
  - 爬坡模型动画（滑块上升→共速→可能滑回）
  - 弹簧释放动画
  - 火箭推进动画

## 4. 核心依赖识别

### 技术依赖

- **渲染方式**：**2D Canvas**（首选）。碰撞场景本质是一维/二维平面运动，用 Canvas 绑定物体运动动画 + 矢量箭头绘制即可；SVG 在大量动画帧下性能不如 Canvas；3D WebGL 不必要。
- **数学/物理计算需求**：
  - 碰撞力学公式（含恢复系数的一般碰撞公式）
  - 动量/动能计算（矢量加法、标量计算）
  - 运动学：匀速运动轨迹、斜面上的运动（爬坡模型）
  - 弹簧势能→动能转化
  - 数值精度要求：守恒验证误差 < 0.1%
- **公式渲染需求**：中等。需在数值面板和教学提示中显示 p = mv、I = Ft 等公式，可用 KaTeX 轻量渲染。

### 交互依赖

- **交互模式**：**参数驱动型 + 模拟演示型（混合）**。用户调节参数 → 设定初始条件 → 播放动画观察结果；核心交互是"调参 + 播放"循环。
- **关键交互**：
  - **滑块**：恢复系数 e（0~1）、质量、速度等连续参数
  - **按钮组**：模型类别切换（碰撞/类碰撞/反冲/人船）、子模型选择
  - **播放控制**：播放/暂停/重置/速度调节
  - **可能的拖拽**：拖拽设置初速度方向和大小（可选增强交互）
- **是否需要实体系统**：**不需要**。物体数量和类型由模型预定义，无需通用实体管理。
- **是否需要命令系统（undo/redo）**：**不需要**。参数调整即时生效，重置即可回到初始状态。

### 与其他模块的关系

- **P-02 运动模拟器**：碰撞涉及物体运动模拟，底层运动学引擎（位移-时间计算、动画循环）可复用。
- **P-14 机械能守恒**：碰撞中的动能损耗分析与机械能守恒密切相关，可共享能量计算与可视化组件（如能量条形图）。
- **P-05 简谐运动与弹簧振子**：弹簧压缩释放模型涉及弹簧力学，弹簧参数和势能计算可复用。

## 5. 竞品分析

### 5.1 PhET Collision Lab（科罗拉多大学）

- **产品形态**：Web（HTML5）+ Java 旧版，免费开源
- **地址**：[phet.colorado.edu/en/simulations/collision-lab](https://phet.colorado.edu/en/simulations/collision-lab)
- **功能特点**：
  - 4个标签页：Intro / Explore 1D / Explore 2D / Inelastic
  - 支持 1D 和 2D 碰撞，可调整球数量、质量、初始条件
  - 弹性系数滑块（0%~100%），实时显示动量和动能变化
  - 支持速度矢量、动能、路径轨迹等显示选项
- **技术实现**：HTML5 Canvas，PhET 自研仿真框架（Scenery 渲染引擎）
- **优点**：碰撞物理准确、支持2D、UI直观、教学社区配套丰富
- **不足**：仅覆盖碰撞模型，不含人船模型、反冲模型、爬坡等类碰撞场景；没有中国高考题型的针对性设计；界面风格偏国外教学风格
- **启示**：恢复系数滑块连续调节是核心交互范式，应借鉴；2D碰撞可作为高级功能；动量矢量箭头的可视化方式值得参考

### 5.2 The Physics Classroom — Collision Carts

- **产品形态**：Web（HTML5），免费
- **地址**：[physicsclassroom.com/interactive/momentum-collisions-and-explosions](https://www.physicsclassroom.com/interactive/momentum-collisions-and-explosions)
- **功能特点**：
  - 1D 小车碰撞模拟，支持弹性/非弹性
  - 近100个动量相关互动模拟
  - 包含碰撞、爆炸等多种场景
- **技术实现**：HTML5 Canvas + JavaScript
- **优点**：场景分类丰富，含爆炸（反冲）模拟
- **不足**：各模拟相互独立，缺乏统一入口；无人船模型；交互较简单
- **启示**：场景覆盖面值得参考，尤其是爆炸/反冲场景的包含

### 5.3 oPhysics — Elastic and Inelastic Collisions

- **产品形态**：Web，免费
- **地址**：[ophysics.com/e2.html](https://ophysics.com/e2.html)
- **功能特点**：
  - 1D 两物体碰撞，支持弹性系数调节
  - 显示速度、动能、动量数值
  - 简洁的单页面模拟
- **技术实现**：HTML5 Canvas（GlowScript/VPython 风格）
- **优点**：极简，加载快，专注单一场景
- **不足**：功能单一，无动画过程控制，无扩展模型
- **启示**：极简方案的参考，但我们需要更丰富的场景覆盖

### 5.4 简易物理（jianyiwuli.cn）

- **产品形态**：Web，中文，免费
- **地址**：[jianyiwuli.cn](https://jianyiwuli.cn/)
- **功能特点**：
  - 面向中国高中物理教学，覆盖动量守恒、碰撞等话题
  - 以动画形式展示物理过程
- **优点**：中文界面，贴合国内教学大纲
- **不足**：交互性有限，以观看动画为主，参数可调性弱；UI较陈旧
- **启示**：内容贴合国内教学是差异化优势方向

### 5.5 CK-12 Collisions Simulation

- **产品形态**：Web（HTML5），免费
- **地址**：[interactives.ck12.org/simulations/physics/collisions](https://interactives.ck12.org/simulations/physics/collisions/app/index.html)
- **功能特点**：
  - 碰撞车模拟（bumper cars），探究动量守恒定律
  - 可调质量和速度
- **优点**：可视化风格活泼，适合学生自主探究
- **不足**：场景单一（仅碰撞车），无反冲/人船等扩展模型
- **启示**：视觉风格可参考，适度活泼的设计有助于学生参与

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：**2D Canvas**（HTML5 Canvas API 或 Konva.js/PixiJS 封装）
  - 理由：所有模型本质是一维运动（碰撞、人船、反冲）或简单二维运动（爬坡），2D Canvas 性能充足且开发效率高
  - 动量矢量箭头、运动轨迹、碰撞闪光等视觉效果均可在 Canvas 上高效实现
- **图表**：**轻量图表库**（如 Chart.js 或 Recharts）用于 v-t 图、能量条形图
- **公式渲染**：**KaTeX**（轻量，渲染速度快）

### 推荐的架构模式

- **参数驱动 + 模拟引擎 + 渲染分离**的三层架构：
  1. **参数层（State）**：使用 Zustand 或类似轻量状态管理，管理所有模型参数（质量、速度、恢复系数等）和模拟状态（播放/暂停/当前时间步）
  2. **物理计算层（Engine）**：纯函数式物理引擎，接收参数输出运动状态序列；碰撞公式、运动学方程、守恒验证等计算逻辑集中于此
  3. **渲染层（View）**：Canvas 渲染器，根据物理引擎输出的状态序列绘制动画帧
- **模型插件化**：每种模型（碰撞/爬坡/弹簧/反冲/人船/冲量）注册为独立模块，共享参数面板框架和动画播放控制，各自实现物理计算和场景渲染
- **技术栈**：React + TypeScript + Canvas（或 PixiJS）+ Zustand + KaTeX

### 关键技术风险或难点

1. **碰撞动画的流畅性**：碰撞瞬间的速度突变需要特殊处理（插值或分段动画），避免视觉上的突兀跳变。需要设计"碰撞过渡帧"（闪光动画 + 短暂减速/加速过渡）。
2. **多模型统一架构**：5大类10+子模型的参数面板和场景渲染差异较大（碰撞是两球相撞，人船是人在船上行走，爬坡是斜面+滑块），需要良好的模型抽象层，避免每个模型完全独立开发导致代码膨胀。
3. **动量矢量可视化的缩放**：动量箭头长度正比于动量值（1 kg·m/s → 10px），当质量或速度范围跨度大时，箭头可能过长或过短，需要自适应缩放策略。
4. **守恒验证精度**：浮点运算累积误差可能导致守恒验证显示 ✗，需要合理设定误差阈值（< 0.1%）并在 UI 上友好展示。
5. **与 P-02/P-14 的复用边界**：运动模拟引擎（动画循环、时间步进）和能量可视化（条形图）可与其他模块共享，但需在仓库规划阶段明确共享层接口，避免过度耦合。
