# P-02 运动模拟器 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程或软件操作基础。核心用户画像是需要在课堂上演示匀变速直线运动、抛体运动、圆周运动等力学运动场景的物理教师。

### 解决什么教学痛点
- **运动过程难以感知**：匀变速直线运动、平抛运动、圆周运动等，学生只看公式没有感性认识，无法建立"运动"的直觉
- **速度/加速度向量不直观**：运动过程中速度方向和大小的变化、加速度的方向（尤其圆周运动的向心加速度），黑板上的静态箭头画不清楚
- **v-t/s-t 图与运动过程脱节**：v-t 图的斜率 = 加速度、图下面积 = 位移等对应关系，学生很难从静态图建立联系
- **多条件对比困难**：不同初速度的平抛轨迹差异、不同摩擦系数的斜面运动差异，黑板上很难同时画出多组并精确对比
- **临界条件理解困难**：竖直圆周运动的最高点临界速度、水平圆周的最大静摩擦力临界等考试重点，纯公式推导缺乏直观支撑

### 核心产品价值
**一句话**：老师选一个运动场景、设几个初始参数，点击播放就能看到物体沿轨迹运动的全过程，同时实时显示速度/加速度向量和 v-t/s-t 图联动，覆盖高中力学运动部分 90% 的教学演示需求。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选运动场景（如"平抛运动"）→ 设参数（初始高度 20m、水平初速度 10m/s）→ 开启速度/加速度向量显示 → 试播放确认效果正确 |
| **课中演示** | 投影仪打开 → 选择场景 → 点击播放，边播放边指出"水平速度不变，竖直速度增大" → 暂停在某时刻，指出 v-t 图上对应点 → 调速到 0.25x 慢放关键阶段 |
| **对比教学** | 进入多物体对比模式 → 设 3 组不同初速度 → 同时播放 → 指出"下落时间相同，水平射程不同"→ 对比 v-t 图曲线 |
| **讲题** | 选竖直圆周运动 → 设初速度为 √(5gR) → 播放观察"刚好通过最高点" → 改为 0.8√(5gR) → 观察"脱轨" → 解释临界条件 |

### 怎么用才是"好用"的
- **3 分钟上手**：打开选场景、设参数、点播放，三步完成一次演示
- **时间轴控制灵活**：播放/暂停/逐步/倒放/调速，老师随时可以停在关键时刻讲解
- **图像联动直观**：v-t 图上的竖线与动画同步移动，当前时刻在图上一目了然
- **投影清晰**：在 1080p 投影仪上轨迹线、向量箭头、数值标注均清晰可辨
- **物理正确**：所有运动轨迹和数值与手算公式完全一致，不能有物理错误

### 交互频率和深度
- **频率**：每节运动学课使用 1-3 次，每次 3-10 分钟
- **深度**：中等 — 主要是参数设置 + 播放控制 + 图像观察，不涉及复杂建模或编辑操作

### 需要实体管理的功能
运动模拟器是**模拟演示型**而非编辑器型，实体管理需求较轻：
- **运动物体**：每个物体有质量、颜色、初始参数等属性，最多 3 个同屏
- **场景环境**：斜面、圆轨道等静态环境元素，由场景选择决定，无需用户手动创建/删除
- 不需要类似 M-01 那样的 Entity Registry / 级联删除等复杂实体系统

### 需要 Undo/Redo 的功能
**不需要**。运动模拟器的核心交互是"设参数 → 播放 → 观察"，没有创建/删除实体的编辑操作。参数修改是即时生效的，用户可以直接改回去。

### 参数驱动的功能
- 初速度 v0 / 加速度 a / 质量 m / 斜面角度 θ / 摩擦系数 μ / 圆周半径 r / 角速度 ω → 运动方程实时更新
- 显示选项（速度向量 / 加速度向量 / 分量 / 轨迹 / 坐标网格）→ 画面实时切换
- 箭头比例（1m/s → 20px，可调）→ 向量箭头长度实时变化

### 模拟/动画驱动的功能
**这是核心交互模式**，几乎所有主功能都是模拟/动画驱动：
- 运动轨迹绘制：按时间步长 0.01s 推进，每 0.02s 采样一个轨迹点
- 向量动画：速度/加速度向量随时间实时更新方向和长度
- 时间轴控制：播放/暂停/逐步/倒放/调速，最大模拟时长 60s
- v-t / s-t 图联动：图像随时间同步绘制，竖线标记当前时刻
- 临界条件检测：竖直圆周运动中检测是否满足通过最高点条件

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：2D Canvas — 运动轨迹、向量箭头、运动物体均为 2D 场景（水平面/斜面/平抛/圆周均可在 2D 平面上展示）
- **图表渲染**：Canvas / Chart.js — v-t / s-t / a-t 图表联动绘制，需要实时追加数据点
- **公式渲染**：KaTeX — 运动方程、关键物理量的 LaTeX 显示

#### 数学/物理计算需求
- **核心**：6 种运动场景的运动方程求解（解析解，非数值积分）
  - 匀速直线：x = v0·t
  - 匀加速直线：x = v0·t + ½at², v = v0 + at
  - 斜面运动：a = g(sinθ - μcosθ)，含上滑/下滑切换（摩擦力方向改变）
  - 平抛运动：x = v0·t, y = ½gt²（含斜面上平抛的落点判定）
  - 水平圆周：v = ωr, a = ω²r, T = 2π/ω
  - 竖直圆周：能量守恒 + 牛顿第二定律联立，各位置速度和向心力
- **中度**：临界条件判定
  - 斜面：tanθ > μ 才能下滑
  - 水平圆周摩擦力提供向心力：mω²r > μmg 时飞出
  - 竖直圆周最高点：v_top = √(gr)（绳模型），v_bottom = √(5gr)
- **轻度**：多物体对比时的参数独立计算（最多 3 组）
- **轻度**：斜面上平抛落点计算（轨迹与斜面交点）

#### 公式渲染需求
- 运动方程（如 x = v0·t + ½at²）
- 关键物理量（落地速度、射程、临界速度等）
- v-t 图的物理量读取说明（斜率=加速度、面积=位移）

### 交互依赖

#### 交互模式
**模拟演示型**（主导）+ **参数驱动型**（辅助）

这是一个典型的模拟演示型应用：
- 核心循环是"设参数 → 播放 → 观察 → 暂停 → 讲解 → 继续/重放"
- 有时间轴控制（播放/暂停/逐步/倒放/调速）
- 有动画帧循环（每帧推进物理状态）
- 有图表联动（v-t/s-t 图同步更新）

同时有参数驱动特征：
- 参数滑块 / 数值输入 → 运动方程和预览实时更新
- 显示选项切换 → 画面实时变化

#### 关键交互
| 交互类型 | 具体场景 |
|---------|---------|
| 按钮组 | 顶部 6 个运动场景选择（点击切换） |
| 滑块 + 数值输入 | 参数设置（初速度/角度/摩擦系数/质量等） |
| 时间轴控制 | 播放/暂停/单步前进/单步后退/倒放/调速（核心交互） |
| 点击轨迹 | 数值读取（点击轨迹任意点显示该时刻物理量） |
| 勾选框 | 显示选项（速度向量/加速度向量/分量/网格/轨迹留存） |
| 颜色选择 | 多物体模式下各物体颜色设定 |

#### 是否需要实体系统
**不需要完整实体系统**。运动模拟器管理的"对象"种类有限（运动物体 + 场景环境），且由场景模板预定义，不需要用户动态创建/删除/组合实体。一个轻量的场景状态管理即可。

#### 是否需要命令系统
**不需要**。没有 Undo/Redo 的需求，参数修改即时生效，用户可以直接回调。

## 5. 竞品分析

### 5.1 PhET Interactive Simulations（科罗拉多大学）
- **形态**：Web（HTML5，浏览器内运行）/ 可下载离线包
- **核心产品**：Projectile Motion、Forces and Motion 系列
- **技术实现**：早期 Java/Flash，现已迁移至 HTML5；使用自研框架 Scenery（SVG/Canvas 混合渲染）；开源（GitHub）
- **优势**：
  - 全球教育领域最知名的物理模拟平台，1亿+ 用户
  - 基于教育研究设计，界面直观，学生自主探索友好
  - 完全免费开源，活跃的社区贡献教学活动
  - 平抛运动模拟支持发射角度、初速度、空气阻力等参数
  - 向量分量显示、轨迹留存功能完善
- **不足**：
  - 每个模拟是独立应用，没有统一的"运动模拟器"覆盖所有运动类型
  - 无 v-t/s-t 图联动（部分模拟有简单图表但非核心设计）
  - 无多物体对比模式（同参数对比需手动操作）
  - 无时间轴倒放、逐步控制
  - 无竖直圆周运动的专项模拟（临界条件分析缺失）
- **启示**：PhET 的"直觉式探索"交互设计值得学习，但 P-02 的差异化在于"6 种场景统一入口 + v-t 图联动 + 多物体对比 + 时间轴精细控制"

### 5.2 Algodoo（Algoryx Simulation AB）
- **形态**：Desktop（Windows/Mac）/ iPad
- **技术实现**：自研物理引擎（SPOOK 线性约束求解器 + SPH 流体），C++ 原生渲染
- **优势**：
  - 2D 物理沙盒，自由度极高，可以"画"出任意场景
  - 实时物理模拟精度高（碰撞、摩擦、重力、浮力、光学均支持）
  - 支持分析图表（速度-时间、力-时间等）
  - 全球 700+ 学术论文引用，150,000+ 共享场景
  - 免费
- **不足**：
  - 非 Web 端，需安装桌面应用
  - 自由度过高 — 教师需要自己搭建场景，不是"选场景 → 设参数 → 播放"的流程
  - 无针对高中物理知识点的预设场景
  - 无 v-t/s-t 图联动的内建功能
  - 界面和操作偏向"游戏/探索"，不适合课堂快速演示
- **启示**：Algodoo 的物理引擎精度和自由度是标杆，但 P-02 走"预设场景 + 参数调节"路线，降低使用门槛

### 5.3 oPhysics（Tom Walsh）
- **形态**：Web（纯浏览器，基于 GeoGebra 引擎）
- **核心产品**：按 Kinematics / Forces / Conservation 等分类的独立模拟
- **技术实现**：基于 GeoGebra 软件制作动画和模拟，HTML5 嵌入
- **优势**：
  - 纯 Web，零安装
  - 按高中物理知识点分类，每个模拟针对一个具体考点
  - 教师 27 年教学经验积累，模拟设计贴合教学
  - 免费
- **不足**：
  - 每个模拟独立页面，无统一界面
  - 基于 GeoGebra 的交互，滑块+图形对应为主，无真正的"动画播放"体验
  - 无时间轴控制、无多物体对比
  - 视觉效果较简陋
  - 无中文版本
- **启示**：知识点粒度的设计思路值得参考，P-02 将多个知识点整合在统一框架下

### 5.4 The Physics Classroom
- **形态**：Web（HTML5 交互模拟）
- **技术实现**：HTML5 Canvas + JavaScript 自研
- **优势**：
  - 包含 Projectile Simulator 和 Uniform Circular Motion 等专题模拟
  - 配套教学文本资源丰富
  - 纯 Web 免费
- **不足**：
  - 各模拟独立、界面风格不一
  - 交互简单，主要是参数调整 + 静态结果展示
  - 无动画播放/时间轴控制
  - 无 v-t 图联动
- **启示**：作为教学辅助工具的定位明确，但交互深度不足

### 5.5 GeoGebra Physics Simulations
- **形态**：Web / Desktop / Mobile
- **技术实现**：GeoGebra 引擎（Java → GWT → JS），社区用户创建的物理模拟
- **优势**：
  - 跨平台、免费、社区生态庞大
  - 用户可自建模拟并分享
  - 数学精确性高
- **不足**：
  - 物理模拟并非原生功能，需用户用数学工具"搭建"
  - 无专门的时间轴控制、v-t 图联动等物理教学专用功能
  - 加载速度慢，GWT 包体积大
- **启示**：GeoGebra 的平台生态值得长期参考，但短期内 P-02 以专业化垂直工具取胜

### 5.7 市场空白与 P-02 定位

**核心发现**：**Web 端统一运动模拟 + v-t 图联动 + 多物体对比的专业物理教学工具存在空白**。

- PhET 是运动模拟标杆但各模拟分散，无统一入口和图像联动
- Algodoo 物理引擎强大但非 Web、自由度过高不适合课堂快速演示
- oPhysics/Physics Classroom/GeoGebra 的运动模拟交互深度不足
- 以上竞品均无"v-t 图实时联动 + 多物体对比 + 时间轴精细控制"的组合
- 国内缺少针对高中物理教材的原生中文运动模拟工具

P-02 的差异化定位：**唯一在 Web 端提供 6 种运动场景统一入口 + v-t/s-t 图实时联动 + 多物体对比 + 时间轴精细控制的运动模拟教学工具**，且原生中文、贴合中国高中物理教材知识点。

## 6. 初步技术判断

### 推荐的渲染技术
- **主渲染**：2D Canvas — 运动轨迹、向量箭头、运动物体、场景背景（斜面/圆轨道等）均为 2D 场景，Canvas 性能和灵活性满足需求
- **图表渲染**：Canvas 自研或 Chart.js — v-t/s-t/a-t 图表需要实时追加数据点并与动画同步，自研 Canvas 图表可控性更好；Chart.js 可作为快速方案
- **公式渲染**：KaTeX — 运动方程和关键物理量的 LaTeX 展示
- 不需要 3D 渲染 — 所有 6 种运动场景均可在 2D 平面上完整表达

### 推荐的架构模式
**模拟演示型架构**：场景状态机 + 物理计算引擎 + 动画帧循环 + 图表联动

```
用户选场景 → 场景状态机初始化 → 设参数 → 物理方程引擎准备
                                          ↓
点击播放 → 动画帧循环 (requestAnimationFrame)
           ├→ 物理引擎：计算当前时刻 t 的 x/y/vx/vy/ax/ay
           ├→ Canvas 渲染器：绘制物体位置 + 轨迹 + 向量箭头
           ├→ 图表渲染器：追加 v-t/s-t 数据点 + 更新竖线标记
           └→ 数值面板：更新实时物理量数值
```

| 系统 | 设计模式 | 说明 |
|------|---------|------|
| 场景管理 | 策略模式（每种场景一个计算策略） | 6 种场景的运动方程独立实现，通过场景选择器切换 |
| 物理计算引擎 | 纯函数（解析解） | 给定参数和时间 t，直接计算位置/速度/加速度，无需数值积分 |
| 动画帧循环 | requestAnimationFrame + 时间控制器 | 支持播放/暂停/逐步/倒放/调速 |
| Canvas 渲染器 | 每帧重绘（物体 + 轨迹 + 向量） | 轨迹点缓存避免重复计算 |
| 图表联动 | 观察者模式（时间轴变化 → 图表更新） | v-t/s-t 图与动画共享同一时间状态 |
| 参数面板 | React 受控组件 + 状态提升 | 参数变化触发场景重置/更新 |
| 多物体管理 | 数组（最多 3 个独立物理状态） | 每个物体独立计算、独立渲染、独立颜色 |

### 6 种场景的渲染差异分析

| 场景 | 渲染特点 | 差异度 |
|------|---------|--------|
| 水平面运动（MOT-001/002） | 水平地面 + 物体平移 | 最简单，一维运动 |
| 斜面运动（MOT-011/012） | 斜面背景 + 物体沿斜面运动 | 中等，需绘制斜面 + 坐标变换 |
| 平抛运动（MOT-021/022） | 初始平台 + 抛物线轨迹 | 中等，二维运动 + 可能含斜面 |
| 水平圆周（MOT-031/032） | 俯视圆形轨道 + 物体绕圆运动 | 中等，需绘制圆形轨道 |
| 竖直圆周（MOT-033） | 竖直圆形轨道 + 绳/杆/轨道 | 较复杂，需处理脱轨判定 |

**结论**：6 种场景的渲染差异主要在背景环境（地面/斜面/圆轨道），物体本身和向量箭头的渲染逻辑可以统一。建议抽象一个统一的场景接口：

```typescript
interface MotionScene {
  id: string;
  params: ParamDefinition[];           // 可调参数定义
  computeState(t: number): PhysicsState; // 给定时刻计算物理状态
  renderBackground(ctx: Canvas2D): void; // 绘制场景背景
  checkCritical(): CriticalResult;       // 临界条件检测
  getGraphTypes(): GraphType[];          // 支持的图表类型
}
```

### 时间轴控制系统（高复用潜力）

时间轴控制是 P-02 的核心交互，且与多个物理模块高度共享（P-05 简谐运动、P-06 波动、P-09 天体运动、P-12 动量碰撞、P-13 电磁感应、P-14 机械能守恒均需要播放/暂停/调速）。建议设计为独立可复用模块：

```typescript
interface TimelineController {
  play(): void;
  pause(): void;
  stepForward(dt: number): void;
  stepBackward(dt: number): void;
  reverse(): void;
  setSpeed(multiplier: number): void; // 0.1x ~ 2x
  seek(t: number): void;
  getCurrentTime(): number;
  getMaxTime(): number;
  onTimeChange(callback: (t: number) => void): void;
}
```

### 与 P-01 受力分析器的融合关系

PRD 明确指出"P-02 与 P-01 在产品中深度融合"。融合点：
- 受力分析结果（力的大小和方向）可以作为运动模拟的输入条件
- 斜面场景中的重力分解、摩擦力方向可以调用 P-01 的受力分析逻辑
- 圆周运动中的向心力/重力/绳张力分析可复用 P-01 的力平衡计算
- **建议**：P-01 和 P-02 在物理计算层共享力学公式库，在 UI 层保持独立但提供"跳转"入口

### 关键技术风险或难点

1. **v-t 图联动的同步精度**：动画帧率与图表更新频率需要严格同步，否则会出现"动画已到某位置但图表未更新"的体验问题。建议共享同一时间源。
2. **斜面上平抛的落点判定**：轨迹与斜面的交点计算需要处理边界条件（平抛轨迹方程与斜面方程联立求解）。
3. **竖直圆周的脱轨判定与动画**：脱轨后的运动需要切换为抛体运动模型，存在状态切换点的连续性问题。
4. **倒放功能**：由于使用解析解（非数值积分），倒放可以直接给负的时间步进，技术上较简单，但 UI 上需要合理处理轨迹擦除逻辑。
5. **多物体对比的性能**：3 组物体各自独立计算 + 渲染 + 图表绘制，需确保在低端设备上不掉帧。
6. **Canvas 自适应**：不同分辨率（尤其 1080p 投影仪）下 Canvas 的 DPI 适配和文字清晰度需要注意。
