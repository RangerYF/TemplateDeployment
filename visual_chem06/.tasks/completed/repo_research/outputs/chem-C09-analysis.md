# C-09 有机化学反应路径图 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中化学教师，主要覆盖高中有机化学（必修+选修）教学场景。
- **解决的教学痛点**：
  - 有机物官能团之间的转化关系是高考有机推断题的核心考点，但传统教学只能靠黑板画线性流程图或学生自己整理笔记，无法直观展现"网状"转化全貌。
  - 学生往往只记住个别反应（如醇→醛），但缺乏对"从 A 物质到 B 物质有几条路径、哪条最短"的全局认知，影响有机推断解题能力。
  - 教师课堂讲解时需要反复在板书上标注反应条件和方程式，耗时且不利于整体展示。
- **核心产品价值**：一句话——**以轮盘式网络图一屏展现全部高中有机物转化关系，教师点击任意两个官能团即可高亮所有可达路径并展示反应条件与方程式，替代手绘板书和静态图片。**

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开反应路径图 → 浏览整体网络结构，确认今天要讲的转化路径（如烯烃→卤代烃→醇→醛→羧酸）→ 点击起终点预览高亮效果 → 截图用于课件或学案 |
| **课中演示** | 投屏打开 → 点击起始节点（如"烯烃"）和目标节点（如"羧酸"）→ 系统高亮所有可达路径，标注最短路径 → 点击路径上的反应连接线，展开显示具体方程式和反应条件 → 切换不同起终点，对比不同转化方案 |
| **复习/讲题** | 讲有机推断题时，投屏展示路径图 → 根据题目条件（已知起始物和产物），让学生先预判路径 → 点击验证路径，展示反应细节 → 借助"最短路径 vs 备选路径"帮助学生理解最优解题思路 |

### 怎么用才是"好用"的？

- **一屏全览**：12 个一级节点 + 50+ 反应路径在一个轮盘式布局中清晰展示，不需要滚动或翻页。
- **路径高亮直观**：点击两个节点后，1 秒内高亮所有可达路径，用颜色区分最短路径和备选路径，教师不需要自己找线。
- **反应细节随点随看**：悬停连接线显示反应类型，点击展开完整方程式和条件（试剂、温度等），不需要另开窗口或查资料。
- **严格限定课纲**：所有反应条件和转化路径限定在高中课纲范围内，教师无需担心"超纲"内容干扰教学。

### 交互频率和深度

- **频率**：中频工具。主要在有机化学单元教学（约 2-3 周集中授课）和高三复习阶段使用，非每课必用。
- **深度**：浅到中等。单次使用通常 1-3 分钟（选路径 → 查看高亮 → 展开方程式），操作简单直接。

## 3. 核心功能点

### 功能清单（按重要性排序）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 轮盘式官能团转化网络图 | 放射状思维导图，中心节点"有机物转化"，12 个一级节点（烷烃、烯烃、炔烃、卤代烃、醇、醛、羧酸、酯、芳香烃、酚、胺、酰胺），50+ 反应路径连接线 |
| P0 | 路径高亮交互 | 点击任意两个节点 → 自动高亮所有可达路径 → 用不同颜色区分最短路径（推荐）和备选路径 |
| P0 | 反应条件与方程式展示 | 每条连接线标注反应类型（氧化、加成等）；鼠标悬停显示完整方程式；点击展开详细条件（试剂、温度等） |
| P1 | 同分异构体模块 | 输入分子式 → 列出所有可能的同分异构体结构；含苯环碳≤15，不含苯环碳≤8；分类展示碳架/位置/官能团/顺反/手性异构（不展示构象异构） |

### 需要实体管理的功能

- **不需要实体管理**。本模块是预定义数据的可视化展示，节点和连接线均来自固定数据集，用户不创建、删除或编辑图中对象。用户只选择节点触发路径查找和高亮。

### 需要 undo/redo 的功能

- **不需要**。本模块是查看/探索型工具，不涉及编辑操作。用户操作可逆性通过"点击其他节点对"或"清除高亮"实现。

### 参数驱动的功能（调参数 → 画面实时变化）

- **路径高亮**：选择不同的起终点节点 → 图上高亮路径实时更新。这是核心交互，但严格说更接近"选择驱动"而非"滑块式参数调整"。
- **同分异构体输入**：输入分子式 → 异构体列表实时更新。

### 模拟/动画驱动的功能

- **无**。本模块是静态网络图的交互式查看器，不涉及时间轴、播放/暂停等动画需求。路径高亮可能有简单的过渡动画（连接线依次点亮），但不构成模拟驱动。

## 4. 核心依赖识别

### 技术依赖

| 维度 | 判断 |
|------|------|
| **渲染方式** | **SVG 为主**。节点数量有限（~12 个一级节点 + 1 个中心节点），连接线 ~50 条，SVG 完全可以胜任且天然支持 DOM 事件绑定（hover/click）、CSS 动画（高亮过渡）、矢量缩放。无需 Canvas 或 WebGL 的性能优势。推荐使用 D3.js 作为 SVG 图形编排引擎。 |
| **数学/物理计算需求** | 低到中等。需要：(1) 图论中的路径查找算法（BFS/DFS 求所有路径、Dijkstra 或 BFS 求最短路径），节点规模极小（12 个），算法复杂度可忽略；(2) 放射状布局算法（D3.js 内置 `d3.forceRadial` 或手动极坐标布局）；(3) 同分异构体枚举算法（P1 功能，需化学规则约束的组合搜索）。 |
| **公式渲染需求** | 中等。反应方程式中包含化学式下标（H₂O）、反应箭头、反应条件上下标注。可用 HTML/CSS 渲染（化学式复杂度低于数学公式），或使用 KaTeX 的 `\ce{}` 化学扩展。 |

### 交互依赖

| 维度 | 判断 |
|------|------|
| **交互模式** | **探索/查看型**。用户在预定义的网络图上点击节点探索路径，不创建或编辑任何内容。与编辑器型、参数驱动型、模拟演示型均不同，是"图/网络探索型"交互。 |
| **关键交互** | 节点点击（选择起/终点）、连接线 hover（预览反应类型）、连接线点击（展开方程式详情）、路径高亮（视觉反馈）、清除/重置选择、分子式输入（同分异构体模块） |
| **是否需要实体系统** | **不需要**。图数据（节点+边+反应条件）来自预定义 JSON 数据文件，运行时只读。 |
| **是否需要命令系统** | **不需要**。无编辑操作，无 undo/redo 需求。 |

## 5. 竞品分析

### 有机化学反应图谱工具（教育类）

#### 百度百科/知乎整理型图谱

- **产品形态**：静态图片（PNG/JPG），散见于各类教学资源网站
- **技术实现**：手绘或 PPT 制作，无交互
- **优点**：在教学中被广泛使用，说明需求真实存在；图谱结构已被师生接受
- **不足**：完全静态，无法交互探索；信息密度受限于画面大小；无法高亮路径；方程式和条件无法按需展开
- **启示**：教师对"有机反应网络图"的心智模型已经建立，轮盘式布局正好匹配这一认知习惯；关键差异化在于交互能力（路径高亮、按需展开）

#### Chemography / Organic Chemistry Map（学术/海外）

- **产品形态**：Web 应用或桌面软件，面向大学有机化学
- **技术实现**：通常为可缩放的矢量图或交互式网页
- **优点**：覆盖范围广，反应类型丰富
- **不足**：面向大学/科研，远超高中课纲范围；英文界面；交互多为简单的缩放和搜索，缺少路径高亮功能
- **启示**：内容需要严格限定高中课纲，"不超纲"是教师强调的核心需求

### D3.js 力导向图 / 网络图示例

#### D3.js Force-Directed Graph

- **产品形态**：开源可视化示例
- **技术实现**：D3.js + SVG，力导向布局算法
- **优点**：布局自动化，节点可拖拽调整，边绑定灵活，开源生态丰富，示例代码多
- **不足**：力导向布局不稳定，每次刷新位置变化；通用示例缺乏化学领域定制
- **启示**：力导向布局适合节点位置不固定的场景，但有机反应路径图更适合**固定的放射状布局**（教师希望布局稳定、可预期），可用 `d3.forceRadial` 或直接手动指定极坐标位置

#### D3.js Radial Layout / Sunburst

- **产品形态**：开源可视化示例
- **技术实现**：D3.js + SVG，放射状/旋日布局
- **优点**：天然适配"轮盘式"需求，节点沿圆周分布，连接线从中心辐射或节点间连接
- **不足**：标准 Sunburst 是层级结构（父-子），而非网络结构（多对多连接），需要定制
- **启示**：可借鉴放射状布局的视觉范式，但数据模型需要改为图（Graph）而非树（Tree），连接线需支持任意两节点间的多对多关系

### 化学思维导图工具

#### XMind / MindMeister / ProcessOn

- **产品形态**：通用思维导图工具（Web/桌面）
- **技术实现**：SVG/Canvas 渲染，树形或自由布局
- **优点**：用户自由创建节点和连接，布局灵活
- **不足**：通用工具缺乏化学语义（无法标注反应条件、无法渲染化学方程式）；树形结构不支持多对多转化关系（如"烯烃"可同时转化为多种物质）；无路径查找/高亮功能
- **启示**：通用思维导图工具无法满足本模块需求，必须做领域定制；但其视觉布局和节点样式可作为设计参考

### 竞品总结对我们的启示

1. **需求真实且市场空白**：教师广泛使用静态有机反应图谱，但目前没有一款产品提供"交互式路径高亮"功能——这是我们的核心差异化。
2. **放射状固定布局优于力导向**：教师需要布局稳定、可预期，而非每次刷新位置变化的力导向图。直接用极坐标手动布局 12 个节点即可。
3. **课纲严格限定**：所有竞品要么面向大学/科研（内容超纲），要么是静态图片（无交互）。高中课纲范围内的精确限定是教学场景的刚需。
4. **图论算法简单**：12 个节点、50+ 条边的规模下，路径查找算法的复杂度可忽略不计，技术风险极低。

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：**SVG**（通过 D3.js 或 React + SVG 直接操作）。理由：
  - 节点数量极少（~13 个），连接线 ~50 条，SVG 性能绰绰有余
  - SVG 元素天然支持 DOM 事件（click/hover/mouseenter），无需额外拾取逻辑
  - SVG 支持 CSS transition/animation，路径高亮动画（颜色变化、描边动画）实现简单
  - 矢量渲染在任何分辨率下清晰，适合投屏场景
- **布局方案**：固定极坐标布局。12 个一级节点沿圆周均匀分布，中心节点为"有机物转化"。不使用力导向算法（避免布局不稳定）。
- **连接线渲染**：SVG `<path>` 元素，使用贝塞尔曲线避免直线交叉。高亮时改变 `stroke` 颜色和 `stroke-width`，可加 `stroke-dasharray` 动画。
- **信息面板**：React 组件（DOM），展示选中反应的方程式和条件。化学方程式可用 KaTeX `\ce{}` 或 HTML+CSS 渲染。
- **同分异构体模块**：DOM 渲染（列表+结构简式），结构简式可用 SVG 或 Canvas 绘制（P1 拓展）。

### 推荐的架构模式

- **数据驱动型查看器架构**（非编辑器架构）：
  - **数据层**：静态 JSON 数据文件，包含节点列表（12 个官能团/物质类别，含名称、官能团、代表分子式）和边列表（50+ 反应路径，含起终节点、反应类型、反应条件、方程式）。数据来源于 `data/C09_有机反应路径数据.md`（待产出）。
  - **图算法层**：轻量图数据结构（邻接表），实现 BFS/DFS 路径查找和最短路径计算。规模极小，无需引入图算法库。
  - **布局层**：极坐标固定布局计算，将 12 个节点映射到圆周上的 (x, y) 坐标。
  - **渲染层**：D3.js / React SVG 渲染节点和连接线，管理高亮状态（选中节点、高亮路径、非高亮路径灰化）。
  - **交互层**：节点点选（选择起/终点）、连接线 hover/click（展示反应详情）、清除选择。
  - **UI 层**：React 组件（信息面板、方程式展示、同分异构体输入与结果列表）。
- **与其他化学模块的关系**：本模块是图/网络可视化型，与其他化学模块（3D 分子查看、参数驱动模拟等）的交互模式差异大，技术栈（SVG/D3.js vs Three.js/WebGL）不同，**共享层较薄**——主要共享 React UI 框架、主题色系统、化学式渲染组件。

### 关键技术风险或难点

| 风险/难点 | 严重性 | 说明 |
|-----------|--------|------|
| **连接线布局与交叉优化** | 中 | 50+ 条连接线在 12 个节点间可能产生大量交叉。固定极坐标布局下，需要优化节点排列顺序以减少交叉，或使用贝塞尔曲线弧线绕行。节点数少，可手动调优。 |
| **路径查找与高亮的用户体验** | 低 | 算法本身无风险（12 节点规模）。UX 上需要清晰表达"最短路径 vs 备选路径"的视觉区分（颜色深浅、线条粗细、动画先后），以及"无可达路径"的友好提示。 |
| **同分异构体枚举算法** | 中（P1 拓展） | 给定分子式枚举所有合法同分异构体是组合数学+化学规则约束问题。碳≤8（不含苯环）或 ≤15（含苯环）的限制控制了搜索空间，但算法仍有一定实现复杂度。可考虑预计算+查表方案（将常见分子式的异构体预存为数据）。 |
| **化学方程式渲染** | 低 | 化学方程式包含下标、上标、反应箭头、条件标注，复杂度低于数学公式。KaTeX `\ce{}` 或自定义 HTML/CSS 均可实现。 |
| **数据文件尚未产出** | 低 | PRD 引用 `data/C09_有机反应路径数据.md` 尚未生成。需要根据高中化学课纲梳理 12 个节点间的 50+ 反应路径、条件和方程式。数据工作量中等，但无技术风险。 |
| **移动端触摸适配** | 低 | SVG 交互在移动端需要处理 touch 事件和目标尺寸（节点/连接线可能偏小）。可通过增大节点触摸区域和使用 tooltip 替代 hover 解决。 |
