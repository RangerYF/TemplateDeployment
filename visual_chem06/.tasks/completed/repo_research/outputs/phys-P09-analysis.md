# P-09 天体运动与引力模拟器 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程或软件操作基础。核心用户画像是需要在课堂上讲解万有引力与天体运动章节的物理教师，该章节涉及开普勒三定律、人造卫星变轨、双星系统、天体追及等内容，是高考物理的必考板块。

### 解决什么教学痛点
- **轨道运动难以想象**：开普勒椭圆轨道的面积定律、近日点远日点速度差异，黑板上的静态图无法展示动态过程
- **变轨过程极度抽象**：霍曼转移轨道涉及两次点火、三条不同轨道，学生很难建立"加速反而变慢"的反直觉理解
- **双星系统绕转不直观**：两星绕质心旋转、质量比与轨道半径比的反比关系，口述难以说清
- **天体追及计算困难**：不同轨道角速度差导致的追及问题是学生失分重灾区，缺乏动态演示工具
- **"高轨低速大周期"规律**：这是天体物理最核心的定性规律，需要通过参数联动让学生直观感受

### 核心产品价值
**一句话**：老师选择天体运动模型、设几个参数，就能看到实时轨道动画，所有物理量（速度、周期、半径、面积扫过率）联动显示，覆盖高中天体物理全部核心考点。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选择天体模型（如椭圆轨道）→ 设参数（半长轴、离心率）→ 预览面积定律效果 → 准备好待演示的变轨场景 |
| **课中演示** | 投影仪打开 → 选"单星系统" → 播放圆轨道动画 → 拖动轨道半径展示"高轨低速大周期" → 切换椭圆轨道展示面积定律 → 暂停标注近日点远日点 |
| **讲题** | 切换到"卫星变轨"模块 → 点击"点火加速"演示低轨→椭圆→高轨全过程 → 标注各轨道速度大小关系 → 切换"追及"模块演示追及时间计算 |

### 怎么用才是"好用"的
- **3 分钟上手**：选模型 → 调参数 → 播放动画，操作极简
- **参数联动直观**：拖动轨道半径滑块，速度/周期/加速度数值同步变化，"高轨低速大周期"一目了然
- **变轨过程可控**：点火加速是显式按钮操作，不是自动播放，老师可以逐步讲解
- **关键数值精确**：速度、周期、T²/a³ 比值与公式计算结果一致
- **投影清晰**：深色太空背景 + 高对比度轨道/天体颜色，1080p 投影仪上清晰可辨

### 交互频率和深度
- **频率**：天体物理章节约 6-8 课时，每节课使用 1-2 次，每次 5-10 分钟
- **深度**：中等偏低 — 主要是选模型 + 调参数 + 播放/暂停动画 + 观察数值变化，不涉及复杂编辑操作

### 需要实体管理的功能
P-09 的实体管理需求**非常轻量**：
- 天体（中心体、卫星/行星、双星）— 仅需位置、质量、速度属性
- 轨道线（圆/椭圆/抛物线/双曲线）— 由参数自动生成，不需用户手动编辑
- 无需创建/删除/选中等编辑器型操作，实体由模型选择自动生成

### 需要 Undo/Redo 的功能
**不需要** — P-09 是纯模拟演示型模块，用户只调参数和控制播放，没有需要撤销的编辑操作。

### 参数驱动的功能
- 中心天体质量 M → 所有轨道物理量联动更新
- 轨道半径 r（圆轨道）→ 速度、周期、角速度、加速度实时计算
- 半长轴 a / 离心率 e（椭圆轨道）→ 近日点/远日点距离和速度
- 低轨半径 r1 / 高轨半径 r2（变轨）→ 各轨道速度计算
- 双星质量比 m1:m2 → 各自轨道半径、共同周期
- 初始角度差 Δθ（追及）→ 追及时间计算

### 模拟/动画驱动的功能
- **圆轨道运动**：匀速圆周运动动画，速度箭头恒定长度
- **椭圆轨道运动**：变速运动（近日点快、远日点慢），面积扫过阴影实时绘制
- **霍曼转移变轨**：三段轨道的连续动画（圆→椭圆→圆），两次点火时刻需要用户触发
- **三宇宙速度**：不同初速对应的轨道类型切换动画（圆→椭圆→抛物线→双曲线）
- **双星绕转**：两星绕质心的同步圆周运动
- **天体追及**：两颗卫星不同角速度运行，标注相遇时刻

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：**2D Canvas**（俯视图视角为主）
  - 天体运动的核心场景（开普勒定律、变轨、双星、追及）本质是平面运动，2D 俯视图最自然
  - 椭圆轨道、面积阴影、速度箭头、轨迹线在 Canvas 上绑制效率高
  - 深蓝/黑色太空背景 + 随机星点用 Canvas 绘制简单高效
- **可选 3D 视角**：卫星绕地球场景可能需要 3D 辅助视角（如倾斜轨道），但非核心需求
- **公式渲染**：KaTeX — 展示万有引力公式、开普勒定律公式、速度公式等

#### 数学/物理计算需求
- **中度**：圆轨道物理量计算（v = √(GM/r), T = 2π√(r³/GM), ω = √(GM/r³), a = GM/r²）
- **中度**：椭圆轨道参数计算（近日点/远日点距离和速度、面积速率、T²/a³ 验证）
- **中度**：霍曼转移轨道速度计算（各轨道各位置速度、两次速度增量 Δv）
- **中度**：双星系统计算（质心位置、各星轨道半径、共同角速度和周期）
- **轻度**：追及时间计算（角速度差 → 追及条件）
- **关键算法**：轨道运动的数值积分 — 对于椭圆轨道的实时动画需要数值方法：
  - 简单方案：开普勒方程求解（解析方法，通过偏近点角 E 与时间的关系）
  - 通用方案：RK4 数值积分（可与 P-02 运动模拟器共享技术底层）
  - 圆轨道：匀速角运动，无需数值积分

#### 公式渲染需求
- 万有引力公式 F = GMm/r²
- 圆轨道系列公式（v, ω, T, a 与 r 的关系）
- 开普勒三定律公式
- 霍曼转移各速度公式
- 双星质心条件和周期公式
- 追及条件公式

### 交互依赖

#### 交互模式
**模拟演示型**（主导）+ **参数驱动型**（辅助）

这是一个典型的模拟演示型应用：
- 核心是动画播放/暂停/调速
- 参数滑块驱动物理量联动
- 特殊交互：变轨场景的"点火加速"按钮
- 无实体编辑、无拖拽创建、无选中/删除操作

#### 关键交互
| 交互类型 | 具体场景 |
|---------|---------|
| 按钮点选 | 模型切换（4 种天体模型） |
| 滑块 | 天体质量、轨道半径、离心率、质量比、角度差等参数 |
| 播放控制 | 播放/暂停/调速（0.1x ~ 5x）、可能需单步 |
| 按钮触发 | "点火加速"/"点火减速"（变轨场景） |
| 画布拖拽 | 视图平移/缩放（观察不同尺度轨道） |
| Hover | 悬停天体/轨道点显示当前物理量 |

#### 是否需要实体系统
**不需要** — 天体和轨道由模型选择自动生成，不需要用户手动创建/管理实体。状态管理一个简单的 Store（当前模型类型 + 参数值 + 动画状态）即可。

#### 是否需要命令系统
**不需要** — 无 Undo/Redo 需求。

## 5. 竞品分析

### 5.1 PhET Gravity and Orbits / Kepler's Laws

- **形态**：Web（HTML5，纯浏览器运行）
- **技术**：HTML5 Canvas，PhET 自有框架（Scenery 渲染库），源码开源于 GitHub
- **优势**：
  - 完全免费、开源，全球教育界广泛使用，NASA 推荐
  - 交互直观：可拖拽天体改变位置，实时看轨道变化
  - 可切换显示引力矢量、速度矢量、轨迹路径
  - 支持"关闭引力"实验，帮助理解引力作用
  - Kepler's Laws 模块专门演示面积定律（等时间扫过等面积的阴影区域）
  - 基于教育研究设计，学生自主探索式学习
- **不足**：
  - 功能覆盖面较窄：没有卫星变轨、双星系统、天体追及等高中考点
  - 没有霍曼转移轨道、三宇宙速度等关键教学场景
  - 偏向"探索发现"而非"教学演示"，不够贴合中国高中物理课纲
  - 界面为英文（有中文翻译但质量一般）
- **启示**：面积定律的阴影标注方式、引力矢量的可视化方式值得参考；但 P-09 需要更贴合中国高中教学大纲的模型覆盖

### 5.2 Solar System Scope

- **形态**：Web（WebGL，需桌面浏览器）/ iOS / Android / Steam
- **技术**：HTML5 + WebGL，3D 渲染，基于 NASA 真实轨道数据
- **优势**：
  - 3D 太阳系全景，视觉震撼，2000 万+全球用户
  - 科学精确：基于 NASA 公布的最新轨道参数
  - 19 种语言支持，3D 百科全书式信息展示
  - 支持夜空观测模拟、时间前后调节
- **不足**：
  - 定位是"太阳系百科"而非"教学工具"，没有教学导向的交互
  - 无参数调节能力（不能修改天体质量看轨道变化）
  - 无开普勒定律验证、无变轨演示、无双星/追及模型
  - 3D 渲染对投影仪性能要求较高
  - 信息展示为主，缺乏物理量联动计算
- **启示**：深蓝太空背景 + 天体渲染的视觉风格可参考；但 P-09 的核心价值在参数可调的教学演示，而非百科展示

### 5.3 Universe Sandbox

- **形态**：Desktop（Steam，付费 $29.99）/ 未来计划移动端
- **技术**：Unity 引擎，N-body 牛顿力学数值模拟，自研物理引擎
- **优势**：
  - N-body 真实物理模拟，精度极高
  - 支持碰撞、潮汐力、洛希极限等高级物理效果

- **不足**：
  - 付费桌面软件，不是 Web 端
  - 功能极其复杂，学习曲线陡峭，不适合中学教学
  - 没有教学导向的引导（如面积定律标注、变轨步骤演示）

- **启示**：N-body 模拟的技术思路可参考（对于双星系统）；但 P-09 定位是教学工具而非沙盒模拟器，需要简化和引导

### 5.4 oPhysics / NAAP Planetary Orbits Lab / Academo Kepler

## 6. 初步技术判断

### 推荐的渲染技术
**2D Canvas 为主**

- 天体运动的核心场景（圆轨道、椭圆轨道、变轨、双星、追及）本质是平面运动，2D 俯视图最自然、最直观
- Canvas 2D 对轨道线、天体圆形、速度箭头、面积阴影扇形的绑制效率高
- 深蓝/黑色太空背景 + 随机星点 + 高对比度轨道/天体颜色，Canvas 完全胜任
- 无需 Three.js/WebGL 的额外复杂度和包体积
- 如果后续需要 3D 辅助视角（如倾斜轨道卫星），可以用 CSS 3D transform 或简单正交投影实现，不需要完整 3D 引擎

### 推荐的架构模式
**参数驱动 + 动画循环架构**

```
用户操作 → 参数 Store 更新 → 物理引擎重算 → 动画帧循环 → Canvas 渲染
                                    ↑
                            时间控制器（播放/暂停/调速）
```

| 层次 | 职责 | 说明 |
|------|------|------|
| 参数层 | 管理当前模型类型、天体参数、动画状态 | Zustand Store，简单状态 |
| 物理计算层 | 纯函数计算：轨道物理量、位置更新 | 与 P-02 共享数值积分底层 |
| 动画控制层 | requestAnimationFrame 循环、时间步进、播放/暂停/调速 | 通用动画框架，可跨模块复用 |
| 渲染层 | Canvas 2D 绑制：天体、轨道、箭头、阴影、数值标注 | 模块专属 |
| UI 层 | 模型选择、参数滑块、数值面板、播放控件 | React 组件 |

### 与其他模块的技术共性

| 共享资产 | 相关模块 | 说明 |
|---------|---------|------|
| 数值积分引擎（Euler/RK4） | P-02 运动模拟器 | 椭圆轨道动画需要数值方法求解运动方程，与 P-02 的抛体/圆周运动共享底层 |
| 动画时间轴控制器 | P-02/P-05/P-06/P-12/P-13/P-14 | 播放/暂停/调速/单步是物理模拟类模块的通用需求 |
| Canvas 2D 渲染基础（箭头/圆/椭圆/文字标注） | P-02/P-12/P-14 | 向量箭头、运动轨迹绑制是共用图元 |
| KaTeX 公式渲染 | 所有物理模块 | 公式展示通用 |
| 参数面板 UI 组件 | 所有物理模块 | 滑块 + 数值输入 + 参数预设 |

### 关键技术风险或难点

1. **椭圆轨道动画的均匀时间步进**：椭圆运动是变速的，不能简单等角度步进。需要求解开普勒方程 M = E - e·sin(E)（M 为平近点角，E 为偏近点角），这是一个超越方程，需要牛顿迭代法求解。替代方案是直接使用 RK4 对引力微分方程做数值积分。
2. **面积定律的阴影绘制**：需要在等时间间隔内绘制扇形区域，并验证面积相等。Canvas 2D 的路径填充可以实现，但椭圆扇形的精确绘制需要注意坐标变换。
3. **时间尺度问题**：天体运动周期可能非常长（地球同步轨道周期约 24 小时），需要大幅加速。动画速度倍率需要覆盖很大范围（如 10⁴x ~ 10⁸x），且不同模型的合理倍率不同。建议将时间尺度归一化为周期的百分比。
4. **变轨交互设计**：霍曼转移需要两次"点火"，如何在时间流动的动画中自然地插入用户触发点，需要精心设计交互流程（建议：到达点火位置时自动暂停，等待用户点击）。
5. **三宇宙速度的轨道类型切换**：初速从 v < v1 到 v > v2 的连续变化，轨道类型从坠落→圆→椭圆→抛物线→双曲线的切换，需要统一的轨道参数化表示（圆锥曲线统一方程 r = p/(1+e·cos θ)）。
6. **双星系统的数值稳定性**：两星绕质心的圆周运动解析解已知，可以直接用公式而不需数值积分。但如果希望展示质量突变时的过渡动画，则需要短暂的数值积分过程。
7. **投影仪显示效果**：深蓝/黑色背景在教室投影仪上可能偏暗，需要确保天体和轨道的颜色对比度足够高。建议天体尺寸适当放大（非真实比例）以保证可见性。
