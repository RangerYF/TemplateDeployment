# C-05 化学键与晶体结构查看器 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中化学教师（主要为选修/选择性必修"物质结构与性质"模块授课教师），不要求有晶体学专业背景或软件操作经验。次要用户为化学竞赛辅导教师和对晶体结构感兴趣的学生。

### 解决什么教学痛点
- **晶体结构空间想象困难**：四大晶体类型（离子晶体、原子晶体、分子晶体、金属晶体）的三维结构仅靠教科书平面图和实物模型难以理解，尤其是配位关系、空隙位置、晶胞边界等概念
- **晶胞计数教学难点**：顶点 1/8、棱 1/4、面心 1/2 的原子归属是高考重点，但黑板上很难直观展示"切割"效果
- **堆积方式抽象**：面心立方密堆积（ABCABC）与六方密堆积（ABABAB）的区别、空间利用率的差异，仅靠讲解极难理解
- **化学键类型区分**：离子键、共价键、金属键、氢键、范德华力在晶体中的表现方式不同，需要在同一个模型中对比展示
- **配位数与空隙关系**：配位数的计算、四面体/八面体空隙的位置和数量关系，是选修模块的核心难点
- **实物模型局限**：学校球棍模型套件数量有限（通常只有 NaCl），无法覆盖 15+ 种晶体，且不支持动态切换显示模式

### 核心产品价值
**一句话**：教师选择一种晶体模型，即可获得可交互旋转的 3D 晶胞，支持球棍/填充/多面体模式切换、晶胞扩展（2x2x2）、堆积动画演示、配位数点击高亮、化学键类型可视化，覆盖高中"物质结构与性质"模块 80% 教学需求。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 从晶体模型库选择目标晶体（如 NaCl）→ 旋转到合适角度 → 切换显示模式（球棍/填充）→ 确认投影效果 |
| **课中演示 — 晶胞教学** | 展示单个晶胞 → 点击"扩展"按钮生成 2x2x2 阵列 → 旋转观察周期性 → 切换到晶胞框架模式 → 展示晶胞参数(a,b,c,α,β,γ) → 演示原子计数（顶点/棱/面心归属高亮） |
| **课中演示 — 堆积方式** | 切换到堆积演示模块 → 播放等径球堆积动画（SC → BCC → FCC → HCP）→ 暂停在 FCC → 高亮四面体空隙和八面体空隙 → 展示空间利用率对比 |
| **课中演示 — 化学键** | 选择冰(Ice)晶体 → 开启化学键类型高亮 → 共价键(蓝色实线) + 氢键(绿色虚线)对比 → 点击水分子查看配位数(4) → 讲解氢键方向性和冰的低密度 |
| **讲题/对比** | 双屏对比 NaCl 与 CsCl → 对比配位数(6:6 vs 8:8) → 对比晶胞含原子数(4 vs 1) → 展示半径比与空隙类型的关系 |
| **拓展教学** | 展示闪锌矿/纤锌矿结构对比 → 讲解多晶型 → 展示钙钛矿的两种晶胞切法 |

### 怎么用才是"好用"的
- **3 分钟上手**：打开即可旋转/缩放查看 3D 晶体模型，无需任何配置
- **模型切换即时**：从模型库切换晶体，加载时间 < 1 秒
- **数据精确可信**：晶胞参数、空间群、原子坐标均来自 ICSD/CRC Handbook，教师可放心用于教学
- **投影清晰**：在 1080p 教室投影仪上，原子球、键、标注文字清晰可辨
- **动画流畅**：堆积方式动画和晶胞扩展动画 >= 30fps

### 交互频率和深度
- **频率**：每学期"物质结构与性质"模块集中使用约 8-12 课时，每课时使用 2-3 次，每次 5-15 分钟
- **深度**：中度偏重 — 核心操作是选模型+旋转+切换显示模式（浅），但配位数标注、空隙展示、晶胞对比等功能需要较深的交互

### 需要实体管理的功能
本模块**不是典型的编辑器型**应用，不需要用户创建/删除实体。核心实体（晶体模型、原子、键）均由数据驱动自动生成。但需要管理以下状态对象：
- 当前选中的晶体模型
- 当前选中的原子（用于配位数高亮）
- 显示模式状态
- 扩展级别（1x1x1 / 2x2x2）
- 化学键类型过滤/高亮状态

### 需要 Undo/Redo 的功能
**不需要** — 本模块是查看器/演示器，用户不做破坏性编辑操作，所有状态变更（切换模型/切换模式/旋转/扩展）都是可逆的即时操作。

### 参数驱动的功能
- 显示模式切换（球棍/填充/多面体/线框）→ 3D 模型实时变化
- 晶胞扩展级别 → 渲染原子数量变化
- 化学键类型筛选 → 键的显示/隐藏
- 堆积动画播放速度

### 模拟/动画驱动的功能
- 等径球堆积方式演示动画（逐层堆积过程，需要播放/暂停/逐步/速度控制）
- 晶胞扩展动画（从单晶胞平滑过渡到 2x2x2 阵列）
- 空隙展示动画（高亮闪烁空隙位置）

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：3D WebGL — Three.js（PRD 明确推荐），用于晶体模型的球棍/填充/多面体/线框渲染
- **渲染管线与 C-02 高度共享**：旋转/缩放/平移（OrbitControls）、光照、原子球体渲染、键的圆柱渲染、Raycasting 点选等基础设施完全一致
- **C-05 额外需求**：周期性边界渲染（晶胞扩展时在边界正确复制原子）、配位多面体渲染（凸包面）、空隙位置标记渲染

#### 数学/晶体学计算需求
- **重度**：分数坐标 → 笛卡尔坐标转换（需要晶胞参数矩阵：a, b, c, α, β, γ → 3x3 变换矩阵）
- **重度**：周期性边界处理 — 晶胞扩展时根据空间群对称操作生成等价原子位置
- **中度**：配位数计算 — 基于距离阈值查找最近邻原子
- **中度**：空隙位置计算 — 四面体空隙和八面体空隙的几何中心
- **中度**：配位多面体构建 — 凸包算法（Convex Hull）生成多面体面
- **轻度**：密度计算（Z × M / (N_A × V_cell)）
- **轻度**：最近邻距离计算（分数坐标距离 → 实际距离）
- **轻度**：空间利用率计算

#### 公式渲染需求
- 晶胞参数标注（a, b, c 数值 + 单位 pm，角度 α, β, γ）
- 原子计数公式（如 "顶点 8 x 1/8 + 面心 6 x 1/2 = 4"）
- 密度公式展示
- 化学键能数据（如氢键 ~20 kJ/mol）
- 需求较轻，普通 HTML/CSS 即可，不一定需要 LaTeX

#### 数据管理需求
- **静态晶体数据**：约 15 个晶体模型的 JSON 数据（晶胞参数 + 空间群 + 原子分数坐标 + 化学键类型 + 配位数 + 教学要点）
- **数据精确性要求极高**：教师明确要求"做有限数量的精确模型"，数据来源为 ICSD/CRC Handbook
- **数据 Schema 已定义**：id, name_cn, formula, level, crystal_system, space_group, lattice_params, atoms, bond_type, coord_number, z, teaching_points
- **可扩展设计**：支持通过添加新 JSON 文件扩展晶体库

### 交互依赖

#### 交互模式
**参数驱动型**（主导）+ **模拟演示型**（辅助）

这是一个查看器/演示器型应用，不是编辑器型：
- 用户从预设模型库中选择晶体（非自由创建）
- 通过切换开关/按钮改变显示模式
- 通过点击原子触发配位数高亮
- 堆积动画是演示型（播放/暂停/速度控制）

#### 关键交互

| 交互类型 | 具体场景 |
|---------|---------|
| 拖拽旋转 | 3D 晶体模型旋转/缩放/平移（OrbitControls） |
| 点选 | 点击原子 → 高亮配位原子 + 显示信息面板 |
| 按钮/开关 | 切换显示模式（球棍/填充/多面体/线框） |
| 按钮 | 晶胞扩展（1x1x1 ↔ 2x2x2） |
| 下拉列表 | 从模型库选择晶体 |
| 播放控制 | 堆积动画的播放/暂停/逐步/速度调节 |
| 复选框/筛选 | 化学键类型显示筛选（离子键/共价键/氢键等） |
| 双屏对比 | 左右分屏对比两种晶体或两种切法 |
| 触摸手势 | 移动端双指缩放、单指旋转 |

#### 是否需要实体系统
**不需要传统编辑器型实体系统** — 无需 Entity Registry、创建/删除/选中/属性编辑的完整流程。但需要轻量的"当前晶体模型"状态管理和"选中原子"高亮状态。

#### 是否需要命令系统
**不需要** — 无破坏性编辑操作，无需 Undo/Redo。

## 5. 竞品分析

### 5.1 专业晶体学软件

#### VESTA (Visualization for Electronic and Structural Analysis)
- **形态**：Desktop（Windows/macOS/Linux），免费
- **技术**：OpenGL 原生渲染，C++ 编写
- **优势**：晶体学领域事实标准工具，支持所有 230 个空间群、CIF/VASP/XSD 等多格式导入、精确的对称操作处理、多面体/等值面/电子密度可视化、发表级图像输出
- **不足**：面向科研人员设计，界面复杂（菜单嵌套深、参数繁多），学习曲线陡峭，无 Web 版，无教学导向功能（无教学要点、无学段标注、无堆积动画）
- **启示**：VESTA 的空间群处理和多面体渲染是技术标杆；C-05 不需要覆盖全部 230 空间群，只需精确处理约 15 个教学常用结构

#### Materials Project (materialsproject.org)
- **形态**：Web 应用
- **技术**：Three.js 3D 渲染（Crystal Toolkit 组件），Python 后端（pymatgen），REST API
- **优势**：海量晶体数据库（15 万+ 材料）、Web 端 3D 查看器、支持对称操作可视化、开源组件库
- **不足**：面向材料科学研究者，非教学工具，Web 3D 查看器功能相对基础（无堆积动画、无教学标注），加载速度受网络影响
- **启示**：证明了 Web 端 Three.js 晶体渲染的可行性和性能足够；其数据 Schema（pymatgen Structure 对象）可参考

#### ICSD Web (Inorganic Crystal Structure Database)
- **形态**：Web 应用（付费订阅）
- **技术**：Jmol/JSmol（Java Applet → JavaScript 移植）
- **优势**：权威无机晶体数据源（28 万+ 结构），3D 结构查看器，CIF 导出
- **不足**：付费昂贵（机构订阅），Web 查看器基于 JSmol 技术陈旧，交互体验差，无教学功能
- **启示**：C-05 的晶体数据精确性应对标 ICSD 级别（数据文档已标注来源为 ICSD）

#### Crystallography Open Database (COD) + Jmol
- **形态**：Web（免费开源数据库 + Jmol 嵌入查看器）
- **技术**：JSmol（JavaScript），MySQL 后端
- **优势**：免费开源、50 万+ 结构、CIF 标准格式
- **不足**：Jmol 技术栈过时、渲染效果差、无教学导向功能
- **启示**：CIF 格式是晶体数据交换标准，C-05 的 JSON Schema 应能与 CIF 字段对应

### 5.2 教学导向产品

#### ChemTube3D (chemtube3d.com, Oxford)
- **形态**：Web
- **技术**：Jmol 嵌入
- **优势**：面向化学教学，有晶体结构专区，配有教学说明
- **不足**：Jmol 渲染效果差，交互有限，无堆积动画，无配位数标注交互
- **启示**：教学导向的内容组织方式（按晶体类型分类 + 教学要点）可参考

#### 国内高中化学教学课件
- **形态**：PPT 内嵌图片/GIF 动画
- **现状**：绝大多数教师使用静态图片或简单 GIF 教学，少数使用 VESTA 截图，几乎无交互式 Web 工具
- **启示**：市场空白明显，教师对交互式晶体教学工具有强烈需求（PRD 教师评价已验证）

### 5.4 市场空白与 C-05 定位

**核心发现**：**面向高中化学教学的 Web 端交互式晶体结构工具不存在**。

- VESTA 是科研标杆但 Desktop Only + 非教学导向
- Materials Project 有 Web 3D 但面向材料科研
- 教学类产品（ChemTube3D）技术陈旧（Jmol），交互能力弱
- 国内高中化学教师几乎无可用的交互式晶体教学工具
- **堆积动画**和**空隙展示**功能在所有竞品中均为空白

C-05 的差异化定位：**唯一面向高中化学教学、Web 端 Three.js 3D 渲染、具备堆积动画+空隙展示+配位数交互+化学键可视化+学段分级的晶体结构查看器**。

## 6. 初步技术判断

### 推荐的渲染技术
**Three.js**（PRD 已明确推荐，与 C-02 分子结构查看器共享渲染管线）

- Three.js 是 Web 3D 事实标准，生态成熟，社区活跃
- 是否使用 @react-three/fiber（R3F）取决于整体化学产品线的框架选型；如果采用 React，R3F 是自然选择
- 对于教学级 3D 渲染（约 15 个晶体模型，单模型最多 ~500 原子在 2x2x2 扩展后），性能完全充裕
- 可复用 Materials Project 的 Crystal Toolkit 开源组件作参考

### 推荐的架构模式
**数据驱动查看器 + 参数状态管理**

```
晶体 JSON 数据 → 坐标转换引擎 → Three.js 场景构建 → 渲染
                                    ↑
用户交互（选模型/切模式/点原子） → 状态管理 → 更新渲染
```

| 层 | 职责 | 说明 |
|---|------|------|
| 数据层 | 晶体模型 JSON | 约 15 个模型的静态数据（晶胞参数 + 原子坐标 + 键信息） |
| 计算层 | 晶体学计算引擎 | 分数坐标→笛卡尔、周期性扩展、配位数计算、空隙定位、多面体构建 |
| 渲染层 | Three.js 场景 | 原子球体 + 键圆柱 + 多面体 + 晶胞框架 + 标注，与 C-02 共享基础管线 |
| 交互层 | 状态管理 + 事件 | 当前模型/显示模式/选中原子/扩展级别等状态 |
| UI 层 | 控制面板 + 信息面板 | 模型选择器、模式切换、晶胞参数展示、配位数信息、教学要点 |

**与 C-02（分子结构查看器）的共享边界**：
- **可共享**：3D 场景基础设施（OrbitControls、光照、相机）、原子球体渲染（InstancedMesh）、键渲染（圆柱/线段）、Raycasting 点选、原子颜色/半径配置、触摸手势支持
- **C-05 专属**：晶胞参数矩阵转换、周期性边界扩展、空间群对称操作、堆积动画系统、空隙计算、配位多面体渲染、晶胞框架渲染

### 关键技术风险或难点

1. **分数坐标→笛卡尔坐标转换精度**：非正交晶系（三方、六方、单斜）的晶胞参数矩阵构建需要精确的三角函数计算（a, b, c, α, β, γ → 3x3 矩阵），实现错误会导致整个模型形变。风险中等，算法成熟但需仔细验证。

2. **周期性边界渲染**：晶胞扩展（2x2x2）时，边界原子的正确复制和键的跨胞连接处理较复杂。需要处理原子在晶胞边界的归属（顶点/棱/面心原子被多个晶胞共享）。

3. **堆积动画的 3D 实现**：等径球逐层堆积动画（尤其是 FCC 与 HCP 的 ABCABC vs ABABAB 区别）需要精心设计动画序列和相机角度，技术难度中等但动画设计工作量大。

4. **配位多面体渲染**：需要根据配位原子位置计算凸包并渲染半透明多面体面，Three.js 支持但需自行实现凸包算法或引入依赖（如 quickhull3d）。

5. **晶胞选取功能（P2）**：在扩展结构中框选区域并判断是否构成有效晶胞，涉及空间群理论的有效性判定，算法复杂度较高，建议 MVP 阶段不实现。

6. **数据精确性验证**：教师明确要求"做有限数量的精确模型"，每个晶体的原子坐标、晶胞参数、配位数都需要与 ICSD 数据交叉验证。这不是技术风险而是质量管控要求。

7. **性能**：2x2x2 扩展后，复杂晶体（如 SiO2）的原子数可达数百个，使用 InstancedMesh 批量渲染应能满足性能要求。若后续扩展到更大晶格（3x3x3+），需要考虑 LOD 或截断渲染。
