# M-06 向量运算演示台 — 模块分析卡片

## 1. 用户场景 & 产品价值

**目标用户**：高中数学教师

**覆盖学段**：
- 高中必修：平面向量（向量概念、加减法、数乘、数量积）
- 高中选修：空间向量与立体几何（空间向量运算、法向量、叉积）

**教学痛点**：
1. **加减法几何意义搞不清**：平行四边形法则和三角形法则，学生在纸上画向量箭头时总搞混"哪个箭头接哪个箭头"
2. **数量积投影不直观**：`a·b = |a||b|cosθ` 中的投影几何意义，黑板上画的虚线歪歪扭扭，学生看不懂
3. **空间向量太抽象**：3D 方向角、法向量、叉积完全依赖空间想象力，部分学生跟不上

**核心产品价值**：让老师在坐标系上直接"画向量、做运算"——拖拽向量端点，所有运算结果和几何辅助线（平行四边形、投影、法向量）实时更新，把抽象的向量运算变为可交互的视觉体验。

## 2. 用户实际使用方式

### 典型操作流程

**课前准备**：
- 从预设向量组中选择合适的例子（如 VEC-011 系列的加法示例），或手动输入坐标创建向量
- 选择 2D/3D 模式（必修课用 2D，选修课用 3D）

**课中演示**：
- **讲加法**：画 a⃗、b⃗ → 点"加法" → 平行四边形自动画出 → 拖动端点看结果实时变化 → 切换三角形法则对比
- **讲数量积**：画 a⃗、b⃗ → 点"数量积" → 投影虚线+投影点自动显示 → 拖动 a⃗ 旋转角度 → 学生看到"90°时投影为0"
- **讲法向量**（3D）：输入平面两个方向向量 → 系统自动计算法向量 → 3D 旋转查看垂直关系
- **讲向量分解**：画 c⃗ 和两个基底 → 点"分解" → 分量自动标出 → 换不同基底看分解系数变化

**讲题**：
- 快速输入题目中的向量坐标，验证学生的计算结果
- 用共线/垂直判定功能演示判定条件

### 怎么用才是"好用"的

- **拖拽直觉化**：拖端点 = 改方向和长度，拖中部 = 平移，所有关联结果实时更新
- **运算可视化清晰**：平行四边形用浅色虚线、结果向量用粗箭头红色、投影用蓝色虚线——视觉层次分明
- **数值始终可见**：坐标分量、模长、角度、运算结果同时显示，不能"光画图没数据"
- **切换流畅**：2D↔3D 一键切换，3D 旋转体验与 M-01 一致

### 交互频率和深度

- **频率**：每堂向量课都会用（必修约8-10课时 + 选修约6-8课时）
- **深度**：中等——主要是"创建向量 → 选运算 → 观察结果 → 拖拽探索"，不需要复杂的实体管理

|--------|------|------|------|

| P0 | 数乘向量 | 参数驱动 | 滑块控制 k 值，向量实时缩放/反向 |

### 功能特征分析

**需要实体管理的功能**：
- 向量的创建/选中/删除/编辑（核心实体）
- 运算结果向量（派生实体，随输入向量变化而自动更新）

**需要 undo/redo 的功能**：
- 向量的创建、删除、位置修改
- 运算模式的切换（低优先级）

**参数驱动型功能（调参数 → 画面实时变化）**：
- 数乘滑块 k → 向量缩放
- 拖拽向量端点 → 所有关联运算结果更新
- 基底向量变化 → 分解系数更新

**模拟/动画驱动型功能**：
- 平行四边形法则的构造动画（可选，非核心）
- 数乘 k 值连续变化的动画演示（可选）

## 4. 核心依赖识别

### 技术依赖

**渲染方式**：
- **2D 模式**：Canvas 2D 或 SVG（需要：坐标系网格、向量箭头、辅助虚线、角度弧线、文字标注）
- **3D 模式**：WebGL（Three.js）—— 坐标轴、向量箭头、平面显示、3D 旋转
- **建议**：2D 用 Canvas/SVG 独立实现（更轻量、交互更精确），3D 复用 M-01 的 Three.js + R3F 体系

**数学计算需求**：
- 向量基本运算：加减、数乘、点积、叉积
- 模长、夹角计算：`|a| = √(x²+y²)`, `cosθ = a·b/(|a||b|)`
- 投影计算：`proj_b(a) = (a·b/|b|²) × b`
- 基底分解：解二元/三元线性方程组 `c = λ₁e₁ + λ₂e₂`
- 共线/垂直判定：行列式判零、点积判零
- 法向量计算：叉积 `n = a × b`
- 计算复杂度低，均为解析解，无需迭代/数值方法

**公式渲染需求**：
- 需要在界面上显示向量符号（a⃗）、运算公式（a·b = |a||b|cosθ）、计算过程
- 可用 KaTeX 轻量渲染，或直接 HTML/CSS 排版（向量符号相对简单）

### 交互依赖

**交互模式**：**编辑器型 + 参数驱动型 混合**
- 编辑器型：向量的创建、选中、拖拽修改（类似 M-01 的点编辑）
- 参数驱动型：运算选择 → 结果实时展示；滑块控制数乘

**关键交互**：
| 交互 | 说明 |
|------|------|
| 拖拽（端点） | 改变向量方向和长度，所有关联结果实时更新 |
| 拖拽（中部） | 平移整个向量（保持方向长度） |
| 拖拽（起点） | 改变起点位置（保持方向和长度） |
| 点选 | 选择向量进行运算 |
| 按钮 | 加法/减法/数乘/点积/分解/判定 |
| 滑块 | 数乘系数 k |
| 坐标输入 | 精确创建向量 |
| 2D/3D 切换 | 模式切换 |

**是否需要实体系统**：
- 是，但比 M-01 简单——只有"向量"一种核心实体类型（+ 派生的"运算结果向量"）
- 不需要 M-01 那样复杂的父子关系和约束系统

**是否需要命令系统（undo/redo）**：
- 需要，用于向量创建/删除/修改的撤回
- 复杂度远低于 M-01

## 5. 竞品分析

### 主要竞品对比

| 工具 | 形态 | 向量加减 | 点积/叉积 | 拖拽交互 | 2D/3D | 教学适用性 |
|------|------|----------|-----------|----------|-------|------------|
| **GeoGebra** | Web/桌面/移动 | 完全支持 | 完全支持 | 完全支持 | 2D+3D | 极高 |
| **Desmos** | Web | 支持 | 支持 | 部分支持 | 2D+3D | 高 |
| **PhET Vector Addition** | Web(HTML5) | 核心功能 | 不支持 | 完全支持 | 仅2D | 极高 |
| **MathBox** | JS 库 | 需开发 | 需开发 | 需开发 | 2D/3D | 视觉极佳 |
| **3Blue1Brown/Manim** | 视频 | 编程实现 | 编程实现 | 不支持 | 2D+3D | 高(仅视频) |

### 竞品详细分析

**GeoGebra**：
- 优点：功能最全面，庞大社区资源库，大量现成向量教学 Applet
- 不足：界面传统、学习曲线陡；3D 渲染效果一般；无法深度定制视觉风格
- 启示：功能完整度的标杆，但我们可以在**教学引导性**和**视觉品质**上超越

**PhET Vector Addition**：
- 优点：拖拽体验最佳，专注向量加法做到极致；教学引导设计精良（探索/实验室模式）
- 不足：只覆盖加法，不支持点积、叉积、分解；仅 2D
- 启示：**交互设计的标杆**——拖拽的手感、视觉反馈、教学模式设计值得深入参考

**Desmos**：
- 优点：界面极简美观，响应极快
- 不足：向量功能较新且不完善；缺少平行四边形法则等教学专用可视化
- 启示：**UI 简洁度**可以参考

**3Blue1Brown/Manim**：
- 优点：动画质量最高，线性代数可视化的"天花板"——网格变形、基底变换动画极其直观
- 不足：是视频制作工具，不支持交互；需 Python 编程
- 启示：**动画叙事方式**值得参考——例如基底分解时先显示网格线，再沿基底方向画平行线找交点

**MathBox**：
- 优点：GPU 加速渲染，视觉品质出版级；基于 Three.js 与现有技术栈兼容
- 不足：开发门槛极高，文档少，维护频率低
- 启示：作为 3D 向量渲染的**技术参考**，但不建议直接依赖

### 对我们的启示

1. **交互体验学 PhET**：拖拽手感、视觉反馈、阈值防抖
2. **视觉效果学 3Blue1Brown**：投影动画、基底分解的网格可视化
3. **功能完整度对标 GeoGebra**：覆盖加减法、数乘、点积、叉积、分解、判定
4. **差异化优势**：专为中国高中教材设计的预设数据 + 教学要点标注 + 投影仪优化的视觉设计

## 6. 初步技术判断

### 推荐的渲染技术

| 模式 | 推荐方案 | 理由 |
|------|----------|------|
| **2D** | Canvas 2D（主）+ SVG/HTML（标注） | 向量箭头、辅助线、角度弧线用 Canvas 绑定更精确；文字标注用 HTML overlay 更清晰；比 3D 引擎轻量 |
| **3D** | Three.js + React-Three-Fiber | 与 M-01 共享渲染管线；OrbitControls 提供旋转/缩放/平移 |

**2D vs 3D 的关系**：
- 2D 模式是**独立实现**（不是 3D 的正交投影），因为 2D 的交互精度要求更高（像素级拖拽），用 3D 引擎做 2D 会引入不必要的复杂度
- 3D 模式复用 M-01 的场景管理、相机控制、坐标系渲染
- 运算引擎（向量计算）2D/3D 共享，仅维度参数不同

### 推荐的架构模式

**参数驱动 + 轻量实体管理 混合架构**：

```
┌─────────────────────────────────────┐
│           UI 层                      │
│  工具栏 / 向量列表 / 属性面板        │
├─────────────┬───────────────────────┤
│  2D 渲染器   │    3D 渲染器          │
│  (Canvas)    │    (Three.js/R3F)    │
├─────────────┴───────────────────────┤
│           交互层                     │
│  向量拖拽 / 运算选择 / 模式切换      │
├─────────────────────────────────────┤
│         向量运算引擎                 │
│  加减 / 数乘 / 点积 / 叉积          │
│  投影 / 分解 / 判定                  │
├─────────────────────────────────────┤
│         状态管理 (Zustand)           │
│  向量列表 / 当前运算 / 运算结果      │
│  Undo/Redo                          │
└─────────────────────────────────────┘
```

**与 M-01 的关系**：
- **可复用**：3D 场景容器（Scene3D）、OrbitControls、坐标系渲染器、基础 Line/Point 渲染组件
- **不可复用**：M-01 的几何体 Builder 系统、截面计算、展开图——这些是立体几何专属
- **建议**：M-06 作为独立模块，在 3D 模式下通过共享组件复用 M-01 的渲染基础设施，但拥有自己的状态管理和运算引擎

### 关键技术风险或难点

| 风险点 | 等级 | 说明 | 应对策略 |
|--------|------|------|----------|
| 2D 拖拽精度与性能 | 中 | 多个向量 + 辅助线 + 标注同时渲染时的性能；端点拖拽的精确拾取 | Canvas 分层渲染（静态层+动态层）；拾取使用距离阈值而非精确碰撞 |
| 2D/3D 双渲染器同步 | 中 | 两套渲染器共享同一状态，需保证一致性 | 状态驱动渲染（Zustand store → 渲染器订阅），渲染器只负责"画"不负责"算" |
| 向量箭头在 3D 中的可见性 | 低 | 旋转时箭头可能被遮挡或变形 | 使用 ConeGeometry 做箭头，或使用 billboard 技术确保始终朝向相机 |
| 基底分解的线性方程组求解 | 低 | 需处理共线基底（无解/无穷解）的边界情况 | 行列式判零 + 用户提示"基底共线，无法分解" |
| 投影可视化的交互细节 | 低 | 投影虚线、投影点、角度弧线的精确绑定 | 参考 M-01 已有的 AngleMeasurementRenderer 和 DistanceMeasurementRenderer |

### 与 M-01 共享 3D 组件的可行性评估

| M-01 组件 | 复用可行性 | 说明 |
|-----------|-----------|------|
| Scene3D.tsx | ✅ 直接复用 | 场景容器、灯光、OrbitControls |
| CoordSystemRenderer.tsx | ✅ 可复用 | 坐标轴、刻度、标签（可能需微调范围参数） |
| SegmentEntityRenderer.tsx | ⚠️ 需改造 | 当前渲染直线段，向量需要加箭头 |
| PointEntityRenderer.tsx | ✅ 可复用 | 向量端点的显示 |
| ToolEventDispatcher.tsx | ✅ 可复用 | 拖拽事件分发机制 |
| selectTool.ts | ⚠️ 需改造 | 拖拽逻辑需适配向量的端点/中部/起点三种拖拽模式 |
| Entity/Store 系统 | ⚠️ 可参考 | 向量的实体模型比几何体简单，可简化后使用 |

## 附：2D/3D 模式决策建议

PRD 明确要求同时支持 2D 和 3D。建议的处理方式：

1. **默认 2D 模式**（覆盖必修课全部内容：加减法、数乘、点积、分解、共线/垂直判定）
2. **可切换 3D 模式**（覆盖选修课内容：空间向量运算、叉积、法向量、方向角）
3. **2D 独立渲染器**（Canvas 2D），不依赖 Three.js，交互更精确、性能更好
4. **3D 复用 M-01 渲染基础设施**，新增向量箭头渲染器和向量专属交互工具
5. **运算引擎统一**，2D/3D 仅维度参数不同
