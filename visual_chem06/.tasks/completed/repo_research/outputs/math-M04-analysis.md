# M-04 三角函数演示台 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中数学老师
- **解决的教学痛点**：三角函数的核心概念"单位圆上的角 → 函数值"在黑板上只能画静态图，老师无法让学生直观看到"拖动角度终边，右侧正弦曲线同步描绘"这一动态过程。五点法作图只靠口述难以让学生理解关键点的取法逻辑。解三角形涉及 SSA 模糊情形（0/1/2 解），静态图无法展示两种解的几何意义。
- **核心产品价值**：一个以单位圆与函数图像实时联动为核心的三角函数专题教学工具——拖动单位圆上的点，右侧三角函数图像同步生成；调节参数滑块，图像实时变换；输入已知条件，自动求解并绘制三角形。把三角函数从"背公式"变成"看得见"的几何体验。

## 2. 用户实际使用方式

### 教师典型操作流程

- **课前准备**：选择预设演示场景（如 TRI-022-A 振幅变化），或输入自定义参数组合，调好初始状态，截图备用
- **课中演示**：
  - **讲三角函数定义**：拖动单位圆上的角度终边，学生看到 sin 值（红色竖线）、cos 值（蓝色横线）、tan 值（金色线段）实时变化，右侧函数图像同步逐步描绘
  - **讲参数影响**：拖动 A/ω/φ/k 四个滑块，学生实时看到 y = A sin(ωx + φ) + k 图像的振幅、周期、相位、平移变化
  - **讲五点法作图**：播放动画，逐步展示一个周期内 5 个关键点的取法
  - **讲公式**：选择辅助角公式场景，动画展示两个分量 a·sinx 和 b·cosx 如何合成为一个正弦波
  - **讲解三角形**：输入已知条件（如 a=6, b=4, A=60°），自动计算并绘制三角形，SSA 情形展示两种解的几何构造
- **讲题**：用解三角形工具验证手算结果，用特殊角值表快速查阅

### 怎样算"好用"

- 拖动单位圆角度时，sin/cos/tan 三条彩色线段和右侧函数图像零延迟同步
- 特殊角（30°/45°/60° 等）自动吸附，显示精确值（√3/2 而非 0.866）
- 角度标注同时显示度数和弧度（可切换）
- 投影仪上颜色可辨，sin（红）/cos（蓝）/tan（金）对比清晰

### 交互频率和深度

- **中高频模块**：三角函数是高中数学的重要章节（必修一+选择性必修一），贯穿整个高中阶段，从基本定义到解三角形
- **交互深度**：中等偏深——核心操作是拖动角度和调参数（浅），但单位圆-图像联动、解三角形多解讨论、辅助角公式合成动画属于深度交互

## 3. 核心功能点

### 功能清单（按重要性排序）

1. **单位圆动态交互** — 拖动终边角度，实时显示 sin/cos/tan 值和三角函数线（正弦线、余弦线、正切线）
2. **单位圆与函数图像联动** — 左侧单位圆 ↔ 右侧函数图像双向同步，虚线连接对应点
3. **参数调节（y = A sin(ωx + φ) + k）** — 四个滑块分别控制振幅、周期、相位、平移，图像实时变化
4. **五点法作图演示** — 动画逐步展示关键点取法，帮助理解作图方法论
5. **解三角形工具** — 支持 SSS/SAS/ASA/AAS/SSA 五种已知条件组合，自动计算并绘制三角形
6. **多函数对比** — sin/cos/tan 三条曲线同时显示，支持变式（sin²x、|sinx| 等）
7. **辅助角公式可视化** — 动画演示 a·sinx + b·cosx 合成过程
8. **特殊角值查看表** — 0°-360° 常见角的三角函数值，支持精确值显示
9. **公式库浏览** — 恒等式、诱导公式、和差公式、二倍角公式等，KaTeX 渲染

### 需要实体管理的功能

- 解三角形中的三角形对象（创建/编辑/多解切换）
- 多函数曲线的管理（显示/隐藏/颜色区分）
- 单位圆上的标注点和辅助线

### 需要 undo/redo 的功能

- 需求不强：本模块以参数驱动型演示为主，非编辑器型
- 可选支持：解三角形的条件修改回退、参数滑块值回退

### 参数驱动的功能（调参 → 画面实时变化）

- **核心模式**：大部分功能都是参数驱动型
- 单位圆角度（拖动 → 三角函数值 + 图像同步）
- A/ω/φ/k 四个参数滑块
- 辅助角公式的 a/b 参数
- 解三角形的已知条件输入

### 模拟/动画驱动的功能

- 五点法作图动画（逐步标点 + 连线）
- 辅助角公式合成动画（分量波 → 合成波）
- 参数变化动画（如 A 从 1→2→3 连续变化）
- 单位圆角度自动旋转动画

## 4. 核心依赖识别

### 技术依赖

**渲染方式**：2D Canvas（主）+ DOM/SVG（辅助 UI）
- 单位圆 + 函数图像都是高频重绘场景（拖动角度逐帧更新），Canvas 性能更适合
- 三角函数图像采样点密集，Canvas 立即模式渲染效率高
- 公式展示、参数面板等用 React DOM
- 解三角形的三角形绘制也适合用 Canvas

**数学/物理计算需求**：
- 三角函数数值计算（Math.sin/cos/tan 原生支持，计算量不大）
- 特殊角精确值映射（30°→ √3/2 等，可用查找表）
- 正弦定理、余弦定理、海伦公式等解三角形计算
- SSA 模糊情形的判别逻辑（0/1/2 解）
- 辅助角公式中的 √(a²+b²) 和 arctan(b/a) 计算
- 函数图像采样与绘制

**公式渲染需求**：
- 大量公式需要展示（恒等式、诱导公式、和差公式、二倍角公式等）
- 特殊角精确值的数学排版（分数、根号）
- 建议使用 KaTeX，与 M-02 共用方案

### 交互依赖

**交互模式**：**参数驱动型为主 + 模拟演示型辅助**
- 主模式：拖动角度/滑块 → 画面实时变化
- 辅模式：五点法和辅助角公式的分步动画演示
- 解三角形部分偏向"输入-计算-输出"的工具型

**关键交互**：
- 拖拽（单位圆上拖动角度终边 — 最核心交互）
- 滑块（A/ω/φ/k 参数调节）
- 输入框（解三角形的已知条件输入）
- 按钮/切换（sin/cos/tan 显示切换、度/弧度切换、动画播放控制）
- 鼠标悬停/吸附（特殊角自动吸附并显示精确值）

**是否需要实体系统**：不需要完整实体系统
- 核心是参数驱动的可视化，不涉及复杂对象管理
- 解三角形部分需要轻量级的三角形对象，但不需要 Entity-Component 架构

**是否需要命令系统（undo/redo）**：不需要
- 参数驱动型工具，用户主要是调参观察，不需要回退历史操作
- 解三角形是"输入即计算"，无需 undo

## 5. 竞品分析

### GeoGebra 三角函数模块

- **产品形态**：Web / 桌面 / 移动端，开源免费
- **技术实现**：GWT 编译为 JS，SVG + Canvas 混合渲染，WebGL（3D 模块）
- **做得好的地方**：
  - 功能极其全面，单位圆、三角函数图像、解三角形都支持
  - 代数窗口与图形窗口双向联动
  - 中文版和大量社区 Applet 资源
  - 支持动画播放（自动旋转角度）
- **不足的地方**：
  - 单位圆与函数图像联动**不是内置一级功能**，依赖社区 Applet，质量参差不齐
  - 界面复杂，学习成本高
  - UI 风格偏传统，视觉吸引力不足
  - GWT 编译产物体积大，加载慢

### Desmos

- **产品形态**：Web / 移动端，免费
- **技术实现**：TypeScript/JS，Canvas + WebGL 混合，MathQuill 公式编辑
- **做得好的地方**：
  - 极致的用户体验，上手零门槛
  - 公式输入所见即所得（MathQuill），业界最佳
  - 滑块响应极其灵敏，60fps 实时更新
  - 分享链接便于教学
- **不足的地方**：
  - 单位圆联动**非内置功能**，需依赖社区 Applet
  - 不支持解三角形专项功能
  - 无中国教材预设和教学要点
  - 无五点法等中国高考专属教学功能

### PhET Trig Tour（科罗拉多大学）

- **产品形态**：Web（HTML5），开源免费
- **技术实现**：基于 PhET 自研 Scenery 渲染库（Canvas + SVG），纯 JS/TS
- **做得好的地方**：
  - 单位圆与函数图像联动**作为核心一级功能**内置
  - 拖拽即学的交互设计
  - 正切渐近线处理优雅（箭头渐隐 + 虚线）
  - 特殊角模式（限制为 30/45/60° 等）
  - 多圈旋转支持（±25.25 圈）
  - 经过严格教学研究验证
- **不足的地方**：
  - **功能极其单一**：仅覆盖基础单位圆探索，无参数调节、公式库、解三角形
  - 视觉风格偏学术，不够现代
  - 不可嵌入第三方平台
  - 无自定义函数能力

### 其他 Web 工具

- **MiniWebTool Unit Circle Visualizer**：特殊角 15° 增量吸附 + 精确分数值显示是亮点
- **Earth in Spoon Trigonometry Explorer**：单位圆 + 直角三角形 + 函数变换三合一，概念关联做得好
- **Math is Fun Interactive Unit Circle**：极简设计 + 配套文字解释，初学者友好

### 国内工具

- **几何画板（GSP）**：桌面端收费软件，国内教师使用率高但无 Web 版，三角函数功能非专项
- **网络画板**：Web 端免费，功能有限，中文原生但社区规模小

### 对我们的启示

1. **最大的差异化机会**：目前没有任何主流工具将"单位圆-函数图像联动"作为内置一级功能 + 同时覆盖参数调节、五点法、解三角形的完整教学流程。GeoGebra/Desmos 依赖社区 Applet，PhET 功能太单一。
2. **借鉴 PhET 的奇异点处理**：tan 渐近线的箭头渐隐 + 虚线效果是标杆，值得直接参考
3. **借鉴 MiniWebTool 的特殊角吸附**：15° 增量吸附 + 精确分数值是核心交互细节
4. **不要做成通用函数绘图工具**：聚焦三角函数专题，与 M-02 函数图像实验室形成差异化——M-02 是通用工具，M-04 是专题教学台
5. **中国市场空白**：国内缺乏高质量 Web 端三角函数教学工具，现代化、移动友好、中文原生是明确的市场机会

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：HTML5 Canvas 2D
  - 理由：单位圆拖动 + 函数图像联动需要高频重绘（每帧重绘单位圆和函数图像），Canvas 立即模式渲染性能足够且开发简洁
  - 解三角形的三角形绘制也适合 Canvas
- **UI 层**：React DOM
  - 理由：参数滑块、公式面板、解三角形输入框、模式切换按钮等控件用 DOM 实现
- **公式渲染**：KaTeX
  - 理由：公式库内容丰富（恒等式、诱导公式、和差公式等），KaTeX 轻量快速，与 M-02 方案统一

### 推荐的架构模式

- **参数驱动型架构**（与 M-02 同类，但更简单）
  - 核心数据流：`角度/参数状态 → 三角函数计算 → Canvas 渲染（单位圆 + 函数图像）`
  - 状态管理：Zustand 管理当前角度、A/ω/φ/k 参数、显示模式、解三角形输入等状态
  - 渲染层：双 Canvas 画布（左侧单位圆 + 右侧函数图像），或单 Canvas 分区绘制
  - 计算层：三角函数值计算（原生 Math）、解三角形求解器、特殊角精确值查找表
- **不需要**：完整实体系统、命令系统（undo/redo）、表达式解析器（参数固定为 A/ω/φ/k，不需要用户输入任意表达式）

### 关键技术风险或难点

1. **单位圆-函数图像联动的同步性**（难度：中，风险：中）
   - 拖动单位圆角度时，右侧图像需要同步"描绘"到当前角度对应位置
   - 需要处理好"描绘中"状态（未到达的区域为虚线/半透明）和"完成"状态的视觉区分
   - 两侧画布之间的虚线对应关系需要精确对齐

2. **特殊角吸附与精确值显示**（难度：低，风险：低）
   - 实现角度吸附（接近特殊角时自动snap）
   - 精确值查找表（有限的 17 个特殊角，直接映射即可）
   - 非特殊角显示数值近似值

3. **解三角形的多解处理**（难度：中，风险：低）
   - SSA 情形需判断 0/1/2 解
   - 两解时需要可视化展示两种构造（同一条件对应两个不同三角形）
   - 计算逻辑本身不复杂，但交互展示需要设计

4. **与 M-02 的功能边界**（风险：低，需设计时注意）
   - M-02 是通用函数绘图工具，三角函数只是其中一种函数类型
   - M-04 的专属价值：单位圆联动、三角函数线（正弦线/余弦线/正切线）、五点法作图、解三角形、公式库、特殊角值表
   - **复用层**：2D Canvas 坐标系渲染引擎、KaTeX 公式组件、参数滑块组件可与 M-02 共用

5. **正切函数的渐近线处理**（难度：低，风险：低）
   - tan(x) 在 ±90° + kπ 处趋近无穷
   - 需要检测不连续点，避免画出穿越全屏的连线
   - 参考 PhET 的箭头渐隐 + 虚线效果
