# V0.3 待决策事项

## D01：曲面体（圆柱/圆锥/球）交互缺失

**现象**：
- 多面体（正方体、长方体、棱锥）的棱线和面已支持 Hover 高亮、左键选中、右键菜单
- 曲面体（圆柱、圆锥、球）的母线、圆弧线、曲面均无法 Hover / 选中 / 右键

**根因**：
- 多面体在 `createBuiltInEntities` 中为每条棱线创建 Segment Entity、每个面创建 Face Entity，渲染器通过 `userData` 支持交互
- 曲面体只创建特征点 Entity，线和面由 `GeometryEntityRenderer` 直接用 Three.js 原语渲染，无 Entity 包装、无 `userData`，raycaster 无法识别

**需要决策**：
1. **是否在 V0.3 范围内解决？** 涉及 builder 结构改造 + Entity 创建逻辑扩展，工作量较大
2. **如果做，采用哪种方案？**
   - **方案A**：为曲面体的线（母线、圆弧）创建 Segment Entity（需要新增 `curve` 类型的 segment 约束），为曲面片创建 Face Entity（需要定义曲面分片策略）
   - **方案B**：不创建 Entity，在 `GeometryEntityRenderer` 中直接给现有 `<Line>` 和 `<mesh>` 加 `userData` + 交互事件，绕过 Entity 体系实现轻量交互
   - **方案C**：延后到 V0.4，当前只保证多面体交互完整

**影响范围**：
- 阶段3（坐标系重构）的"选面定 Z 轴"功能：如果曲面体的面不可选中，则曲面体无法使用该功能
- 面上取点：曲面体无法使用"面上取点"
- 用户体验一致性：多面体和曲面体交互能力不对等

## D02：自定义点拖拽无边界约束

**现象**：
- 线上取的点可以拖到线段之外（不限于线段两端点之间）
- 面上取的点可以拖到面的多边形边界之外（虽然仍在面所在的无限平面内）

**根因**：
- 线上的点：拖拽时直接设置 `positionOverride` 覆盖约束位置，无任何边界检查（V0.2 已有行为）
- 面上的点：阶段2 新增了面平面投影（投影到面的无限平面），但未做多边形边界 clamp

**需要决策**：
1. **是否需要限制？** 当前行为允许用户自由拖拽，灵活但不严谨
2. **如果做，约束到什么程度？**
   - **方案A**：线上点 clamp 到 t∈[0,1]，面上点 clamp 到多边形内部（需要点在多边形内判断 + 最近边界点投影）
   - **方案B**：仅做视觉提示（拖出边界时改变颜色/样式），不强制约束
   - **方案C**：保持现状，用户自由拖拽

**复杂度评估**：
- 线上 clamp：简单，只需限制 t 参数
- 面上 clamp：中等，需要实现多边形内点判断 + 最近边界点投影算法
