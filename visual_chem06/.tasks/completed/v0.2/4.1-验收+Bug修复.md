## Bug 清单

### Bug #1：Maximum update depth exceeded（无限更新循环）

**现象**：
```
Maximum update depth exceeded. This can happen when a component repeatedly
calls setState inside componentWillUpdate or componentDidUpdate. React limits
the number of nested updates to prevent infinite loops.
```

**可能原因分析**：

1. **SceneContent 的 EntityStore selector**（最可能）：
   ```typescript
   // Scene3D.tsx:87
   const entities = useEntityStore((s) => Object.values(s.entities));
   ```
   `Object.values()` 每次调用产生新数组引用 → Zustand 浅比较判定为"变化" → 触发重渲染 → 渲染器内部通过 Command/Store 写入 → 再次触发 → 无限循环。

2. **渲染器内 useEntityStore selector**：
   如 `useBuilderResult` Hook 内的 selector 可能每次返回新引用。

3. **ToolEventDispatcher 或渲染器在 render 阶段意外触发 store 写入**。

**修复方案**：

对 `Object.values()` selector 使用 Zustand 的 `useShallow` 或自定义相等比较：
```typescript
import { useShallow } from 'zustand/react/shallow';

// 方案A：useShallow（推荐）
const entities = useEntityStore(useShallow((s) => Object.values(s.entities)));

// 方案B：自定义 selector，返回 entities Record 本身，在组件内 Object.values
const entitiesMap = useEntityStore((s) => s.entities);
const entities = useMemo(() => Object.values(entitiesMap), [entitiesMap]);
```

**状态**：✅ 已修复

### Bug #1 修复（2026-03-09）
- **根因**：`useEntityStore((s) => Object.values(s.entities))` 每次调用 `Object.values()` 产生新数组引用，Zustand 浅比较判定为变化 → 重渲染 → 循环
- **修复**：selector 改为返回 `s.entities` Record 本身（引用稳定），组件内用 `useMemo` 派生排序数组
- **文件**：`src/components/scene/Scene3D.tsx`
- **验证**：`pnpm lint` + `pnpm tsc --noEmit` 通过

### Bug #2 修复（2026-03-09）—— 所有工具点击无反应 + Escape 无效
- **根因**：
  1. PointEntityRenderer 的 `<Html>` 标签渲染为 DOM 覆盖层，默认 `pointerEvents: 'auto'`，拦截了鼠标事件，导致底层 Canvas/R3F raycaster 收不到点击 → ToolEventDispatcher 不触发 → 所有工具失效
  2. `shortcuts.ts` 只处理 Ctrl+Z/Y/Escape 三种按键，未将其他键盘事件（Delete/Backspace 等）转发给 active tool 的 `onKeyDown`
- **修复**：
  1. PointEntityRenderer: 所有非编辑状态的标签 div 设置 `pointerEvents: 'none'`（鼠标穿透到 Canvas）；`<Html>` wrapper 也根据编辑状态切换 pointerEvents；双击编辑改为在 3D mesh `onDoubleClick` 上触发
  2. CustomPointRenderer: 额外添加不可见命中扩大体积（r=0.12），与 BuiltIn 点保持一致
  3. `shortcuts.ts`: 新增通用键盘事件转发——非特殊按键一律转发给 `activeTool.onKeyDown`；Escape 处理前先让当前 tool 处理（重置内部状态）
- **文件**：`src/components/scene/renderers/PointEntityRenderer.tsx`、`src/editor/shortcuts.ts`
- **验证**：`pnpm lint` + `pnpm tsc --noEmit` 通过

### Bug #3 修复（2026-03-09）—— Escape/键盘快捷键在 StrictMode 下丢失
- **根因**：React 18 StrictMode 下 effect 执行顺序为 mount→unmount→mount。`useEditorInit` 用 `useRef` 守卫包裹了 `initEditor()` + `setupShortcuts()`，第一次 unmount 调用 `teardownShortcuts()` 移除监听器，第二次 mount 因 ref 守卫跳过了所有逻辑 → 键盘监听器永久丢失
- **修复**：`initEditor()` 已有模块级 `initialized` 守卫（天然幂等），移除 useRef 守卫，让 `initEditor()` 和 `setupShortcuts()` 都在 effect 内执行。每次 mount 都绑定监听器，每次 unmount 都解绑
- **文件**：`src/editor/init.ts`
- **验证**：`pnpm lint` + `pnpm tsc --noEmit` 通过

### Bug #4 修复（2026-03-09）—— 截面选点不全部高亮 + 截面多边形渲染错误
- **根因**：
  1. CrossSectionTool 使用 `select()`（单选替换），选第二个点时第一个点的高亮丢失
  2. `computeEdgeIntersections` 返回的交点按棱迭代顺序排列，非多边形环绕顺序。FaceMesh 的三角扇渲染需要有序点，无序点导致截面面渲染错乱
- **修复**：
  1. SelectionStore 新增 `addToSelection()` 方法（累加选中），CrossSectionTool 改用此方法
  2. 新增 `sortIntersectionsByAngle()`：计算质心 → 建立平面坐标系 → 按极角排序交点 → 确保多边形正确环绕
- **文件**：`src/editor/store/selectionStore.ts`、`src/editor/tools/crossSectionTool.ts`
- **验证**：`pnpm lint` + `pnpm tsc --noEmit` 通过
- **B8 说明**：CoordSystemTool 的点击逻辑正确创建了 coordinateSystem Entity，但渲染器（CoordSystemRenderer）是阶段5的任务，当前无可见效果属预期行为

### Bug #5 修复（2026-03-09）—— builtIn 实体删除无反馈
- **根因**：SelectTool 的 onKeyDown 检测到 builtIn 实体时直接 `return`，无任何用户反馈
- **修复**：
  1. 新建 `notificationStore.ts`（轻量 Zustand store，含 show/dismiss + 自动消失）
  2. Scene3D 新增 `SceneNotification` 组件（底部居中黑色半透明 toast）
  3. SelectTool 删除 builtIn 时通过动态 import 调用 `show('无法删除几何体自带的点、线、面')`
- **文件**：`src/components/scene/notificationStore.ts`（新建）、`src/components/scene/Scene3D.tsx`、`src/editor/tools/selectTool.ts`
- **验证**：`pnpm lint` + `pnpm tsc --noEmit` 通过

### Bug #6 修复（2026-03-09）—— 拖拽顶点时卡住拖不动
- **根因**：R3F 的 `onPointerMove` 在 `<group>` 上只有当鼠标射线命中子 mesh 时才触发。拖拽过程中鼠标移到空白区域（无 mesh）→ 事件停止 → 拖拽卡死
- **修复**：重写 ToolEventDispatcher，pointerDown 命中 mesh 后切换到 window 级 `pointermove`/`pointerup` 监听。通过存储的 camera 和 dragPlane 在 DOM 事件中完成射线-平面投影。pointerUp 时自动清理监听器
- **文件**：`src/components/scene/ToolEventDispatcher.tsx`
- **验证**：`pnpm lint` + `pnpm tsc --noEmit` 通过
