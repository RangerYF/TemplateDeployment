## 任务目标

将展开图（UnfoldingPanel）、三视图（ThreeViewPanel）、度量显示（MeasurementDisplay）三个"只读展示"模块的数据源从旧 `useGeometryStore` 切换到 `EntityStore`，同时将面板显示开关（`unfoldingEnabled` / `threeViewEnabled`）从旧 Store 迁移到独立的 UI 状态管理。

## 技术要点

### EntityStore 读取几何体参数的标准模式

```typescript
// 标准读取方式（避免 Zustand selector 返回新引用导致无限循环）
const entities = useEntityStore((s) => s.entities);
const geometry = useMemo(() => {
  const geo = Object.values(entities).find(
    (e) => e.type === 'geometry'
  );
  return geo ?? null;
}, [entities]);

const geometryType = geometry?.properties.geometryType;
const params = geometry?.properties.params;
```

**注意 Zustand 无限循环陷阱**：阶段5/6已多次遇到 selector 返回新对象/数组引用导致 `Maximum update depth exceeded`。标准修复方式：selector 只取 `s.entities` 引用，派生数据用 `useMemo` 计算。

### 展开图标签映射

旧系统的 `labelOverrides: Record<number, string>` 是按顶点索引映射的自定义标签。新系统中，顶点标签存储在 Entity point 的 `properties.label` 中。

映射方式：
```typescript
function getVertexLabelMap(entities: Record<string, Entity>): Record<number, string> {
  const map: Record<number, string> = {};
  for (const entity of Object.values(entities)) {
    if (entity.type === 'point' && entity.properties.builtIn && entity.properties.source?.type === 'vertex') {
      map[entity.properties.source.vertexIndex] = entity.properties.label;
    }
  }
  return map;
}
```

这个函数可以直接替代旧的 `labelOverrides`，且不需要额外的 Store 字段。

### MeasurementDisplay 参数格式差异

旧系统中 `params` 是 `GeometryParams` 对象（按类型索引的参数集合），需要 `params[currentType]` 取当前类型的参数。

新系统中 `geometry.properties.params` 直接就是当前类型的参数对象，不需要再索引：
```typescript
// 旧：calculate(currentType, params[currentType] as Record<string, number>)
// 新：calculate(geometryType, params as Record<string, number>)
```

### uiStore 设计原则

`uiStore` 只存放**纯 UI 面板状态**，不存放任何实体或业务数据：
- `unfoldingEnabled: boolean` — 展开图面板是否展开
- `threeViewEnabled: boolean` — 三视图面板是否展开
- 未来可扩展：其他纯 UI 状态（面板折叠、布局偏好等）

## 注意事项

1. **engine/ 目录不动**：`unfold()` / `project()` / `calculate()` 的输入签名不变（geometryType + params），只改调用方获取参数的来源
2. **labelOverrides 的完整替代**：展开图中的标签显示必须与 V0.1 完全一致，测试重命名顶点后展开图是否正确反映
3. **Zustand 无限循环防范**：所有新的 EntityStore selector 都使用"取 entities 引用 + useMemo 派生"模式
4. **CalcStepsModal 无需改动**：它是纯 props 驱动的展示组件
5. **旧 Store 不删除**：仍有旧渲染系统（GeometryRenderer 等）依赖 `positionOverrides` 等字段，阶段8统一清理
6. **导出功能**：`exportSvgAsPng()` 是纯 DOM 操作，不依赖 Store，无需改动
