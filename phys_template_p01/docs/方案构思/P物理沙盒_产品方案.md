## 维度一：沙盒分组与融合方案

### 1.1 分组方案

| 沙盒/工具 | 包含模块 | 分组依据 | 内部联动方式 |
|-----------|---------|---------|------------|
| **力学沙盒** | P-01 受力分析 + P-02 运动模拟 + P-12 动量守恒 + P-14 机械能守恒 | 同一物体的力→运动→能量→碰撞构成完整因果链；教学中这四个主题频繁串联 | 受力→加速度→运动方程→能量变化→碰撞前后，共用同一个场景对象 |
| **电磁学沙盒** | P-08 电磁场可视化 + P-13 电磁感应 | 带电粒子在场中运动(P-08)和导体在场中运动产生EMF(P-13)是同一物理域的两面；共享磁场环境 | 场环境配置共享；P-08的粒子轨迹可过渡到P-13的导体棒运动 |
| **振动波动沙盒** | P-05 简谐运动 + P-06 波动演示 | SHM是波的微观基础；教材上振动→波是连续章节；共享数学模型 y=Asin(ωt+φ) | 单质点振动(P-05)→多质点排列形成波(P-06)，一键过渡 |
| **光学工具**（专题） | P-03 光学实验台 | 光学现象（折射/干涉/衍射）与力学/电磁无交互耦合；渲染方式独特（像素级强度） | 独立模块 |
| **热力学工具**（专题） | P-07 热力学模拟器 | 微观粒子+气体状态方程是完全独立的物理体系；液柱/气缸是独立考点 | 独立模块 |
| **天体运动工具**（专题） | P-09 天体运动 | 虽共享数值积分引擎，但教学场景独立（开普勒/变轨/双星不会与斜面/碰撞混讲） | 独立模块 |

**分组原则**：

1. **教学场景耦合 > 技术架构相似**。老师备课时会自然串联"受力→运动→能量"，不会串联"斜面+天体"
2. 融合的标准：同一堂课中老师可能在模块间切换 ≥ 3次
3. 独立的标准：教学中几乎不与其他模块在同一场景下使用

### 1.2 沙盒内部联动设计

#### 力学沙盒联动链

```
P-01 受力分析 ──合力/加速度──→ P-02 运动模拟
      ↑                              │
      │ 碰撞后重新受力                │ 运动状态(v,x,t)
      │                              ↓
P-12 动量守恒 ←──碰撞事件──── P-14 机械能守恒
      │                              ↑
      └──── 碰后速度 ────────────────┘
```

**联动机制**：同一个物体对象贯穿四个模块。老师在受力分析阶段操作的物体，释放后自动进入运动模拟，运动过程中实时计算能量变化，遇到碰撞触发动量守恒计算。

#### 电磁学沙盒联动

```
P-08 场环境配置 ──磁场参数──→ P-13 导体棒运动
(电场线/磁感线)              (EMF/电流/安培力)
```

**联动机制**：P-08配置的均匀磁场参数(B方向和大小)直接传递给P-13的导体棒模型。

#### 振动波动沙盒联动

```
P-05 单质点SHM ──"展开为波"──→ P-06 多质点波动
(x-t图)                      (波形图+振动图)
```

**联动机制**：P-05的振动参数(A, ω, φ)直接作为P-06的波源参数。

### 1.3 独立模块的存在形式

独立模块（光学/热力学/天体）称为**专题工具**。与沙盒共享以下基础设施：

- 参数面板组件
- 时间轴控制器
- Canvas坐标系
- KaTeX公式展示
- 导出PNG功能

区别：沙盒有"场景对象"概念（物体可在模块间传递），专题工具没有。

## 维度二：使用场景与操作流程

### 2.1 四个具体场景

| # | 场景名 | 频率 | 老师要做什么 | 工具提供什么 | 结果 |
|---|--------|------|------------|------------|------|
| S1 | **讲概念——展示分析结果** | 高频(每节课) | 投影仪上展示"斜面上的物体受哪些力、合力多大"，辅助口头讲解 | 选模型→设条件→自动受力分析，受力图+数值一键展示 | 学生看到清晰的受力图和计算结果 |
| S2 | **讲例题——复现题目条件** | 高频(每节课) | 把一道具体题的条件输入（斜面30°, μ=0.3, m=2kg），看受力、运动、能量 | 从预设模板加载→微调参数→一键分析 | 题目条件可视化，数值与手算一致 |
| S3 | **动态探究——改变参数** | 中频(每周2-3次) | 课上问"如果角度从30°变成45°会怎样？"然后实时拖动滑块 | 参数实时响应，受力图/运动轨迹/v-t图即时更新 | 学生看到参数变化的物理效果 |
| S4 | **过程回放——关键帧讲解** | 低频(复习课) | 暂停在某个时刻，讲解"此刻速度多大、加速度多大、动能多少" | 时间轴拖拽 + 暂停态数值面板 + 点击任意位置读数 | 学生理解运动过程中每个瞬间的物理量 |

### 2.2 操作步骤流程

#### S1 讲概念——展示分析结果（4步）

1. **选场景**：首页点击"力学沙盒"→ 顶部默认在"受力分析"Tab
2. **选模型**：从左侧模型列表点击"斜面"→ 画布出现斜面和物块，自动显示默认受力
3. **设条件**：左侧面板勾选"粗糙面"、设角度 → 受力图实时更新
4. **展示结果**：点击"正交分解"按钮 → 分解动画播放 → 数值面板显示各力大小和合力

#### S2 讲例题——复现题目条件（4步）

1. **选预设**：首页"力学沙盒"→ 顶栏"场景库"→ 搜索"斜面下滑"或按章节浏览
2. **调参数**：修改预设中的质量、角度、摩擦因数为题目条件
3. **分析**：受力图自动更新 → 数值面板显示各力大小 → 与题目答案核对
4. **演示运动**：Tab切到"运动"→ 点击播放 → v-t图同步绘制 → 暂停讲解关键点

#### S3 动态探究——改变参数（3步）

1. **搭建基础场景**（同S1或S2）
2. **实时调参**：拖动角度滑块从30°到45° → 画布上斜面几何实时变化 → 受力图/运动结果同步更新
3. **观察变化**：数值面板中各力大小实时跳变，学生直观看到"角度变大→沿斜面分力变大→加速度变大"

#### S4 过程回放——关键帧讲解（3步）

1. **播放运动**：在运动Tab点击播放，物体运动过程完整播放一遍
2. **拖动时间轴**：将时间轴拖到关键时刻（如碰撞瞬间、最高点等）
3. **读取数值**：暂停态下数值面板显示当前时刻所有物理量，点击轨迹上任意点也可读数

### 2.3 "讲概念"vs"讲例题"在操作流程上的差异

| 维度 | 讲概念(S1) | 讲例题(S2) |
|------|-----------|-----------|
| **入口** | 选模型→默认参数 | 选预设/手动输入精确参数 |
| **核心操作** | 调条件看受力变化 | 快速出结果核对答案 |
| **信息密度** | 可能只看受力图 | 力+运动+能量多个Tab切换 |
| **是否需要不同UI** | 不需要 | 同一套流程自然覆盖 |

**设计结论**：不需要两套UI。讲概念时老师停留在"受力"Tab不切换；讲例题时老师会切换多个Tab看不同维度。同一套界面自然覆盖两种需求。

## 维度三：交互与信息展示设计

### 3.1 物理量的视觉语言

| 物理量 | 颜色 | 形状 | 动画特征 | 标注位置 |
|--------|------|------|---------|---------|
| **重力 G** | 深红 #C0392B | 实线箭头↓ | 恒定不闪烁 | 箭头末端右侧 |
| **支持力 N** | 蓝色 #2980B9 | 实线箭头⊥面 | 恒定 | 箭头末端 |
| **摩擦力 f** | 橙色 #E67E22 | 虚线箭头沿面 | 方向可能变化时闪烁提示 | 箭头末端 |
| **弹力 T** | 绿色 #27AE60 | 实线箭头沿绳/弹簧 | 弹簧伸缩时长度变化 | 箭头末端 |
| **合力 F合** | 紫色 #8E44AD | 粗实线箭头 | 出现时有脉冲放大动画 | 箭头末端 |
| **速度 v** | 天蓝 #3498DB | 实心三角头箭头 | 长度随v实时缩放 | 箭头上方 |
| **加速度 a** | 红色 #E74C3C | 空心三角头箭头 | 长度随a实时缩放 | 箭头上方 |
| **动量 p** | 青色 #1ABC9C | 粗实线箭头 | 碰撞前后长度突变 | 物体上方 |
| **动能 Ek** | 红色条 | 条形图柱 | 高度实时变化 | 柱顶数值 |
| **势能 Ep** | 蓝色条 | 条形图柱 | 高度实时变化 | 柱顶数值 |
| **总能 E总** | 灰色线 | 水平参考线 | 守恒时不变/不守恒时下降 | 线右端数值 |

**区分同类物理量规则**：

- 同类力用**同色系不同明度**（如多个弹力用绿色深浅区分）
- 每个力箭头旁标注力的名称缩写（G、N、f、T）
- 多物体时，物体用不同底色圆/方块区分（物体A=蓝底，物体B=橙底）

```
左侧参数面板（抽屉式，点击展开/收起，默认收起以最大化画布）
```

**信息过载管理策略**：

| 规则 | 说明 |
|------|------|
| **默认只显示当前Tab对应信息** | 选"受力"Tab时，主画布只显示力箭头，右侧只显示力列表 |
| **叠加层开关** | 右上角有 ☑力 ☑速度 ☑轨迹 ☑能量 四个开关，可手动叠加其他层 |
| **信息密度三档** | 简洁(只有箭头) / 标准(箭头+数值) / 详细(箭头+数值+公式) |
| **画布优先** | 参数面板默认收起（抽屉式），调完参数后自动收起 |

### 3.3 2D vs 3D 决策

| 模块/场景 | 维度 | 理由 |
|-----------|------|------|
| 斜面、水平面、碰撞 | **纯2D** | 运动在一个平面内，2D足够 |
| 平抛运动 | **纯2D** | 侧视图就是标准教学视角 |
| 竖直圆周 | **纯2D** | 侧视图 |
| 水平圆周 | **2D俯视图 + 侧视小图** | 俯视看轨道，右上角小窗侧视看力 |
| 天体运动 | **2D俯视图** | 轨道平面运动 |
| 磁场中粒子运动 | **纯2D** | 垂直纸面的磁场，运动在纸面内 |
| 电磁感应导轨 | **纯2D** | 标准教学示意图就是2D |
| 光学 | **纯2D** | 光路图天然2D |

**结论：本阶段全部2D，不引入3D。** 水平圆周等需要空间感的场景，用"主视图+辅助小图"的方式补充，而非3D渲染。理由：

- 零学习成本（不需要教老师旋转3D视角）
- 技术复杂度可控（不引入Three.js）
- 与物理教材/试卷的示意图风格一致

## 维度四：场景搭建与预设体系

### 4.1 预设体系设计

#### 三级预设组织

| 层级 | 维度 | 示例 | 粒度 |
|------|------|------|------|
| **L1 物理模型** | 按力学模型分类 | 水平面、斜面、悬挂、圆周、碰撞… | 约15个模型 |
| **L2 典型场景** | 每个模型下的常见条件组合 | 斜面-匀速上滑、斜面-加速下滑、斜面-临界静止… | 每模型3-8个场景 |
| **L3 经典例题** | 精确到具体参数的题目复现 | "2kg物体在30°斜面μ=0.3从静止下滑" | 可扩展 |

#### 场景可穷举性分析

高中物理题目的场景**高度模式化**。高考物理的本质是模型识别——每道题都是把文字条件映射到有限的物理模型上。以力学为例：

| 模型 | 下级场景 | 穷举性 |
|------|---------|--------|
| 水平面 | 匀速/加速/减速 × 光滑/粗糙 × 单体/连接体 | 约12种组合，**可穷举** |
| 斜面 | 上滑/下滑/静止 × 光滑/粗糙 × 有无初速 | 约12种，**可穷举** |
| 悬挂 | 单绳/双绳/绳+杆 × 静止/圆周 | 约8种，**可穷举** |
| 圆周运动 | 水平(转盘/锥摆/管道) / 竖直(绳/杆/轨道) | 约10种，**可穷举** |
| 碰撞 | 完全弹性/非弹性/完全非弹性 × 追及/对碰 | 约6种，**可穷举** |
| 平抛 | 水平抛出/斜面上抛出 | 约4种，**可穷举** |

L1+L2预设能覆盖约85%的高中物理教学场景。覆盖不到的15%主要是多物体复合场景（如"传送带上的物体飞出后做平抛落到斜面上"），这些可通过后续"场景拼接"能力扩展。

### 4.2 手动搭建流程（从零开始，4步）

| 步骤 | 操作 | 界面反馈 | 耗时估计 |
|------|------|---------|---------|
| ① 选模型 | 点击左侧"新建场景"→弹出模型选择卡片（大图标+名称） | 画布立即出现该模型的默认场景（默认参数+自动受力分析） | 2秒 |
| ② 设物体 | 设置物体数量、类型（物块/小球）、质量 | 物体出现在画布上，大小按质量比缩放 | 5秒 |
| ③ 设条件 | 左侧面板设角度/摩擦因数/初速度/弹簧劲度系数等 | 画布实时更新几何外观和受力图 | 10秒 |
| ④ 开始演示 | 点击播放按钮 | 物体开始运动，v-t图/能量条同步更新 | 1秒 |

**总耗时约18秒**，目标是让老师在30秒内完成任何场景搭建。

### 4.3 快捷操作设计

| 快捷设计 | 说明 | 覆盖场景 |
|---------|------|---------|
| **智能默认值** | 选"斜面"模型后，自动预设θ=30°、m=1kg、μ=0——最常见教学参数 | 所有场景 |
| **条件一键切换** | "光滑⇄粗糙"、"有初速⇄从静止"、"水平⇄倾斜"用Toggle按钮 | 高频条件变更 |
| **参数联动推断** | 设了"匀速"运动状态→自动反算所需摩擦因数并锁定 | 平衡条件题 |
| **最近参数记忆** | 上次使用的质量/角度/摩擦因数自动填充到新场景 | 连续讲题 |
| **数值直接输入** | 滑块旁的数值框可直接键入精确值（如μ=0.25） | 精确复现题目 |

### 4.4 场景保存与复用

| 功能 | 设计 |
|------|------|
| **保存** | 点击工具栏"保存场景"→输入名称→自动保存所有参数+显示状态+当前时刻 |
| **存储位置** | 浏览器 localStorage（单机使用），后续可扩展到云端 |
| **组织方式** | 按保存时间倒序 + 支持文件夹分组（老师可建"高三一轮复习"等文件夹） |
| **检索** | 名称搜索 + 按模型类型筛选 |
| **分享** | 导出为JSON文件 → 另一台电脑导入（不依赖账户系统） |
| **预设场景容量** | 不限制数量，localStorage上限前自动提示 |

### 5.2 完整操作流程走查

**场景：老师要讲"质量2kg的物体在倾角30°粗糙斜面上，动摩擦因数0.3，从静止开始下滑"**

| 步骤 | 老师操作 | 系统响应 | 耗时 |
|------|---------|---------|------|
| **1. 进入沙盒** | 首页点击"力学沙盒" | 打开力学沙盒，默认Tab在"受力" | 1s |
| **2. 选模型** | 左侧模型列表点击"斜面" | 画布出现30°斜面+1kg物块（默认参数），自动显示默认受力图（G、N） | 1s |
| **3. 调参数** | 左侧面板：m改为2kg，θ确认30°，μ改为0.3，v₀确认0 | 受力图实时更新：摩擦力f出现（因μ>0），各力数值更新 | 5s |
| **4. 查看受力结果** | 点击"正交分解"按钮 | 正交分解动画播放（G分解为沿斜面和垂直斜面分量），数值面板显示所有力的大小和合力 | 3s |
| **5. 切到运动** | 点击顶部"运动"Tab→点击▶播放 | 物块沿斜面加速下滑；右侧v-t图同步绘制直线(匀加速)；画布上速度箭头实时增长；力箭头降为半透明保留 | 10s |
| **6. 暂停讲解** | 点击⏸暂停在t=2s处 | 画面定格；数值面板显示 t=2s, v=4.7m/s, s=4.7m；可拖动时间轴回看 | 30s |
| **7. 切到能量** | 点击"能量"Tab | 右侧切换为能量条形图：Ek(红)、Ep(蓝)、W_f(灰色损耗)；总机械能参考线 | 即时 |
| **8. 回看完整过程** | 时间轴拖回t=0→点击▶重新播放 | 运动重新开始，能量条形图同步动态变化 | 10s |

**总耗时约1分钟**（不含老师口头讲解时间）。

### 5.3 受力分析的展示设计

受力分析的展示遵循PRD定义：选模型→设条件→系统自动计算并一次性显示受力结果。

**受力图展示内容**：

| 元素 | 说明 |
|------|------|
| **力箭头** | 各力按颜色编码同时显示（G深红、N蓝、f橙），长度按比例 |
| **数值标注** | 每个力箭头末端显示力的名称和大小（如"G=19.6N"） |
| **正交分解** | 点击"正交分解"按钮 → G分解为沿斜面分量和垂直斜面分量（虚线箭头+直角标记），动画时长0.8s |
| **合力显示** | 点击"显示合力"按钮 → 合力箭头（紫色粗线）出现，脉冲放大后稳定 |
| **力列表** | 右侧信息面板列出所有力的大小：G=19.6N, N=17.0N, f=5.1N, F合=4.7N |
| **条件矛盾提示** | 若约束条件不合理（如静摩擦力超过最大值），红色提示框说明 |

### 5.4 受力→运动的联动

**切换方式**：点击顶部"运动"Tab

| 切换后的变化 | 说明 |
|-------------|------|
| 力箭头降为0.3透明度 | 受力分析结果作为"背景层"保留，让学生看到"是这些力导致了这个运动" |
| 出现速度箭头(天蓝)和加速度箭头(红) | 初始v=0所以速度箭头长度为0 |
| 右侧面板切换为v-t图 | 空白坐标系，等待播放后绘制 |
| 底栏时间轴激活 | 受力Tab时时间轴禁用（静态分析），运动Tab时启用 |

**播放后同屏效果**：

```
┌──────────────────────────────────────────────┐
│                    ↑N(淡蓝0.3)               │
│                   /                          │
│        □──→v(天蓝)  ──→a(红)                 │
│       /↓G(淡红0.3)  ←f(淡橙0.3)             │
│      / ___物块沿斜面加速下滑___                │

│   /_________________________________________ │
└──────────────────────────────────────────────┘
```

- 力箭头作为半透明背景，展示"力→运动"的因果关系
- 速度箭头跟随物块实时增长
- 轨迹用虚线点留痕
- 老师可通过叠加层开关完全关闭力箭头（只看运动）

### 5.5 信息密度管理

| Tab状态 | 主画布默认显示 | 右侧面板显示 | 可手动叠加 |
|---------|--------------|-------------|-----------|
| **受力** | 力箭头 + 标注 | 力列表 + 数值 | 网格、坐标轴 |
| **运动** | 速度箭头 + 轨迹 | v-t图 / s-t图 | 力(0.3透明度)、加速度箭头 |
| **能量** | 物体运动动画 | 能量条形图 + 总能线 | 力、速度、轨迹 |
| **动量** | 动量箭头 + 碰撞动画 | 动量数值 + 守恒验证 | 力、速度、能量条 |

**自动降密规则**：

1. 切换Tab时，自动只显示该Tab的核心信息，其他层默认关闭
2. 手动开启的叠加层在切换Tab后自动关闭（回到干净状态）
3. 右侧信息面板始终只显示当前Tab对应的图表，避免多图表堆叠

## 汇总：新方案 vs 现有PRD的关键差异

| 维度 | 现有PRD方案 | 新沙盒方案 | 变化原因 |
|------|-----------|-----------|---------|
| **产品形态** | 12个独立HTML工具 | 3个沙盒 + 3个专题工具 | 力/运动/能量/动量在教学中是一体的 |
| **模块关系** | 各模块独立，无联动 | 沙盒内模块共享场景对象，受力→运动→能量自动串联 | 老师不应在4个工具间切换同一道题 |
| **信息展示** | 固定布局（左参数+中画布+右图表） | Tab驱动的动态布局 + 叠加层开关 + 信息密度三档 | 避免同时展示4维信息导致过载 |
| **受力分析** | 选模型→自动出结果（PRD原始设计） | 保持一致：选模型→设条件→自动受力分析 | 遵循PRD |
| **场景搭建** | 选模型→调参数 | 三级预设体系(模型→场景→例题) + 快捷操作 + 场景保存 | 手动建模太慢，预设覆盖80%需求 |
| **参数面板** | 始终展开在左侧 | 抽屉式，调完自动收起 | 最大化投影仪上的画布面积 |
| **导航** | 12个独立入口 | 首页6卡片 + 最近使用 + 场景库搜索 | 降低认知负担 |
| **力→运动过渡** | 需从P-01退出再进P-02 | Tab切换，力箭头自动作为运动的背景层保留 | 保持因果链可见 |
| **3D** | 未明确 | 全2D，空间感用主视图+辅助小图补充 | 零学习成本，与教材风格一致 |
| **预设粒度** | 模型级（"斜面"） | 三级：模型→场景→例题，支持教材章节和物理模型双维度浏览 | 老师最高频操作是"复现一道题" |
