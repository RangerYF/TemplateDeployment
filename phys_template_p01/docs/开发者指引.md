# 预设开发指引

## 概述

本文档说明如何参照"水平面物块受力"范例，在 `src/domains/` 下开发一个新的物理预设。

**核心原则：域隔离**——所有实现只在 `src/domains/{域}/` 内完成，不修改 `core/`、`shell/`、`renderer/` 的代码。注册 API 由 core 层提供，域代码只需调用注册函数。

---

## 开发流程（5 步）

```
Step 1: 注册实体类型    → domains/{域}/entities/
Step 2: 实现求解器      → domains/{域}/solvers/
Step 3: 实现实体渲染器  → domains/{域}/renderers/
Step 4: 实现视角渲染器  → domains/{域}/viewports/（如需新视角）
Step 5: 编写预设 JSON   → domains/{域}/presets/

最后：在 domains/{域}/index.ts 中导入并调用所有注册函数
```

## Step 1：注册实体类型

**目标**：定义一种新的物理实体（如物块、斜面、弹簧等）。

**参考文件**：`src/domains/mechanics/entities/block.ts`

**API 签名**：

```typescript
import { entityRegistry } from '@/core/registries/entity-registry';

entityRegistry.register({
  type: string,                    // 实体类型标识，如 'block'
  category: EntityCategory,        // 'object' | 'surface' | 'connector' | 'field' | ...
  label: string,                   // 显示名称，如 '物块'
  defaultProperties: Record<string, unknown>,  // 默认属性值
  paramSchemas: ParamSchema[],     // 可调参数定义（驱动面板渲染）
  hitTest: (entity, point, transform) => HitTestResult | null,  // 点击检测
  createEntity: (overrides?) => Entity,  // 工厂函数
});
```

**代码模板**：

```typescript
// src/domains/mechanics/entities/slope.ts
import { entityRegistry } from '@/core/registries/entity-registry';
import type { Entity, SliderParamSchema } from '@/core/types';

export function registerSlopeEntity(): void {
  entityRegistry.register({
    type: 'slope',
    category: 'surface',
    label: '斜面',
    defaultProperties: {
      angle: 30,    // 度
      length: 2,    // m
      friction: 0,
    },
    paramSchemas: [
      {
        key: 'angle', label: '倾角', type: 'slider',
        min: 5, max: 85, step: 1, default: 30, unit: '°',
      } satisfies SliderParamSchema,
    ],
    hitTest: (entity, point) => {
      // 判断 point 是否在斜面附近（用 pointOnLine）
      // ...
      return null;
    },
    createEntity: (overrides) => {
      const entity: Entity = {
        id: `slope-${crypto.randomUUID().slice(0, 8)}`,
        type: 'slope',
        category: 'surface',
        transform: overrides?.transform ?? { position: { x: 0, y: 0 }, rotation: 0 },
        properties: { angle: 30, length: 2, friction: 0, ...overrides?.properties },
        label: overrides?.label ?? '斜面',
      };
      return entity;
    },
  });
}
```

**注意事项**：

- `hitTest` 接收物理坐标，需根据 `entity.transform.position` 和属性计算碰撞区域
- block 的坐标约定：`transform.position` 是底边中心，矩形向上延伸
- 几何工具函数可从 `@/core/physics/geometry` 导入：`pointInRect`、`pointInCircle`、`pointOnLine`

## Step 2：实现求解器

**目标**：实现一个物理求解器，根据场景实体和关系计算力、运动状态等。

**参考文件**：`src/domains/mechanics/solvers/block-on-surface.ts`

**API 签名**：

```typescript
import { solverRegistry } from '@/core/registries/solver-registry';
import type { SolverFunction } from '@/core/registries/solver-registry';

// SolverFunction 统一签名
type SolverFunction = (
  scene: SceneDefinition,   // 场景（实体Map + 关系数组 + 参数值）
  time: number,             // 当前时刻 t（秒）
  dt: number,               // 时间步长（解析解为0）
  prevResult: PhysicsResult | null,  // 上一帧结果（数值积分用）
) => PhysicsResult;

solverRegistry.register({
  id: string,                        // 唯一标识，如 'mech-block-on-horizontal-surface'
  label: string,                     // 显示名称
  pattern: {
    entityTypes: string[],           // 场景中必须存在的实体类型
    relationType: string,            // 必须存在的关系类型（'none' 表示无关系）
    qualifier?: Record<string, string>,  // 额外判定条件
  },
  solveMode: 'analytical' | 'numerical',
  solve: SolverFunction,
  integrator?: 'semi-implicit-euler' | 'velocity-verlet' | 'rk4',  // 数值积分器
  priority?: number,                 // 匹配优先级（越小越优先，默认100）
  eventDetectors?: EventDetector[],  // 事件检测器列表
});
```

**返回值 PhysicsResult 结构**：

```typescript
{
  time: number,
  forceAnalyses: Map<EntityId, ForceAnalysis>,   // 每个实体的受力分析
  motionStates: Map<EntityId, MotionState>,       // 每个实体的运动状态
  energyStates?: Map<EntityId, EnergyState>,      // 可选：能量状态
}
```

**ForceAnalysis 结构**：

```typescript
{
  entityId: EntityId,
  forces: Force[],       // 独立力列表
  resultant: Force,      // 合力
  decomposition?: OrthogonalDecomposition,  // 正交分解（斜面场景需要）
}
```

**Force 结构**：

```typescript
{
  type: ForceType,       // 'gravity' | 'normal' | 'friction' | 'tension' | ...
  label: string,         // 标注文本，如 'G'、'N'
  magnitude: number,     // 大小（N）
  direction: Vec2,       // 单位方向向量
  applicationPoint?: Vec2,  // 作用点（默认实体中心）
}
```

**注意事项**：

- 解析解求解器：`solveMode: 'analytical'`，dt 参数为 0，任何时刻返回相同结果（静力平衡）或公式计算结果
- 数值积分求解器：`solveMode: 'numerical'`，需指定 `integrator`，使用 `prevResult` 做递推
- `qualifier` 用于区分同一实体组合下的不同场景（如 `{surface: 'horizontal'}` vs `{surface: 'inclined'}`）
- 从 `scene.entities` 中按 type 查找实体：`Array.from(scene.entities.values()).find(e => e.type === 'block')`

## Step 3：实现实体渲染器

**目标**：定义实体在 Canvas 上的绘制方式。

**参考文件**：
- `src/domains/mechanics/renderers/block-renderer.ts`
- `src/domains/mechanics/renderers/surface-renderer.ts`

**API 签名**：

```typescript
import { rendererRegistry } from '@/core/registries/renderer-registry';

// 渲染器签名
type EntityRenderer = (
  entity: Entity,
  result: PhysicsResult | null,
  ctx: RenderContext,
) => void;

rendererRegistry.registerEntity({
  entityType: string,         // 对应实体类型
  renderer: EntityRenderer,
  layer: RenderLayer,         // 绘制层级
});
```

**RenderLayer 枚举**（按绘制顺序）：

```
'background' → 'surface' → 'field' → 'object' → 'connector' → 'overlay'
```

**坐标转换工具**：

```typescript
import { worldToScreen, worldLengthToScreen } from '@/renderer/coordinate';

// 物理坐标 → 画布像素坐标（Y 轴自动翻转）
const screenPos = worldToScreen(physicsPos, ctx.coordinateTransform);

// 物理长度 → 像素长度
const screenWidth = worldLengthToScreen(physicsWidth, ctx.coordinateTransform);
```

**绘制 primitives**：

```typescript
import { drawRect, drawLine } from '@/renderer/primitives/shapes';
import { drawArrow } from '@/renderer/primitives/arrow';
import { drawTextLabel } from '@/renderer/primitives/text-label';
```

**注意事项**：

- 所有绘制前先用 `worldToScreen` 转换坐标
- 物理坐标 Y 向上，画布 Y 向下，`worldToScreen` 已处理翻转
- `RenderContext.ctx` 是 `CanvasRenderingContext2D`，可直接调用 Canvas 2D API
- 渲染器函数中应 `ctx.ctx.save()` / `ctx.ctx.restore()` 保护状态

## Step 4：实现视角渲染器

**目标**：实现特定视角（如受力、运动、能量）的覆盖层渲染。

**参考文件**：`src/domains/mechanics/viewports/force-viewport.ts`

**API 签名**：

```typescript
import { rendererRegistry } from '@/core/registries/renderer-registry';

type ViewportRenderer = (
  data: ViewportData,                    // 视角数据（从 PhysicsResult 提取）
  entities: Map<EntityId, Entity>,       // 所有实体
  ctx: RenderContext,
) => void;

rendererRegistry.registerViewport('force', forceViewportRenderer);
```

**ViewportData 结构**（discriminated union）：

```typescript
| { type: 'force';    data: { analyses: ForceAnalysis[] } }
| { type: 'motion';   data: { motionStates: MotionState[] } }
| { type: 'energy';   data: { energyStates: EnergyState[] } }
| { type: 'momentum'; data: { momenta: [...] } }
| { type: 'field';    data: { fieldEntities: [...] } }
| { type: 'circuit';  data: { emf?, current?, ... } }
```

**力箭头绘制要点**：

- 力的方向是物理方向（`direction: Vec2`），绘制时注意 Y 轴翻转
- 箭头起点用 `worldToScreen` 转换实体中心
- 箭头终点在屏幕坐标系下计算：`screenTo.y = screenFrom.y - direction.y * length`
- 标注位置：箭头中点偏移 12px，避免压在箭头上

**颜色约定**：

| 力类型 | 颜色 | 色值 |
|--------|------|------|
| gravity | 红色 | `#E53E3E` |
| normal | 蓝色 | `#3182CE` |
| friction | 橙色 | `#DD6B20` |
| tension | 绿色 | `#27AE60` |
| resultant | 绿色虚线 | `#38A169` |

**注意事项**：

- 如果只需已有视角（如 force），无需新增视角渲染器
- 新视角类型需在 `core/types.ts` 的 `ViewportType` 中添加（这是少数需要修改 core 的情况）

## Step 5：编写预设 JSON

**目标**：声明一个完整的物理场景，包含实体、关系、参数和视角配置。

**参考文件**：`src/domains/mechanics/presets/horizontal-block.json`

**PresetData 字段**：

```jsonc
{
  "id": "P01-FM002-inclined-block",     // 预设唯一标识
  "version": "1.0",
  "name": "斜面·从静止下滑",
  "description": "物块从斜面顶端由静止开始下滑",
  "category": "P-01",                    // PRD 模块编号
  "supportedViewports": ["force", "motion"],
  "defaultViewport": "force",
  "displayConfig": {
    "scale": 150,                        // 物理单位 → 像素
    "origin": { "x": 400, "y": 400 }    // 物理原点在画布上的像素位置
  },
  "entities": [                          // 实体定义（用 ref 标识）
    { "ref": "block-A", "type": "block", "properties": {...}, "transform": {...} },
    { "ref": "slope-1", "type": "slope", "properties": {...}, "transform": {...} }
  ],
  "relations": [                         // 关系定义（用 ref 引用实体）
    { "type": "contact", "sourceRef": "block-A", "targetRef": "slope-1", "properties": {...} }
  ],
  "paramGroups": [                       // 参数面板分组
    {
      "key": "block-params", "label": "物体属性",
      "params": [
        { "key": "mass", "label": "质量", "type": "slider", ... ,
          "targetEntityId": "block-A", "targetProperty": "mass" }
      ]
    }
  ],
  "paramValues": { "mass": 2 },          // 参数默认值
  "solveMode": "analytical",
  "duration": 5,                         // 模拟时长（秒），0=静态
  "solverQualifier": { "surface": "inclined" }  // 求解器匹配的额外条件
}
```

**注意事项**：

- `entities` 中的 `ref` 是预设内部引用标识，PresetLoader 会映射为真实 EntityId
- `paramGroups.params[].targetEntityId` 使用 `ref` 值（加载时自动替换）
- `duration: 0` 表示静态场景（无时间演化），Simulator 不会自动播放
- `solverQualifier` 帮助 SolverRegistry 区分同一实体组合下的不同求解器

## 注册入口

每个域的 `index.ts` 统一导入并调用所有注册函数：

```typescript
// src/domains/mechanics/index.ts
import { registerBlockEntity } from './entities/block';
import { registerSlopeEntity } from './entities/slope';
import { registerBlockOnSurfaceSolver } from './solvers/block-on-surface';
import { registerBlockOnSlopeSolver } from './solvers/block-on-slope';
import { registerBlockRenderer } from './renderers/block-renderer';
import { registerSurfaceRenderer } from './renderers/surface-renderer';
import { registerSlopeRenderer } from './renderers/slope-renderer';
import { registerForceViewport } from './viewports/force-viewport';

export function registerMechanicsDomain(): void {
  registerBlockEntity();
  registerSlopeEntity();
  registerBlockOnSurfaceSolver();
  registerBlockOnSlopeSolver();
  registerBlockRenderer();
  registerSurfaceRenderer();
  registerSlopeRenderer();
  registerForceViewport();
}
```

注册函数在 `src/main.tsx` 中应用启动时调用，早于任何预设加载。

## 目录结构约定

```
domains/{域}/
├── entities/       — 实体类型注册（每个实体一个文件）
├── solvers/        — 求解器实现（每个场景一个文件）
├── renderers/      — 实体渲染器（每个实体类型一个文件）

```

## 下一个预设建议：斜面·从静止下滑

与"水平面物块受力"的差异点：

### 1. 新增 slope 实体

- 新建 `entities/slope.ts`
- 属性：`angle`（倾角）、`length`（斜面长度）、`friction`（摩擦因数）
- hitTest：使用 `pointOnLine`，线段从 position 沿 angle 方向延伸 length
- 渲染：一条斜线 + 底部斜线填充

### 2. 新增斜面求解器

- 新建 `solvers/block-on-slope.ts`
- pattern: `{ entityTypes: ['block', 'slope'], relationType: 'contact', qualifier: { surface: 'inclined' } }`
- 物理计算（无摩擦）：
  ```
  θ = slope.angle（弧度）
  G = mg，方向竖直向下
  N = mg·cosθ，方向垂直斜面向上
  合力 = mg·sinθ，方向沿斜面向下
  a = g·sinθ，沿斜面向下
  ```
- 需要正交分解（`ForceAnalysis.decomposition`）：
  - axis1：沿斜面方向
  - axis2：垂直斜面方向
- `solveMode: 'analytical'`，位移公式 `s = ½at²`
- `duration > 0`，有时间演化

### 3. 渲染差异

- slope-renderer：斜线绘制，需根据 angle 旋转
- force-viewport：力箭头沿斜面方向（不再只是水平/竖直），需要用 direction 向量计算
- motion-viewport（可选）：速度箭头沿斜面方向

### 4. 预设 JSON 差异

- `duration` > 0（有运动过程）
- `supportedViewports` 增加 `"motion"`
- `solverQualifier: { surface: "inclined" }`
- `paramGroups` 增加斜面倾角参数

## 常见问题与调试

### Q: 预设加载报错"实体类型未注册"

确认 `registerXxxEntity()` 在 `index.ts` 中被调用，且 `main.tsx` 中 `registerXxxDomain()` 在 `simulator.loadPreset()` 之前执行。

### Q: 求解器匹配不到

检查 `solverRegistry.register()` 的 `pattern` 字段：
- `entityTypes` 是否与预设 JSON 中的实体类型一致
- `relationType` 是否与预设 JSON 中的关系类型一致
- 如果使用了 `qualifier`，预设 JSON 中需要有对应的 `solverQualifier`

### Q: 渲染不出来

1. 确认 `rendererRegistry.registerEntity()` 已调用
2. 检查坐标转换是否正确（`worldToScreen` Y 轴翻转）
3. 确认 `layer` 设置正确（物体用 `'object'`，表面用 `'surface'`）

### Q: 力箭头方向反了

物理方向中 `direction.y = -1` 表示向下。绘制时屏幕坐标的终点需要：
```typescript
screenTo.y = screenFrom.y - direction.y * arrowLength;  // 注意减号（Y轴翻转）
```

### Q: 参数联动没生效

确认预设 JSON 中 `paramGroups.params[]` 的 `targetEntityId`（使用实体的 `ref`）和 `targetProperty`（对应 `entity.properties` 中的字段名）正确。
