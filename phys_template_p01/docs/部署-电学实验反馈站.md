# 电学实验反馈站部署说明

## 目标

本次反馈站只开放 `P-04 电学实验` 相关内容，用于给教师或学生做体验反馈。

当前范围：

- 首页
- 电学实验模板库
- P-04 预设实验
- P-04 专题页

当前不开放：

- 自由搭建
- 力学、热学、P-02 电磁场等其他模块

---

## 推荐方案

不要单独拆第二个仓库，直接基于当前仓库做“双部署”：

- 正式版：继续使用默认构建
- 反馈版：使用 `electric-feedback` 模式构建

这样做的好处：

- 不会复制代码和依赖
- 不会影响主分支后续继续开发
- 反馈结束后可以直接关闭反馈站，成本低

---

## 代码开关

仓库已增加反馈版构建模式：

```bash
pnpm dev:electric-feedback
pnpm build:electric-feedback
```

反馈模式下会自动执行以下限制：

- 只注册电磁域能力
- 只显示 `P-04` 分类
- 屏蔽 `/#builder`
- 非 `P-04` 预设 hash 链接会回退到模板库

对应环境变量：

```bash
VITE_APP_SCOPE=electric-feedback
```

## 本地验证

```bash
pnpm dev:electric-feedback
```

建议至少验证以下入口：

- `/#`
- `/#gallery`
- `/#meter-error`
- `/#measure-emf-compare`
- `/#half-deflection-compare`
- `/#ohmmeter-midpoint-compare`
- `/#meter-reading-trainer`
- `/#voltage-resistance-method`
- `/#current-resistance-method`
- `/#meter-conversion`

再验证一个不应开放的入口：

- `/#builder`

预期结果：

- 首页不显示“自由搭建”
- 模板库只显示 `P-04 电学实验`
- 访问 `/#builder` 不进入自由搭建页面

## Vercel 部署

### 方案 A：新建一个反馈站项目

最推荐。直接在 Vercel 里新建一个项目，仍然指向当前仓库，但使用反馈版构建命令。

建议配置：

- Framework Preset: `Vite`
- Install Command: `pnpm install`
- Build Command: `pnpm build:electric-feedback`
- Output Directory: `dist`
- Root Directory: `.`

优点：

- 反馈站和正式站互不干扰
- 可以单独绑定一个体验域名
- 可以单独看访问数据和错误日志

### 方案 B：同一个项目切换环境变量

不推荐作为线上反馈方案，只适合临时预览。

如果必须复用已有项目，需要把构建命令改成：

```bash
pnpm build
```

并在 Vercel 环境变量里设置：

```bash
VITE_APP_SCOPE=electric-feedback
```

但这会影响默认产物，容易和正式版混淆。

## 推荐访问方式

反馈站首页建议只放一个主入口：

- “进入电学实验模板库”

如果你要直接发给体验用户，也可以附上几个深链：

- `/#gallery`
- `/#meter-error`
- `/#measure-emf-compare`
- `/#meter-conversion`

因为当前使用 hash 路由，静态托管不需要额外 rewrite。

---

## 反馈周期建议

建议先做一轮 3 到 5 天的小范围反馈。

推荐对象：

- 2 到 3 位高中物理老师
- 2 到 3 位教研或内容同学
- 2 到 4 位熟悉电学实验的学生

重点不是覆盖人数，而是尽快确认三件事：

- 用户能否在 30 秒内理解入口和玩法
- 用户能否完成核心实验任务
- 页面中的术语、公式、数值反馈是否符合教材预期

---

## 上线前检查清单

- 反馈版首页没有“自由搭建”入口
- 模板库只显示 `P-04`
- 专题页都能返回模板库
- 至少抽查 3 个模板实验和 3 个专题页
- 移动端打开时没有明显遮挡或布局错位
- 体验文档和反馈收集表已发给测试用户

---

## 建议的发布节奏

1. 先部署一个内部链接给研发和产品自测。
2. 自测通过后再复制一个外部分享链接给体验用户。
3. 收到第一轮反馈后，优先修高频问题，再决定是否开放第二轮。
