# 修改计划：P-13 电磁感应

更新时间：2026-04-08

对照文档：

- `docs/需求文档md/P物理教具产品线 · 产品需求文档.md`

## 1. 目标

对照产品需求和模型数据，把当前 `P-13 电磁感应` 从“已有独立入口和两个基础样例”推进到“主干模型完整、终态分析可讲、界面结构统一、可以按产品口径验收”的状态。

本计划只覆盖 P-13，不包含 P-04 电路搭建器和 P-08 电场与磁场可视化器的继续扩展。

---

## 2. 对照结论

### 2.1 产品需求里的目标

总 PRD 对 P-13 的目标不是单一演示，而是一个完整模块，核心要求包括：

- 6 类模型入口：
  - 线圈切割磁场（动生）
  - 磁通量变化（感生）
  - 单棒模型
  - 双棒模型
  - 竖直导轨
  - 自由组装
- 课堂主功能：
  - 楞次定律可视化
  - 单棒运动终态分析
  - 双棒模型分析
  - 含电容 / 含电源特殊情况
  - 任意时刻的安培力方向分析
- 界面口径：
  - 顶部模型选择
  - 左侧参数区
  - 中央演示区
  - 下方 `v-t / i-t` 图
  - 右下角终态数值区
- 验收重点：
  - 插入 / 拔出时感应电流方向判断正确
  - 无摩擦双棒模型满足共速与动量守恒
  - 含电容终态时 `I = 0`，且 `U电容 = BLv终`

### 2.2 模型数据里的落地粒度

`P13 电磁感应 · 模型数据` 已把 P-13 细化到可实现的编号级：

- `EMI-001` 磁棒-线圈楞次定律
- `EMI-011~013` 单棒模型族
  - 纯电阻
  - 含电源
  - 含电容
- `EMI-021~023` 双棒模型族
  - 无摩擦
  - 有摩擦
  - 含电容
- `EMI-031` 竖直导轨单棒

同时已经明确了：

- 主要参数口径
- 核心公式
- 终态表达式
- 分步分析顺序

这意味着 P-13 的后续工作不该再停留在“补一个 demo”，而应该按编号逐支补齐。

### 2.3 当前代码现状

当前仓库已经完成的部分：

- 已有 `/#p13` 独立入口页。
- 已有 2 个可进入的模型：
  - `P13-BASE-001` 矩形线框穿过匀强磁场
  - `EMI-001` 磁棒-线圈楞次定律
- 已有 P-13 核心计算基座：
  - 磁通量
  - 感应电动势
  - 感应电流
  - 安培力样本
- 已有 `EMI-001` 专用逐步分析页面。

当前明显缺失的部分：

- `EMI-011~013` 未落地
- `EMI-021~023` 未落地
- `EMI-031` 未落地
- 还没有真正的“力-电-运动耦合”终态求解
- 还没有统一的 `v-t / i-t` 图和终态结果区
- 还没有 P-13 的自由组装入口
- 当前 `#p13` 仍更像“模型卡片页”，还不是总 PRD 里的完整产品界面

### 2.5 核心差距

现在 P-13 的主要问题不是“完全没有基础”，而是：

- 已有基础演示，但主干模型覆盖率太低
- 已有方向判断，但缺少终态分析
- 已有入口，但还不是产品化模块界面
- 已有局部物理链路，但还没形成统一教学链路

## 3. 修改原则

### 原则 1：先补主干模型，再做自由组装

P-13 的教学价值首先来自标准题型。单棒、双棒、竖直导轨如果没补齐，自由组装会先把复杂度拉高，而不是提升产品完成度。

### 原则 2：先做真实耦合，再做更多壳层文案

P-13 和 P-08 不同，关键难点不在“有没有场景卡”，而在“安培力是否反作用到速度，终态是否真的算出来”。

### 原则 3：统一界面结构，不让每个模型各写一套页面

单棒、双棒、竖直导轨都需要：

- 参数区
- 动画区
- 小型电路拓扑区
- `v-t / i-t` 图
- 终态数值区
- `分析受力` 分步入口

这部分应抽成 P-13 通用壳层，而不是每个页面重复拼接。

### 原则 4：先按教材理想模型收口

首轮实现统一采用高中物理常见理想化口径：

- 忽略自感和辐射
- 忽略导轨接触电阻的高阶变化
- 优先按题目给定的纯电阻 / 理想电容 / 理想电源处理

先保证课堂公式和动画口径一致，再决定是否扩展更复杂的真实效应。

## 4. 修改范围

### 本轮纳入

- P-13 产品入口与模型分组重构
- 单棒模型族 `EMI-011~013`
- 双棒模型族 `EMI-021~023`
- 竖直导轨 `EMI-031`
- 通用 `分析受力` 分步能力
- 统一的 `v-t / i-t` 图与终态数值区
- P-13 验收清单与体验文档

### 本轮后置

- 完整的任意拓扑自由组装
- 自感、非线性阻尼、复杂接触电阻等扩展物理
- 与 P-04 电路 builder 的深度合并

说明：

- 总 PRD 提到“磁通量变化（感生）”独立模型。
- 当前更细的 P13 模型数据没有单独给出这一支编号，而是把楞次定律教学集中在 `EMI-001`。
- 因此本轮建议先把 `EMI-001` 作为感生类教学入口，不额外拆一个新模型，除非产品明确要求“动生 / 感生”必须各有独立按钮。

## 5. 分阶段计划

### 当前基线：Phase 1 ~ Phase 2 已完成

已完成内容：

- P-13 独立入口
- `P13-BASE-001` 基础线框样例
- P-13 核心感应计算层
- `EMI-001` 磁棒-线圈楞次定律逐步分析

当前基线可以保留，不需要返工重做。

### Phase 3：统一产品壳层 + `EMI-011` 单棒基础

#### 目标

把 P-13 从“两张卡片 + 两个样例”推进到“有统一教学骨架的模块”，并补上最关键的单棒基础模型。

#### 改动项

- 重构 `/#p13` 首页：
  - 按产品口径分成 6 类模型位
  - 当前可用模型和后续占位模型分组展示
- 抽 P-13 通用页面骨架：
  - 左侧参数区
  - 中央演示区
  - 下方图表区
  - 右下角终态结果区
- 落地 `EMI-011 单棒基础（纯电阻电路）`
- 新增单棒动力学耦合：
  - `ε = BLv`
  - `i = ε / (R + R棒)`
  - `F安 = BIL`
  - 安培力真实反作用到速度
- 补 `分析受力` 通用流程：
  - 速度方向
  - EMF 方向
  - 电流方向
  - 安培力方向
- 增加 `v-t / i-t` 图联动

#### 预期输出

- `EMI-011` 可独立进入和验收
- 单棒基础模型具备真实减速过程
- P-13 页面结构首次与总 PRD 对齐

#### 验收

- 无摩擦时，速度衰减趋势与文档公式一致
- 调节 `B / L / R / m / v0` 后，动画、图表和结果区同步变化
- 点击 `分析受力` 后，四步方向判断稳定正确

### Phase 4：补齐 `EMI-012 / EMI-013` 单棒特殊情况

#### 目标

在 `EMI-011` 的物理和界面基线上，把单棒模型族补完整。

#### 改动项

- 扩展 `EMI-012 单棒 + 含电源`
- 扩展 `EMI-013 单棒 + 含电容`
- 为单棒模型加入小型电路拓扑视图
- 统一终态口径输出：
  - `v终`
  - `I终`
  - `U电容`
- 在结果区补“终态解释文案”：
  - 为什么匀速
  - 为什么静止
  - 为什么 `I -> 0`

#### 验收

- 含电源模型中，终态速度能按 `v终 = ε0 / (BL)` 给出
- 含电容模型中，电流最终趋于 0
- 含电容模型中，结果区给出 `U电容 = BLv终`

### Phase 5：补齐双棒模型和竖直导轨

#### 目标

完成 P-13 的主干题型覆盖，使其达到“能支撑高中电磁感应主流课堂”的程度。

#### 改动项

- 落地 `EMI-021 双棒基础（无摩擦）`
- 落地 `EMI-022 双棒（有摩擦）`
- 落地 `EMI-023 双棒 + 含电容`
- 落地 `EMI-031 竖直导轨`
- 双棒场景增加双对象速度曲线与共速判定
- 竖直导轨场景增加重力 / 安培力平衡提示
- 结果区支持：
  - `v1 / v2 / v终`
  - 动量守恒校验
  - 终态电流

#### 验收

- 无摩擦双棒模型最终共速
- 共速值与 `v终 = (m1v1 + m2v2) / (m1 + m2)` 一致
- 竖直导轨终态速度与 `v终 = mg(R+R棒) / (B²L²)` 一致
- 竖直导轨终态电流与 `i终 = mg / (BL)` 一致

### Phase 6：自由组装 MVP + 文档验收收口

#### 目标

在主干模型稳定后，补一个受控范围内的 P-13 builder MVP，并把模块做成“可发布、可验收、可说明”的状态。

#### 改动项

- 新增 P-13 builder MVP：
  - 磁场区域
  - 导轨
  - 单棒 / 双棒
  - 电阻 / 电容 / 电源
- builder 范围先限制在“模板化半自由组装”：
  - 不追求任意拓扑
  - 优先支持单棒和双棒常见课堂结构
- 增加 P-13 文档闭环：
  - 验收清单
  - 教师体验文档
  - 阶段自查文档

#### 验收

- 老师能从 `/#p13` 进入标准模型，也能进入 builder MVP
- 至少可以搭出“纯电阻单棒”和“含电源单棒”两类结构
- P-13 有单独的验收清单，不再混在总状态文档里描述

## 7. 风险与前置确认

### 需要产品确认

- “磁通量变化（感生）”是否必须做成独立模型入口
- P-13 的自由组装是否要求首轮就支持任意拓扑
- 图表区是否固定为底部双图，还是允许切换页签

### 技术风险

- 单棒和双棒的“安培力反作用到速度”会引入新的数值稳定性问题
- 含电容和含电源分支如果直接写死在页面里，后续会很难维护，应该尽早抽成 P-13 专用计算层
- builder 一旦做成完全自由拓扑，复杂度会迅速接近 P-04，需要提前限制范围

---

## 8. 建议先做的最小闭环

如果本轮只能做一条最小主线，建议顺序是：

1. 做 P-13 通用页面骨架
2. 做 `EMI-011 单棒基础` 的真实动力学耦合
3. 补 `分析受力`、`v-t / i-t` 图和终态结果区
4. 跑一轮单棒模型验收

这样改动量可控，但对产品完成度提升最大，也能给后续 `EMI-012~013`、`EMI-021~023`、`EMI-031` 提供统一底座。
