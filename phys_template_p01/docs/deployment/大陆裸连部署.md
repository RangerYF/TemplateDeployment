# 大陆裸连部署

这套改动把生产访问从 `*.vercel.app` 拆开，改成：

- `Vercel` 继续做预览和临时链接
- 自定义域名承接正式流量
- GitHub Actions 构建 `dist/`
- 通过 `SSH` 发布到你自己的服务器
- `Nginx` 直接托管静态文件，必要时前面再挂 CDN

## 新增文件

- `.github/workflows/build-static-artifact.yml`
- `.github/workflows/deploy-mainland-server.yml`
- `ops/deploy/mainland/install_release.sh`
- `ops/deploy/mainland/nginx-site.conf`

## 适用场景

- 你要保留 Vercel 作为预览环境
- 正式访问不想继续暴露 `*.vercel.app`
- 当前项目是静态前端，构建产物在 `dist/`
- 你有一台可通过 `SSH` 登录的服务器

## 前提条件

- 一个自定义域名
- 一台服务器
- 如果服务器或 CDN 使用中国大陆节点，需要先完成 ICP 备案
- GitHub 仓库可启用 Actions

## GitHub Secrets

在仓库 `Settings -> Secrets and variables -> Actions` 中新增：

- `DEPLOY_HOST`: 服务器 IP 或域名
- `DEPLOY_USER`: SSH 登录用户名
- `DEPLOY_SSH_KEY`: 私钥内容，建议单独生成部署用密钥
- `DEPLOY_PORT`: 可选，默认 `22`
- `DEPLOY_ROOT`: 可选，默认 `/var/www/phys-template`
- `DEPLOY_KNOWN_HOSTS`: 可选，建议填 `ssh-keyscan -H your-host` 的输出

## 服务器目录

部署完成后默认目录结构如下：

```text
/var/www/phys-template/
├── current -> /var/www/phys-template/releases/20260407010101-abcdef0
└── releases/
    ├── 20260407010101-abcdef0
    ├── 20260407121010-1234567
    └── ...
```

工作流会保留最近 5 个发布版本。

## Nginx 配置

把 `ops/deploy/mainland/nginx-site.conf` 复制到服务器，按实际域名修改：

- `server_name`
- `root`

这份配置已经包含了：

- SPA 路由回退到 `index.html`
- `assets/` 长缓存
- `index.html` 不缓存

## DNS 与访问路径

推荐做法：

- `preview.your-domain.com` 继续给 Vercel
- `www.your-domain.com` 指向你的生产服务器或 CDN
- 如果前面接 CDN，源站就指向这台服务器

## 使用方式

1. 先在服务器上装好 `nginx`
2. 让 `www.your-domain.com` 先能解析到服务器
3. 配置好上面的 GitHub Secrets
4. 在 GitHub Actions 里手动运行 `build-static-artifact`
5. 确认 `dist` 产物正常
6. 再手动运行 `deploy-mainland-server`
7. 在 `app_scope` 中选择当前要发的版本

## app_scope 说明

工作流复用仓库里现有的 `pnpm build:vercel` 逻辑，支持：

- `electric-feedback`
- `builder-feedback`
- `builder-free-feedback`

默认值是 `electric-feedback`。

## 建议

- 如果目标是真正稳定的中国大陆访问，优先用大陆服务器或大陆 CDN，并完成备案
- 如果暂时不备案，可先用香港服务器或海外 CDN，这样通常能裸连，但稳定性和时延不如大陆节点
- 正式对外只发自定义域名，不再发 `*.vercel.app`
