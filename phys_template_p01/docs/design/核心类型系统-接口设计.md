# 核心类型系统 · TypeScript 接口设计

> 阶段2最终产出。所有类型均定义在 `core/types.ts`，各域在 `domains/` 下扩展具体类型。

---

## 模块 1：基础向量与几何类型

```typescript
// ─── 二维向量 ───
interface Vec2 {
  x: number;
  y: number;
}

// ─── 矩形区域（用于场区域、命中检测、边界框） ───
interface Rect {
  x: number;      // 左上角 x
  y: number;      // 左上角 y
  width: number;
  height: number;
}

// ─── 空间变换（实体在场景中的位姿） ───
interface Transform {
  position: Vec2;  // 物理坐标系下的位置
  rotation: number; // 弧度，逆时针为正
}
```

**设计决策**：
- `Vec2` 是可变对象（非 class），在 60fps 循环中避免 GC 压力
- `Rect` 轴对齐矩形，足够覆盖场区域和命中检测；旋转形状的命中检测在渲染层用 Transform 反变换处理
- `rotation` 用弧度而非角度，与 Canvas API 和三角函数一致

## 模块 2：Entity 基础接口

```typescript
// ─── 实体标识 ───
type EntityId = string;   // 格式建议：`${type}-${nanoid(6)}`，如 "block-a3f2k1"
type EntityType = string; // 如 'block' | 'slope' | 'bfield'，各域自行扩展

// ─── 实体类别 ───
type EntityCategory =
  | 'object'      // 物体：物块、小球、小车、带电粒子、导体棒
  | 'surface'     // 表面：水平面、斜面、弧形轨道、导轨
  | 'connector'   // 连接件：弹簧、轻绳、轻杆、滑轮
  | 'field'       // 场：重力场、匀强电场、匀强磁场
  | 'constraint'  // 约束源：铰接点、固定点、滑动轨道
  | 'instrument'; // 观测工具：速度传感器、能量计、电压表

// ─── 实体基础接口 ───
interface Entity<TProps extends Record<string, unknown> = Record<string, unknown>> {
  readonly id: EntityId;
  readonly type: EntityType;
  readonly category: EntityCategory;
  transform: Transform;
  properties: TProps;
  label?: string;           // 场景中显示的名称，如"物体A"
}
```

**设计决策**：
- **泛型 `TProps`** 兼顾灵活性和类型安全：`core/types.ts` 用宽松默认值 `Record<string, unknown>`，各域可窄化为具体类型（如 `Entity<BlockProps>`），在域内获得完整类型提示
- `id` 和 `type` 为 `readonly`，创建后不可变
- `category` 冗余但有用：注册机制和UI可按类别筛选，不需要每个域维护类别映射表
- `label` 可选，用于 UI 显示和力标注（"物体A的重力"）
- `transform` 可变：Simulator 每帧更新位置

**域扩展示例**（`domains/mechanics/entities/block.ts`）：
```typescript
interface BlockProps {
  mass: number;         // kg
  width: number;        // m（渲染用）
  height: number;       // m
  charge?: number;      // C（跨域预设时可带电荷）
  initialVelocity?: Vec2;
}

type BlockEntity = Entity<BlockProps>;
```

## 模块 3：Relation 接口

```typescript
// ─── 关系标识 ───
type RelationId = string;
type RelationType = string; // 如 'contact' | 'connection' | 'field-effect' | 'containment'

// ─── 关系基础接口 ───
interface Relation<TProps extends Record<string, unknown> = Record<string, unknown>> {
  readonly id: RelationId;
  readonly type: RelationType;
  readonly sourceEntityId: EntityId;  // 关系的主动方（物块→斜面中的物块）
  readonly targetEntityId: EntityId;  // 关系的被动方（物块→斜面中的斜面）
  properties: TProps;
}
```

**设计决策**：
- **有方向性**：`source → target`。语义约定：
  - 接触关系：`source` = 物体，`target` = 表面（"物块在斜面上"）
  - 连接关系：`source` = 物体，`target` = 连接件（"物块挂在弹簧上"）
  - 场作用：`source` = 场，`target` = 受作用物体（"磁场作用于带电粒子"）
  - 包含关系：`source` = 容器，`target` = 被包含物体（"小球在小车内"）
- 命名用 `sourceEntityId/targetEntityId` 而非 `from/to`，语义更明确
- 泛型 `TProps` 与 Entity 一致，域内可窄化

**域扩展示例**（接触关系）：
```typescript
interface ContactProps {
  friction: number;      // 动摩擦因数 μ
  normalDirection: Vec2; // 法线方向（指向物体一侧）
}

type ContactRelation = Relation<ContactProps>;
```

**求解器匹配查询**：
```typescript
// 求解器通过 (实体类型组合 + 关系类型) 匹配，Relation 的 type 字段直接用于查询
// 例：匹配 "block + slope + contact" → slope-contact-solver
// 具体匹配 API 在阶段3设计
```

## 模块 4：参数 Schema 类型

```typescript
// ─── 参数控件类型 ───
type ParamControlType = 'slider' | 'input' | 'toggle' | 'select';

// ─── 单个参数 Schema（联合类型，按控件类型区分） ───

interface ParamSchemaBase {
  key: string;         // 参数的程序标识，如 "mass"、"angle"
  label: string;       // 面板显示名称，如 "质量"
  group?: string;      // 所属分组名，如 "物体属性"、"表面属性"
  targetEntityId?: EntityId;  // 该参数影响哪个实体（参数变更时定位更新目标）
  targetProperty?: string;    // 映射到实体 properties 中的字段名
}

interface SliderParamSchema extends ParamSchemaBase {
  type: 'slider';
  min: number;
  max: number;
  step: number;
  default: number;
  unit: string;        // 如 "kg"、"m/s"、"°"
  precision?: number;  // 显示小数位数，默认根据 step 推断
}

interface InputParamSchema extends ParamSchemaBase {
  type: 'input';
  min?: number;        // 可选范围约束
  max?: number;
  default: number;
  unit: string;
  precision?: number;
}

interface ToggleParamSchema extends ParamSchemaBase {
  type: 'toggle';
  default: boolean;
  labelOn?: string;    // 开启时显示文本，如 "粗糙"
  labelOff?: string;   // 关闭时显示文本，如 "光滑"
}

interface SelectParamSchema extends ParamSchemaBase {
  type: 'select';
  options: Array<{ value: string; label: string }>;
  default: string;
}

type ParamSchema = SliderParamSchema | InputParamSchema | ToggleParamSchema | SelectParamSchema;

// ─── 参数分组（驱动面板分区渲染） ───
interface ParamGroup {
  key: string;          // 分组标识
  label: string;        // 分组标题，如 "物体属性"
  params: ParamSchema[];
}

// ─── 参数值运行时容器 ───
type ParamValues = Record<string, number | boolean | string>;

// ParamLinkage 已在阶段7发散审查中删除（Phase 1 用命名约定 `-toggle`/`-coeff` 实现联动）
  formula: string;      // 计算公式标识（如 "friction_for_uniform_velocity"）
}
```

**设计决策**：
- **联合类型**而非单一接口 + 可选字段：`SliderParamSchema | InputParamSchema | ...` 让 TS 编译器按 `type` 字段自动窄化，避免运行时 `if (schema.type === 'slider')` 时类型丢失
- `targetEntityId + targetProperty`：参数面板改参数后，知道要更新哪个实体的哪个属性，再触发求解器重算。这比"参数变更后全局重算"更精确
- `ParamLinkage` 已在阶段7发散审查中删除，Phase 1 用命名约定（`-toggle`/`-coeff` 后缀）实现联动
- `unit` 是必填字段（slider/input），物理量必须有单位

## 模块 5：PhysicsResult 接口

```typescript
// ─── 力的类型标识（用于颜色映射） ───
type ForceType =
  | 'gravity'       // 重力 G — 深红 #C0392B
  | 'normal'        // 支持力 N — 蓝色 #2980B9
  | 'friction'      // 摩擦力 f — 橙色 #E67E22
  | 'tension'       // 弹力/绳张力 T — 绿色 #27AE60
  | 'electric'      // 电场力 qE — 金色 #F39C12
  | 'lorentz'       // 洛伦兹力 qvB — 品红 #9B59B6
  | 'ampere'        // 安培力 BIL — 品红 #9B59B6
  | 'resultant'     // 合力 F合 — 紫色 #8E44AD
  | 'spring'        // 弹簧弹力 — 绿色 #27AE60
  | 'custom';       // 域扩展的自定义力

// ─── 单个力 ───
interface Force {
  type: ForceType;
  label: string;            // 标注文本，如 "G=19.6N"
  magnitude: number;        // 大小（N）
  direction: Vec2;          // 单位方向向量
  applicationPoint?: Vec2;  // 力的作用点（物理坐标），默认实体中心
}

// ─── 正交分解结果 ───
interface OrthogonalDecomposition {
  axis1: Vec2;          // 第一个轴方向（如沿斜面方向）
  axis2: Vec2;          // 第二个轴方向（如垂直斜面方向）
  components: Array<{
    force: Force;        // 原始力
    component1: number;  // 在 axis1 方向的分量大小（带符号）
    component2: number;  // 在 axis2 方向的分量大小（带符号）
  }>;
}

// ─── 单个实体的受力分析 ───
interface ForceAnalysis {
  entityId: EntityId;
  forces: Force[];                          // 所有独立力
  resultant: Force;                         // 合力
  decomposition?: OrthogonalDecomposition;  // 正交分解（仅斜面等非正交场景需要）
}

// ─── 单个实体的运动状态 ───
interface MotionState {
  entityId: EntityId;
  position: Vec2;
  velocity: Vec2;
  acceleration: Vec2;
  angularVelocity?: number;   // rad/s（圆周运动等）
  trajectory?: Vec2[];        // 历史轨迹点（运动视角绘制轨迹线）
}

// ─── 单个实体的能量状态 ───
interface EnergyState {
  entityId: EntityId;
  kineticEnergy: number;           // 动能 Ek
  potentialEnergies: Array<{       // 各种势能（可叠加）
    type: string;                  // 如 'gravitational' | 'elastic' | 'electric'
    label: string;                 // 如 "重力势能"
    value: number;
  }>;
  totalEnergy: number;             // 总机械能
  workDone?: Array<{               // 各力做功
    forceType: ForceType;
    label: string;
    value: number;
  }>;
}

// ─── 求解器完整输出（某一时刻） ───
interface PhysicsResult {
  time: number;                              // 当前时刻 t（秒）
  forceAnalyses: Map<EntityId, ForceAnalysis>; // 各实体受力分析
  motionStates: Map<EntityId, MotionState>;    // 各实体运动状态
  energyStates?: Map<EntityId, EnergyState>;   // 各实体能量状态（仅能量视角需要时计算）
}
```

**设计决策**：
- **`ForceType` 是开放联合类型**：用 `| 'custom'` 兜底，各域可在自己的代码中定义更多力类型并在视觉映射表中注册颜色。但核心力类型统一定义在 core，确保颜色语言一致
- **PhysicsResult 提供完整数据，不按视角拆分**：由视角层自行从中提取所需数据。这样求解器无需知道当前视角，职责单一
- **`Map<EntityId, ...>`** 而非数组：求解器和渲染器频繁按 entityId 查找，Map O(1) 查找优于数组遍历
- **`trajectory` 放在 MotionState 中**：运动视角需要轨迹线绘制，轨迹是随时间累积的，由 Simulator 维护并追加到 MotionState 中
- **`energyStates` 可选**：并非所有场景都需要能量分析（如纯受力分析场景），按需计算
- **解析解 vs 数值积分的统一**：两者都输出同一个 `PhysicsResult` 结构。解析解求解器可以一次算出多个时刻的结果（返回 `PhysicsResult[]`），数值积分求解器逐帧输出单个 `PhysicsResult`。Simulator 层统一管理

## 模块 6：SimulationState 接口

```typescript
// ─── 模拟状态枚举 ───
type SimulationStatus = 'idle' | 'running' | 'paused' | 'finished';

// ─── 求解模式 ───
type SolveMode = 'analytical' | 'numerical';

// ─── 积分器类型 ───
type IntegratorType = 'semi-implicit-euler' | 'velocity-verlet' | 'rk4';

// ─── 时间轴状态 ───
interface TimelineState {
  currentTime: number;   // 当前时刻（秒）
  duration: number;      // 模拟总时长（秒）
  playbackRate: number;  // 播放速率（1.0=正常，0.5=半速，2.0=倍速）
  dt: number;            // 时间步长（秒），数值积分用，如 1/60
}

// ─── 场景定义（静态数据，预设加载后不变） ───
interface SceneDefinition {
  entities: Map<EntityId, Entity>;
  relations: Relation[];
  paramGroups: ParamGroup[];
  paramValues: ParamValues;
}

// ─── 模拟全局状态 ───
interface SimulationState {
  status: SimulationStatus;
  solveMode: SolveMode;
  integrator: IntegratorType;       // 数值模式下使用的积分器
  timeline: TimelineState;
  scene: SceneDefinition;           // 场景静态定义
  currentResult: PhysicsResult | null;  // 当前帧的求解结果
  resultHistory: PhysicsResult[];   // 历史结果（时间轴拖拽回放用）
}
```

**设计决策**：
- **`SceneDefinition` 与 `PhysicsResult` 分离**：场景定义是静态输入（实体、关系、参数），PhysicsResult 是动态输出（力、速度、位置）。清晰的输入/输出边界
- **`resultHistory`**：时间轴拖拽需要跳到任意时刻，存储历史结果。解析解场景可以不存历史（随时重算），数值积分场景需要存储。内存控制由 Simulator 层管理（如仅保留最近 N 帧或按时间间隔采样）
- **`solveMode + integrator`** 组合确定求解策略：预设 JSON 声明使用哪种模式和积分器
- **`entities` 用 `Map`**：频繁按 ID 查找实体，Map 比数组高效
- **`relations` 用数组**：关系数量少且需要遍历匹配，数组足够
- **`paramValues` 平铺在顶层**：参数变更是最频繁的用户操作，平铺访问快

## 模块 7：视角层类型

```typescript
// ─── 视角类型 ───
type ViewportType = 'force' | 'motion' | 'energy' | 'momentum' | 'field' | 'circuit';

// ─── 信息密度 ───
type InfoDensity = 'compact' | 'standard' | 'detailed';
// compact  = 只有箭头/图形
// standard = 箭头 + 数值标注
// detailed = 箭头 + 数值 + 公式

// ─── 视角层状态 ───
interface ViewportState {
  primary: ViewportType;           // 当前主视角
  overlays: ViewportType[];        // 叠加层列表（透明度 0.3 渲染）
  density: InfoDensity;
}

// ─── 视角层数据（各视角从 PhysicsResult 提取的数据切片） ───

// 受力视角数据
interface ForceViewportData {
  analyses: ForceAnalysis[];       // 各实体的受力分析
}

// 运动视角数据
interface MotionViewportData {
  motionStates: MotionState[];     // 各实体的运动状态（含轨迹）
  // v-t / s-t 图表数据由渲染器从 motionStates + resultHistory 中提取
}

// 能量视角数据
interface EnergyViewportData {
  energyStates: EnergyState[];     // 各实体的能量状态
  systemTotalEnergy?: number;      // 系统总能量（守恒系统为常数）
}

// 动量视角数据
interface MomentumViewportData {
  momenta: Array<{
    entityId: EntityId;
    momentum: Vec2;                // p = mv
    label: string;
  }>;
  systemTotalMomentum?: Vec2;      // 系统总动量
}

// 场视角数据
interface FieldViewportData {
  fieldEntities: Array<{
    entityId: EntityId;
    fieldType: string;             // 如 'electric' | 'magnetic' | 'gravitational'
    region: Rect;                  // 场的作用区域
    direction: Vec2;               // 场的方向
    magnitude: number;             // 场强大小
  }>;
}

// 电路视角数据
interface CircuitViewportData {
  emf?: number;                    // 感应电动势
  current?: number;                // 电流大小
  currentDirection?: Vec2;         // 电流方向
  voltages?: Array<{
    label: string;
    value: number;
  }>;
}

// 视角数据联合类型
type ViewportData =
  | { type: 'force'; data: ForceViewportData }
  | { type: 'motion'; data: MotionViewportData }
  | { type: 'energy'; data: EnergyViewportData }
  | { type: 'momentum'; data: MomentumViewportData }
  | { type: 'field'; data: FieldViewportData }
  | { type: 'circuit'; data: CircuitViewportData };
```

**设计决策**：
- **Tagged union（`type` + `data`）**：视角数据用 discriminated union，TS 编译器可以按 `type` 自动窄化到具体数据类型
- **视角数据从 PhysicsResult 提取**：每种视角的渲染器知道如何从 `PhysicsResult` 中提取自己需要的 `ViewportData`，这个提取逻辑在各域的 `viewports/` 中实现
- **动量和场视角数据独立定义**：虽然动量可以从 `MotionState` 计算 `p=mv`，但视角层需要的是已组装好的展示数据，提取逻辑不该在渲染器中

## 模块 8：渲染相关类型

```typescript
// ─── 坐标变换参数 ───
interface CoordinateTransform {
  scale: number;            // 物理单位到像素的缩放比，如 100 表示 1m=100px
  origin: Vec2;             // 物理坐标原点在画布上的像素位置
  // 物理坐标 → 画布像素：px = (phys - offset) * scale + origin
  // 画布像素 → 物理坐标：phys = (px - origin) / scale + offset
}

// ─── 渲染上下文 ───
interface RenderContext {
  ctx: CanvasRenderingContext2D;
  canvas: HTMLCanvasElement;
  coordinateTransform: CoordinateTransform;
  viewport: ViewportState;
  selectedEntityId: EntityId | null;  // 当前选中的实体（高亮渲染）
  dt: number;                         // 帧间隔，用于动画插值
}

// ─── 命中检测结果 ───
interface HitTestResult {
  entityId: EntityId;
  entityType: EntityType;
  hitPoint: Vec2;             // 命中位置（物理坐标）
  distance: number;           // 命中点到实体中心/边界的距离（用于排序优先级）
}

// ─── 箭头样式 ───
type ArrowHeadStyle = 'solid' | 'hollow';  // 实心三角 / 空心三角
type LineStyle = 'solid' | 'dashed';       // 实线 / 虚线

// ─── 物理量视觉样式 ───
interface VisualStyle {
  color: string;              // 十六进制颜色，如 "#C0392B"
  lineWidth: number;          // 线宽（像素）
  lineStyle: LineStyle;
  arrowHead: ArrowHeadStyle;
  opacity: number;            // 0-1，叠加层用 0.3
}

// ─── 力的视觉语言映射表 ───
// 预定义的力类型 → 视觉样式映射（来自产品方案 3.1）
type ForceVisualMap = Record<ForceType, VisualStyle>;

// ─── 物理量视觉语言映射表（扩展到非力的物理量） ───
interface PhysicsVisualMap {
  forces: ForceVisualMap;
  velocity: VisualStyle;       // 速度箭头：天蓝 #3498DB，实心三角头
  acceleration: VisualStyle;   // 加速度箭头：红色 #E74C3C，空心三角头
  momentum: VisualStyle;       // 动量箭头：青色 #1ABC9C，粗实线
}
```

**设计决策**：
- **`CoordinateTransform`** 最简设计：只需 scale + origin 就能完成物理坐标 ↔ 画布像素的双向转换。不引入矩阵变换，对本项目场景足够
- **`RenderContext`** 是渲染器函数的统一入参：每种实体的渲染器签名为 `(entity: Entity, result: PhysicsResult, ctx: RenderContext) => void`
- **`HitTestResult`** 包含 `distance`：多个实体重叠时，按距离排序选择最近的
- **`ForceVisualMap`** 类型化映射表：确保每种力类型都有对应的视觉样式，遗漏时 TS 编译报错

## 场景验证

### 验证1：斜面物块受力

场景：2kg 物块在 30° 斜面上，μ=0.3，从静止下滑。

```typescript
// ─── 实体定义 ───

const block: Entity<BlockProps> = {
  id: 'block-001',
  type: 'block',
  category: 'object',
  transform: { position: { x: 2, y: 1.5 }, rotation: 0 },
  properties: { mass: 2, width: 0.5, height: 0.5 },
  label: '物块',
};

const slope: Entity<SlopeProps> = {
  id: 'slope-001',
  type: 'slope',
  category: 'surface',
  transform: { position: { x: 0, y: 0 }, rotation: 0 },
  properties: { angle: Math.PI / 6, length: 4, baseWidth: 3.46, height: 2 },
  label: '斜面',
```

## 设计总结

### 核心 `types.ts` 导出清单

| 模块 | 导出类型 | 数量 |
|------|---------|------|
| 基础几何 | `Vec2`, `Rect`, `Transform` | 3 |
| 实体 | `EntityId`, `EntityType`, `EntityCategory`, `Entity` | 4 |
| 关系 | `RelationId`, `RelationType`, `Relation` | 3 |
| 参数Schema | `ParamControlType`, `ParamSchemaBase`, `SliderParamSchema`, `InputParamSchema`, `ToggleParamSchema`, `SelectParamSchema`, `ParamSchema`, `ParamGroup`, `ParamValues` | 9 |
| 物理结果 | `ForceType`, `Force`, `OrthogonalDecomposition`, `ForceAnalysis`, `MotionState`, `EnergyState`, `PhysicsResult` | 7 |
| 模拟状态 | `SimulationStatus`, `SolveMode`, `IntegratorType`, `TimelineState`, `SceneDefinition`, `SimulationState` | 6 |
| 视角层 | `ViewportType`, `InfoDensity`, `ViewportState`, `ForceViewportData`, `MotionViewportData`, `EnergyViewportData`, `MomentumViewportData`, `FieldViewportData`, `CircuitViewportData`, `ViewportData` | 10 |
| 渲染 | `CoordinateTransform`, `RenderContext`, `HitTestResult`, `ArrowHeadStyle`, `LineStyle`, `VisualStyle`, `ForceVisualMap`, `PhysicsVisualMap` | 8 |
| **合计** | | **51** |

### 关键设计原则总结

1. **泛型 + 域扩展**：`Entity<T>` 和 `Relation<T>` 用泛型保证 core 层灵活、域层类型安全
2. **Tagged Union**：`ParamSchema`、`ViewportData` 用 discriminated union，TS 编译器自动窄化
3. **Map 优于 Array**：PhysicsResult 中按 EntityId 索引的数据用 Map，O(1) 查找
4. **输入/输出分离**：`SceneDefinition`（输入）与 `PhysicsResult`（输出）职责清晰
5. **可选字段节制**：只在确实"有些场景不需要"时才用可选（如 `decomposition?`、`energyStates?`），避免运行时到处做 null check
