## 1. SolverPattern 匹配模式设计

### 1.1 选型决策

**选择方案B：结构化对象匹配**

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| A. 字符串 `"block+slope+contact"` | 简洁 | 无类型安全，易拼写错误，无法表达可选条件 | 淘汰 |
| B. 结构化对象 | 类型安全，IDE补全，声明式可检索 | 稍繁琐 | ✅ 选用 |
| C. 谓词函数 `(scene) => boolean` | 最灵活 | 丢失声明式能力，无法做注册表检索/展示 | 淘汰 |

### 1.2 SolverPattern 类型定义

```typescript
/**
 * 求解器匹配模式 —— 声明该求解器能处理什么样的实体+关系组合
 *
 * 匹配规则：场景中必须同时存在 pattern 声明的所有 entityTypes，
 * 且存在至少一个 relationType 指定类型的关系连接这些实体。
 */
interface SolverPattern {
  /** 需要的实体类型集合（场景中必须全部存在） */
  entityTypes: EntityType[];

  /** 需要的关系类型（场景中必须存在该类型的关系连接上述实体） */
  relationType: RelationType;

  /**
   * 可选：额外判定条件。用于区分同一 entityTypes+relationType 组合下的不同物理场景。
   * 例：同为 block+slope+contact，但需要区分"下滑"和"上滑"时，用 qualifier 区分。
   * 设计为简单键值对而非谓词函数，保持声明式可序列化。
   */
  qualifier?: Record<string, string>;
}
```

### 1.3 匹配算法

```typescript
function matchSolver(
  pattern: SolverPattern,
  scene: SceneDefinition
): boolean {
  // 1. 检查 entityTypes：场景中是否包含所有 pattern 要求的实体类型
  const sceneEntityTypes = new Set(
    Array.from(scene.entities.values()).map(e => e.type)
  );
  const allTypesPresent = pattern.entityTypes.every(t => sceneEntityTypes.has(t));
  if (!allTypesPresent) return false;

  // 2. 检查 relationType：场景中是否存在该类型的关系
  const hasRelation = scene.relations.some(r => r.type === pattern.relationType);
  if (!hasRelation) return false;

  // 3. 检查 qualifier（如果有）：由调用者自定义匹配逻辑
  //    qualifier 的匹配策略：预设 JSON 中声明 solverQualifier，与 pattern.qualifier 做全键匹配
  if (pattern.qualifier) {
    // qualifier 匹配逻辑在 PresetData 中通过 solverQualifier 字段提供
    // 此处由 SolverRegistry.match() 内部处理
  }

  return true;
}
```

## 2. SolverFunction 统一签名设计

### 2.1 设计决策

**核心问题**：解析解和数值积分的求解器签名不同（解析解给定 t 直接算，数值积分基于前一帧迭代），如何统一？

**选择：在 `SolverFunction` 层面统一签名，由 `SolveMode` 区分行为**

```typescript
/**
 * 统一求解器签名
 *
 * @param scene    - 场景静态定义（实体、关系、参数）
 * @param time     - 当前时刻 t（秒）
 * @param dt       - 时间步长（秒）。解析解模式下可忽略此参数
 * @param prevResult - 前一帧的求解结果。解析解模式下为 null
 * @returns 当前时刻的完整求解结果
 */
type SolverFunction = (
  scene: SceneDefinition,
  time: number,
  dt: number,
  prevResult: PhysicsResult | null,
) => PhysicsResult;
```

**统一签名的运作方式**：

| 求解模式 | `time` 的使用 | `dt` 的使用 | `prevResult` 的使用 |
|---------|--------------|------------|-------------------|
| 解析解 `analytical` | 直接代入公式计算 t 时刻的状态 | 忽略 | 忽略（每次独立计算） |
| 数值积分 `numerical` | 作为当前时间戳记录 | 积分步长 | 基于前一帧状态迭代 |

**设计理由**：
- Simulator 调用求解器时无需做 `if (mode === 'analytical')` 分支，统一传入四个参数
- 解析解求解器内部只用 `scene` 和 `time`，忽略 `dt` 和 `prevResult`
- 数值积分求解器内部使用全部四个参数
- 签名简洁，不需要函数重载或联合类型

### 2.2 解析解预计算支持

解析解场景可以预计算整条时间线（优化 seekTo 和参数变更体验）：

```typescript
/**
 * 可选的批量预计算接口。解析解求解器可以额外实现此方法。
 * Simulator 在 loadPreset / updateParam 后调用，一次性计算整条时间线。
 *
 * @param scene    - 场景静态定义
 * @param duration - 总时长（秒）
 * @param sampleRate - 采样率（每秒帧数），如 60
 * @returns 整条时间线的求解结果数组（按时间排序）
 */
type SolverPrecompute = (
  scene: SceneDefinition,
  duration: number,
  sampleRate: number,
) => PhysicsResult[];
```

### 2.3 积分器接口

```typescript
/**
 * 数值积分器接口
 * 将 ODE 的积分逻辑与求解器的物理方程分离
 */
interface Integrator {
  /**
   * 执行一步积分
   * @param state   - 当前状态向量 [x, y, vx, vy, ...]
   * @param t       - 当前时刻
   * @param dt      - 时间步长
   * @param derivFn - 导数函数：给定状态和时间，返回导数向量
   * @returns 新的状态向量
   */
  step(
    state: number[],
    t: number,
    dt: number,
    derivFn: (state: number[], t: number) => number[],
  ): number[];
}
```

三种积分器的实现策略（已在技术栈选型中确定）：

| 积分器 | 适用场景 | 特性 |
|--------|---------|------|
| `SemiImplicitEuler` | 大多数场景的默认选择 | 辛积分器，保能量，一阶精度 |
| `VelocityVerlet` | 周期运动（弹簧振子、天体） | 辛积分器，保能量，二阶精度 |
| `RK4` | 非守恒 ODE（电磁感应阻尼） | 高精度四阶，不保能量 |

## 3. 三大注册表 API

### 3.1 EntityRegistry（实体注册表）

```typescript
// ─── 实体注册信息 ───
interface EntityRegistration {
  /** 实体类型标识 */
  type: EntityType;

  /** 实体类别 */
  category: EntityCategory;

  /** 显示名称（UI用），如 "物块"、"斜面" */
  label: string;

  /** 默认属性值（创建新实体时使用） */
  defaultProperties: Record<string, unknown>;

  /** 该实体类型的可调参数定义（驱动参数面板渲染） */
  paramSchemas: ParamSchema[];

  /**
   * 命中检测函数
   * 判断一个点（物理坐标）是否命中该类型的实体
   */
  hitTest: (
    entity: Entity,
    point: Vec2,
    transform: CoordinateTransform,
  ) => HitTestResult | null;

  /**
   * 工厂函数：创建该类型的新实体实例（Phase 2 搭建器使用）
   * 自动生成 id，填入默认属性
   */
  createEntity: (overrides?: Partial<{
    transform: Transform;
    properties: Record<string, unknown>;
    label: string;
  }>) => Entity;
}

// ─── EntityRegistry API ───
interface EntityRegistry {
  /** 注册一个实体类型 */
  register(config: EntityRegistration): void;

  /** 获取指定类型的注册信息 */
  get(type: EntityType): EntityRegistration | undefined;

  /** 获取指定类别的所有注册信息（如获取所有 'object' 类别） */
  getByCategory(category: EntityCategory): EntityRegistration[];

  /** 获取所有已注册的实体类型 */
  getAll(): EntityRegistration[];

  /** 检查某类型是否已注册 */
  has(type: EntityType): boolean;
}
```

**设计决策**：
- **单例 + 可替换**：全局导出一个默认实例 `entityRegistry`，测试时可创建独立实例
- **`createEntity` 工厂函数**：Phase 2 搭建器需要从工具栏点击"添加物块"时创建实体，工厂函数封装了 id 生成 + 默认属性填充
- **`hitTest` 注册在实体类型上**：每种实体形状不同（矩形、三角形、圆形、区域），命中检测函数由各域在注册时提供

### 3.2 SolverRegistry（求解器注册表）

```typescript
// ─── 求解器注册信息 ───
interface SolverRegistration {
  /** 求解器唯一标识，如 'slope-contact-solver' */
  id: string;

  /** 显示名称（调试用），如 "斜面接触求解器" */
  label: string;

  /** 匹配模式 */
  pattern: SolverPattern;

  /** 求解模式：解析解 / 数值积分 */
  solveMode: SolveMode;

  /** 数值积分模式下推荐的积分器（由注册时指定，非预设JSON） */
  integrator?: IntegratorType;

  /** 求解函数（统一签名） */
  solve: SolverFunction;

  /**
   * 可选：解析解批量预计算函数
   * 仅 solveMode === 'analytical' 的求解器可提供
   */
  precompute?: SolverPrecompute;

  /**
   * 优先级（数值越小越优先）
   * 多个求解器匹配同一场景时的执行顺序
   * 默认 100
   */
  priority?: number;

  /**
   * 事件检测函数列表（可选）
   * 求解器注册时声明自己需要检测的事件
   */
  eventDetectors?: EventDetector[];
}

// ─── SolverRegistry API ───
interface SolverRegistry {
  /** 注册一个求解器 */
  register(config: SolverRegistration): void;

  /**
   * 匹配场景：返回所有匹配的求解器（按 priority 升序排列）
   * @param scene - 场景定义
   * @param qualifier - 可选的 qualifier（来自预设 JSON）
   */
  match(
    scene: SceneDefinition,
    qualifier?: Record<string, string>,
  ): SolverRegistration[];

  /** 按 id 获取指定求解器 */
  get(id: string): SolverRegistration | undefined;

  /** 获取所有已注册的求解器 */
  getAll(): SolverRegistration[];
}
```

**设计决策**：

**多求解器组合策略 → 选择方案2：每个预设绑定一个"完整求解器"**

| 方案 | 描述 | 结论 |
|------|------|------|
| 方案1：细粒度求解器（每个力一个求解器，最终合并） | 灵活但需要合并逻辑，物理量依赖关系复杂 | 淘汰 |
| 方案2：完整求解器（一个求解器负责整个场景的全部力） | 与"每个预设硬编码方程"的设计一致，实现简单 | ✅ 选用 |

理由（来自讨论记录第五条）：每个预设的物理方程是人写的（不是通用引擎推导的），一个求解器完整处理一个场景的所有受力+运动+能量计算，职责清晰。同类预设（如所有斜面题）复用同一个求解器，只是参数不同。

**积分器由求解器注册时指定**（而非预设 JSON 声明）：
- 积分器选择是求解器实现细节（Velocity Verlet 适合弹簧振子是因为物理方程特性）
- 预设 JSON 只需声明 `solveMode: 'analytical' | 'numerical'`
- 求解器内部根据自己的物理特性选择最优积分器

### 3.3 RendererRegistry（渲染器注册表）

```typescript
// ─── 实体渲染器签名 ───
type EntityRenderer = (
  entity: Entity,
  result: PhysicsResult | null,
  ctx: RenderContext,
) => void;

// ─── 视角层渲染器签名 ───
type ViewportRenderer = (
  data: ViewportData,
  entities: Map<EntityId, Entity>,
  ctx: RenderContext,
) => void;

// ─── 渲染层级（z-order） ───
type RenderLayer =
  | 'background'   // 背景层：网格、坐标轴
  | 'surface'      // 表面层：斜面、水平面、导轨
  | 'field'        // 场层：电场/磁场区域（半透明填充+符号阵列）
  | 'object'       // 物体层：物块、小球、导体棒
  | 'connector'    // 连接件层：弹簧、绳、杆
  | 'overlay';     // 叠加层：力箭头、轨迹线、标注（视角层渲染器）

// ─── 实体渲染器注册信息 ───
interface EntityRendererRegistration {
  entityType: EntityType;
  renderer: EntityRenderer;
  /** 渲染层级，决定绘制顺序 */
  layer: RenderLayer;
}

// ─── RendererRegistry API ───
interface RendererRegistry {
  /** 注册实体渲染器 */
  registerEntity(config: EntityRendererRegistration): void;

  /** 注册视角层渲染器 */
  registerViewport(viewportType: ViewportType, renderer: ViewportRenderer): void;

  /** 获取实体渲染器 */
  getEntityRenderer(entityType: EntityType): EntityRendererRegistration | undefined;

  /** 获取视角层渲染器 */
  getViewportRenderer(viewportType: ViewportType): ViewportRenderer | undefined;

  /**
   * 获取所有实体渲染器，按 layer 排序（用于渲染循环的绘制顺序）
   * 顺序：background → surface → field → object → connector → overlay
   */
  getEntityRenderersSorted(): EntityRendererRegistration[];
}
```

**设计决策**：
- **渲染层级由注册时指定**：用 `RenderLayer` 替代数字 z-order，语义明确且不易冲突
- **渲染循环按层级顺序绘制**：先画表面（斜面）→ 再画场区域 → 再画物体 → 最后画视角叠加层
- **选中状态高亮由渲染循环统一处理**：渲染循环在绘制每个实体后，检查是否为 `selectedEntityId`，若是则额外绘制高亮边框。避免每个渲染器重复实现选中逻辑
- **叠加层透明度由渲染循环设置**：渲染循环在绘制叠加视角前，设置 `ctx.globalAlpha = 0.3`，视角渲染器不需要自行处理透明度

### 3.4 注册表实现模式

三个注册表共用同一实现模式：

```typescript
// 注册表工厂（伪代码，展示实现模式）
function createRegistry<T extends { /* 按具体注册表定义 */ }>() {
  const _store = new Map<string, T>();

  return {
    register(config: T) {
      const key = /* 从 config 提取唯一标识 */;
      if (_store.has(key)) {
        console.warn(`[Registry] Duplicate registration: ${key}`);
      }
      _store.set(key, config);
    },
    get(key: string) { return _store.get(key); },
    getAll() { return Array.from(_store.values()); },
    has(key: string) { return _store.has(key); },
  };
}
```

## 4. Simulator 运行循环

### 4.1 Simulator 公开接口

```typescript
interface Simulator {
  // ─── 生命周期 ───
  /** 加载预设，初始化场景和求解器 */
  loadPreset(preset: PresetData): void;

  /** 卸载当前场景，清理资源 */
  unload(): void;

  // ─── 播放控制 ───
  /** 开始模拟（从当前时刻播放） */
  play(): void;

  /** 暂停模拟 */
  pause(): void;

  /** 重置到 t=0 */
  reset(): void;

  // ─── 时间轴 ───
  /** 跳到指定时刻 */
  seekTo(time: number): void;

  /** 设置播放速率（0.5 / 1.0 / 2.0） */
  setPlaybackRate(rate: number): void;

  // ─── 参数交互 ───
  /** 参数变更（用户调整参数面板时调用） */
  updateParam(paramKey: string, value: ParamValues[string]): void;

  // ─── 状态读取 ───
  /** 获取当前模拟状态 */
  getState(): SimulationState;

  /** 获取当前帧的求解结果 */
  getCurrentResult(): PhysicsResult | null;

  // ─── 事件订阅 ───
  /** 订阅模拟事件 */
  on(event: SimulatorEvent, handler: SimulatorEventHandler): void;

  /** 取消订阅 */
  off(event: SimulatorEvent, handler: SimulatorEventHandler): void;
}
```

### 4.2 内部运行循环

```
┌─────────────────────────────────────────────────────────────────┐
│                   Simulator 主循环（每帧）                       │
│                                                                 │
│  requestAnimationFrame(tick)                                    │
│       │                                                         │
│       ▼                                                         │
│  ① 计算 dt                                                      │
│     rawDt = (now - lastFrameTime) / 1000                       │
│     dt = rawDt * playbackRate                                   │
│     clamp(dt, 0, maxDt)  // 防止大步长                           │
│       │                                                         │
│       ▼                                                         │
│  ② 推进时间                                                      │
│     newTime = currentTime + dt                                  │
│     if (newTime >= duration) → newTime = duration, 标记 finished │
│       │                                                         │
│       ▼                                                         │
│  ③ 求解                                                         │
│     ┌─ analytical ──────────────────────────────┐               │
│     │ result = solver.solve(scene, newTime, 0, null)            │
│     │ （或从 precomputedResults 中二分查找最近帧）  │               │
│     └───────────────────────────────────────────┘               │
│     ┌─ numerical ───────────────────────────────┐               │
│     │ result = solver.solve(scene, newTime, dt, prevResult)     │
│     │ resultHistory.push(result)                │               │
│     └───────────────────────────────────────────┘               │
│       │                                                         │
│       ▼                                                         │
│  ④ 事件检测                                                      │
│     for each detector in solver.eventDetectors:                 │
│       event = detector.check(prevResult, result, scene)         │
│       if (event) → handleEvent(event)                           │
│       │                                                         │
│       ▼                                                         │
│  ⑤ 状态更新                                                      │
│     currentResult = result                                      │
│     timeline.currentTime = newTime                              │
│     更新实体 transform（从 motionStates 同步位置）                 │
│       │                                                         │
│       ▼                                                         │
│  ⑥ 通知渲染                                                      │
│     emit('frame', { time: newTime, result })                    │
│     → RenderLoop 收到事件后执行渲染管线                            │
│       │                                                         │
│       ▼                                                         │
│  ⑦ 循环                                                         │
│     if (status === 'running') requestAnimationFrame(tick)       │
│     if (status === 'finished') emit('finished')                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.4 关键行为设计

#### seekTo 实现

```typescript
seekTo(time: number): void {
  const clampedTime = Math.max(0, Math.min(time, this.timeline.duration));

  if (this.solveMode === 'analytical') {
    // 解析解：直接计算 t 时刻的结果，O(1)
    if (this.precomputedResults) {
      // 从预计算结果中二分查找最近帧
      this.currentResult = findNearestFrame(this.precomputedResults, clampedTime);
    } else {
      this.currentResult = this.activeSolver.solve(this.scene, clampedTime, 0, null);
    }
  } else {
    // 数值积分：从 resultHistory 中查找最近帧
    this.currentResult = findNearestFrame(this.resultHistory, clampedTime);
    // 注意：只能跳到已计算过的帧。如果 clampedTime > 最后一帧的时间，
    // 则需要从最后一帧继续积分到目标时刻
  }

  this.timeline.currentTime = clampedTime;
  this.triggerRender();  // 暂停态下也要渲染
}
```

#### updateParam 实现

```typescript
updateParam(paramKey: string, value: ParamValues[string]): void {
  // 1. 更新参数值
  this.scene.paramValues[paramKey] = value;

  // 2. 找到参数对应的实体和属性（从 ParamSchema 中获取映射关系）
  const schema = findParamSchema(this.scene.paramGroups, paramKey);
  if (schema?.targetEntityId && schema?.targetProperty) {
    const entity = this.scene.entities.get(schema.targetEntityId);
    if (entity) {
      (entity.properties as Record<string, unknown>)[schema.targetProperty] = value;
    }
  }

  // 3. 重置模拟（参数变更后需从 t=0 重新求解）
  this.timeline.currentTime = 0;
  this.resultHistory = [];

  // 4. 如果是解析解且有预计算，重新预计算
  if (this.solveMode === 'analytical' && this.activeSolver.precompute) {
    this.precomputedResults = this.activeSolver.precompute(
      this.scene, this.timeline.duration, 60
    );
  }

  // 5. 计算 t=0 的结果并渲染
  this.currentResult = this.activeSolver.solve(this.scene, 0, 0, null);
  this.triggerRender();
}
```

const leaveSurfaceDetector: EventDetector = {
  id: 'slope-leave-surface',
  eventType: 'leave-surface',

      if (normalForce && normalForce.magnitude <= 0) {

          type: 'leave-surface',

// 电磁感应求解器注册的事件检测器
const reachTerminalDetector: EventDetector = {
  id: 'em-reach-terminal',
  eventType: 'reach-terminal',
  check(prev, curr, scene) {
    if (!prev) return null;
    // 检查速度变化率是否足够小（趋近终态）
    for (const [entityId, motion] of curr.motionStates) {
      const prevMotion = prev.motionStates.get(entityId);
      if (!prevMotion) continue;
      const dv = Math.hypot(
        motion.velocity.x - prevMotion.velocity.x,
        motion.velocity.y - prevMotion.velocity.y,
      );
      if (dv < 1e-6) {
        return {
          type: 'reach-terminal',
          time: curr.time,
          entityId,
          description: '导体棒趋近终态速度',
          data: { terminalVelocity: Math.hypot(motion.velocity.x, motion.velocity.y) },
        };
      }
    }
    return null;
  },
};

## 6. 时间轴控制接口

### 6.1 设计决策

**时间轴直接集成在 Simulator 中**，不独立为 `TimelineController`。

理由：
- 时间轴的所有操作（play/pause/seekTo/setPlaybackRate）都直接影响 Simulator 状态
- 独立为 Controller 会增加一层不必要的间接层
- Zustand store 为 UI 提供响应式数据绑定已经足够

### 6.2 时间轴相关数据

```typescript
// UI 组件需要的数据（从 SimulationStore 读取）
interface TimelineUIData {
  currentTime: number;    // 当前时刻（秒）
  duration: number;       // 总时长（秒）
  progress: number;       // 进度百分比 0-1（= currentTime / duration）
  status: SimulationStatus;  // idle | running | paused | finished
  playbackRate: number;   // 播放速率
}
```

### 6.3 交互行为

| 操作 | 解析解场景 | 数值积分场景 |
|------|-----------|-------------|
| 播放 | 每帧调用 `solver.solve(scene, t, 0, null)` | 每帧调用 `solver.solve(scene, t, dt, prev)` |
| 暂停 | 停止 rAF，保持当前渲染 | 同左 |
| 拖拽时间轴 | 实时调用 `seekTo(t)` → 直接计算渲染 | 从 `resultHistory` 读取最近帧渲染 |
| 变速 | 修改 `playbackRate`，下帧生效 | 同左 |
| 到达终点 | 状态变为 `finished`，自动暂停 | 同左 |
| 重置 | `seekTo(0)` | `seekTo(0)`，`resultHistory` 保留 |

### 6.4 时间离散策略

- **底层按帧离散**：`dt = 1/60`（数值积分固定步长），解析解按连续 t 计算
- **UI 显示连续值**：`t=1.234s`，精度两位小数
- **拖拽对齐**：拖拽时间轴时，数值积分场景对齐到最近的已计算帧

solverRegistry.register({
  id: 'slope-contact-solver',
  label: '斜面接触求解器',
  pattern: { entityTypes: ['block', 'slope'], relationType: 'contact' },
  solveMode: 'analytical',
  solve(scene, time, _dt, _prev) {
    // 从场景中提取实体和关系
    const block = findEntityByType(scene, 'block')!;
    const slope = findEntityByType(scene, 'slope')!;
    const contact = findRelationByType(scene, 'contact')!;

    const m = (block.properties as { mass: number }).mass;
    const θ = (slope.properties as { angle: number }).angle;
    const μ = (contact.properties as { friction: number }).friction;
    const g = 9.8;

    // 受力分析
    const G = m * g;
    const N = G * Math.cos(θ);
    const f = μ * N;
    const F_net = G * Math.sin(θ) - f;
    const a = F_net / m;  // 沿斜面加速度

    // 运动方程（解析解：匀加速直线运动）
    const s = 0.5 * a * time * time;              // 位移（沿斜面）
    const v = a * time;                            // 速度
    const sx = s * Math.cos(θ);                    // x方向位移
    const sy = -s * Math.sin(θ);                   // y方向位移（向下为负）

    // 构造 PhysicsResult
    return {
      time,
      forceAnalyses: new Map([
        [block.id, {
          entityId: block.id,
          forces: [

            direction: F_net >= 0 ? { x: Math.cos(θ), y: -Math.sin(θ) } : { x: -Math.cos(θ), y: Math.sin(θ) },

    };
  },
  precompute(scene, duration, sampleRate) {
    const results: PhysicsResult[] = [];
    const dt = 1 / sampleRate;
    for (let t = 0; t <= duration; t += dt) {
      results.push(this.solve(scene, t, 0, null));
    }
    return results;
  },
  eventDetectors: [leaveSurfaceDetector],
});

solverRegistry.register({
  id: 'em-single-bar-solver',
  label: '电磁感应单棒求解器',
  pattern: {
    entityTypes: ['conductor-bar', 'bfield', 'rail'],
    relationType: 'field-effect',
    qualifier: { model: 'single-bar-induction' },
  },
  solveMode: 'numerical',
  integrator: 'rk4',  // 非守恒 ODE，用 RK4
  solve(scene, time, dt, prevResult) {
    const bar = findEntityByType(scene, 'conductor-bar')!;
    const bfield = findEntityByType(scene, 'bfield')!;
    const rail = findEntityByType(scene, 'rail')!;

    const m = (bar.properties as any).mass;
    const B = (bfield.properties as any).magnitude;
    const L = (rail.properties as any).spacing;
    const R = (rail.properties as any).resistance;

    // 获取前一帧状态（首帧用初始值）
    let v: number;
    let x: number;
    if (prevResult) {
      const prevMotion = prevResult.motionStates.get(bar.id)!;
      v = prevMotion.velocity.x;
      x = prevMotion.position.x;
    } else {
      v = (bar.properties as any).initialVelocity.x;
      x = bar.transform.position.x;
    }

    // ODE: m·dv/dt = -B²L²v/R（安培力方向与运动方向相反）
    // 使用 RK4 积分器
    const state = [x, v];  // 状态向量 [位置, 速度]
    const derivFn = (s: number[], _t: number) => {
      const [_x, vel] = s;
      const dvdt = -(B * B * L * L * vel) / (m * R);
      return [vel, dvdt];  // [dx/dt, dv/dt]
    };

    const newState = rk4Integrator.step(state, time, dt, derivFn);
    const [newX, newV] = newState;

    // 计算当前帧的物理量
    const emf = B * L * Math.abs(newV);        // 感应电动势 ε = BLv
    const I = emf / R;                          // 电流 I = ε/R
    const F_ampere = B * I * L;                 // 安培力 F = BIL
    const a = -F_ampere / m * Math.sign(newV);  // 加速度（减速方向）

    return {
      time,
      forceAnalyses: new Map([
        [bar.id, {
          entityId: bar.id,
          forces: [
            {
              type: 'ampere',
              label: `F安=${F_ampere.toFixed(3)}N`,
              magnitude: F_ampere,
              direction: { x: -Math.sign(newV), y: 0 },
            },
          ],
          resultant: {
            type: 'resultant',
            label: `F合=${F_ampere.toFixed(3)}N`,
            magnitude: F_ampere,
            direction: { x: -Math.sign(newV), y: 0 },
          },
        }],
      ]),
      motionStates: new Map([
        [bar.id, {
          entityId: bar.id,
          position: { x: newX, y: bar.transform.position.y },
          velocity: { x: newV, y: 0 },
          acceleration: { x: a, y: 0 },
          trajectory: prevResult
            ? [...(prevResult.motionStates.get(bar.id)?.trajectory ?? []), { x: newX, y: bar.transform.position.y }]
            : [{ x: newX, y: bar.transform.position.y }],
        }],
      ]),
      // 能量状态（可选）
      energyStates: new Map([
        [bar.id, {
          entityId: bar.id,
          kineticEnergy: 0.5 * m * newV * newV,
          potentialEnergies: [],
          totalEnergy: 0.5 * m * newV * newV,
          workDone: [
            { forceType: 'ampere', label: '安培力做功', value: -(0.5 * m * (bar.properties as any).initialVelocity.x ** 2 - 0.5 * m * newV * newV) },

        }],
      ]),
    };
  },

### 10.2 关键设计原则总结

1. **只加不改**：新域通过副作用导入 (`import '@/domains/em'`) 完成注册，不修改 core 层或已有域代码
2. **统一签名**：`SolverFunction` 四参数统一签名，Simulator 无需区分解析解/数值积分
3. **声明式匹配**：`SolverPattern` 结构化对象，类型安全且可序列化
4. **完整求解器**：每个求解器负责一个场景的全部物理计算，与"每个预设硬编码方程"的设计一致
5. **注册时定积分器**：积分器是求解器的实现细节，由注册时指定，非预设 JSON 声明
6. **层级化渲染**：`RenderLayer` 枚举确定绘制顺序，选中高亮和叠加层透明度由渲染循环统一处理
7. **事件驱动**：事件检测器由求解器注册时提供，事件-动作映射可由预设 JSON 定义
8. **预设自描述**：`PresetData` 完整包含场景定义+元信息+求解配置，"加功能点=加JSON文件"
