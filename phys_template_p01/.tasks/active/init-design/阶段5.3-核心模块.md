## 上下文

5.2已完成类型系统（51个接口）和设计系统集成。本批次实现核心业务模块：三大注册表提供插件化扩展能力，数学工具提供物理计算基础，Zustand Store 连接 UI 层和引擎层。

## 实施任务

### T6: 三大注册表实现

**需要创建的文件**：
- `src/core/registries/entity-registry.ts` — EntityRegistry
- `src/core/registries/solver-registry.ts` — SolverRegistry
- `src/core/registries/renderer-registry.ts` — RendererRegistry
- `src/core/registries/index.ts` — 统一导出

**实现依据**：`注册机制与引擎设计.md` 中三大注册表的 API 定义

**EntityRegistry 关键 API**：
- `registerEntity(type, config)` — 注册实体类型
- `getRegistration(type)` — 查询注册信息
- `getAllTypes()` — 获取所有已注册类型
- `createEntity(type, overrides?)` — 创建实体实例（生成 EntityId、合并默认属性）

**SolverRegistry 关键 API**：
- `registerSolver(pattern, solver, options?)` — 注册求解器
- `findSolver(scene)` — 根据场景匹配求解器（SolverPattern 结构化匹配）
- `getAllPatterns()` — 获取所有已注册的 pattern

**RendererRegistry 关键 API**：
- `registerEntityRenderer(entityType, renderer)` — 注册实体渲染器
- `registerViewportRenderer(viewportType, renderer)` — 注册视角渲染器
- `getEntityRenderer(type)` — 查询实体渲染器
- `getViewportRenderer(type)` — 查询视角渲染器

**设计关注点**：
- [x] 注册表内部存储用 Map
- [x] "只加不改"原则：重复注册同一 key 应 warn 而非静默覆盖
- [x] SolverRegistry 的 findSolver 需实现 SolverPattern 匹配逻辑（entityTypes 集合匹配 + relationType 匹配 + qualifier 可选匹配）
- [x] 注册表应为单例或通过 Zustand store 管理（采用全局单例 + createXxxRegistry() 工厂函数方案）

### T7: core/physics/ 基础数学工具

**需要创建的文件**：
- `src/core/physics/vec2.ts` — Vec2 向量运算函数
- `src/core/physics/geometry.ts` — 几何判断（pointInRect, pointInCircle, lineIntersection 等）
- `src/core/physics/integrators.ts` — Semi-implicit Euler、Velocity Verlet 积分器
- `src/core/physics/index.ts` — 统一导出

**Vec2 运算**（最小集）：
- add, subtract, scale, dot, cross2D, magnitude, normalize, rotate, fromAngle, lerp

**几何判断**（hit test 所需最小集）：
- pointInRect, pointInCircle, pointOnLine（带阈值）

**积分器**（骨架，阶段6验证）：
- semiImplicitEuler(state, acceleration, dt) → newState
- velocityVerlet(state, accelerationFn, dt) → newState

**设计关注点**：
- [x] 所有函数为纯函数，不修改输入参数
- [x] Vec2 操作使用 `{ x: number, y: number }` 接口（对齐 core/types.ts 的 Vec2）
- [x] 积分器签名需对照阶段3设计文档中 IntegratorType 的定义
- [x] 不需要 RK4（留给需要的域求解器自行实现）

### T13: Zustand Store 设计

**需要创建的文件**：
- `src/store/simulation-store.ts` — 模拟状态 store
- `src/store/index.ts` — 统一导出

**Store 管理的状态**：
- `simulationState: SimulationState` — 当前模拟状态
- `paramValues: ParamValues` — 当前参数值
- `selectedEntityId: EntityId | null` — 当前选中实体
- `viewportState: ViewportState` — 当前视角状态

**Store 暴露的 actions**：
- `loadPreset(preset)` — 加载预设（委托给 Simulator）
- `updateParam(key, value)` — 更新参数值
- `selectEntity(id)` — 选中实体
- `play() / pause() / reset()` — 时间线控制
- `switchPrimaryViewport(type)` — 切换主视角（清空叠加层+重置密度）
- `toggleOverlayViewport(type)` — 切换叠加视角

**设计关注点**：
- [x] Store 是 UI 层和引擎层之间的桥梁
- [x] 参数更新需触发引擎重算（静态模式立即重算，运动模式在下一帧生效）— updateParam 同步更新 scene.paramValues
- [x] 视角切换逻辑对照阶段4视角状态机设计 — switchPrimaryViewport 清空叠加层+重置密度，toggleOverlayViewport 最多3层+自动降密度
