## 实施约束与前置条件

### 已锁定的设计产出（阶段1-4）

本阶段是**首次编写实际代码**，所有实现必须严格对照以下设计文档：

**阶段1 · 技术栈**：
- React 18 + Zustand + 原生 Canvas 2D + Vite 6 + TypeScript strict + Tailwind CSS + pnpm
- ESLint（`@typescript-eslint/eslint-plugin` + React hooks 规则）+ Prettier
- Husky pre-commit hook：`pnpm lint && pnpm tsc --noEmit`
- 配套依赖：class-variance-authority、clsx、tailwind-merge、lucide-react、@radix-ui/*
- 数学库手写于 `core/physics/`，总量 300-500 行 TS

**阶段2 · 类型系统（51个接口）**：
- 8个类型模块：向量/几何 → 实体/关系 → 参数 → 物理结果 → 场景 → 视角 → 可视化 → 模拟状态
- 关键类型：`Vec2`、`Rect`、`Transform`、`Entity<T>`、`Relation<T>`、`ParamSchema`（discriminated union）、`PhysicsResult`（Map 索引）、`ViewportData`（tagged union）、`RenderContext`、`SimulationState`

**阶段3 · 注册与引擎（28个接口）**：
- 三大注册表：`EntityRegistry`、`SolverRegistry`、`RendererRegistry`
- 统一求解器签名：`SolverFunction(scene, time, dt, prevResult)`
- `SolverPattern` 结构化对象匹配（entityTypes + relationType + qualifier）
- `Simulator` 运行循环：loadPreset → 求解循环 → 渲染管线
- `PresetLoader`：JSON → ref 映射为 EntityId → 注入 Entity System → 匹配求解器
- `RenderLayer` 枚举绘制顺序：background → surface → field → object → connector → overlay
- `EventDetector` 由求解器注册时提供

**阶段4 · 预设Schema与视角层（4个新类型）**：
- `PresetData` 完整字段规范（version/meta/scene/solverConfig/events/displayConfig/supportedViewports/level）
- 视角状态机：切换主视角清空叠加层+重置密度
- `extractViewportData` 6种视角的数据提取逻辑
- 预设开发指南

### 核心原则

1. **骨架优先**：本阶段只搭骨架，不实现具体物理域逻辑（域逻辑留给阶段6）
2. **设计文档即规范**：类型定义照搬阶段2文档，注册表 API 照搬阶段3文档，不自行"改进"
3. **design_guid 复用**：UI 组件和样式 token 从 `design_guid/` 引入，不重新实现
4. **可运行**：每个子任务完成后项目必须能 `pnpm dev` 启动，不留半成品

### 10. Simulator 运行循环骨架 ✅

**已创建的文件**：
- `src/core/engine/simulator.ts` — Simulator 模块（createSimulator 工厂函数 + 全局 simulator 单例）
- `src/core/engine/index.ts` — 统一导出

**实现依据**：`注册机制与引擎设计.md` 中 Simulator 的完整接口设计

**核心管道**：
```
loadPreset(preset) → 初始化 SceneDefinition + 匹配求解器
step(dt) → 调用 SolverFunction → 更新 PhysicsResult → 触发事件检测
```

**关键方法（ISimulator 接口）**：
- `loadPreset(preset: PresetData)` — 加载预设 JSON（调用 PresetLoader）
- `step(dt: number)` — 单步物理计算（含时间推进+求解+事件检测+状态更新+帧通知）
- `play() / pause() / reset()` — 时间线控制
- `seekTo(time) / setPlaybackRate(rate)` — 时间轴操作
- `updateParam(key, value)` — 参数变更（更新实体属性+重置+重新求解 t=0）
- `getState() / getCurrentResult()` — 状态读取
- `on(event, handler) / off(event, handler)` — 事件订阅
- `unload()` — 卸载场景

**设计关注点**：
- [x] Simulator 不直接管理 requestAnimationFrame（由 render-loop.ts 管理，Simulator 只暴露 step）
- [x] step 内部：计算有效dt → 推进时间 → 调用 SolverFunction → 执行 EventDetector → 更新状态 → emit('frame')
- [x] 时间线控制通过内部状态管理 SimulationStatus（idle/running/paused/finished）
- [x] 阶段5只搭骨架和管道，不实现具体求解逻辑（无求解器时使用 stub 兜底）
- [x] 解析解支持预计算（precompute），seekTo 用二分查找最近帧
- [x] updateParam 支持嵌套属性路径（`x.y` 格式，setNestedProperty）

**加载流程**：
1. 校验 PresetData（实体ref唯一性、类型已注册、关系ref引用合法、参数targetEntityId引用合法、defaultViewport在supportedViewports中）
2. 创建实体实例 + 构建 ref→EntityId 映射
3. 创建关系实例（替换 sourceRef/targetRef 为真实 EntityId）
4. 替换 paramGroups 中 targetEntityId 的 ref 为真实 EntityId
5. 组装 SceneDefinition
6. 通过 SolverRegistry 匹配求解器
7. 替换 eventActions 中 entityId 的 ref 为真实 EntityId
8. 返回 PresetLoadResult

**types.ts 新增类型（10个）**：
- PresetEntityDef、PresetRelationDef、EventAction、EventActionMapping、PresetData
