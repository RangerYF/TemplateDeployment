## 任务清单

### T6：编写预设 JSON

**创建文件**：`src/domains/mechanics/presets/horizontal-block.json`

**内容**：严格遵循 PresetData 类型定义，参照阶段6文档中的完整 JSON 示例。

**关键字段**：
- id: `"P01-FM001-horizontal-block"`
- supportedViewports: `["force"]`
- defaultViewport: `"force"`
- displayConfig: `{ scale: 150, origin: { x: 400, y: 400 } }`
- entities: block-A（mass:2, width:0.5, height:0.5, position:{x:0,y:0.5}）+ surface-1（length:6, position:{x:-3,y:0}）
- relations: contact（block-A → surface-1）
- paramGroups: 质量滑块（targetEntityId: "block-A", targetProperty: "mass"）
- solveMode: `"analytical"`
- duration: `0`（静态场景）
- solverQualifier: `{ "surface": "horizontal" }`

**注意**：
- `duration: 0` 表示静态场景，Simulator 不会自动播放
- block-A 的 position.y = 0.5（物块底边贴地面，物块高度 0.5m）
- 预设 JSON 中的实体用 `ref` 标识，PresetLoader 会映射为真实 EntityId

### T7：打通端到端管道

**修改文件**：
- `src/domains/mechanics/index.ts` — 填充 `registerMechanicsDomain()` 导入并调用所有注册函数
- `src/main.tsx` 或 `src/shell/App.tsx` — 确保应用启动时加载默认预设

#### mechanics/index.ts 填充

```typescript
import { registerBlockEntity } from './entities/block';
import { registerSurfaceEntity } from './entities/surface';
import { registerBlockOnSurfaceSolver } from './solvers/block-on-surface';
import { registerBlockRenderer } from './renderers/block-renderer';
import { registerSurfaceRenderer } from './renderers/surface-renderer';
import { registerForceViewport } from './viewports/force-viewport';

export function registerMechanicsDomain(): void {
  registerBlockEntity();
  registerSurfaceEntity();
  registerBlockOnSurfaceSolver();
  registerBlockRenderer();
  registerSurfaceRenderer();
  registerForceViewport();
}
```

#### 应用启动流程

在 main.tsx 或 App.tsx 中：
1. 调用 `registerMechanicsDomain()`
2. 导入预设 JSON
3. 调用 `simulator.loadPreset(preset)`
4. 启动渲染循环

**需要确认现有 App.tsx / main.tsx 的启动逻辑**，在合适的位置插入预设加载。
