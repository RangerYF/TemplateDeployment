# 阶段6：参考实现（端到端打通）

> 目标：实现"水平面物块受力"这一最简预设，打通从预设加载到画布渲染到参数联动的完整链路，作为两位开发者的开发范本。

---

## 实施约束与前置条件

### 已就绪的骨架代码（阶段5产出）

本阶段在阶段5搭好的骨架上填充**第一个具体物理域实现**，验证整个架构端到端可用。

**可用的注册 API**（`src/core/registries/`）：
- `entityRegistry.register(config: EntityRegistration)` — 注册实体类型（type, category, label, defaultProperties, paramSchemas, hitTest, createEntity）
- `solverRegistry.register(config: SolverRegistration)` — 注册求解器（id, pattern, solver, solveMode, integratorType?, precompute?, eventDetectors?）
- `rendererRegistry.registerEntity(config: EntityRendererRegistration)` — 注册实体渲染器（entityType, renderer, layer）
- `rendererRegistry.registerViewport(viewportType, renderer: ViewportRenderer)` — 注册视角渲染器

**可用的引擎管道**（`src/core/engine/`）：
- `simulator.loadPreset(preset)` → 调用 PresetLoader → 匹配求解器 → 初始化状态
- `simulator.step(dt)` → 调用 SolverFunction → 更新 PhysicsResult → 事件检测
- `simulator.updateParam(key, value)` → 更新实体属性 → 重置 → 重新求解 t=0

**可用的渲染基础设施**（`src/renderer/`）：
- `drawArrow(ctx, from, to, style)` — 力箭头绘制
- `drawTextLabel(ctx, position, text, style)` — 数值标注
- `drawRect(ctx, rect, style)` / `drawLine(ctx, from, to, style)` — 基础形状
- `worldToScreen(point, coordSystem)` / `screenToWorld(point, coordSystem)` — 坐标转换
- `createRenderLoop(config)` — 渲染循环（按 RenderLayer 排序 + 叠加层 alpha）

**可用的 UI 骨架**（`src/shell/`）：
- `ParamPanel` — schema 驱动的参数面板（接收 ParamSchema[] + ParamValues，自动选择控件）
- `CanvasContainer` — 画布容器（管理 Canvas ref + resize + devicePixelRatio）
- `TimelineBar` — 播放/暂停/重置 + 时间显示

**可用的 Store**（`src/store/`）：
- `useSimulationStore` — updateParam, play/pause/reset, switchPrimaryViewport 等

### 核心原则

1. **域隔离验证**：全部实现只在 `src/domains/mechanics/` 内，**不修改 core/、shell/、renderer/ 的代码**——这是架构设计的核心验证目标
2. **最简场景**：水平面物块受力是解析解（G=mg, N=mg），无运动、无时间演化，是最简单的端到端验证
3. **范本导向**：代码结构和注释要清晰到开发者看完就能照着做"斜面下滑"预设
4. **对照预设Schema**：预设 JSON 严格遵循阶段4定义的 PresetData 字段规范

### 1. 注册实体类型：block（物块）

**需要创建的文件**：
- `src/domains/mechanics/entities/block.ts`

**实现内容**：
调用 `entityRegistry.register()` 注册 block 实体类型：

```typescript
{
  type: 'block',
  category: 'object',
  label: '物块',
  defaultProperties: {
    mass: 1,       // kg
    width: 0.5,    // m
    height: 0.5,   // m
  },
  paramSchemas: [
    { key: 'mass', label: '质量', type: 'slider', min: 0.1, max: 10, step: 0.1, default: 1, unit: 'kg' },
  ],
  hitTest: (entity, point, transform) => { /* pointInRect 判断 */ },
  createEntity: (overrides?) => { /* 生成 EntityId，合并默认属性 */ },
}
```

**设计关注点**：
- [ ] hitTest 使用 `pointInRect`（从 `core/physics/geometry` 导入），需根据 entity.transform 计算矩形范围
- [ ] createEntity 需生成唯一 EntityId（可用 `block-${counter}` 或 crypto.randomUUID）
- [ ] block 的 properties.width/height 用于渲染时的尺寸，物理计算只用 mass

**需要创建的文件**：
- `src/domains/mechanics/entities/surface.ts`

**实现内容**：
调用 `entityRegistry.register()` 注册 surface 实体类型：

```typescript
{
  type: 'surface',
  category: 'surface',
  label: '水平面',
  defaultProperties: {
    length: 6,    // m，水平面长度
    friction: 0,  // 动摩擦因数
  },
  paramSchemas: [],  // 水平面本身无用户可调参数（摩擦因数通过关系属性配置）
  hitTest: (entity, point, transform) => { /* 判断点是否在水平线附近 */ },
```

**设计关注点**：

- [ ] 水平面的渲染：一条实线 + 底部斜线填充（标准物理教材画法）

### 3. 实现求解器：block-on-surface

**需要创建的文件**：
- `src/domains/mechanics/solvers/block-on-surface.ts`

**SolverPattern**：
```typescript
{
  entityTypes: ['block', 'surface'],
  relationType: 'contact',
  qualifier: { surface: 'horizontal' },  // 区分水平面 vs 斜面
}
```

**SolverFunction 实现**（解析解，无时间演化）：
```
输入：scene（含 block 实体的 mass 属性）
计算：
  g = 9.8 m/s²
  G = m * g（重力，方向竖直向下）
  N = m * g（支持力，方向竖直向上）
  合力 = 0（静力平衡）
输出：PhysicsResult
  - forceAnalyses: Map<EntityId, ForceAnalysis>
    - block 的 ForceAnalysis: { forces: [G向下, N向上], netForce: {x:0, y:0} }
  - motionStates: Map<EntityId, MotionState>
    - block 的 MotionState: { position, velocity: {x:0,y:0}, acceleration: {x:0,y:0} }
  - energyStates: Map<EntityId, EnergyState>（本预设可选）
```

**注册选项**：
```typescript
{
  id: 'mech-block-on-horizontal-surface',
  pattern: { entityTypes: ['block', 'surface'], relationType: 'contact', qualifier: { surface: 'horizontal' } },
  solver: blockOnSurfaceSolver,
  solveMode: 'analytical',
  // 无 integratorType（解析解不需要积分器）
  // 无 eventDetectors（静力平衡无事件）
}
```

**设计关注点**：
- [ ] 这是解析解求解器：任何时刻 t 的结果都一样（静力平衡），step(dt) 每次返回相同 PhysicsResult
- [ ] ForceVector 需要包含：magnitude, direction（Vec2），label（"G"/"N"），color（用于渲染）
- [ ] 确保 forceAnalyses 的 key 是真实 EntityId（由 PresetLoader 映射后的）
- [ ] 参数变更时（质量改变），下一次 step 应反映新的力值——由 Simulator.updateParam 触发 reset+re-solve

### 4. 实现实体渲染器

**需要创建的文件**：
- `src/domains/mechanics/renderers/block-renderer.ts`
- `src/domains/mechanics/renderers/surface-renderer.ts`

**block-renderer**：
- 注册到 `rendererRegistry.registerEntity({ entityType: 'block', renderer, layer: 'object' })`
- 绘制内容：填充矩形（浅色填充 + 深色边框），内部显示 label
- 使用 `drawRect`（从 renderer/primitives 导入）+ `worldToScreen` 坐标转换
- 矩形尺寸由 entity.properties.width/height 决定，位置由 entity.transform.position 决定

**surface-renderer**：
- 注册到 `rendererRegistry.registerEntity({ entityType: 'surface', renderer, layer: 'surface' })`
- 绘制内容：水平线 + 底部斜线填充（标准物理教材风格的地面画法）
- 使用 `drawLine` + Canvas 2D pattern fill

**设计关注点**：
- [ ] 渲染器签名：`(entity: Entity, result: PhysicsResult | null, ctx: RenderContext) => void`
- [ ] 坐标转换：所有绘制前先将物理坐标转为画布像素坐标
- [ ] block 矩形应有圆角和阴影，使其看起来像教具而非纯几何图形
- [ ] surface 的斜线填充间距约 6-8px，角度约 -45°

### 5. 实现受力视角渲染器

**需要创建的文件**：
- `src/domains/mechanics/viewports/force-viewport.ts`

**注册**：
- `rendererRegistry.registerViewport('force', forceViewportRenderer)`

**ViewportRenderer 签名**：
```typescript
(data: ViewportData, entities: Map<EntityId, Entity>, ctx: RenderContext) => void
```

**渲染内容**（参照阶段4视角层设计 §5.3 受力视角）：
- 从 `data`（type='force'）中读取 `forceAnalyses`
- 遍历每个实体的 ForceAnalysis：
  - 每个 ForceVector → 调用 `drawArrow(ctx, from, to, style)` 绘制力箭头
    - 箭头起点：实体中心
    - 箭头终点：起点 + 力方向 × 缩放系数（力值 → 箭头长度映射）
    - 颜色约定：重力红色、支持力蓝色（由 ForceVector.color 或 ForceVisualMap 确定）
  - 每个力箭头旁 → 调用 `drawTextLabel` 绘制数值标注
    - standard 密度格式：`"G=19.6N"`
    - detailed 密度格式：`"G=mg=2×9.8=19.6N"`（本阶段可只实现 standard）
  - 合力不为零时 → 绘制合力箭头（虚线，绿色）

**信息密度适配**（阶段4 表4.2）：
| 渲染元素 | compact | standard | detailed |
|---------|---------|----------|----------|
| 力箭头 | 有 | 有 | 有 |
| 数值标注 | 无 | "G=19.6N" | "G=mg=2×9.8=19.6N" |
| 正交分解虚线 | 无 | 有 | 有+分量标注 |

**设计关注点**：
- [ ] 力箭头长度缩放：需要一个映射函数 forceToLength(magnitude)，确保小力可见、大力不溢出画布
- [ ] 标注位置：箭头中点偏移，避免标注压在箭头上
- [ ] 本预设（水平面）不涉及正交分解，但渲染器应预留接口以便斜面预设复用
- [ ] 颜色方案参考 `VisualStyle` / `ForceVisualMap`，或暂用约定色（gravity: #E53E3E, normal: #3182CE）

### 7. 打通端到端管道（集成联调）

**需要修改的文件**：
- `src/domains/mechanics/index.ts` — 填充注册逻辑（导入并调用所有注册函数）
- `src/main.tsx`（或 `src/shell/App.tsx`）— 确保应用启动时加载默认预设

**registerMechanicsDomain() 填充**：
```typescript
import { registerBlockEntity } from './entities/block';
import { registerSurfaceEntity } from './entities/surface';
import { registerBlockOnSurfaceSolver } from './solvers/block-on-surface';
import { registerBlockRenderer } from './renderers/block-renderer';
import { registerSurfaceRenderer } from './renderers/surface-renderer';
import { registerForceViewport } from './viewports/force-viewport';

export function registerMechanicsDomain(): void {
  // 实体
  registerBlockEntity();
  registerSurfaceEntity();
  // 求解器
  registerBlockOnSurfaceSolver();
  // 渲染器
  registerBlockRenderer();
  registerSurfaceRenderer();
  // 视角
  registerForceViewport();
}
```

**应用启动流程**：
```
main.tsx: registerMechanicsDomain() → import preset JSON → simulator.loadPreset(preset) → renderLoop.start()
```

**联调验证点**：
- [ ] PresetLoader.validate() 返回 valid=true
- [ ] SolverRegistry.match() 成功匹配到 block-on-surface 求解器
- [ ] simulator.step(0) 返回正确的 PhysicsResult（forceAnalyses 含 G 和 N）
- [ ] ParamPanel 显示"质量"滑块（值 = 2）
- [ ] Canvas 渲染：水平面 + 物块 + 力箭头（G 向下红色 + N 向上蓝色）+ 数值标注
- [ ] 拖动质量滑块 → store.updateParam → simulator.updateParam → 力值更新 → 箭头和标注实时变化
