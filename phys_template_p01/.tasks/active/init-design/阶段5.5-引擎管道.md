# 阶段5.5：引擎管道（Simulator + PresetLoader）

> 批次5/6 | 依赖：5.3已完成 | 包含：T10 Simulator骨架 + T11预设加载器

## 上下文

5.3已完成三大注册表、数学工具和 Store。本批次搭建引擎核心管道：Simulator 运行循环和 PresetLoader 预设加载器，打通 "JSON预设 → 实体创建 → 求解器匹配 → 物理计算 → 状态更新" 的完整链路骨架。

## 实施任务

### T10: Simulator 运行循环骨架

**需要创建的文件**：
- `src/core/engine/simulator.ts` — Simulator 类/模块
- `src/core/engine/index.ts` — 统一导出

**实现依据**：`注册机制与引擎设计.md` 中 Simulator 的完整接口设计

**核心管道**：
```
loadPreset(preset) → 初始化 SceneDefinition + 匹配求解器
step(dt) → 调用 SolverFunction → 更新 PhysicsResult → 触发事件检测
render(ctx) → 按 RenderLayer 遍历渲染器 → 绘制到 Canvas
```

**关键方法**：
- `loadPreset(preset: PresetData)` — 加载预设 JSON（调用 PresetLoader）
- `step(dt: number)` — 单步物理计算
- `start() / pause() / reset()` — 时间线控制
- `getCurrentState()` — 获取当前 SimulationState

**设计关注点**：
- [ ] Simulator 不直接管理 requestAnimationFrame（由 render-loop.ts 管理，Simulator 只暴露 step）
- [ ] step 内部：取当前 SceneDefinition → findSolver → 调用 SolverFunction → 更新 PhysicsResult → 执行 EventDetector
- [ ] 时间线控制通过 Zustand store 管理 SimulationStatus（idle/running/paused/finished）
- [ ] 阶段5只搭骨架和管道，不实现具体求解逻辑

### T11: 预设加载器

**需要创建的文件**：
- `src/core/engine/preset-loader.ts` — PresetLoader

**实现依据**：`注册机制与引擎设计.md` 中 PresetLoader 的加载流程 + `预设Schema与视角层设计.md` 中 PresetData 字段规范

**加载流程**：
1. 读取 PresetData JSON
2. 解析 `scene.entities` 和 `scene.relations`，将 ref 字符串映射为真实 EntityId
3. 通过 EntityRegistry 创建实体实例
4. 构建 SceneDefinition
5. 通过 SolverRegistry 匹配求解器
6. 初始化 ParamValues（从 entities 的 defaultProperties 和 params 中收集）
7. 初始化 ViewportState（从 supportedViewports 设置默认主视角）

**设计关注点**：
- [ ] ref → EntityId 映射表在加载时创建，后续事件映射和渲染都通过此表引用
- [ ] 验证 PresetData 的 version 字段（Schema 版本兼容性）
- [ ] 错误处理：注册表中找不到实体类型/求解器时给出有意义的错误信息
- [ ] 阶段5用一个最简测试 JSON（仅含 meta + 空 entities）验证加载流程
