### 需要补充的内容

#### 2.1 新增预设标准流程（引用开发者指引）

```
新增预设时，必须阅读并遵循 `docs/开发者指引.md` 中的 5 步流程。
```

#### 2.2 命名规范

- **文件命名**：实体 `{type}.ts`、求解器 `{scene-description}.ts`、渲染器 `{type}-renderer.ts`、视角 `{viewport}-viewport.ts`、预设 `{scene-description}.json`
- **预设 ID 格式**：`{模块号}-{模型ID}-{变体}`，如 `P01-FM001-horizontal-block`
- **求解器 ID 格式**：`{domain}-{description}`，如 `mech-block-on-horizontal-surface`

#### 2.3 视觉常量约束

```
力的颜色必须使用 `src/core/visual-constants.ts` 中的 FORCE_COLORS，禁止在渲染器中硬编码力颜色。
```

#### 2.4 公共代码修改流程

当前 F2 只说"修改需两位开发者协商"，需要明确：
- 什么情况下域开发者可能需要修改公共代码（如新增 EntityCategory、新增 ViewportType）
- 修改前需要做什么（提 issue / 在任务文档记录需求）
- 如何避免冲突（追加不修改、不删已有导出）

#### 2.5 类型扩展约定

- `core/types.ts` 只能追加，不能修改已有接口
- 域特有类型放在域目录下，不放 core/

#### 2.6 依赖管理

- Phase 1 不新增外部 npm 依赖（当前依赖已够用）
- 如确需新增，需在任务文档记录理由

### T7 ✅ 缺失约束补充

预演中发现一个潜在问题需要补充到开发者指引：

**问题**：预设 JSON 中 `paramGroup.params[].default` 的值单位与 `entity.properties` 的值单位可能不一致。例如电荷量滑块显示 `2 μC`，但实体 `charge` 属性存储 `2e-6` C。当前 PresetLoader 直接将滑块值写入实体属性，不做单位转换。

**处理方案**：在预设 JSON 中，实体 properties 的值应与滑块值使用相同的数量级。即 `charge: 2`（μC）而非 `charge: 2e-6`（C），求解器中自行处理单位换算。这与力学域一致（mass 滑块 default=2，实体 mass=2，单位 kg 统一）。

→ 已在 em 预演代码中按此方案修正。
