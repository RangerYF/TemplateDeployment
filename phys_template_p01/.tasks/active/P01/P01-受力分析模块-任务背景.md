## 二、模块目标

为高中物理教师提供覆盖**受力分析全部典型题型**的预设场景库，教师可以：

1. 从预设库中加载经典受力分析场景
2. 通过参数面板调整质量、摩擦因数、角度、力的大小等
3. 在「受力视角」下观察力的箭头、合力、正交分解
4. 在「运动视角」下观察运动轨迹和 v-t / s-t 图
5. 在「能量视角」下观察动能、势能的变化

**3 分钟内复现一道受力分析题的完整过程。**

## 六、开发依赖与约束

### 6.1 已有基础设施

- **引擎核心**：`src/core/engine/` — SimulationEngine、SolverRegistry、PresetLoader
- **类型系统**：`src/core/types.ts` — 51 个 TypeScript 接口
- **视觉常量**：`src/core/visual-constants.ts` — FORCE_COLORS
- **注册入口**：`src/domains/mechanics/index.ts`

### 6.2 开发标准流程（5 步）

1. **注册实体类型** → `entities/{type}.ts`
2. **实现求解器** → `solvers/{scene-name}.ts`
3. **实现实体渲染器** → `renderers/{type}-renderer.ts`
4. **实现视角渲染器**（如需新视角）→ `viewports/{viewport}-viewport.ts`
5. **编写预设 JSON** → `presets/{scene-name}.json`

### 6.3 质量门禁

- `src/` 变更：执行 `pnpm lint && pnpm tsc --noEmit`
- 命中数学计算：追加手算验证
- 预设 JSON 须通过 `PresetLoader.validate()` 校验

### 6.4 禁止事项

- 禁止修改 `src/domains/em/` 下任何文件
- 禁止修改 `src/core/types.ts` 已有接口（只能追加）
- 禁止硬编码力的颜色（必须引用 FORCE_COLORS）
- 禁止新增外部 npm 依赖（Phase 1 约束）
