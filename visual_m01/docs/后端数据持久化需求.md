# 后端数据持久化需求（MVP）

> 基于 V0.2 编辑器架构，MVP 阶段需要持久化的数据及其服务功能。

---

## 一、数据模型

### Project 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识（主键） |
| userId | string | 所属用户 ID（外键，关联 User） |
| name | string | 作品名称 |
| thumbnail | string (URL) | 3D 视图截图 |
| createdAt | timestamp | 创建时间 |
| updatedAt | timestamp | 最后更新时间 |
| sceneData | JSON | entityStore 快照 |

### sceneData 结构（entityStore 快照）

```typescript
{
  entities: {
    [id: string]: {
      id: string;
      type: 'geometry' | 'point' | 'segment' | 'face' | 'coordinateSystem' | 'circumSphere' | 'circumCircle';
      visible: boolean;
      properties: { ... }  // 按 type 不同，包含几何参数、点约束、线面定义等
    }
  },
  nextId: number,
  activeGeometryId: string | null
}
```

这一个 JSON 包含了用户在编辑器中创建的所有内容：几何体及其参数、辅助点线面、坐标系、标签等。

---

## 二、用户可体验的功能

| 功能 | 依赖的数据 | 说明 |
|------|-----------|------|
| **保存作品** | sceneData, name | 编辑完点保存，所有几何操作持久化 |
| **加载作品** | sceneData | 下次登录恢复完整场景（几何体、辅助元素、标签） |
| **作品列表** | name, thumbnail, updatedAt | 缩略图预览，按时间排序 |
| **分享链接** | id, sceneData | 通过作品 ID 生成链接，他人可查看同一场景 |
| **跨设备同步** | 全部 | 电脑编辑，平板打开也能看 |
| **多作品管理** | id, name, createdAt | 创建、重命名、删除多个作品 |
