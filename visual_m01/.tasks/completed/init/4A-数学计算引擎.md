## 架构设计

### 1. 精确值表示（SymbolicValue）

数学教学需要展示精确值而非浮点数。设计一个轻量的符号值系统：

```typescript
// src/engine/math/types.ts

/** 精确值：同时持有 LaTeX 展示形式和数值近似 */
interface SymbolicValue {
  /** LaTeX 字符串（用于 KaTeX 渲染） */
  latex: string;
  /** 数值近似 */
  numeric: number;
}

/** 计算步骤 */
interface CalcStep {
  /** 步骤说明（中文） */
  label: string;
  /** 公式/表达式（LaTeX） */
  latex: string;
}

/** 单项计算结果（体积或表面积） */
interface MeasureResult {
  /** 精确值 */
  value: SymbolicValue;
  /** 计算步骤序列 */
  steps: CalcStep[];
}

/** 几何体完整计算结果 */
interface CalculationResult {
  volume: MeasureResult;
  surfaceArea: MeasureResult;
}
```

**设计决策**：不做 AST 级别的符号运算（过于复杂），而是在每个计算器内手动构造 LaTeX 字符串。对于 V1 的 6 种基础几何体，公式模式固定，手动构造完全可控且精确。

### 2. 计算器注册表

与 Builder 注册表模式一致：

```typescript
// src/engine/math/index.ts
type Calculator = (params: Record<string, number>) => CalculationResult;
const calculators: Record<GeometryType, Calculator> = { ... };

export function calculate(type: GeometryType, params: Record<string, number>): CalculationResult;
```

### 3. KaTeX 渲染

需要新增 `katex` 依赖来渲染 LaTeX 公式。在 React 中通过 `dangerouslySetInnerHTML` + `katex.renderToString()` 渲染，或使用 `react-katex` 封装组件。

**方案选择**：直接使用 `katex` 包 + 自行封装轻量 `<TeX>` 组件（避免 `react-katex` 年久失修的风险）。

### 4. 坐标系设计

**交互流程**：
1. 左侧"辅助功能"面板开启"坐标系"开关
2. 点击一个顶点作为原点
3. 系统自动确定坐标轴方向：
   - **长方体**：选中顶点处的 3 条相邻棱方向作为 X/Y/Z 轴（自然正交基）
   - **曲面体**：原点在特征点处，Y 轴沿几何体对称轴，X/Z 取标准方向
4. 渲染坐标轴（红/绿/蓝 = X/Y/Z）+ 刻度标记
5. 每个顶点旁显示坐标值 `(x, y, z)`

**数据模型**：
```typescript
interface CoordinateSystem {
  /** 原点顶点索引 */
  originIndex: number;
  /** 原点 3D 坐标 */
  origin: Vec3;
  /** 三个轴方向（单位向量） */
  axes: [Vec3, Vec3, Vec3]; // X, Y, Z
  /** 各顶点在此坐标系下的坐标 */
  vertexCoords: Vec3[];
}
```

### 5. 外接球设计

**长方体外接球**：
- 球心 = 长方体几何中心 = `((A + C₁) / 2)`
- 半径 R = `√(l² + w² + h²) / 2`
- 渲染为半透明线框球（wireframe sphere）

**数据模型**：
```typescript
interface CircumscribedSphere {
  center: Vec3;
  radius: number;
  /** 半径精确值（LaTeX） */
  radiusLatex: string;
}
```

### 6. 外接圆设计（三点定圆）

**通用算法**：给定 3D 空间中不共线的 3 点 A、B、C：
1. 求平面法向量 n = (B-A) × (C-A)
2. 在该平面内求外接圆圆心（联立两条中垂线）
3. 半径 R = |圆心 - A|

**渲染**：在 3D 空间中生成圆环几何体（`RingGeometry` 或点序列 `<Line>`），旋转至正确平面。

### T4A.4 圆锥体积 / 表面积计算器

**目标**：实现圆锥的 V 和 S 计算，含 π 和 √ 的精确值。

**产出文件**：
- `src/engine/math/calculators/cone.ts`

**公式**：
- V = πr²h / 3
- 母线长 l = √(r² + h²)
- S_侧 = πrl
- S_底 = πr²
- S = πr² + πrl = πr(r + l)

**计算步骤示例（V, r=1.5, h=2）**：
1. "体积公式" → `V = \frac{1}{3}\pi r^2 h`
2. "代入数值" → `V = \frac{1}{3}\pi \times 1.5^2 \times 2`
3. "化简" → `V = \frac{1}{3}\pi \times 4.5`
4. "计算结果" → `V = 1.5\pi \approx 4.71`

**验收**：
- V 精确值含 π，近似值正确
- S 精确值含 π 和 √，母线长计算正确
- 手算验证：r=1.5, h=2 → l=√(2.25+4)=√6.25=2.5 → S=π×1.5×(1.5+2.5)=6π ≈ 18.85

### T4A.10 外接圆（三点定圆）

**目标**：用户选择 3 个顶点，在这 3 点所在平面上画外接圆并显示半径。

**产出文件**：
- `src/engine/math/circumscribedCircle.ts` — 三点定圆算法
- `src/components/scene/CircumCircle.tsx` — 外接圆 3D 渲染

**修改文件**：
- `src/store/useGeometryStore.ts` — 新增外接圆状态
- `src/components/scene/Scene3D.tsx` — 集成 CircumCircle

**算法**（三点定圆，3D 空间）：
1. 输入 A, B, C（Vec3）
2. 求平面法向量 `n = (B-A) × (C-A)`，单位化
3. 求 AB 中点 M1，AB 方向 d1 = B - A
4. 求 AC 中点 M2，AC 方向 d2 = C - A
5. 中垂面交线（在平面内的中垂线）→ 联立求圆心 O
6. R = |O - A|

**交互流程**：
- 开启"外接圆"模式后，依次点击 3 个顶点
- 选满 3 个后自动渲染外接圆
- 点击第 4 个顶点时，替换最早选择的（或重置）

**渲染**：
- 在 3 点平面内绘制圆环（64段折线）
- 圆心标记 + 半径值标签
- 可取消/重选

**Store 扩展**：
```typescript
circumCircleEnabled: boolean;
circumCircleVertices: number[];  // 选中的 3 个顶点索引
setCircumCircleEnabled: (enabled: boolean) => void;
addCircumCircleVertex: (index: number) => void;
resetCircumCircle: () => void;
```

**验收**：
- 长方体底面 A/B/C → 外接圆 = 底面对角线所定圆，R = √(l²+w²)/2
- 选 3 个不共面顶点（如 A/B/A₁）→ 圆正确渲染在 ABB₁A₁ 面上
- 半径值与手算一致

## 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 精确值表示 | LaTeX string + numeric 双持有 | 不需要 CAS 级别的符号运算，手动构造 LaTeX 足够可控 |
| 数学渲染 | KaTeX | 轻量快速，SSR 友好，社区活跃，适合教学场景 |
| KaTeX 封装 | 自制 `<TeX>` 组件 | react-katex 维护不活跃，自封装代码量极小 |
| 计算器模式 | 注册表 + 纯函数 | 与 Builder 注册表一致，方便 4B 阶段铺开 |
| 坐标轴方向 | 按顶点邻接棱确定 | 符合高中数学"以顶点为原点、棱为轴"的教学惯例 |
| 外接球范围 | 4A 仅长方体 | 圆锥外接球公式不常用，4B 铺开时再处理 |
| 外接圆算法 | 三点定圆（通用） | 与几何体类型无关，任意 3 点可用 |
| MeasurementDisplay 位置 | Canvas 外层 HTML | 与 ContextMenu3D 一致，避免在 R3F 管线内处理 DOM |

---

## 风险与注意事项

1. **KaTeX CSS 引入**：KaTeX 需要引入 CSS 文件（`katex/dist/katex.min.css`），需确认 Vite 打包正确处理。
2. **精确值边界情况**：当参数产生无理数（如 √17/2）时，LaTeX 需要正确表示分数+根号的组合。需要覆盖测试。
3. **坐标系轴方向**：并非所有几何体顶点都有 3 条正交邻接棱（如棱锥顶点 P），需要有 fallback 策略。4A 阶段只处理长方体和圆锥，长方体天然正交，圆锥使用标准方向。
4. **外接圆三点共线**：3 点共线时无法定圆，需要检测并提示用户。
5. **外接圆平面渲染**：圆环需要从 XY 平面旋转到 3 点所在平面，旋转计算需要注意数值稳定性。
6. **Store 膨胀**：4A 新增较多状态，考虑是否需要拆分为单独的 `useAuxiliaryStore`。初步决策：暂时保持单 store，4B 时评估。
7. **参数变化联动**：参数变化时，外接球/外接圆/坐标系都需要重新计算。需确认与 `resetAnnotations` 的时机协调。
