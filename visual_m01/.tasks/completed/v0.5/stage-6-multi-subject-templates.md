# 第6阶段：多学科模板体系 & AI 预留

**所属版本**：V0.5
**风险等级**：L1（常规风险）
**流程路径**：MODE 0 → MODE 3 → MODE 4 → MODE 5 → MODE 6
**前置依赖**：阶段1-5已完成

## 目标

1. 将工作台从"仅数学几何体"扩展为"多学科模板体系"，支持 学科 → 模块 → 模板 三级结构
2. 创建完整的模板配置文件，覆盖数学/物理/化学三个学科的所有模块和模板
3. 工作台按学科 Tab 切换展示，已实现的模板可直接使用，未实现的标记"即将上线"
4. 预留 AI 意图推荐和 AI 生成的接口位置

## 未来 AI 功能说明（本阶段仅预留位置，不实现）

1. **AI 模板推荐**：用户输入教学意图 → 后端大模型接口 → 返回匹配模板列表
2. **AI 生成作品**：无匹配模板或用户不满意时 → 后端大模型接口 → 直接生成符合意图的 scene_data

本阶段在工作台顶部预留一个搜索/意图输入框位置，UI 上显示"AI 助手即将上线"的占位提示。

### 6.1 模板配置文件

**要做什么**：创建学科/模块/模板三级配置数据。

**涉及文件**：
- `src/config/templates.ts` — 新建

**数据结构**：
```typescript
interface Template {
  id: string;                    // 如 'm01-cube'
  name: string;                  // 如 '正方体'
  icon?: string;                 // lucide icon 名或自定义 SVG 组件名
  status: 'available' | 'coming'; // 是否已实现
  geometryType?: GeometryType;   // M-01 模板专属，对应已有几何体
}

interface Module {
  id: string;                    // 如 'm01'
  code: string;                  // 如 'M-01'
  name: string;                  // 如 '立体几何展示台'
  description: string;           // 一句话描述
  priority: 'P0' | 'P1' | 'P2' | 'P3';
  status: 'available' | 'coming';
  templates: Template[];
}

interface Subject {
  id: string;                    // 如 'math'
  name: string;                  // 如 '数学'
  icon: string;
  modules: Module[];
}

export const SUBJECTS: Subject[]
```

**覆盖范围**：
- 数学：6 个模块（M-01 ~ M-06），M-01 的 13 种几何体为 available，其余 coming
- 物理：14 个模块（P-01 ~ P-14），全部 coming
- 化学：9 个模块（C-02 ~ C-09），全部 coming

每个模块下列出主要模板（不需要列所有 500+ 场景，每个模块挑 3-8 个代表性模板即可）。
