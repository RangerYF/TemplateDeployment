# 🎯 V0.5 渐进式开发计划 — 用户系统 & 作品管理

## 📌 版本目标
用户可以登录系统，在工作台管理自己的作品（新建、打开、删除），在编辑器中保存作品到云端，下次打开时恢复上次编辑状态。

## 🔄 开发流程总览
```
阶段1：基础设施（路由 + API客户端 + 认证状态）
  → 阶段2：登录与鉴权（登录页 + Token管理 + 路由守卫）
    → 阶段3：工作台页面（作品列表 + 搜索排序 + 新建/删除/打开）
      → 阶段4：编辑器持久化（保存/加载 + 自动保存 + 缩略图截取）
        → 阶段5：模板推荐 & Mock（工作台模板卡片 + 使用模板新建 + Mock模式整合）
          → 阶段6：多学科模板体系（学科Tab + 模块分组 + AI预留）
```

## 📋 串行执行阶段

### 第1阶段：基础设施搭建 ✅ 已完成

**目标**：项目具备多页面路由能力、统一的 API 请求层和认证状态管理，为后续所有功能提供底层支撑。

**主要任务**：
1. 安装依赖（react-router-dom、及其他必要库）
2. 搭建路由结构：登录页、工作台页、编辑器页三个路由
3. 改造 `main.tsx` 和 `App.tsx`，将现有编辑器包裹在路由中
4. 创建统一 API 客户端（基于 fetch 封装，支持 Bearer Token 自动注入、错误统一处理）
5. 创建认证 Store（Zustand）：Token 存储/读取（localStorage）、登录态判断、登出清理
6. 环境变量配置：API Base URL 通过 `.env` / Vite 环境变量管理

**子任务链路**：
安装依赖 → 环境变量配置 → API 客户端封装 → 认证 Store → 路由结构搭建 → 改造入口文件

**涉及文件范围**：
- `package.json` - 新增 react-router-dom 依赖
- `.env.development` / `.env.production` - API Base URL 配置
- `src/lib/api/` - API 客户端、请求/响应类型定义
- `src/editor/store/authStore.ts` - 认证状态管理
- `src/main.tsx` - 包裹 BrowserRouter
- `src/App.tsx` - 改造为路由出口
- `src/router/` - 路由配置与路由守卫
- `vite.config.ts` - 开发环境 API 代理配置（待 Base URL 确认后）

**验收标准**：
✅ 浏览器访问 `/login` 显示占位登录页，`/workspace` 显示占位工作台页，`/editor` 显示现有编辑器
✅ API 客户端可发起请求并自动携带 Token（若存在）
✅ 认证 Store 能正确读写 localStorage 中的 Token
✅ `pnpm lint && pnpm tsc --noEmit` 通过

**本阶段产出**：
路由骨架 + API 客户端 + 认证 Store。下一阶段在此基础上实现登录页 UI 和鉴权流程。

### 第4阶段：编辑器持久化与保存策略 ✅ 已完成

**目标**：编辑器能从后端加载已有作品数据进行编辑，支持手动保存、定时自动保存和离开保存，保存时同步更新缩略图。

**主要任务**：
1. 编辑器路由改造：从 URL 获取 `projectId`，加载对应作品的 `scene_data`
2. scene_data 序列化/反序列化：entityStore 的 `getSnapshot()` ↔ 后端 `scene_data` 字段的映射
3. 手动保存按钮（编辑器顶栏添加保存按钮，调用 PUT 接口更新 scene_data）
4. 保存状态指示（未保存/保存中/已保存）
5. 定时自动保存（检测变更后定时触发，避免无变更时重复保存）
6. 离开页面保存（beforeunload 事件 + 路由离开拦截）
7. 缩略图截取（从 Three.js Canvas 截图生成图片数据，保存时作为 thumbnail 上传）
8. 新建作品流程打通：从工作台新建 → 编辑器创建空场景 → 首次保存时调用 POST 或 PUT
9. 作品名称编辑（编辑器内可修改作品名称）

**子任务链路**：
scene_data 序列化方案确认 → 编辑器加载已有作品 → 手动保存 + 保存状态指示 → 缩略图截取 → 定时自动保存 → 离开页面保存 → 新建作品流程 → 作品名称编辑

**涉及文件范围**：
- `src/editor/store/` - 可能新增 projectStore（当前作品信息、保存状态、自动保存逻辑）
- `src/editor/store/entityStore.ts` - 确认 getSnapshot/loadSnapshot 与 scene_data 的兼容性
- `src/components/layout/TopBar.tsx` - 添加保存按钮、保存状态、作品名称
- `src/components/scene/Scene3D.tsx` - 缩略图截取（Canvas toDataURL）
- `src/lib/api/projects.ts` - 复用已有 API 函数
- `src/App.tsx` / 编辑器页面组件 - 加载逻辑、路由参数处理

**验收标准**：
✅ 从工作台打开已有作品，编辑器正确恢复该作品的几何体和场景状态
✅ 点击保存按钮，作品数据成功写入后端，保存状态正确显示
✅ 编辑后等待一段时间，自动保存触发（可在网络面板观察到 PUT 请求）
✅ 关闭/刷新页面前触发保存
✅ 保存后工作台的作品缩略图已更新
✅ 新建作品 → 编辑 → 保存 → 返回工作台 → 重新打开，数据完整
✅ `pnpm lint && pnpm tsc --noEmit` 通过

**本阶段产出**：
编辑器与后端完全打通，作品数据可靠持久化。V0.5 核心功能全部完成。
