# 第5阶段：模板推荐 & Mock 模式

**所属版本**：V0.5
**风险等级**：L1（常规风险）
**流程路径**：MODE 0 → MODE 3 → MODE 4 → MODE 5 → MODE 6
**前置依赖**：阶段1-4已完成

## 目标

1. 工作台默认展示几何体模板卡片，用户 hover 后点击"使用模板"即可新建作品并进入编辑器
2. 移除原有的"新建作品"命名弹窗（使用模板 = 新建）
3. 整合 Mock 模式，使后端未就绪时可完整体验全部 UI 流程

## 背景说明

当前工作台的核心交互是"用户作品列表"，但实际使用场景中，用户第一步应是选择一个几何体模板开始创建。模板数据来自已有的 `GEOMETRY_LIST`（13 种几何体）和 `GEOMETRY_GROUPS`（4 个分组），无需后端接口。

### 5.3 "使用模板"流程实现

**要做什么**：点击"使用模板"后，创建作品并跳转编辑器，编辑器初始化为对应几何体。

**涉及文件**：
- `src/pages/WorkspacePage.tsx` — 处理 onUse 回调
- `src/pages/EditorPage.tsx` — 支持从 URL 参数或 state 接收初始几何体类型

**流程**：
```
用户点击"使用模板"（如选择"圆锥"）
  → createProject({ name: "圆锥" })  // 以几何体名称作为默认作品名
  → navigate(`/editor/${project.id}`, { state: { geometryType: 'cone' } })
  → EditorPage 检测到 state.geometryType
  → 如果 scene_data 为 null（新建作品），使用指定几何体类型初始化（而非默认正方体）
```

**init.ts 改造**：
- `initEditor()` 当前硬编码创建 `cube`，需支持传入 `geometryType` 参数
- 新增签名：`initEditor(geometryType?: GeometryType)`，默认 `'cube'`
