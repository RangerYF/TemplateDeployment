## 技术要点

### 旧组件内联策略

**CoordinateAxes 内联**（最复杂，~158行）：
- 旧组件使用 `useGeometryStore.positionOverrides` 计算顶点位置
- 新版本：通过 EntityStore 中 builtIn point 的 `computePointPosition()` 获取位置
- 核心渲染逻辑（XYZ 轴线 + Html 标签 + 坐标值文字）直接移入 CoordSystemRenderer

**CircumSphere 内联**（简单，~30行）：
- 仅渲染一个 `<mesh>` + `<sphereGeometry>` + `<meshStandardMaterial transparent opacity={0.15}>`
- 加上 Html 标签显示半径值

**CircumCircle 内联**（简单，~48行）：
- 渲染一个 `<mesh>` 使用 `RingGeometry` 或自定义 geometry
- 加上 Html 标签显示半径值

### engine/ 中 types/scene.ts 的依赖处理

`src/engine/math/crossSection.ts` 和 `sphereCrossSection.ts` 引用了 `PointRef` 和 `CustomPoint` 类型。检查方案：

1. 如果这两个 engine 函数**仍被新系统调用**（如 CrossSectionTool 或 CreateCrossSectionCommand 内部）→ 将 PointRef/CustomPoint 类型移到 `src/engine/types.ts` 或 `src/editor/entities/types.ts`
2. 如果这两个 engine 函数**已被新系统完全替代**（CreateCrossSectionCommand 自己实现了截面计算）→ 可以连同 engine 函数一起清理

需在执行前确认。

### AuxiliaryTools 清理后的结构

清理后 AuxiliaryTools 应只包含：
- 辅助功能开关区（坐标系/外接球/外接圆 — 通过 EntityStore + Command）
- 展开图/三视图开关（通过 uiStore）
