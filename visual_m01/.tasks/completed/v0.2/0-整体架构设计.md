### T0.2 七系统职责边界与依赖关系设计

**目标**：明确 Entity / Command / Event / Tool / Selection / Property / Renderer 各系统的职责、对外接口、系统间调用方向。

**动作**：
1. 定义每个系统的职责边界（做什么、不做什么）

**产出**：系统依赖关系图 + 各系统职责说明

### T0.3 Entity 类型体系完整定义

**目标**：定义所有实体类型的 properties 结构，包括字段名、类型、引用关系。

**动作**：
1. 设计 Entity 基础接口（id, type, properties, visible）
2. 定义 geometry 实体 properties（几何体参数 + BuilderResult 缓存策略）
3. 定义 point 实体 properties（统一模型：顶点/棱上点/面上点/坐标输入点）
4. 定义 segment 实体 properties（端点引用、颜色、虚实、长度）
5. 定义 coordinateSystem 实体 properties（Singleton，原点引用、轴方向）
6. 定义 circumSphere / circumCircle 实体 properties
7. 定义 crossSection 实体 properties
8. 明确 Singleton 与普通实体的区分规则
9. 确保所有 properties 可 JSON 序列化

**产出**：完整的 Entity 类型定义表

---

### T0.4 数据流全链路设计

**目标**：设计从用户操作到渲染更新的完整数据流链路。

**动作**：
1. 设计主链路：用户操作 → Tool 捕获 → Command 封装 → EntityStore 变更 → Event 广播 → Renderer 更新 / Inspector 刷新
2. 设计连续操作（拖拽/滑块）的 Command 粒度控制方案
3. 设计依赖更新流程（被引用实体变化时，引用方的重算触发机制）
4. 设计 Undo/Redo 数据流
5. 设计几何体类型切换的"重置性操作"处理流程

**产出**：数据流链路图 + 各典型操作的流程描述

---

### T0.5 系统间接口契约定义
