## 子任务清单

### 批次A：三个 EntityRenderer

#### T5.1 CoordSystemRenderer

**目标**：坐标系 Entity 的 3D 渲染。

**产出文件**：`src/components/scene/renderers/CoordSystemRenderer.tsx`

**实现内容**：
- 接收 `entity: Entity<'coordinateSystem'>`
- **数据获取**：
  - 从 EntityStore 获取 `originPointId` 对应的 Point Entity
  - 从 Point Entity 的 constraint 提取 `vertexIndex`（坐标系仅支持 `constraint.type === 'vertex'` 的顶点）
  - 从 `useBuilderResult(geometryId)` 获取 BuilderResult
  - 从 EntityStore 获取 geometry Entity 的 `geometryType`
- **计算**：
  - 调用 `buildCoordinateSystem(geometryType, builderResult, vertexIndex)` 获取 `CoordinateSystem`
  - 考虑 positionOverride：如果原点顶点被拖拽，坐标系原点位置需跟随
- **渲染**：
  - 复用现有 `CoordinateAxes` 组件（传入计算结果）
  - 或直接内联渲染：轴线(R/G/B) + 刻度 + 轴标签(X/Y/Z) + 顶点坐标值标签
  - 样式：X=红(#ef4444), Y=绿(#22c55e), Z=蓝(#3b82f6)
- **注册**：调用 `registerRenderer('coordinateSystem', CoordSystemRenderer)`

---

#### T5.2 CircumSphereRenderer

**目标**：外接球 Entity 的 3D 渲染。

**产出文件**：`src/components/scene/renderers/CircumSphereRenderer.tsx`

**实现内容**：
- 接收 `entity: Entity<'circumSphere'>`
- **数据获取**：
  - 从 EntityStore 获取 geometry Entity 的 `geometryType` + `params`
  - 从 `useBuilderResult(geometryId)` 获取 BuilderResult（用于计算球心位置，考虑顶点拖拽）
- **计算**：
  - 调用 `computeCircumscribedSphere(geometryType, params)` 获取 `{ center, radius }`
  - 球体几何类型返回 null → 不渲染
- **渲染**：
  - 复用现有 `CircumSphere` 组件或内联渲染
  - 紫色(#8b5cf6) 线框球，opacity=0.3
  - 半径值标签（显示精确值 radiusLatex）
- **注册**：调用 `registerRenderer('circumSphere', CircumSphereRenderer)`

---

#### T5.3 CircumCircleRenderer

**目标**：外接圆 Entity 的 3D 渲染。

**产出文件**：`src/components/scene/renderers/CircumCircleRenderer.tsx`

**实现内容**：
- 接收 `entity: Entity<'circumCircle'>`
- **数据获取**：
  - 从 Entity.properties.pointIds 获取 3 个 Point Entity ID
  - 通过 `usePointPosition` Hook 获取三个点的实时 3D 位置（含 positionOverride 考虑）
- **计算**：
  - 调用 `computeCircumscribedCircle(p1, p2, p3)` 获取 `{ center, radius, normal }`
  - 三点共线时返回 null → 不渲染
- **渲染**：
  - 复用现有 `CircumCircle` 组件或内联渲染
  - 橙色(#f59e0b) 圆环，64段分割
  - 半径值标签（显示精确值 radiusLatex）
  - 通过法向量计算平面内圆周点
- **注册**：调用 `registerRenderer('circumCircle', CircumCircleRenderer)`

#### T5.6 InspectorRegistry 框架

**目标**：建立实体类型 → Inspector 组件的注册表（为阶段6的完整 Property 系统打基础）。

**产出文件**：`src/components/panels/inspectors/index.ts`

**实现内容**：
```typescript
type InspectorComponent = React.ComponentType<{ entity: Entity }>;
const inspectorRegistry: Record<string, InspectorComponent> = {};

function registerInspector(type: EntityType, component: InspectorComponent): void;
function InspectorPanel(): JSX.Element | null {
  const primaryId = useSelectionStore(s => s.primaryId);
  const entity = useEntityStore(s => primaryId ? s.getEntity(primaryId) : undefined);
  if (!entity) return null;
  const Inspector = inspectorRegistry[entity.type];
  if (!Inspector) return null;
  return <Inspector entity={entity} />;
}
```

## 技术要点

### 坐标系原点索引推导

CoordinateSystem Entity 存储的是 `originPointId`（Point Entity ID），但 `buildCoordinateSystem()` 需要 `originIndex`（顶点索引）。转换方式：

```typescript
const pointEntity = getEntity(originPointId);
const constraint = pointEntity.properties.constraint;
// 坐标系仅支持 vertex 约束的点
if (constraint.type === 'vertex') {
  const originIndex = constraint.vertexIndex;
  const coordSystem = buildCoordinateSystem(geometryType, builderResult, originIndex);
}
```

### 外接圆的响应式更新

外接圆引用 3 个 Point Entity，当这些点的位置变化（拖拽、参数变化）时，渲染器需自动更新：
- 使用 `usePointPosition` Hook 获取三个点的实时位置
- `useMemo` 依赖三个位置 → `computeCircumscribedCircle` 自动重算

### Entity 可见性 vs 存在性

ARCH 设计中，辅助功能的开/关对应 Entity 的创建/删除（而非 visible 属性切换）：
- 开启坐标系 = 创建 CoordinateSystem Entity
- 关闭坐标系 = 删除 CoordinateSystem Entity
- 好处：undo 恢复时语义清晰（undo 删除 = 恢复实体）

### 旧组件复用策略

旧的 `CoordinateAxes.tsx` / `CircumSphere.tsx` / `CircumCircle.tsx` 可以：
- **方案A**：新 Renderer 内部调用旧组件（传入计算结果），最小改动
- **方案B**：将旧组件渲染逻辑内联到新 Renderer 中，一步到位

推荐**方案A**：旧组件的渲染逻辑已验证正确（V0.1），内部调用避免重写渲染细节，降低视觉回退风险。阶段8清理时再决定是否合并。
