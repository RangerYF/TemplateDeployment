# C-08 酸碱滴定与pH模拟器 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中化学教师，覆盖高中必修（酸碱中和滴定基础）和高中选修（分析化学方法、缓冲溶液）学段。
- **解决的教学痛点**：
  - 酸碱滴定 pH 曲线是高考重点，但传统教学只能用教材上的静态曲线图，无法动态展示不同滴定类型的曲线差异，也无法让学生直观感受"突跃"过程。
  - 指示剂选择的合理性（变色范围与等当点的关系）是常见易错点，静态图无法直观展示指示剂变色范围与曲线突跃区间的叠加关系，以及由此导致的终点误差方向。
  - 缓冲溶液概念抽象（"为什么加酸/碱后 pH 变化不大"），缺乏对比可视化手段。
  - 不同滴定类型（强酸滴强碱 vs 强碱滴弱酸等）的曲线差异需要叠加对比才能有效教学，纸质教材无法实现。
- **核心产品价值**：一句话——**通过参数可调的 pH-V 滴定曲线实时绘制，配合指示剂变色范围叠加和终点误差标注，让教师在课堂上直观演示滴定分析的核心概念。**

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 选择要讲解的滴定类型 → 调整浓度和体积参数 → 预览曲线效果 → 记录关键数据点用于课件 |
| **课中演示** | 投屏打开模拟器 → 选择滴定类型（如强碱滴弱酸）→ 设定浓度/体积 → 实时绘制 pH-V 曲线 → 指示曲线上的关键标注点（起始pH、半当量点、等当点、突跃范围）→ 叠加指示剂变色范围色带 → 分析终点误差方向 |
| **对比教学** | 在同一坐标系叠加 2-3 条不同类型的滴定曲线 → 对比起始pH、突跃范围、等当点pH的差异 → 解释为什么不同滴定类型需要不同指示剂 |
| **缓冲溶液教学** | 切换到缓冲溶液演示 → 选择缓冲体系（醋酸-醋酸钠/氨-氯化铵/血液缓冲）→ 对比缓冲溶液与纯水加酸碱后的 pH 变化柱形图 |

### 怎么用才是"好用"的？

- **快**：选择滴定类型后曲线即时绘制，无等待。参数调整后曲线实时更新。
- **直观**：突跃范围用色带标注一目了然；指示剂变色范围叠加在曲线上，误差方向（偏大/偏小）有明确标注。
- **对比方便**：同一坐标系叠加多条曲线，颜色区分，无需切换页面。
- **教学导向**：关键点（半当量点、等当点）有文字标注和理论 pH 值，而非需要教师口头补充。

### 交互频率和深度

- **频率**：中低频工具。主要用于酸碱中和滴定章节（高中必修约 2-3 课时）和选修分析化学部分，非全学期常用。
- **深度**：浅度交互。单次使用约 3-5 分钟（选类型 → 看曲线 → 讲标注 → 对比），操作路径短。

## 3. 核心功能点

### 功能清单（按重要性排序）

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | pH-V 曲线实时绘制 | 选择滴定类型 + 设定浓度体积 → 实时绘制曲线，曲线光滑连续 |
| P0 | 曲线关键点标注 | 起始 pH、半当量点（弱酸/弱碱时 pH = pKa）、等当点/终点、突跃范围（色带） |
| P0 | 指示剂变色范围叠加 | 在曲线图上叠加 4 种指示剂（石蕊/酚酞/甲基橙/甲基红）的变色范围色带 |
| P0 | 滴定终点误差分析 | 标注等当点理论 pH、指示剂变色点 pH，当两者不重合时标注误差方向（偏大/偏小） |
| P1 | 滴定曲线对比 | 同一坐标系叠加最多 3 条不同类型的滴定曲线，颜色区分 |
| P1 | 预设滴定组合 | 4 种预设组合（强酸滴强碱/强碱滴强酸/强碱滴弱酸/强酸滴弱碱），含推荐指示剂 |
| P1 | 基准物质概念说明 | 界面中简要说明滴定剂标定原理，提及 Na₂CO₃ 和草酸 |
| P2 | 缓冲溶液演示 | 3 种预设缓冲体系（醋酸-醋酸钠/氨-氯化铵/血液缓冲），双柱形图对比 pH 变化 |

### 需要实体管理的功能

- **不需要实体管理**。本模块无需创建/选中/删除/编辑对象。用户操作的是"选择滴定类型"和"调整参数"，产出的是自动生成的曲线图表，无用户自建实体概念。

### 需要 undo/redo 的功能

- **不需要**。本模块是参数驱动的可视化工具，操作可逆性通过"重新选择类型"或"重置参数"实现，无需命令系统。

### 参数驱动的功能（调参数 → 画面实时变化）

- **浓度/体积参数**：调整初始浓度和体积 → 曲线形状和关键点位置实时变化。
- **滴定类型选择**：切换类型 → 曲线整体重绘。
- **指示剂叠加开关**：勾选/取消指示剂 → 色带叠加/移除。

### 模拟/动画驱动的功能

- **轻度动画需求**：PRD 动画规范（3.4节）适用于本模块，但由于已删除滴定操作动画，实际动画需求仅限于曲线绘制过程的渐进动画（可选）。主要是静态曲线展示，非时间轴驱动的模拟。

## 4. 核心依赖识别

### 技术依赖

| 维度 | 判断 |
|------|------|
| **渲染方式** | **DOM + Chart 库**。pH-V 曲线是标准的 2D 折线图/曲线图，使用 Chart.js / Recharts / ECharts 等图表库即可满足。缓冲溶液演示的柱形图同理。不需要 Canvas 自绘或 3D WebGL。 |
| **数学计算需求** | 中等。需要：(1) 根据酸碱类型、浓度、体积计算每个滴加点的 pH 值（强酸强碱为解析公式，弱酸弱碱涉及 Ka/Kb 的迭代近似或近似公式）；(2) 等当点、半当量点、突跃范围的自动计算；(3) 缓冲溶液 Henderson-Hasselbalch 方程的计算。计算量不大，纯前端可实时完成。 |
| **公式渲染需求** | 轻度。化学式下标（NaOH、CH₃COOH）和 pH 公式（pH = pKa + log[A⁻]/[HA]）需要简单渲染，可用 HTML/CSS 上下标或轻量 KaTeX 片段。 |

### 交互依赖

| 维度 | 判断 |
|------|------|
| **交互模式** | **参数驱动型**。用户选择类型+调参数 → 系统自动计算并绘制曲线。无编辑器特征，无模拟时间轴。 |
| **关键交互** | 下拉选择（滴定类型）、滑块/输入框（浓度、体积）、复选框（指示剂叠加开关、曲线叠加开关）、图表 hover 提示（显示具体 pH 值和体积）、Tab 切换（曲线绘制 / 曲线对比 / 缓冲演示） |
| **是否需要实体系统** | **不需要**。无用户自建对象。 |
| **是否需要命令系统** | **不需要**。无 undo/redo 需求。 |

## 5. 竞品分析

### PhET pH Scale（phet.colorado.edu）

- **产品形态**：Web / 桌面应用（Java/HTML5），免费教育模拟器
- **技术实现**：HTML5 Canvas 自研渲染引擎（Scenery），内置物理/化学模型
- **优点**：交互设计精良，拖拽式操作直觉化；酸碱浓度可视化做得好（分子数量动态变化）；品牌知名度高，教师信任度高
- **不足**：pH Scale 模块侧重 pH 概念理解（酸碱强弱对比），不提供滴定曲线绘制功能；无指示剂变色范围叠加；无终点误差分析；无中文本地化
- **启示**：PhET 的交互设计理念（极简参数 + 实时反馈）值得参考；但其 pH Scale 模块与滴定曲线是不同定位，我们的模块更聚焦于滴定分析教学

### 虚拟滴定实验（各教育平台）

- **产品形态**：Web，通常嵌入教育平台（如 NOBOOK、仰望星空等）
- **技术实现**：Canvas 2D 动画 + 简单数学模型
- **优点**：提供完整的滴定操作模拟（添加试剂、摇锥形瓶、观察颜色变化），沉浸感强
- **不足**：重操作模拟轻数据分析，曲线展示往往是结果展示而非实时绘制；指示剂选择和终点误差分析通常缺失；教师评价"点击拖拽图标并非真正的交互性"
- **启示**：PRD 已明确删除滴定操作动画，我们的定位是"数据分析工具"而非"操作模拟器"，这是正确的差异化方向

### 在线 pH 计算器（如 wolfram|alpha、chemicalaid.com）

- **产品形态**：Web，工具型
- **技术实现**：后端计算 + 前端图表渲染
- **优点**：计算精确，支持任意酸碱组合和复杂体系
- **不足**：面向化学专业，非教学场景；无曲线标注（突跃范围、指示剂叠加）；无教学引导；界面复杂
- **启示**：计算精度可参考（Henderson-Hasselbalch 公式和强酸强碱精确公式），但产品形态完全不同

### 竞品总结对我们的启示

1. **定位差异化明确**：竞品要么是概念理解工具（PhET），要么是操作模拟器（虚拟实验），要么是专业计算器——我们聚焦"pH 曲线绘制 + 教学标注 + 终点误差分析"是一个空白的细分定位。
2. **轻量即优势**：作为小品模块，无需追求全面性，4 种滴定类型 + 4 种指示剂 + 3 种缓冲体系的预设数据完全够用。
3. **标注是核心价值**：曲线本身任何图表库都能画，差异化在于教学标注（突跃范围色带、指示剂变色范围叠加、误差方向箭头），这是教师最需要的功能。

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：DOM + React 图表库。推荐 Recharts 或 ECharts（React 封装）。理由：(1) pH-V 曲线是标准 2D 折线图，图表库原生支持；(2) 曲线标注（reference line、area band）图表库均有内置支持；(3) 指示剂色带叠加可用图表库的 ReferenceArea / markArea 组件实现；(4) 缓冲溶液柱形图同样是图表库原生场景。
- **不需要 Canvas/WebGL**：无 3D 需求，无复杂自绘需求。标准图表库 + DOM 即可覆盖全部渲染场景。
- **公式渲染**：化学式上下标用 HTML `<sub>`/`<sup>` 即可，Henderson-Hasselbalch 等公式如有展示需求可用轻量 KaTeX。

### 推荐的架构模式

- **参数驱动型轻量工具架构**：
  - **数据层**：静态内置数据（4 种滴定类型参数、4 种指示剂数据、3 种缓冲体系数据），JSON 常量即可，无需独立数据文件。
  - **计算层**：pH 计算引擎（核心模块），输入为滴定类型+浓度+体积，输出为 pH-V 数据点数组。包含：强酸强碱解析公式、弱酸弱碱近似公式（Henderson-Hasselbalch）、等当点/半当量点/突跃范围自动检测算法。
  - **图表层**：React 图表组件，接收数据点数组 + 标注配置 → 渲染曲线 + 标注。
  - **UI 层**：React 组件（滴定类型选择、参数输入面板、指示剂勾选、Tab 切换）。
- **与其他模块的关系**：本模块高度独立，无需与其他化学模块共享核心层。唯一可共享的是 UI 组件库（按钮、滑块、面板布局等通用组件）。

### 关键技术风险或难点

| 风险/难点 | 严重性 | 说明 |
|-----------|--------|------|
| **弱酸/弱碱滴定 pH 计算精度** | 低 | 弱酸弱碱体系的 pH 计算涉及 Ka 的多次迭代求解（三次方程），但中学教学场景对精度要求不高，可用 Henderson-Hasselbalch 近似公式，仅在等当点附近用精确解。成熟公式，无算法风险。 |
| **曲线突跃范围自动检测** | 低 | 需要从 pH-V 数据点中自动识别突跃区间（dPH/dV 变化率最大的区间）。算法简单：计算一阶差分，取峰值前后一定范围即可。 |
| **图表标注的视觉清晰度** | 低 | 多条曲线叠加 + 多个指示剂色带 + 关键点标注可能导致视觉拥挤。需要合理的颜色方案和图层管理。通过交互控制（勾选显隐）可缓解。 |
| **整体风险评估** | **极低** | 这是一个数据量小（内置常量）、计算简单（解析公式）、渲染轻量（图表库）、交互简单（选择+滑块）的小品模块。无 3D 渲染、无实体系统、无复杂状态管理、无外部数据依赖。开发周期预计短，技术风险极低。 |
