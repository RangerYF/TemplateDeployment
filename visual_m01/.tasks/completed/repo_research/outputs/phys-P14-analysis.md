# P-14 机械能守恒 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程或软件操作基础。核心用户画像是需要讲解力学能量守恒定理的高中物理教师，覆盖从自由落体到复杂多体系统的全部机械能模型。

### 解决什么教学痛点
- **能量转化不可见**：动能、势能、弹性势能的相互转化是抽象的数值变化，黑板上只能列公式，学生缺乏直观感知
- **多模型难以对比**：自由落体、斜面滑块、弹簧振子、轻杆圆周等模型在公式层面类似但物理图景差异大，逐个手工推导效率低
- **摩擦损耗理解困难**：有摩擦时总机械能递减、摩擦热递增的动态过程无法在黑板上呈现
- **弹性势能不可直接测量**：弹簧压缩/伸长过程中弹性势能的变化规律学生难以建立直觉
- **多物体能量分配**：阿特伍德机、轻杆双球等模型中，多个物体的能量如何分配和转化，口述难以清晰

### 核心产品价值
**一句话**：老师选一个能量模型、调几个参数，物体运动动画和能量条形图同步实时变化，一眼看清"动能增 = 势能减"，覆盖高中机械能守恒全部核心模型，支持有/无摩擦对比。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选择模型（如"斜面滑块"）→ 设参数（质量2kg、角度30°、斜面长5m）→ 切换光滑/有摩擦模式预览效果 → 记录关键时刻截图备课 |
| **课中演示** | 投影仪打开 → 点击播放，学生看到物体运动 + 右侧能量条实时变化 → 暂停在某位置，指出"此时 Ek = X, Ep = Y, 总和不变" → 切换有摩擦模式，学生对比总能量递减 |
| **讲题** | 选择"阿特伍德机"模型 → 设 m1=3kg, m2=2kg → 播放动画，指出"m1 下降势能减少、m2 上升势能增加、两物体动能增加" → 暂停验证公式 (m1-m2)gh = (1/2)(m1+m2)v² |

### 怎么用才是"好用"的
- **3 分钟上手**：选模型 → 调参数 → 点播放，三步完成一次演示
- **能量条形图是核心**：右侧能量条（动能红色、势能蓝色、弹性势能绿色、总能灰色）必须在运动全程实时更新，且在投影仪上一目了然
- **暂停即分析**：暂停时数值放大显示，并自动验证"Ek + Ep = E总 ✓"
- **有/无摩擦一键对比**：无需重新设参数，一键切换即可对比

### 交互频率和深度
- **频率**：每节机械能守恒课使用 2-4 次，每次 3-10 分钟
- **深度**：中等偏轻 — 主要是参数调整 + 播放/暂停 + 模式切换，无复杂编辑操作

### 哪些功能需要实体管理（创建/选中/删除/编辑对象）
**不需要传统实体管理**。P-14 是典型的"模拟演示型"模块，不涉及实体的创建/删除/选中/编辑。物体（球/滑块/弹簧等）由模型预设生成，用户只调参数，不手动创建或删除对象。

### 哪些功能需要 undo/redo
**不需要 undo/redo**。参数修改即时生效且可逆（调回来即可），动画通过时间轴控制前进/后退，无需命令系统。

### 哪些功能是参数驱动的（调参数 → 画面实时变化）
- 所有模型的物理参数（质量 m、初始高度 h、初速度 v0、角度 θ、弹簧系数 k、摩擦因数 μ 等）→ 运动轨迹和能量曲线实时重算
- 有/无摩擦切换 → 能量条行为变化
- 重力势能零点拖动 → Ep 数值和条形图高度变化
- 重力加速度 g 可调 → 所有计算实时更新

### 哪些功能是模拟/动画驱动的（播放/暂停/逐步）
- 所有 6 种模型的运动过程动画（物体位移、弹簧形变、摆的摆动、杆的旋转等）
- 能量条形图的实时随动更新
- 时间轴控制（播放/暂停/单步/调速/倒放）

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：2D Canvas 或 SVG — 所有运动场景（自由落体、斜面、弹簧、单摆、轻杆圆周、阿特伍德机）均为 2D 侧视图
- **能量条形图**：Canvas 自绘或 Chart.js/D3.js — 右侧垂直柱状图实时更新，需要高帧率同步
- **公式渲染**：KaTeX — 暂停时展示能量公式和数值

#### 数学/物理计算需求
- **中度**：各模型的运动方程数值积分
  - 自由落体/竖直上抛：解析解（匀变速运动公式）
  - 斜面滑块：解析解（含摩擦力的匀变速直线运动）
  - 阿特伍德机：解析解，a = (m1-m2)g/(m1+m2)
  - 弹簧振子：SHM 解析解 x = x0·cos(ωt)，有阻尼时需数值积分
  - 单摆：小角度近似解析解，大角度需数值积分（如 RK4）
  - 轻杆双球圆周：需数值积分（非线性耦合运动方程）
- **轻度**：各种能量的实时计算
  - Ek = (1/2)mv²
  - Ep = mgh
  - Ep弹 = (1/2)kx²
  - Wf = μmg·cosθ·d
  - E总 = Ek + Ep + Ep弹
- **轻度**：向心力条件判断（轻杆双球模型最高点临界条件）

#### 公式渲染需求
- 暂停时展示当前时刻的能量值
- 守恒验证等式："Ek + Ep = E总 ✓"
- 各模型的核心公式（如 (m1-m2)gh = (1/2)(m1+m2)v²）

### 交互依赖

#### 交互模式
**模拟演示型**（主导）+ **参数驱动型**（辅助）

这是一个典型的模拟演示型应用：
- 核心交互是"播放 → 观察 → 暂停 → 分析"
- 无实体创建/删除/选中
- 无 Undo/Redo 需求
- 参数通过滑块/输入框驱动，修改后重置动画

#### 关键交互

| 交互类型 | 具体场景 |
|---------|---------|
| 按钮组点选 | 顶部模型选择（6 种模型切换） |
| 滑块 | 质量/高度/角度/弹簧系数/摩擦系数/初速度/g 值等 |
| 数字输入框 | 精确参数设置 |
| 播放控制 | 播放/暂停/单步前进/后退/调速/倒放 |
| 拖拽 | 重力势能零点拖动选择 |
| 切换开关 | 光滑/有摩擦模式切换 |

#### 是否需要实体系统（统一的对象模型）
**否** — P-14 中的"物体"由模型预设固定，无用户自定义创建/删除需求。状态管理只需维护：当前模型类型 + 参数值 + 模拟时间 + 运动状态。

#### 是否需要命令系统（undo/redo）
**否** — 参数修改可逆（滑块滑回来即可），动画通过时间轴回退，无需命令栈。

## 5. 竞品分析

### 5.1 PhET Energy Skate Park（核心竞品）

- **形态**：Web（HTML5），全平台浏览器直接运行，支持移动端
- **开发者**：University of Colorado Boulder，PhET Interactive Simulations
- **技术**：HTML5 Canvas + SceneryStack 自研渲染框架，JavaScript
- **核心功能**：
  - 滑板场景中的能量守恒演示
  - 实时能量条形图（动能/势能/热能）和饼图
  - 可自定义轨道形状（拖拽控制点建轨道）
  - 可调质量、摩擦系数
  - 支持不同行星（不同 g 值）和太空场景
  - 速度/加速度矢量显示
- **优势**：
  - 全球最知名的物理教育模拟工具，用户量极大
  - 免费开源，教学资源丰富（大量配套教案和 worksheet）
  - 自由度高 — 可自建任意形状轨道
  - 能量条形图 + 饼图双可视化
  - 多行星/太空探索，趣味性强
- **不足**：
  - 仅"滑板公园"单一场景，无法演示阿特伍德机、弹簧振子、轻杆圆周等经典物理模型
  - 未提供斜面滑块、定滑轮双物体等高中常见计算模型
  - 无有/无摩擦同屏对比功能
  - 无弹性势能可视化（仅重力势能 + 动能 + 热能）
  - 不贴合中国高中物理教材的具体模型需求
- **启示**：能量条形图的实时同步更新是核心竞争力验证，用户直觉就是"看柱子高低变化理解能量转化"

### 5.2 The Physics Classroom — Work-Energy Bar Charts Interactive

- **形态**：Web（HTML5），纯浏览器

- **核心功能**：

  - 偏练习工具而非演示工具，无运动动画
  - 无实时物理模拟
  - 场景固定，不可调参数
- **启示**：能量条形图概念在教学中被广泛接受和使用，是通用教学语言

### 5.3 SimuFísica — Conservation of Mechanical Energy Simulator

- **形态**：Web + 移动端（Android/iOS）+ 桌面（Windows/Linux）
- **开发者**：巴西 Universidade Federal de Rondônia 物理系
- **技术**：多平台原生开发，运动方程数值积分（非线性 ODE 求解）
- **核心功能**：
  - 弹簧-重力系统的机械能守恒模拟

  - 可调弹簧系数、质量、初始位移等参数

- **优势**：
  - 基于严格物理方程的数值模拟，精度高
  - 多平台覆盖（Web + 移动 + 桌面），离线可用

  - 仅弹簧+重力单一系统，模型覆盖面很窄

  - 无有/无摩擦对比功能

### 5.6 市场空白与 P-14 定位

**核心发现**：**覆盖中国高中物理全部经典机械能模型的 Web 端工具不存在。**

- PhET Energy Skate Park 是能量教学的标杆，但仅覆盖"滑板公园"单一场景
- SimuFísica 和 oPhysics 仅覆盖弹簧振子模型
- Physics Classroom 偏练习工具，无实时模拟

- 无竞品提供有/无摩擦同屏对比功能

P-14 的差异化定位：**唯一在 Web 端以能量条形图为核心可视化手段、覆盖 6 种中国高中物理经典机械能模型、支持有/无摩擦一键对比的机械能守恒教学工具**。

## 6. 初步技术判断

### 推荐的渲染技术

**2D Canvas（主）+ Chart.js/自研 Canvas 图表（能量条形图）+ KaTeX（公式）**

- 所有 6 种模型均为 2D 侧视图场景（自由落体、斜面、弹簧、单摆、阿特伍德机、轻杆圆周），无需 3D
- Canvas 2D 性能充裕，帧率可达 60fps
- 能量条形图需要与运动动画高帧率同步，建议同一 Canvas 上下文或使用 requestAnimationFrame 同步更新
- KaTeX 用于暂停时展示公式和精确值

### 推荐的架构模式

**模拟演示型架构**：参数状态 → 物理引擎（解析解/数值积分）→ 动画帧循环 → 双区域渲染（运动场景 + 能量条形图）

```
用户调参 → PhysicsEngine 重置状态
用户按播放 → AnimationLoop 启动
  每帧: PhysicsEngine.step(dt) → 更新物体位置/速度 → 计算各能量值
        → SceneRenderer 绘制运动场景
        → EnergyBarRenderer 绘制能量条形图
        → DataPanel 更新数值显示
用户按暂停 → AnimationLoop 暂停 → 放大显示数值 + 守恒验证
```

核心架构组件：

| 组件 | 职责 | 设计要点 |
|------|------|---------|
| ModelRegistry | 6 种模型的注册与切换 | 策略模式，每种模型实现统一接口 |
| PhysicsEngine | 运动方程求解 + 能量计算 | 解析解优先，需数值积分的模型用 RK4 |
| AnimationLoop | 帧循环 + 时间轴控制 | requestAnimationFrame + dt 累积，支持变速/暂停/单步 |
| SceneRenderer | 运动场景绘制 | Canvas 2D，每种模型一个 Renderer |
| EnergyBarRenderer | 能量条形图绘制 | Canvas 2D 或 Chart.js，与运动场景同步 |
| ParameterStore | 参数状态管理 | Zustand 或 React useState，参数变更触发 Engine 重置 |

### 关键技术风险或难点

1. **轻杆双球圆周的运动方程**：两个质量不同的球通过刚性杆连接做竖直面圆周运动，运动方程为非线性 ODE，需用 RK4 等数值积分方法，且需处理向心力约束和杆的拉力/推力切换
2. **有阻尼弹簧振子的数值稳定性**：阻尼系数较大时需注意数值积分的稳定性，避免能量发散
3. **能量条形图与运动动画的同步性**：两个渲染区域必须在同一帧内更新，任何不同步都会导致视觉上的"能量不守恒"错觉
4. **多场景对比模式的性能**：双面板同屏运行两个物理模拟 + 两组能量条形图，帧率可能下降
5. **大角度单摆的精确模拟**：大角度时 sinθ ≈ θ 近似失效，需用完整非线性方程数值求解
6. **重力势能零点拖动**：用户拖动零点后，所有 Ep 值需实时重算并更新条形图，涉及坐标系变换
