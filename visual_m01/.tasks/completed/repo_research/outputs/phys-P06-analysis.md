# P-06 波动与振动演示台 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中物理老师，不要求有任何编程基础。核心用户画像是需要讲授机械波章节（人教版选择性必修第一册）的高中物理教师。

### 解决什么教学痛点
- **波形图与振动图混淆**：学生普遍分不清"某时刻所有质点的位移-位置图（波形图）"和"某质点随时间的位移-时间图（振动图）"，粉笔图无法动态展示两者的对应关系
- **干涉稳定性理解困难**：同频叠加产生稳定干涉、异频叠加产生不稳定图样，这一关键判据仅靠静态图片无法体会
- **驻波形成过程抽象**：入射波与反射波的叠加形成驻波，波节不动、波腹振幅最大，整个"驻"的过程需要慢放动画才能看清
- **多普勒效应缺乏直觉**：声源移动时波前压缩/拉伸的过程，黑板上画不出动态效果
- **横波与纵波对比**：纵波的疏密变化用粉笔图根本画不清楚

### 核心产品价值
**一句话**：老师选择波动演示模块、调几个参数，就能看到实时动画：波的传播、质点振动、波的叠加干涉、驻波形成、多普勒效应，点击任意质点即可查看其振动图，彻底解决"波形图 vs 振动图"的教学痛点。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 选择"波动与振动演示台" → 选择"单列波传播"模块 → 调好频率、振幅参数 → 准备好几个预设场景（驻波基频/二次谐波/三次谐波） |
| **课中演示 — 波形图与振动图** | 播放横波传播动画 → 点击波形图上某个质点 → 下方自动显示该质点的振动图 → 切换不同质点对比相位差 → 暂停/逐帧展示某时刻波形 |
| **课中演示 — 干涉** | 切换到"两列波叠加"模块 → 设置两列波同频率 → 观察稳定干涉图样（加强点/减弱点位置固定）→ 改为不同频率 → 观察图样不稳定、来回变化 → 对比讲解干涉条件 |
| **课中演示 — 驻波** | 切换到"驻波演示"模块 → 显示入射波（蓝色虚线）和反射波（红色虚线）→ 慢放观察合成驻波（绿色实线）→ 波节（黑色圆点）和波腹（黄色圆点）自动标注 → 切换基频/二次谐波/三次谐波预设 |
| **讲题** | 给定波速和时间间隔 → 使用"波形平移"模块展示波形移动 Δx = vΔt → 两个时刻波形叠加对比 → 辅助判断传播方向 |

### 怎么用才是"好用"的
- **即点即看**：点击波形图上任意质点，下方振动图立刻切换，相位关系准确
- **参数实时响应**：拖动频率/振幅/波速滑块，波形动画无卡顿地实时更新
- **对比清晰**：同频 vs 异频叠加的差异一眼可辨；入射波/反射波/合成驻波用不同颜色和线型区分
- **慢放控制**：驻波形成过程可慢放（0.25x/0.5x），清晰看到"驻"的过程

### 交互频率和深度
- **频率**：每节课使用 2-4 次，每次 3-10 分钟
- **深度**：中等 — 主要是选模块 + 调参数 + 点质点 + 播放/暂停，不涉及复杂建模或编辑操作

### 需要实体管理的功能
本模块**不需要传统实体管理**。没有"创建/选中/删除/编辑"对象的需求。核心对象是波本身，由参数完全确定，不需要用户手动创建或删除。唯一的"选中"行为是点击波形图上的质点来查看其振动图。

### 需要 Undo/Redo 的功能
**不需要**。所有操作都是参数调整和动画播放控制，无破坏性操作，无需撤销。

### 参数驱动的功能
- 振幅 A、频率 f、波速 v、质点数量 N → 波形图实时变化
- 两列波各自的振幅/频率/相位差 → 叠加结果实时变化
- 入射波振幅/频率/波速/弦长 → 驻波形态实时变化
- 波源频率/波速/波源速度 → 多普勒效应可视化实时变化
- 波速/时间间隔 → 波形平移距离实时变化

### 模拟/动画驱动的功能
**全部核心功能都是动画驱动的** — 这是本模块的最大特征：
- 横波/纵波传播动画（60fps 连续渲染）
- 波叠加动画（同频稳定 / 异频不稳定变化）
- 驻波形成动画（入射波+反射波+合成波同步运动）
- 多普勒效应波前扩散动画（声源移动 + 同心圆弧不断产生）
- 需要播放/暂停/慢放/逐帧控制

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **主渲染**：2D Canvas — 波形图、振动图、叠加图、驻波图均为 2D 曲线/质点渲染，Canvas 是最合适的选择（高频重绘、大量质点、连续动画）
- **辅助渲染**：2D Canvas — 多普勒效应的波前同心圆弧扩散动画
- **公式渲染**：LaTeX — 波动方程 y=Asin(ωt-kx)、干涉条件、驻波方程、多普勒公式等

> **关键性能考量**：所有动画模块均要求 60fps 连续重绘。横波传播需绘制 30-60 个质点及其连线；驻波需同时绘制 3 条波形（入射/反射/合成）；多普勒需不断绘制扩散的同心圆弧。Canvas 2D 完全能胜任此规模（远未达到 WebGL 的必要性阈值），但需注意以下优化：
> - 使用 `requestAnimationFrame` 驱动动画循环
> - 避免每帧创建新对象（对象池复用）
> - 波形曲线用 Path 一次性绘制，不逐点 fillRect

#### 数学/物理计算需求
- **轻度**：正弦函数计算 y = A sin(ωt - kx)，JavaScript Math.sin 即可
- **轻度**：叠加计算 y = y1 + y2，逐点相加
- **轻度**：驻波方程 y = 2A cos(kx) sin(ωt)
- **轻度**：多普勒频率 f' = f₀v/(v ± vs)
- **轻度**：波长 λ = v/f，周期 T = 1/f，角频率 ω = 2πf，波数 k = 2π/λ
- **无需额外数学库**：所有计算均为基础三角函数和四则运算

#### 公式渲染需求
- 波动方程：`y(x,t) = A \sin(\omega t - kx)`
- 驻波方程：`y = 2A \cos(kx) \sin(\omega t)`
- 多普勒公式：`f' = f_0 \frac{v}{v \pm v_s}`
- 干涉条件文字标注
- 波节/波腹位置公式

### 交互依赖

#### 交互模式
**模拟演示型**（主导）+ **参数驱动型**（辅助）

这是一个典型的模拟演示型应用：
- 核心是动画播放和观察
- 用户通过调节参数改变动画表现
- 通过点击质点获取关联信息（振动图）
- 无实体创建/编辑/删除流程
- 无 Undo/Redo 需求

#### 关键交互

| 交互类型 | 具体场景 |
|---------|---------|
| 点选 | 点击波形图上的质点 → 切换显示该质点的振动图 |
| 滑块 | 振幅/频率/波速/相位差/波源速度等参数调节 |
| 按钮 | 演示模块选择（6 个模块切换）、预设场景切换（驻波共振模式） |
| 播放控制 | 播放/暂停/慢放(0.25x/0.5x/1x)/逐帧 |
| Hover | 波形图上质点的 hover 高亮（提示可点击） |

#### 是否需要实体系统
**否** — 无需统一的实体模型。波和质点由参数完全确定，不需要用户管理的对象。

#### 是否需要命令系统
**否** — 无破坏性操作，参数调整即时生效，无需撤销。

### 与 P-05 简谐运动的关系
波动的每个质点都在做简谐运动，P-06 的"单质点振动图"本质上就是 P-05 的简谐运动曲线。两者在模型上存在共享可能：
- **共享层**：简谐运动的数学模型 y = A sin(ωt + φ)、动画时间轴控制、参数面板组件
- **差异层**：P-05 聚焦单个振子（弹簧振子/单摆）的动力学特征（加速度、回复力、能量），P-06 聚焦多个质点的波传播与叠加
- **建议**：提取共享的"简谐运动数学模型"和"动画时间控制器"为公共模块，但不强制要求两模块紧耦合

## 5. 竞品分析

### 5.1 PhET — Wave on a String & Wave Interference

#### Wave on a String
- **形态**：Web（HTML5），全平台浏览器运行
- **技术**：PhET 自研框架 Scenery（基于 Canvas/SVG），开源（CC-BY 4.0）
- **功能**：弦上波的传播、反射（固定端/自由端/无端）、阻尼、张力调节、慢放、标尺和计时器
- **优势**：
  - 交互极其直观 — 拖动端点直接"抖"绳子生成波
  - 提供手动/振荡器/脉冲三种模式，覆盖多种教学场景
  - 界面简洁美观，游戏化设计降低使用门槛
  - 开源、免费、全球广泛采用
- **不足**：
  - 仅单列波，不支持两列波叠加/干涉
  - 无驻波专项展示（需手动调出）
  - 无波形图与振动图的联动对比
  - 无多普勒效应
- **启示**：PhET 的"拖动生成波"交互体验值得参考；但 P-06 的差异化在于波形图-振动图联动和干涉稳定性对比

#### Wave Interference
- **形态**：Web（HTML5）
- **技术**：同 Scenery 框架
- **功能**：水波/声波/光波的干涉、单缝/双缝衍射、可放置屏幕观察图样
- **优势**：跨介质对比（水/声/光）、双缝干涉图样直观
- **不足**：
  - 聚焦衍射/干涉图样，非一维波形分析
  - 无波形图 vs 振动图联动
  - 无驻波、无多普勒效应
- **启示**：P-06 更聚焦高中物理的一维机械波教学，与 PhET Wave Interference 的 2D 波面可视化互补而非竞争

### 5.2 Falstad Ripple Tank

- **形态**：Web（WebGL），也有 Windows/Mac/iOS 版本
- **技术**：WebGL（原 Java Applet 移植），开源（GitHub: pfalstad/ripplegl）
- **功能**：2D 波面模拟，支持干涉、衍射、折射、共振、相控阵列、多普勒效应
- **优势**：
  - 功能极其丰富 — 涵盖几乎所有波动现象
  - WebGL 性能出色，可模拟大面积 2D 波面
  - 提供 3D 视角切换
  - 预设场景丰富（Example 菜单）
  - 可交互拖动波源
- **不足**：
  - 界面老旧（Java 时代风格），操作不直观
  - 面向 2D 波面模拟，不适合高中一维机械波教学
  - 无波形图 vs 振动图联动
  - 无教学引导，学习曲线陡
  - 参数太多，非教学优化
- **启示**：Falstad 展示了 WebGL 在波动模拟中的强大能力，但 P-06 的定位是"教学优化的一维波动演示"，不需要 2D 波面的复杂度

### 5.6 市场空白与 P-06 定位

**核心发现**：**面向高中一维机械波教学、功能完整的 Web 工具存在空白**。

- PhET 的波动模块分散在多个独立 sim 中，缺乏波形图-振动图联动和干涉稳定性对比
- Falstad 功能强大但面向 2D 波面，非教学优化
- oPhysics 各模块功能单一，缺乏一站式体验
- 没有任何现有工具同时提供"波形图-振动图联动 + 同频/异频叠加稳定性对比 + 驻波分解显示 + 多普勒效应"的完整教学套件

P-06 的差异化定位：**唯一面向高中物理一维机械波教学的一站式 Web 演示工具**，核心差异在于波形图-振动图联动、干涉稳定性直观对比、驻波入射/反射/合成分解显示。

## 6. 初步技术判断

### 推荐的渲染技术

**HTML5 Canvas 2D**

- 所有功能模块（横波/纵波/叠加/驻波/多普勒/波形平移）均为 2D 曲线和质点动画，Canvas 2D 完全胜任
- 60fps 连续动画是核心需求，Canvas 的 `requestAnimationFrame` + 逐帧清除重绘是标准模式
- 质点数量级为 30-60 个，波形曲线为简单正弦函数采样，计算量极小
- 多普勒效应的同心圆弧绘制用 Canvas arc() 即可
- **不需要 WebGL** — 场景复杂度远未达到 GPU 加速的必要性阈值
- **不需要 SVG** — 高频重绘场景下 Canvas 性能优于 SVG DOM 操作
- 公式标注可用 KaTeX 渲染后叠加在 Canvas 之上（HTML 层）

### 推荐的架构模式

**参数驱动 + 动画循环**（模拟演示型架构）

```
┌─────────────────────────────────────────────┐
│              模块选择器（6 个模块）            │
├───────────────┬─────────────────────────────┤
│               │                             │
│  参数面板      │     Canvas 渲染区            │
│  (滑块/输入)   │   ┌─ 波形图（上半区）──────┐ │
│               │   │  质点可点击           │ │
│  显示选项      │   ├─ 振动图（下半区）──────┤ │
│  (复选框)      │   │  选定质点的 y-t 图    │ │
│               │   └────────────────────┘ │
│               │                             │
├───────────────┴─────────────────────────────┤
│         动画控制栏（播放/暂停/慢放/逐帧）       │
└─────────────────────────────────────────────┘
```

核心架构要素：
1. **WaveModel（波动数学模型）**：纯计算层，给定参数和时间 t，输出所有质点位移数组。支持单波、叠加、驻波、多普勒等模式。可复用于所有子模块。
2. **AnimationLoop（动画循环控制器）**：管理 `requestAnimationFrame`、时间步进（支持慢放倍速）、播放/暂停/逐帧状态机。
3. **WaveRenderer（Canvas 渲染器）**：将 WaveModel 输出的数据绘制为波形图/振动图/波前圆弧。不同子模块对应不同渲染策略。
4. **ParameterPanel（参数面板组件）**：React 组件，滑块/输入框/预设按钮，双向绑定 WaveModel 参数。
5. **ModuleSwitcher（模块切换器）**：6 个演示模块的路由/切换逻辑，切换时重置参数和动画状态。

### 关键技术风险或难点

| 风险/难点 | 等级 | 说明 | 应对策略 |
|-----------|------|------|---------|
| 60fps 动画性能 | 低 | 质点数 30-60，正弦计算极轻量，Canvas 2D 完全胜任 | 标准 rAF 循环；避免每帧 GC；Path 批量绘制 |
| 波形图-振动图联动精度 | 中 | 点击质点后振动图的相位必须与波形图严格一致，否则教学错误 | 统一时间源；振动图从 WaveModel 同一时间轴采样；增加单元测试验证相位关系 |
| 纵波疏密可视化 | 中 | 纵波的质点沿传播方向振动导致疏密交替，视觉效果需要仔细调试才能清晰 | 参考 Physics Classroom 的纵波渲染方式；质点间距变化 + 颜色深浅双重编码 |
| 多普勒效应波前绘制 | 中 | 波前圆弧需要随时间不断产生、扩散、消失，同时声源在移动，计算逻辑稍复杂 | 维护波前圆弧数组，每个波前记录产生时刻和产生位置；每帧根据当前时间计算半径；超出可视范围移除 |
| 多模块状态管理 | 低 | 6 个子模块共享动画控制但参数不同 | 每个模块独立的参数 Config；共享 AnimationLoop；模块切换时 reset |
| 与 P-05 的模型共享 | 低 | 简谐运动模型可共享但不强制 | 先独立开发，后续重构时提取公共模块 |
