# C-07 化学反应速率与平衡模拟器 — 模块分析卡片

## 1. 用户场景 & 产品价值

### 目标用户
高中化学教师（主要为必修模块"化学反应原理"授课教师），无需具备动力学或编程背景。次要用户为化学竞赛辅导教师（涉及反应级数、基元反应等拓展内容）和需要自主复习平衡移动题型的学生。

### 解决什么教学痛点
- **微观碰撞过程不可见**：有效碰撞、过渡态、活化能等概念纯靠语言描述极难建立直觉。教师普遍反映"分子活化、有效碰撞、过渡态、活化能、键的断裂和生成"需要可视化支撑
- **平衡建立过程抽象**：正反应速率逐渐减小、逆反应速率逐渐增大、最终 v(正)=v(逆) 的动态过程，黑板上只能画静态 v-t 图，学生难以体会"动态平衡"的含义
- **勒夏特列原理理解困难**：改变浓度/温度/压强后平衡如何移动、新平衡如何建立，是高考核心考点但也是最大难点。学生容易混淆"速率突变→新平衡"的因果链
- **v-t 图、c-t 图判读能力弱**：高考大量考查"条件改变后速率-时间图/浓度-时间图"的判读，但教师手画不精确，学生缺乏动态理解
- **工业反应条件选择的"权衡"难讲**：合成氨为什么选 500°C 而非更低？"速率与转化率的权衡"概念抽象
- **等效平衡判断**：恒温恒容/恒温恒压下等效平衡的判断规则是高考高频难点，纯文字/公式讲解效果差
- **基元反应与反应级数（选修）**：决速步如何决定总速率、零级/一级/二级反应曲线差异，需要对比可视化

### 核心产品价值
**一句话**：教师选择一个预设反应体系，即可观看微观分子碰撞动画与宏观图表（v-t、c-t 等多类图表）的实时联动，通过滑块改变浓度/温度/压强/催化剂，直观演示勒夏特列原理下的平衡移动与新平衡建立过程，覆盖高中化学平衡全部核心知识点。

## 2. 用户实际使用方式

### 教师典型操作流程

| 场景 | 操作流程 |
|------|---------|
| **课前准备** | 打开浏览器 → 从反应库选择合成氨反应 → 预览微观动画和图表 → 确认投影效果 |
| **课中演示 — 反应速率概念** | 选择 Na₂S₂O₃+H₂SO₄ 演示反应 → 分组对比浓度/温度对速率的影响 → 观看气泡/沉淀生成速率的可视差异 |
| **课中演示 — 微观碰撞** | 选择合成氨反应 → 播放容器内分子碰撞动画 → 观察有效碰撞闪光标记 → 调高温度滑块 → 观察分子运动加快、有效碰撞频率增加 → 开启催化剂 → 观察活化能峰降低 |
| **课中演示 — 平衡建立** | 选择 N₂+3H₂⇌2NH₃ → 从初始状态播放 → 观察 v-t 图上正反应速率下降、逆反应速率上升直至相等 → 同步观察 c-t 图上反应物浓度下降、产物浓度上升至稳定 |
| **课中演示 — 勒夏特列原理** | 平衡建立后 → 拖动"增大 N₂ 浓度"滑块 → 观察 v-t 图上正反应速率突增 → 图表标注条件改变竖线 → 新平衡逐渐建立 → 重复演示升温/加压/催化剂 |
| **讲题 — v-t 图/c-t 图判读** | 展示某条件改变后的图表 → 暂停在关键帧 → 标注速率变化原因 → 对照学生做题中的常见错误 |
| **讲题 — 等效平衡** | 设置两个对照容器（1mol N₂+3mol H₂ vs 2mol NH₃）→ 播放两者各自达到平衡 → 对比平衡组成相同 → 切换到恒温恒压条件对比 |
| **拓展教学 — 反应级数** | 选择基元反应模块 → 以 2NO+O₂→2NO₂ 为例 → 展示两步机理 → 标注决速步 → 对比零级/一级/二级反应的 c-t 曲线差异 |
| **拓展教学 — K 与温度** | 打开 K-T 曲线面板 → 对比放热反应（合成氨：K 随 T 升高减小）与吸热反应（C+CO₂：K 随 T 升高增大）→ 展示范特霍夫方程计算器 |

### 怎么用才是"好用"的
- **3 分钟上手**：选反应 → 播放 → 拖滑块改变条件，即完成核心演示
- **微观-宏观联动直观**：微观碰撞动画与宏观图表同步更新，教师无需手动切换
- **图表实时绘制**：图表随动画进度实时绘制（非预设静态图），条件改变时刻自动标注竖线
- **数据精确可信**：10 个反应体系的热力学参数、平衡常数均来自 NIST/CRC Handbook
- **投影清晰**：在 1080p 投影仪上，分子碰撞动画、图表曲线、标注文字清晰可辨
- **动画可控**：支持播放/暂停/逐步/速度调节（0.5x/1x/2x），教师可在关键帧停留讲解

### 交互频率和深度
- **频率**：高中"化学反应原理"模块集中使用约 10-15 课时，每课时使用 2-4 次，每次 5-15 分钟
- **深度**：中重度 — 核心操作（选反应+播放+改条件）是浅层交互，但等效平衡对照、K-T 曲线、反应级数对比等功能需要较深交互

### 需要实体管理的功能
本模块**不是编辑器型**应用，用户不自由创建/删除对象。核心实体（反应体系、分子、图表曲线）均由数据驱动自动生成。需要管理以下状态对象：
- 当前选中的反应体系
- 当前条件参数（温度、浓度、压强、催化剂开关）
- 模拟运行状态（播放/暂停/当前时间步）
- 微观动画中的粒子状态（位置、速度、碰撞事件）
- 图表数据序列（随时间累积的浓度/速率数据点）
- 等效平衡对照容器状态（双容器模式下）

### 需要 Undo/Redo 的功能
**不需要** — 本模块是模拟演示器，用户操作（选反应/改条件/播放暂停）都是即时可逆的状态变更，无破坏性编辑操作。条件改变后可通过"重置"按钮回到初始状态。

### 参数驱动的功能
- 温度滑块 → 分子运动速度变化 + 有效碰撞频率变化 + 平衡常数变化 + 图表更新
- 浓度滑块 → 微观动画中分子数量变化 + 图表更新
- 压强调节（容器体积缩放） → 分子密度变化 + 图表更新
- 催化剂开关 → 活化能峰降低 + 达到平衡时间缩短
- 动画速度（0.5x/1x/2x） → 播放速度变化

### 模拟/动画驱动的功能
- 微观碰撞动画（容器内多个分子的随机运动、碰撞检测、有效碰撞判定、过渡态展示）— 需播放/暂停/逐步/速度控制
- 平衡建立过程动画（v-t/c-t 图表实时绘制曲线）
- 条件改变后的平衡移动动画（旧平衡→扰动→新平衡的完整过程）
- 堆积动画中的气泡/沉淀生成速率可视对比
- 等效平衡双容器并行模拟动画
- 基元反应决速步演示动画

## 4. 核心依赖识别

### 技术依赖

#### 渲染方式
- **微观碰撞动画**：2D Canvas（主推）— 容器内分子的随机运动与碰撞属于 2D 粒子模拟，Canvas 性能足够且实现简洁。也可考虑 SVG（分子数量较少时），但 Canvas 在大量粒子场景下性能更优
- **宏观图表**：SVG 或 Canvas — 使用图表库（如 D3.js 或轻量图表库）绘制 v-t、c-t、p-t 等曲线图，支持实时绘制、多曲线叠加、双 Y 轴、条件改变标注线
- **能量曲线**：SVG — 反应坐标-能量图属于静态+参数驱动型图表（催化剂开关改变活化能峰高度）
- **UI 控件**：DOM — 滑块、按钮、下拉选择、信息面板
- **无需 3D WebGL**：本模块的微观碰撞动画是 2D 俯视示意图（容器截面），不是 3D 分子模拟

#### 数学/物理计算需求
- **中度**：粒子运动模拟 — 2D 随机运动、碰撞检测（圆-圆碰撞）、弹性碰撞速度计算。与 P-07 热力学粒子模拟技术相似，但规模更小（几十个分子即可）
- **中度**：化学动力学方程求解 — 速率方程 v=k[A]^m[B]^n 的数值积分（简单欧拉法或 RK4），用于驱动 c-t 曲线的实时计算
- **中度**：平衡计算 — 给定初始浓度和 K 值，求解平衡浓度（ICE 表法 + 一元二次/三次方程求解）
- **轻度**：范特霍夫方程计算 — ln(K₂/K₁)=-ΔH°/R×(1/T₂-1/T₁)
- **轻度**：吉布斯自由能计算 — ΔG°=-RTlnK
- **轻度**：浓度商 Qc 的实时计算与 Kc 比较

#### 公式渲染需求
- 化学方程式渲染（下标、可逆箭头 ⇌、气体/沉淀标记）
- 速率表达式 v=k[A]^m[B]^n
- 平衡常数表达式 Kp/Kc
- 范特霍夫方程、吉布斯自由能公式
- 等效平衡判断规则的公式表达
- **需求中等**：建议使用 KaTeX 进行数学公式渲染（化学方程式可用 HTML+CSS 处理下标上标）

#### 数据管理需求
- **静态反应数据**：约 10 个反应体系的 JSON 数据（方程式、ΔH、K 值随温度变化、速率参数、勒夏特列演示参数、教学要点）
- **数据已详细定义**：数据文档提供了完整的数据结构（id, name_cn, equation, level, delta_H, Kp/Kc, rate_factors, equilibrium_shift）
- **图表数据格式已定义**：c-t 曲线 JSON（initial/equilibrium 浓度 + curves 配色）、v-t 曲线 JSON（扰动前后速率 + 新平衡速率）
- **数据来源**：NIST Chemistry WebBook, CRC Handbook

### 交互依赖

#### 交互模式
**模拟演示型**（主导）+ **参数驱动型**（辅助）

这是一个模拟器/演示器型应用：
- 用户从预设反应库中选择反应体系（非自由创建）
- 通过播放/暂停/逐步控制动画进度
- 通过滑块/开关改变条件参数，触发平衡移动模拟
- 图表随模拟过程实时绘制

#### 关键交互

| 交互类型 | 具体场景 |
|---------|---------|
| 下拉选择 | 从反应库选择反应体系（合成氨/SO₃/酯化/NO₂二聚等） |
| 滑块 | 温度调节（连续）、浓度调节（各物种独立）、容器体积（压强） |
| 开关按钮 | 催化剂开/关 |
| 播放控制 | 播放/暂停/逐步前进/速度调节（0.5x/1x/2x） |
| 重置按钮 | 回到初始状态重新模拟 |
| 图表交互 | 鼠标悬停查看某时刻的精确数值（tooltip）、缩放查看细节 |
| 标签切换 | 切换不同图表类型（v-t/c-t/p-t 等） |
| 双容器模式 | 等效平衡对照时的双容器并排展示 |
| 触摸手势 | 移动端滑块拖拽、图表缩放 |

#### 是否需要实体系统
**不需要** — 无需 Entity Registry 或创建/删除/编辑的完整实体管理。仅需轻量的"当前反应"状态和"模拟参数"状态管理。

#### 是否需要命令系统
**不需要** — 无破坏性操作，无需 Undo/Redo。"重置"按钮即可回到初始状态。

## 5. 竞品分析

### 5.1 PhET Reactions & Rates

- **形态**：Web（HTML5）/ 桌面（Java 遗留版），免费开源
- **技术**：HTML5 Canvas + PhET 自研框架（Scenery 渲染引擎），纯前端无后端
- **优势**：
  - 微观碰撞动画非常直观 — 容器内分子随机运动，碰撞时有视觉反馈
  - 温度滑块实时改变分子运动速度
  - 活化能柱状图同步显示
  - 教学导向设计，上手极简
  - 多语言支持（含中文）
- **不足**：
  - 仅支持单一虚拟反应（A+BC→AB+C），不支持真实化学反应体系
  - 无宏观图表联动（无 v-t、c-t 图表）
  - 无勒夏特列原理的系统演示
  - 无平衡常数、浓度商等定量概念
  - 无等效平衡功能
  - 无基元反应/反应级数内容
- **启示**：微观碰撞动画的设计（分子运动+碰撞闪光+活化能同步）是非常好的参考范本；但需大幅扩展到真实反应体系和宏观图表联动

### 5.2 PhET Equilibrium (Reactions & Rates → Equilibrium 部分)

- **形态**：Web（HTML5），免费开源
- **技术**：同上 PhET 框架
- **优势**：
  - 展示了化学平衡建立的动态过程
  - 浓度柱状图实时变化
  - 简洁直观的 UI
- **不足**：
  - 同样基于虚拟反应，非真实化学体系
  - 图表功能极简（仅柱状图，无 v-t/c-t 曲线）
  - 无勒夏特列原理的条件改变演示
  - 无温度/压强变量
  - 无定量计算（K 值、Qc）
- **启示**：平衡建立过程的动画设计可参考，但 C-07 需要在定量性和真实反应体系上远超 PhET

### 5.3 虚拟实验室化学反应模块（国内）

- **形态**：Web / 客户端，多为付费
- **代表**：NB 实验、仿真物理实验室、Nobook 虚拟实验室
- **优势**：对接国内教材和高考考点，有中文支持
- **不足**：
  - 多为视频录播或固定动画序列（非实时模拟）
  - 交互性弱（多为"看"而非"调"）
  - 很少有微观碰撞动画
  - v-t/c-t 图表多为预设静态图片
  - 封闭系统，无法灵活组合条件
- **启示**：国内市场对化学平衡交互教学工具有需求，但现有产品交互能力和模拟真实度远不够

### 5.4 其他参考

#### ChemCollective (Virtual Labs, Carnegie Mellon)
- **形态**：Web（Java Applet 遗留 → 部分 HTML5 迁移）
- **优势**：基于真实化学数据的虚拟实验
- **不足**：技术陈旧，UI 过时，无微观碰撞动画
- **启示**：真实化学数据驱动模拟的思路正确

| Web 端免费 | ✅ | ✅ | ❌(多付费) | **✅** |

### 5.6 市场空白与 C-07 定位

**核心发现**：**将微观碰撞动画与宏观图表实时联动、覆盖真实反应体系的 Web 化学平衡模拟器不存在**。

- PhET 在微观碰撞动画方面是设计标杆，但仅用虚拟反应、无宏观图表联动
- 国内产品对接高考考点但交互能力弱、多为静态展示
- 等效平衡演示、浓度商 Qc 与 Kc 对比、K-T 曲线在所有竞品中均为空白
- 基于真实热力学/动力学数据（NIST/CRC）的定量模拟在教学类产品中极为罕见

C-07 的差异化定位：**唯一基于真实反应数据（10 个体系）、微观碰撞动画+宏观多类图表实时联动、支持全条件变量（浓度/温度/压强/催化剂）交互的化学平衡教学模拟器，覆盖高考化学平衡全部核心考点**。

## 6. 初步技术判断

### 推荐的渲染技术

**混合渲染方案**：
- **微观碰撞动画**：HTML5 Canvas 2D — 粒子数量在几十到百级别，Canvas 性能充裕且实现灵活。使用 requestAnimationFrame 驱动动画循环
- **宏观图表**：SVG（D3.js 或类似图表库）— 支持实时绘制曲线、双 Y 轴、条件改变标注线、tooltip 交互。若性能需求更高可考虑 Canvas 图表库（如 Chart.js）
- **能量曲线**：SVG — 反应坐标-能量图为参数驱动静态图
- **UI/控制面板**：DOM + CSS — 滑块、按钮、选择器、信息面板

### 推荐的架构模式

**数据驱动模拟器 + 双层渲染（微观+宏观）**

```
反应体系 JSON 数据 → 模拟引擎（动力学求解器）→ 双层渲染
                         ↑                         ├→ 微观层：Canvas 粒子动画
用户交互（选反应/改条件） → 状态管理              ├→ 宏观层：SVG 图表实时绘制
                                                    └→ 能量层：SVG 能量曲线
```

| 层 | 职责 | 说明 |
|----|------|------|
| 数据层 | 反应体系 JSON | 约 10 个反应的热力学/动力学参数（ΔH、K、速率参数、勒夏特列演示参数） |
| 模拟引擎 | 动力学求解 + 粒子模拟 | 速率方程数值积分（c-t 驱动）+ 2D 粒子碰撞模拟（微观动画驱动） |
| 微观渲染层 | Canvas 2D 粒子动画 | 分子运动、碰撞检测、有效碰撞闪光、过渡态高亮、活化分子标记 |
| 宏观渲染层 | SVG 图表 | v-t/c-t/p-t 等多类图表，实时绘制，条件改变标注，双 Y 轴 |
| 能量渲染层 | SVG 能量曲线 | 反应坐标-能量图，活化能峰，催化剂效果 |
| 交互层 | 状态管理 + 事件 | 当前反应/条件参数/模拟状态/播放控制 |
| UI 层 | 控制面板 + 信息面板 | 反应选择、条件滑块、播放控制、Qc vs Kc 面板、教学要点 |

**与 P-07（热力学气体分子模拟器）的技术相似性**：
- **可参考但不共享**：P-07 的容器内分子随机运动与碰撞动画（Canvas 2D 粒子模拟）在技术实现上与 C-07 微观碰撞动画高度相似 — 粒子位置更新、圆-圆碰撞检测、弹性碰撞速度计算
- **C-07 专属**：有效碰撞判定（取向+能量阈值）、过渡态动画、键断裂/生成动画、活化分子标记、化学反应事件（分子 A+B → C+D 的物种转换）
- **跨学科不合并**：虽然粒子渲染管线相似，但化学和物理属于不同产品线，不做代码合并。技术方案可互相借鉴

### 关键技术风险或难点

1. **微观-宏观一致性**：微观碰撞动画中的反应事件频率必须与宏观 c-t/v-t 曲线数据保持一致，否则两层展示会产生矛盾。需要将宏观动力学方程的求解结果映射为微观碰撞概率参数。这是最大的技术难点。

2. **粒子模拟的化学正确性**：与物理粒子模拟不同，化学碰撞需要考虑取向因子和活化能阈值，有效碰撞后还需要实现物种转换（反应物分子消失、产物分子出现）。动画表现需要兼顾科学性和直观性。

3. **实时图表绘制性能**：v-t/c-t 曲线需要随模拟进度实时追加数据点并重绘。当模拟步长多（数千步）时，SVG 路径节点过多可能影响性能，需要考虑路径简化或 Canvas 回退方案。

4. **多图表同步与布局**：PRD 要求支持 6 种图表类型 + 双 Y 轴 + 多曲线叠加 + 条件改变标注。图表系统的通用性和布局灵活性设计工作量较大。

5. **等效平衡的双容器模拟**：需要同时运行两个独立的模拟实例并排展示，对状态管理和渲染性能有一定要求。

6. **条件改变的过渡动画**：改变条件后，微观层（分子数量突变/运动速度突变）和宏观层（速率突变曲线）的过渡需要设计合理的动画衔接，避免突兀。

7. **反应级数模块的数据复杂性**：零级/一级/二级反应需要不同的速率方程和数值积分方式，且教学上需要准确展示其 c-t 曲线的差异。算法本身简单但需要仔细验证正确性。

8. **NO₂/N₂O₄ 颜色变化**：PRD 数据中提到压强增大时"先加深后变浅"的两阶段效果，需要精心设计颜色渐变动画以体现"先浓缩后平衡移动"的过程。
