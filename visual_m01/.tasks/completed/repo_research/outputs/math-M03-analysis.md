# M-03 解析几何画板 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中数学老师
- **覆盖学段**：高中必修（直线与圆的位置关系）、高中选修（椭圆/双曲线/抛物线 — 圆锥曲线）
- **解决的教学痛点**：
  - 圆锥曲线在黑板上画不准确（椭圆像鸡蛋、双曲线歪扭、渐近线对不齐）
  - 焦点/准线/离心率等抽象概念缺乏动态可视化，学生"脑子里没有画面"
  - 焦点弦旋转、弦长变化、面积变化等高考核心操作无法在黑板上动态展示
  - 圆锥曲线统一定义（离心率从 0→2 的连续演变）难以用静态图表达
- **核心产品价值**：让抽象的坐标计算变成看得见的图形操作 — 标准精确的曲线绘制 + 关键元素自动标注 + 交互式参数调节 + 实时计算反馈

## 2. 用户实际使用方式

- **场景1 — 讲椭圆定义**：选择椭圆，拖动动点 P，屏幕实时显示 |PF₁| + |PF₂| 恒等于 2a，学生直观理解定义
- **场景2 — 讲焦点弦（高考高频）**：画过焦点的直线，拖动旋转，实时看弦长变化、焦点三角形面积变化，通径位置弦长最短 — 学生自己"看出来"结论
- **场景3 — 讲离心率**：拖动 e 滑块从 0→2，图形从圆→椭圆→抛物线→双曲线平滑过渡，理解圆锥曲线统一性
- **场景4 — 讲直线与圆的位置关系**：画圆和直线，拖动直线平移/旋转，d 与 r 的大小关系实时更新（相交/相切/相离）
- **怎么用才"好用"**：曲线标准精确、关键元素自动标注无需手算、直线添加方便（输入方程或图上点选）、弦长面积自动计算且与手算一致
- **交互频率和深度**：课堂演示为主，每节课可能用 10-20 分钟；交互深度中等 — 主要是选择曲线类型 + 调参数 + 画直线 + 观察结果

## 3. 核心功能点

### 功能清单（按重要性排序）

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 曲线选择与绘制 | 选择圆/椭圆/双曲线/抛物线，立即绘制标准曲线 |
| P0 | 参数调节与实时更新 | 滑块调参（a/b/p/r 等），曲线与所有关联元素同步更新 |
| P0 | 自动标注关键元素 | 焦点、准线、对称轴、顶点、渐近线自动绘制与标注（可单独开关） |
| P0 | 直线工具 | 输入方程 / 点击两点 / 过特殊点+斜率，三种方式添加直线 |
| P0 | 交点计算与标注 | 直线与曲线交点自动求解并标注坐标 |
| P0 | 弦长自动计算 | 两交点间弦长实时计算并标注，直线可拖动旋转 |
| P1 | 焦点弦专题模式 | 过焦点直线可旋转，实时显示弦长/焦半径/焦点三角形面积/倾斜角，通径位置高亮 |
| P1 | 离心率连续变化演示 | e 从 0→2+ 滑块，圆→椭圆→抛物线→双曲线平滑过渡（亮点功能） |
| P1 | 直线与圆位置关系 | 专题模式：d 与 r 实时比较，交点/切点/弦长自动显示 |
| P1 | 面积计算 | 选定封闭区域（焦点三角形/弓形），自动计算面积并着色 |
| P1 | 动点轨迹 | 设定运动规则（距离和/差/等），播放动画描绘轨迹 |
| P2 | 光学性质可视化 | 椭圆/抛物线的反射性质动画（选学拓展） |

### 功能性质分类

- **需要实体管理（创建/选中/删除/编辑）的功能**：
  - 直线工具（用户可添加多条直线，需要选中、删除、编辑）
  - 曲线对象（虽然通常只有一条主曲线，但可能叠加多条辅助曲线）
  - 交点标注（随直线/曲线变化自动更新，但也是可管理的对象）
  - 动点与轨迹（创建动点、设定规则、生成轨迹）
  - **判定：需要轻量级实体管理，但复杂度远低于立体几何编辑器**

- **需要 undo/redo 的功能**：
  - 添加/删除直线
  - 修改曲线参数
  - 添加/删除动点
  - **判定：需要 undo/redo，但操作种类有限，命令系统可较简单**

- **参数驱动的功能（调参数 → 画面实时变化）**：
  - 曲线参数调节（a/b/p/r 滑块）— **核心交互**
  - 离心率连续变化滑块
  - 直线斜率/截距调节
  - 焦点弦倾斜角调节

- **模拟/动画驱动的功能（播放/暂停/逐步）**：
  - 动点轨迹播放
  - 光学性质动画
  - 离心率演变动画（也可手动拖滑块）

## 4. 核心依赖识别

### 技术依赖

- **渲染方式**：**2D Canvas（推荐）** 或 SVG
  - 纯 2D 坐标系场景，无 3D 需求
  - Canvas 适合高精度曲线绘制（圆锥曲线需 1000+ 采样点平滑渲染）
  - 需要高效的实时重绘（参数拖动时 60fps 更新曲线）
  - SVG 在大量采样点时性能可能不足，但交互事件处理更方便
  - **建议方案**：Canvas 绘制曲线 + 轻量 DOM/SVG 叠加层处理标注和交互控件

- **数学计算需求**：**中高**
  - 圆锥曲线方程求解（二次方程组联立）
  - 交点计算（直线与圆锥曲线联立，涉及判别式/韦达定理）
  - 弦长公式计算
  - 焦半径公式
  - 面积计算（三角形面积、可能涉及数值积分）
  - 点到直线距离
  - 坐标变换（焦点在 x/y 轴的切换）
  - 离心率连续变化时的曲线类型切换（需要统一参数化表示）

- **公式渲染需求**：**中等**
  - 标准方程显示（如 x²/25 + y²/9 = 1）
  - 焦点坐标、准线方程、离心率等数值显示
  - 弦长精确值（含根号形式）
  - 可用 KaTeX 或 MathJax 轻量渲染

### 交互依赖

- **交互模式**：**混合型（参数驱动为主 + 轻量编辑器型）**
  - **参数驱动**占主导：选曲线类型 → 调滑块 → 观察变化，这是最核心的使用流程
  - **编辑器型**成分：直线工具（添加/选中/删除/编辑直线对象）、动点轨迹（创建动点对象）
  - 但"编辑器"复杂度远低于 M-01 立体几何展示台：对象种类少（曲线+直线+点），无 3D 变换，无复杂嵌套

- **关键交互**：
  - 滑块：参数调节（核心交互，需丝滑响应）
  - 点选：选择曲线类型、选中已有直线/点
  - 拖拽：拖动直线旋转/平移、拖动动点
  - 输入框：输入直线方程、设定动点规则
  - 播放控件：动点轨迹播放/暂停

- **是否需要实体系统**：**需要，但可轻量化**
  - 对象类型有限：Curve（主曲线）、Line（直线，可多条）、Point（交点/动点）、Annotation（标注）
  - 不需要复杂的层级/分组/变换栈
  - 核心是"一条主曲线 + N 条辅助直线 + 自动计算的交点/标注"

- **是否需要命令系统（undo/redo）**：**需要，轻量级**
  - 主要操作：添加/删除直线、修改曲线参数、添加/删除动点
  - 操作种类少，命令系统可简化

## 5. 竞品分析

### GeoGebra（核心竞品）

- **产品形态**：Web / 桌面 / 移动全平台
- **技术实现**：Java 内核（后迁移到 Web），Canvas + 自研 CAS 计算引擎
- **解析几何能力**：
  - 完整的圆锥曲线支持（方程输入 / 图形化构造）
  - 自动计算交点、弦长、面积
  - 支持参数滑块和动画
  - 支持轨迹（Locus）功能
  - 内置 CAS 系统可做符号计算
- **优点**：功能极其强大，数学完备性高，免费
- **不足**：
  - 界面复杂，学习曲线陡峭，对非技术型老师不友好
  - 功能过于通用，缺乏针对中国高考题型的专题模式（如焦点弦专题）
  - 没有"离心率连续演变"这样的教学导向功能
  - 中文本地化不够深入

### 几何画板（Geometer's Sketchpad）

- **产品形态**：桌面软件（Windows/Mac）
- **技术实现**：原生桌面应用
- **解析几何能力**：支持圆锥曲线构造和变换，支持轨迹
- **优点**：经典工具，教师群体认知度高
- **不足**：
  - 已停止更新，无 Web 版本
  - 坐标系下的解析几何功能不如 GeoGebra 丰富
  - 不支持直接输入方程

### Desmos

- **产品形态**：Web（免费）
- **技术实现**：Canvas + 自研数学引擎
- **解析几何能力**：通过方程输入绘制任意曲线，支持参数滑块
- **优点**：界面极简美观，方程输入体验极佳，免费
- **不足**：
  - 纯方程驱动，无"选择曲线类型"的教学引导
  - 不自动标注焦点/准线等几何要素
  - 缺乏针对教学的专题模式
  - 无交点/弦长自动计算

### 对我们的启示

1. **差异化定位**：不做通用数学工具（GeoGebra 已无法超越），做**面向高中教学场景的专题工具**
2. **教学导向设计**：焦点弦专题模式、离心率连续演变、直线与圆位置关系专题 — 这些都是竞品没有或做得不好的
3. **极简交互**：学 Desmos 的简洁，但加上教学引导（选曲线类型→自动标注→专题模式）
4. **无需 CAS**：不做符号计算系统，只做高中解析几何范围内的数值/公式计算即可
5. **投影仪友好**：大字体、高对比度、粗线条 — 这是所有竞品的共同短板

## 6. 初步技术判断

### 推荐的渲染技术

- **主渲染**：2D Canvas（HTML5 Canvas API）
  - 高性能曲线绘制（1000+ 采样点逐帧重绘无压力）
  - 参数拖动时平滑刷新
  - 可用 Path2D 缓存曲线路径
- **标注层**：SVG 或 DOM 叠加层
  - 坐标标签、方程文本、交互控件用 DOM/SVG 更方便
  - 避免 Canvas 文本渲染的锯齿和交互不便
- **公式渲染**：KaTeX（轻量、快速）
- **无需 Three.js / WebGL**：纯 2D 场景

### 推荐的架构模式

- **参数驱动 + 轻量实体管理**的混合架构：
  - **核心层**：坐标系管理（缩放/平移/网格）+ 曲线渲染引擎 + 数学计算引擎
  - **实体层**：轻量对象模型（Curve / Line / Point / Annotation），支持增删改查
  - **命令层**：简单的 undo/redo 栈（操作种类少，不需要复杂命令模式）
  - **交互层**：滑块绑定 + 鼠标事件（点选/拖拽）
  - **专题模式层**：焦点弦模式、离心率演变模式、直线与圆模式 — 各自是独立的"预设场景+交互规则"

### 与 M-02 函数图像实验室的关系

- **高度共享的能力**：
  - 2D 坐标系（网格、刻度、缩放、平移）— **完全相同**
  - 曲线绘制引擎（参数方程 → 采样点 → Canvas 路径）— **高度相似**
  - 参数滑块与实时更新机制 — **完全相同**
  - 交点计算（直线与曲线联立）— 可复用
  - 公式渲染 — 相同需求
- **M-03 独有的能力**：
  - 圆锥曲线专用计算（焦点/准线/离心率/焦半径/焦点弦公式）
  - 专题模式（焦点弦/离心率演变/位置关系）
  - 直线工具（M-02 可能也有，但 M-03 更核心）
  - 动点轨迹
  - 面积计算（封闭区域着色）
- **结论**：M-02 和 M-03 **强烈建议同仓库**，共享坐标系+曲线引擎+参数交互层，各自实现专题功能

### 关键技术风险或难点

1. **离心率连续演变的平滑过渡**：e=1 时从椭圆到抛物线的拓扑变化（封闭→开放），需要统一参数化表示（如焦点-准线统一定义），渲染上需处理抛物线在 e=1 附近的平滑过渡
2. **高精度交点计算**：直线与圆锥曲线联立求解需要数值稳定的二次方程求解器，特别是判别式接近 0（相切）时的精度处理
3. **弦长精确值显示**：需要一定的符号计算能力（化简根号表达式），或退而求其次只显示小数近似值
4. **动点轨迹的流畅绘制**：动点按规则运动时需要高效的轨迹采样和渲染，避免卡顿
5. **面积计算**：弓形等非多边形区域的面积需要数值积分，但高中范围内可用公式简化
