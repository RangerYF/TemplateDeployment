# M-05 概率统计模拟器 — 模块分析卡片

## 1. 用户场景 & 产品价值

- **目标用户**：高中数学老师
- **解决的教学痛点**：概率统计是高中数学中最"反直觉"的部分——学生听说"抛硬币正面概率是 0.5"，可实际抛 10 次出了 7 次正面；"大数定律说频率趋近概率"，但到底多大才够？"正态分布曲线下面积 = 1"是什么意思？这些概念必须通过**大量重复实验**才能建立直觉，而课堂上手工实验时间极其有限（抛 100 次硬币就要好几分钟）。
- **核心产品价值**：一个快速模拟 + 实时可视化的概率统计教学工具。点一下按钮，1 秒模拟 10000 次实验，频率直方图实时更新，学生亲眼看到频率从波动到稳定的全过程，比任何说教都有效。

## 2. 用户实际使用方式

### 教师典型操作流程

- **课前准备**：从实验类型菜单选择要演示的模拟（如"抛硬币"、"正态分布"），预设好参数（投掷次数、分布参数等），可能准备好数据集用于回归分析演示
- **课中演示**：
  - 讲大数定律：先模拟 10 次（频率波动大），再 100 次、10000 次，学生看到频率趋于稳定
  - 讲正态分布：拖动 μ 和 σ 滑块，实时看曲线平移和"胖瘦"变化，一键显示 68-95-99.7 法则
  - 讲回归分析：输入散点数据，自动拟合回归直线，手动增删散点观察直线变化
  - 蒙特卡罗估算 π（亮点演示）：随机投点动画，实时更新 π 估算值
- **讲题**：用模拟器验证手算结果，用统计图表辅助解题分析

### 怎样算"好用"

- 模拟速度要快——10000 次实验 1 秒内出结果
- 图表标准规范——频率直方图纵轴是"频率/组距"（教材标准）、正态曲线平滑
- 动画可控——慢速看过程、快速看结果，可暂停在任意节点
- 投影仪友好——1080p 投影下图表、标注清晰可辨

### 交互频率和深度

- **中频模块**：概率统计在高中必修和选修中各有一个完整单元，使用频率低于函数但高于向量
- **交互深度**：浅到中——核心操作是"选实验 → 调参数 → 点开始 → 看结果"，比编辑器型模块简单，但回归分析的散点编辑和正态分布的区间查询属于中度交互

## 3. 核心功能点

### 功能清单（按重要性排序）

1. **抛硬币模拟** — N 次模拟 + 频率折线图 + 理论值参考线（大数定律核心演示）
2. **正态分布可视化** — μ/σ 滑块实时调节钟形曲线 + 68-95-99.7 法则着色 + 自定义区间概率
3. **蒙特卡罗估算 π** — 随机投点动画 + 实时估算值 + 误差收敛曲线（亮点功能）
4. **频率分布直方图** — 手动输入/预设数据集 + 可调组距 + 纵轴"频率/组距" + 面积验证
5. **掷骰子模拟** — 单/多骰子 + 频率分布柱状图 + 理论分布叠加对比
6. **线性回归分析** — 散点图 + 最小二乘法拟合 + 相关系数 r + 残差可视化 + 手动增删散点
7. **二项分布可视化** — n/p 参数滑块 + 概率分布柱状图 + 正态近似叠加
8. **摸球实验** — 有放回/无放回 + 超几何分布对比
9. **大数定律动画** — 逐步增加实验次数 + 频率收敛动画 + 多曲线叠加
10. **古典概型可视化** — 样本空间图示 + 事件着色 + 条件概率
11. **几何概型扩展** — 会面问题、布丰投针等经典问题可视化
12. **茎叶图** — 手动输入数据 + 背靠背对比

### 需要实体管理的功能

- **极轻量**：本模块主要是"选实验 → 看结果"的模拟演示型，不像编辑器型模块需要管理复杂对象
- 回归分析的散点需要增删管理（最接近实体管理的场景）
- 多条频率曲线的叠加显示/隐藏
- 预设数据集的选择管理

### 需要 undo/redo 的功能

- **需求较弱**：大部分操作是"重置 → 重新模拟"，不需要撤销
- 回归分析中散点的增删可考虑支持 undo
- 参数调节通常通过滑块回拖即可"撤销"

### 参数驱动的功能（调参 → 画面实时变化）

- 正态分布 μ/σ 滑块 → 曲线实时变形
- 二项分布 n/p 滑块 → 柱状图实时变化
- 超几何分布 N/M/n 参数调节
- 频率直方图组距调节
- 回归分析散点位置拖动 → 回归直线实时更新

### 模拟/动画驱动的功能（播放/暂停/逐步）

- **核心模式**：这是本模块与其他模块最大的差异——大量功能是"运行模拟 → 观察动态过程"
- 抛硬币/掷骰子的逐次模拟动画（前 100 次可见动画，之后快速模式）
- 蒙特卡罗投点的逐点/批量动画
- 大数定律的频率收敛动画
- 模拟速度控制（慢/中/快/极快）
- 暂停/继续/重置

## 4. 核心依赖识别

### 技术依赖

**渲染方式**：2D Canvas + SVG/DOM 混合
- Canvas 适合：投点动画（大量点的实时绘制）、频率折线图（高频更新）
- SVG 适合：统计图表（柱状图、正态曲线、散点图）、坐标轴标注、交互元素（hover/click）
- **推荐**：SVG 为主渲染（统计图表天然适合 SVG）+ Canvas 辅助（蒙特卡罗投点等高性能场景），或统一使用图表库（如 D3.js 兼顾两者）

**数学/物理计算需求**：
- 随机数生成（均匀分布、正态分布等）
- 组合数计算 C(n,k)（二项分布、超几何分布概率公式）
- 正态分布密度函数 & 累积分布函数（区间概率计算）
- 最小二乘法线性回归（回归系数 b、a，相关系数 r）
- 统计量计算（均值、方差、标准差、中位数、众数）
- 计算复杂度较低，不需要专门的数学引擎

**公式渲染需求**：
- 概率公式展示（P(X=k) = C(n,k)·p^k·(1-p)^(n-k) 等）
- 回归方程 ŷ = bx + a
- 统计量公式
- 建议使用 KaTeX

### 交互依赖

**交互模式**：**模拟演示型为主 + 参数驱动型辅助**
- 主模式：选择实验 → 设参数 → 点开始 → 观察动画/结果（模拟演示）
- 辅模式：拖滑块 → 分布曲线实时变化（参数驱动，用于正态分布、二项分布等）
- 轻度编辑：回归分析中散点的增删拖动

**关键交互**：
- 按钮（开始实验 / 暂停 / 重置 — 最核心）
- 滑块（参数调节 — μ/σ/n/p 等）
- 输入框（精确参数值、数据输入）
- 下拉菜单（实验类型切换、预设数据集选择）
- 鼠标悬停（图表数据读取 — 频率、概率等精确值）
- 点击/拖拽（回归散点的增删拖动）
- 速度控制（动画速度切换）

**是否需要实体系统**：不需要
- 本模块是"模拟演示型"，核心交互是"运行 → 观察"，不涉及复杂对象的创建/选中/编辑/删除
- 回归散点管理可用简单数组状态实现

**是否需要命令系统（undo/redo）**：不需要（或极轻量）
- 大部分操作通过"重置"回到初始状态
- 回归散点编辑如需撤销，简单操作栈即可

## 5. 竞品分析

### Seeing Theory（Brown University）

- **产品形态**：Web 应用（纯前端），开源
- **技术实现**：D3.js（SVG 渲染），纯静态前端
- **做得好的地方**：
  - 视觉设计精美，动画流畅自然，是统计可视化的标杆
  - 6 大单元 18 个交互可视化，课程体系完整、渐进递进
  - 将抽象统计概念转化为直观可操作的动画体验
- **不足的地方**：
  - 内容固定，不支持用户自定义数据输入
  - 无中文版，中国教师使用有门槛
  - 不针对任何具体教材，缺乏教学要点提示
- **对我们的启示**：其"渐进式单元结构"和"交互式直觉培养"的设计理念值得借鉴；D3.js 是统计可视化的事实标准

### GeoGebra Probability Calculator

- **产品形态**：Web + 桌面 + 移动端
- **技术实现**：GWT 编译的 JavaScript，自有渲染引擎
- **做得好的地方**：
  - 内置概率计算器视图，支持正态、二项、泊松等多种分布
  - 可视化分布曲线并计算指定区间概率
  - 有中文版和大量中文社区教学资源
- **不足的地方**：
  - 界面复杂，学习曲线陡峭
  - 概率统计是附属功能，深度有限
  - 无蒙特卡罗模拟等动态演示
- **对我们的启示**：避免功能堆砌导致界面复杂化；概率统计作为专题工具比全能工具更有价值

### PhET Plinko Probability

- **产品形态**：Web 应用（HTML5）
- **技术实现**：HTML5 Canvas/SVG，PhET 自有框架
- **做得好的地方**：
  - 弹珠落钉板的游戏化设计，直觉性极强
  - "理论 vs 实验"对比展示是核心教学模式
  - 基于大量教育研究设计，教学效果有验证
- **不足的地方**：
  - 功能单一，仅覆盖二项分布
  - 无法扩展到其他分布或自定义场景
- **对我们的启示**："理论值 vs 实验频率"的对比展示是概率教学的核心模式，所有模拟实验都应包含这一设计

### R Psychologist 交互式统计可视化

- **产品形态**：Web 应用（系列独立页面）
- **技术实现**：D3.js + React，SVG 渲染
- **做得好的地方**：
  - 参数滑块实时联动可视化，交互设计优秀
  - 专业深度大，被高校教学广泛引用
- **不足的地方**：
  - 面向有统计基础的用户，入门门槛高
  - 各工具独立，缺乏整体串联
- **对我们的启示**：参数滑块实时联动是统计可视化的核心交互范式

### 对我们的启示总结

1. **"理论 vs 实验"对比是核心教学模式**：所有模拟实验都应同时显示理论值参考线和实验频率，让学生直观感受"频率趋近概率"
2. **D3.js/SVG 是统计图表的成熟方案**：Seeing Theory、R Psychologist 都基于 D3.js，证明其在统计教育可视化领域的成熟度
3. **差异化在"教学场景化"**：按中国人教版教材分类、内置教学要点、预设数据集贴近高考——这是所有国际竞品做不到的
4. **动画速度控制是刚需**：慢速看过程（建立直觉）+ 快速看结果（验证理论），两种模式缺一不可
5. **功能覆盖要全但交互要简**：PRD 规划的 9 个功能覆盖了高中概率统计全部教学内容，但每个功能的交互应该极简——"选 → 调 → 点 → 看"

## 6. 初步技术判断

### 推荐的渲染技术

- **统计图表**：D3.js（SVG 渲染）或 Chart.js/ECharts
  - 理由：统计图表（柱状图、折线图、散点图、正态曲线）是 SVG 的强项——支持精确的坐标轴标注、hover 交互、动画过渡；D3.js 在统计可视化领域有大量成熟实践
  - 备选：如果团队更熟悉封装型图表库，ECharts 或 Chart.js 可以更快上手，但自定义深度不如 D3.js
- **蒙特卡罗投点**：HTML5 Canvas 2D
  - 理由：大量随机点（10 万+）的实时绘制需要 Canvas 的性能
- **UI 层**：React DOM
  - 理由：参数面板、按钮、滑块、下拉菜单等控件用 DOM 实现
- **公式渲染**：KaTeX
  - 理由：概率公式、回归方程等数学表达式的标准渲染

### 推荐的架构模式

- **模拟演示型架构**（区别于 M-01 的编辑器型、M-02 的参数驱动型）
  - 核心数据流：`用户操作(选实验/调参/点开始) → 模拟引擎(随机采样/计算) → 图表渲染(实时更新)`
  - 状态管理：React 状态（或 Zustand）管理实验类型、参数值、模拟结果、动画状态
  - 模拟引擎：Web Worker 中运行大规模模拟（避免阻塞 UI），通过 postMessage 逐步推送结果
  - 渲染层：图表组件库 + Canvas 投点渲染
  - 实验切换：实验类型菜单驱动，切换实验 → 切换参数面板 + 可视化视图（类似路由）

### 关键技术风险或难点

1. **大规模模拟的性能与动画流畅性**（难度：中，风险：中）
   - 10 万次模拟本身计算量不大（纯 JS 毫秒级完成），关键是**动画展示**——逐次/逐批模拟并实时更新图表
   - 需要平衡"动画效果"和"出结果速度"：前 100 次逐次动画，之后批量更新
   - **建议**：大规模模拟放 Web Worker，通过可控帧率推送中间结果到主线程渲染

2. **蒙特卡罗投点的大量点渲染**（难度：中，风险：低）
   - 10 万个点的 Canvas 绑图没有性能问题
   - 逐点动画需要 requestAnimationFrame 控制帧率
   - 点的圆内/圆外判定和着色是简单的几何计算

3. **统计图表的规范性**（难度：低，风险：中）
   - 频率直方图纵轴必须是"频率/组距"（教材标准、考试常考），不是频率也不是频次——这是一个容易出错的细节
   - 正态曲线必须平滑（足够密的采样点）
   - 图表在投影仪上的可读性（字号、颜色对比度）

4. **实验类型多导致的代码组织**（难度：中，风险：低）
   - 9 个功能/实验类型，每个有独立的参数面板和可视化视图
   - 需要良好的模块化设计：通用的模拟框架 + 实验类型插件
   - 可用策略模式或组件化解决

5. **与其他模块的复用性**（风险：低，需设计时注意）
   - 本模块与其他 5 个数学模块差异较大——它是"模拟演示型"，其他多为"编辑器型"或"参数驱动型"
   - 可复用的组件：2D 图表渲染（坐标系、轴标注、网格）、滑块组件、KaTeX 公式渲染
   - 表达式解析器（M-02/M-03/M-04 核心需求）在本模块中不需要
