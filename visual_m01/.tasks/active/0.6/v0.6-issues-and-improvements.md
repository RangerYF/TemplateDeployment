# V0.6 AI 推荐功能 — 现存问题与改进方向

> 创建时间：2026-03-23
> 背景：V0.6 四个阶段已完成，整体链路跑通（输入 → AI 推荐 → 点击加载 3D 场景），但实际使用中仍有明显体验问题。

## 一、作品数量不足

**现状**：108 个作品，覆盖 9 种几何体类型。

**问题**：
- 用户输入一个知识点，LLM 只能从 108 个中选，匹配度可能不高
- 某些几何体类型严重偏科：正方体 29 个、棱锥/棱柱各 23 个，而圆锥仅 1 个、正四面体仅 2 个、球 4 个
- 缺失常见教学场景：如正方体展开图、三视图、旋转体、组合体等
- 无"纯知识点演示"作品（如"什么是二面角"的概念可视化），全部源自真题

**影响**：推荐结果可能不匹配用户意图，或反复推荐同一批作品。

## 二、真题选项"爆炸"问题

**现状**：19 个作品来源于选择题，每道选择题有 4 个选项（A/B/C/D）。

**问题**：
- 选择题的每个选项可能描述不同的几何关系（如选项 A 说"FG // 平面 AA₁D₁D"，选项 B 说"EF // 平面 BC₁D₁"）
- 当前生成脚本可能把所有选项的几何含义都画在同一个 scene_data 中
- 导致场景内容过于密集、视觉上杂乱、教学重点不聚焦
- 用户看到的不是"一个清晰的几何演示"，而是"一道题目的所有答案堆在一起"

**理想状态**：每个选项拆成独立作品，或只可视化正确答案相关的几何关系。

## 三、作品质量参差不齐

**现状**：阶段 2（人工校验）被跳过，仅做了简单抽检。

**问题**：
- 部分作品的度量标注（角度弧线、距离虚线）可能定位不准确
- 部分场景的辅助线/辅助面绘制不完整或多余
- 截面作品的多边形顶点顺序可能导致渲染异常
- 坐标系作品的轴方向可能不直观
- 没有系统性的质量评估，不知道哪些作品有问题

## 四、作品描述和标签不够丰富

**现状**：每个作品的 `description` 仅一句话（如"正方体中过中点的线面平行关系"），`tags` 来自自动提取。

**问题**：
- LLM 匹配时信息不足，description 太短无法精确理解作品内容
- tags 可能有遗漏或不准确（自动生成）
- 缺少"适用场景"描述（如"适合讲解线面平行判定定理"）
- 缺少"教学目标"字段（如"学生能理解中位线与平面的平行关系"）

**影响**：LLM 推荐准确度受限于 meta 信息质量。

## 五、Prompt 注入全量 meta 的性能与成本问题

**现状**：每次推荐请求将 108 个 meta 全部注入 System Prompt（约 6-8K tokens）。

**问题**：
- 随着作品增加，Prompt 会越来越长，增加延迟和 API 费用
- Haiku 的上下文窗口虽够用，但信息密度降低可能影响匹配质量
- 无预筛选机制，LLM 每次都要遍历全部 108 个作品

**改进方向**：
- 前端先按关键词/几何体类型粗筛，再将候选子集发给 LLM 精排
- 或使用 embedding 向量做相似度召回，LLM 只做 rerank

## 六、推荐结果缺少反馈闭环

**现状**：用户看到推荐结果后只能点击加载或换个描述重试。

**问题**：
- 无法知道推荐结果是否真的帮到了用户
- 无"不相关"反馈机制
- 无推荐历史记录
- 无法基于用户反馈优化 Prompt 或匹配策略

## 七、场景类型分布不均

**现状分布**：

| 场景类型 | 数量 | 说明 |
|---------|------|------|
| comprehensive（综合） | 36 | 最多，但"综合"标签太泛，匹配时不够精确 |
| volume / surface-area | 10 / 10 | 体积/面积类较多 |
| skew-angle / dihedral / line-face-angle | 8 / 7 / 7 | 角度类还行 |
| point-distance / parallel | 7 / 7 | 距离/平行关系 |
| circumsphere | 7 | 外接球 |
| perpendicular | 5 | 垂直关系偏少 |
| skew-distance | 2 | 异面距离严重不足 |
| coordinate | 2 | 坐标系方法偏少 |

**问题**：
- "comprehensive"类型太泛，36 个作品标记为综合，LLM 难以精确区分
- 异面距离、坐标法等高频考点作品太少
- 缺少"截面"作为独立场景类型（当前截面可能归入 comprehensive）

## 九、部署环节待验证

**现状**：API 代理层代码已写好但尚未部署。

**待办**：
- Vercel 环境变量 `AI_API_KEY` / `AI_BASE_URL` 尚未配置
- Edge Function 在生产环境的行为未测试
- 代理目标 `edumindai.nocoder.win` 的稳定性和限流策略未知
